Pages-Hub

集中式 GitHub Pages 部署中心，管理多個靜態網站專案的統一路由、CI/CD 管線與子網域管理。

GitHub Pages CI/CD Static Sites GitHub Actions
View on GitHub

概述

Pages-Hub 是集中式編排層，用於管理多個 GitHub Pages 靜態網站專案。負責統一路由、CI/CD 管線協調與部署生命週期管理，為所有發布在 yu-codes.github.io 下的子專案提供單一控制平面。

目錄結構

pages-hub/
├── .github/
│   └── workflows/
│       ├── deploy-portfolio.yml    # 作品集網站部署
│       ├── deploy-docs.yml         # 文件網站部署
│       ├── sync-subprojects.yml    # 跨倉庫同步
│       └── cleanup.yml             # 過期部署清理
├── config/
│   ├── routes.json                 # URL 路由設定
│   ├── projects.json               # 已註冊子專案
│   └── domains.json                # 自訂網域對應
├── scripts/
│   ├── deploy.sh                   # 部署編排器
│   ├── validate.sh                 # 部署前驗證
│   └── rollback.sh                 # 回滾機制
├── templates/
│   ├── 404.html                    # 共用 404 頁面
│   └── redirect.html               # URL 重導範本
├── public/                          # 聚合部署產出
├── README.md
└── CNAME                            # 自訂網域設定

功能特色

  • 多專案編排：從單一中心註冊與管理多個靜態站點，各自獨立部署生命週期
  • 統一路由：乾淨 URL 路徑（/portfolio/、/docs/ 等）透過靜態檔案結構路由至各子專案建置產出
  • CI/CD 管線協調：GitHub Actions 工作流程在子專案變更時觸發，搭配跨倉庫 dispatch 自動重建
  • 部署驗證：部署前檢查建置產出完整性、壞連結與資源可用性
  • 回滾機制：透過 Git 歷史快速回滾至前一版部署，確保零停機恢復
  • 過期清理：自動清理孤立部署與過期建置產物
