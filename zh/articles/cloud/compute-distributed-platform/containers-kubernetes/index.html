容器技術與 Kubernetes 已成為現代雲端應用部署的事實標準。容器提供輕量級、可攜帶的應用打包方式，而 Kubernetes 則提供了大規模容器編排與管理的平台。

AWS 服務實作

ECR（Elastic Container Registry）

  • AWS 管理的容器映像倉庫，類似 Docker Hub，可推送與拉取 Docker 映像
  • 每個區域每個帳號都有一個登錄庫，內有多個儲存庫
  • 每個儲存庫可以有多個映像，映像可以有多個標籤

映像管理功能：

  • 映像標籤：支援可變和不可變標籤
  • 跨區域/跨帳號複製：自動同步映像到其他區域或帳號
  • 映像掃描：自動掃描容器映像的安全漏洞
  • 生命週期政策：自動清理舊映像

安全功能：

  • 支援 KMS 加密
  • IAM 細粒度存取控制
  • VPC 端點支援私有存取

ECS（Elastic Container Service）概述

  • 容器編排服務，可建立、執行、管理 Docker 容器叢集
  • 支援 EC2 啟動類型（自管 EC2）與 Fargate 啟動類型（無伺服器）

ECS 核心概念

Task Definition（任務定義）： 定義單一或多個容器的設定內容：

  • 映像來源
  • 記憶體和 CPU
  • 埠口映射
  • 環境變數
  • 儲存配置

IAM Role 類型：

類型說明
ECS Task IAM Role給容器使用，讓其可存取 AWS 資源（如 S3、DynamoDB）
Task Execution IAM Role給 EC2 執行階段使用，用於拉映像、發送 log 至 CloudWatch

Service Definition（服務定義）：

  • 用來部署與維護指定數量的 ECS 任務
  • 可搭配 CLB/ALB
  • 執行在 EC2 叢集或 Fargate 上

ECS 啟動類型比較

項目EC2 啟動類型Fargate 啟動類型
管理方式自行管理 EC2 實例，需安裝 ECS AgentServerless，ECS 管理底層 EC2
配置完全控制底層基礎設施僅需定義容器資源（CPU/記憶體）
適用場景長時間、可預測工作量間歇或彈性工作
成本較低（可用 Reserved/Spot）較高
啟動時間較慢較快

ECS Agent：

  • 安裝於 EC2 實例中
  • 負責回報資源與任務狀態
  • 接收 ECS 控制指令

模式說明

ECS Service Auto Scaling

  • Target Tracking：維持指標在目標值
  • Step Scaling：階梯式調整
  • Scheduled Scaling：排程調整

常用指標：

  • ECSServiceAverageCPUUtilization
  • ECSServiceAverageMemoryUtilization
  • ALBRequestCountPerTarget

ECS 服務發現

  • 與 Route 53 整合
  • 自動註冊和取消註冊服務
  • 支援 DNS 和 SRV 記錄

ECS Anywhere

  • 在本地伺服器或虛擬機上運行 ECS 任務
  • 使用 ECS Agent 連接到 AWS
  • 統一管理雲端和本地容器工作負載

EKS（Elastic Kubernetes Service）

  • AWS 託管的 Kubernetes（K8s）叢集服務
  • 適合已有 K8s 經驗或使用原生 K8s 的團隊
  • 可運行在 AWS、Outposts、EKS Anywhere、Local Zones
  • 控制平面運行在 AWS 託管的 VPC 中，整合 AWS 服務：ECR、ELB、IAM、VPC

控制平面特性：

  • AWS 管理多個 AZ，etcd 跨多個 AZ 分散
  • 透過 ENI 注入客戶 VPC

EKS 節點類型：

類型說明
自管節點您建立和管理的 EC2
託管節點群組AWS 管理的 EC2 群組
Fargate Pods無伺服器容器

EKS 進階功能

安全性：

  • IAM Roles for Service Accounts (IRSA)：為 Pod 分配 IAM 角色
  • Pod 安全群組：直接對 Pod 套用安全群組
  • Secrets 加密：使用 KMS 加密 etcd 中的 Secrets

負載平衡：

  • AWS Load Balancer Controller 支援 ALB/NLB
  • 支援 Ingress 資源自動配置 ALB

監控：

  • Container Insights 提供叢集和 Pod 指標
  • 與 CloudWatch 和 X-Ray 整合

EKS Anywhere：

  • 在本地資料中心運行 EKS
  • 使用相同的工具和 API
  • 支援 VMware vSphere

容器基礎概念（AWS 視角）

容器化特點：

  • 與虛擬機不同：容器使用主機作業系統
  • 容器不如虛擬機隔離
  • 容器僅包含應用程式和相依套件
  • 容器引擎（Docker）讓容器相信它們有自己的作業系統

映像與容器：

  • Docker 映像：用於建立容器的藍圖，透過 Dockerfile 建立
  • 映像分為多層，只有頂層（讀寫層）可修改
  • 映像儲存在容器登錄庫（如 Docker Hub、ECR）

容器安全最佳實踐

  1. 使用官方基底映像
  2. 定期掃描安全漏洞
  3. 使用不可變標籤
  4. 最小化映像大小
  5. 使用 Fargate 簡化管理：無需管理基礎設施
  6. 使用 awsvpc 網路模式：更好的網路隔離
  7. 配置適當的 Task 資源：避免資源浪費
  8. 實施 Service Auto Scaling：自動調整任務數量
  9. 分離 Task Role 和 Execution Role：遵循最小權限原則
  10. 啟用 ECR 映像掃描：主動發現安全漏洞
  11. 使用 IRSA（EKS）：為 Pod 配置最小權限
