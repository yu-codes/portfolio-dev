邊緣運算

邊緣運算（Edge Computing）是將運算、儲存和網路資源從集中式雲端資料中心延伸到靠近資料來源或使用者的位置。這種架構模式解決了延遲、頻寬和資料主權等挑戰。

核心概念

邊緣運算的定義

位置說明延遲運算能力
雲端（Cloud）集中式資料中心50-200ms無限
近邊緣（Near Edge）區域性小型資料中心10-50ms中高
遠邊緣（Far Edge）靠近使用者/設備1-10ms中
裝置邊緣（Device）終端設備本身< 1ms低

驅動因素

因素說明範例
超低延遲即時回應需求自動駕駛、AR/VR
頻寬節省減少資料傳輸影像分析、IoT
資料主權資料不離開區域GDPR、資料在地化
離線能力網路中斷仍可運作工廠設備、遠端
規模效率分散處理減輕中心壓力CDN、遊戲

邊緣 vs 雲端 vs 地端

面向邊緣雲端地端
位置靠近使用者遠端資料中心企業機房
延遲最低較高中
擴展性有限無限固定
管理複雜度高（分散）低（集中）中
成本模型按位置按用量固定投資
運算能力有限無限固定

應用場景

智慧工廠

  • 感測器即時資料收集
  • 邊緣 ML 即時品質檢測
  • 異常即時告警（< 100ms）
  • 歷史資料上傳雲端分析
  • 離線模式確保產線不停

影像串流分析

  • 攝影機 → 邊緣伺服器
  • 本地 ML 推論（人臉/物件偵測）
  • 僅傳送結果（非全影像）到雲端
  • 頻寬節省 90%+
  • 隱私保護（影像不離開本地）

遊戲與 AR/VR

  • 5G MEC 超低延遲渲染
  • 邊緣伺服器遊戲狀態同步
  • 雲端遊戲串流
  • AR 即時空間運算
