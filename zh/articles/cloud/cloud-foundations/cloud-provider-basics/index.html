雲端供應商

主要雲端平台核心服務

在選擇雲端平台時，理解各主要供應商的服務版圖、核心優勢與生態系定位是做出正確技術決策的第一步。目前全球公有雲市場由三大供應商主導：Amazon Web Services（AWS）、Microsoft Azure 與 Google Cloud Platform（GCP）。它們各自擁有超過百項服務，覆蓋運算、儲存、網路、資料庫、AI/ML、安全與開發工具等領域。本文提供對這三大平台的全面比較與核心服務導覽。

核心概念

雲端供應商的角色：雲端供應商是計算資源的「公用事業」提供者。如同電力公司提供電力，雲端供應商提供運算能力、儲存空間與網路頻寬。使用者無需了解背後的實體基礎設施如何運作，只需關注如何消費這些服務來解決業務問題。

服務組織架構：

  • 運算：處理業務邏輯的核心（VM、容器、Serverless）。
  • 儲存：資料持久化的各種形態（物件、區塊、檔案）。
  • 資料庫：結構化與非結構化資料管理（RDS、DynamoDB、Cosmos DB）。
  • 網路：連接所有元件的骨幹（VPC、負載均衡、DNS）。
  • 安全與身份：存取控制與資料保護（IAM、KMS、WAF）。
  • 開發與部署：CI/CD、容器註冊與應用託管。

計費模型的差異：雖然三家都標榜「按量付費」，但計費粒度、折扣機制與定價策略各有不同。AWS 的 Reserved Instances、Azure 的 Reserved VM Instances 與 GCP 的 Committed Use Discounts 都是長期承諾換取折扣的機制，但條款細節差異顯著。

帳戶與組織管理：

  • AWS 使用 Organizations 管理多帳戶，以 OU（組織單位）進行層級化管理。
  • Azure 採用 Tenant > Management Group > Subscription > Resource Group 的階層。
  • GCP 使用 Organization > Folder > Project 的結構。

架構總覽

三大供應商各有其核心服務對應關係：

運算服務：

  • AWS EC2 / Azure Virtual Machines / GCP Compute Engine — 虛擬機器
  • AWS Lambda / Azure Functions / GCP Cloud Functions — 無伺服器函式
  • AWS ECS+EKS / Azure AKS / GCP GKE — 容器編排

儲存服務：

  • AWS S3 / Azure Blob Storage / GCP Cloud Storage — 物件儲存
  • AWS EBS / Azure Managed Disks / GCP Persistent Disk — 區塊儲存
  • AWS EFS / Azure Files / GCP Filestore — 檔案儲存

資料庫服務：

  • AWS RDS / Azure SQL Database / GCP Cloud SQL — 關聯式資料庫
  • AWS DynamoDB / Azure Cosmos DB / GCP Firestore — NoSQL
  • AWS ElastiCache / Azure Cache for Redis / GCP Memorystore — 快取

網路服務：

  • AWS VPC / Azure VNet / GCP VPC — 虛擬私有網路
  • AWS ELB / Azure Load Balancer / GCP Cloud Load Balancing — 負載均衡
  • AWS Route 53 / Azure DNS / GCP Cloud DNS — DNS 管理

各平台的獨特優勢：

  • AWS：最早進入市場，社群資源最豐富，第三方整合最多。Marketplace 生態系強大。
  • Azure：Active Directory 整合無可替代，Hybrid Cloud（Azure Arc）策略成熟。對企業既有 Microsoft 授權有優惠。
  • GCP：BigQuery 在數據分析領域領先，GKE 是最佳的託管 Kubernetes，Vertex AI 整合完善。全球網路品質最佳。

應用場景

  • 新創公司起步：通常選擇 AWS（生態最成熟、教學資源最多）或 GCP（免費額度慷慨、BigQuery 免費額度高）。
  • 企業數位轉型：若已使用 Microsoft 技術堆疊（AD、Office 365、.NET），Azure 是自然選擇。
  • 數據密集型應用：BigQuery 的無伺服器特性與按掃描量計費模型，讓 GCP 在大數據場景極具吸引力。
  • AI/ML 平台：三家都有完整的 ML 平台（SageMaker、Azure ML、Vertex AI），選擇取決於團隊熟悉度與既有資料位置。
  • 多雲策略：使用 Terraform 等 IaC 工具搭配各供應商的 Kubernetes 服務，實現核心工作負載的跨雲可攜性。
