容錯設計

容錯設計（Fault Tolerance）是指系統在部分元件發生故障時，仍能繼續正常運作的能力。在分散式雲端環境中，「故障是常態而非例外」——硬體會損壞、網路會中斷、軟體會出錯。容錯設計的目標是讓這些不可避免的故障對使用者透明。

故障類型分類

容錯架構層級

第一層：實例級容錯

  • Auto Scaling Group 自動替換不健康實例
  • 健康檢查（HTTP、TCP、自訂）
  • 自動重啟策略

第二層：可用區域級容錯

  • 跨 AZ 部署（最少 2 個 AZ）
  • 資料跨 AZ 同步複寫
  • 負載平衡器自動故障轉移

第三層：區域級容錯

  • 多區域部署（Active-Active 或 Active-Passive）
  • 全球負載平衡（DNS 或 Anycast）
  • 跨區域資料複寫（同步或非同步）

應用場景

金融交易系統

  • 需求：零資料遺失、近零停機
  • 架構：同步複寫 + Active-Active
  • 跨 AZ：資料庫同步複寫
  • 跨 Region：半同步複寫 + 自動故障轉移
  • 監控：毫秒級異常偵測

全球 SaaS 應用

  • 多區域 Active-Active 部署
  • 全球負載平衡（按延遲路由）
  • 資料最終一致性（CRDT 或事件溯源）
  • 區域獨立（一個區域故障不影響其他）

IoT 邊緣場景

  • 邊緣節點本地容錯（離線模式）
  • 資料緩衝與重試上傳
  • 中心端多 AZ 部署
  • 最終一致性的資料同步
