高可用性（High Availability，HA）是指系統能在預定時間內持續提供服務的能力，通常以可用性百分比（如 99.99%）來衡量。在雲端架構中，實現高可用性需要在多個層級實施冗餘、故障偵測與自動復原機制。

核心概念

可用性的定義與量測

可用性公式： $$Availability = \frac{MTBF}{MTBF + MTTR} \times 100%$$

術語說明
MTBF平均故障間隔時間
MTTR平均修復時間
Uptime系統正常運行時間
Downtime系統停機時間

N 個九的含義

組合可用性計算

串聯系統（所有元件都需正常）： $$A_{total} = A_1 \times A_2 \times … \times A_n$$

並聯系統（任一元件正常即可）： $$A_{total} = 1 - (1-A_1) \times (1-A_2) \times … \times (1-A_n)$$

架構計算結果
單一 99.9% 服務0.99999.9%
兩個 99.9% 串聯0.999 × 0.99999.8%
兩個 99.9% 並聯1-(0.001)²99.9999%
三層各 99.9% 串聯0.999³99.7%

應用場景

三層 Web 應用 HA

  1. 前端：CloudFront/CDN + S3（多 AZ 冗餘）
  2. 應用：ALB + Auto Scaling（跨 3 AZ）
  3. 資料：RDS Multi-AZ + ElastiCache Multi-AZ
  4. 監控：健康檢查 + 自動告警 + 自動復原

全球 SLA 99.99% 架構

  • 兩個 Region Active-Active
  • 全球負載平衡（延遲路由）
  • 資料庫全球複寫（Aurora Global/Spanner）
  • 每個 Region 內跨 3 AZ
  • Chaos Engineering 定期驗證

無伺服器高可用

  • Lambda/Cloud Functions 自動跨 AZ
  • DynamoDB Global Tables 全球複寫
  • API Gateway 內建 HA
  • 無需管理基礎設施層面的 HA
