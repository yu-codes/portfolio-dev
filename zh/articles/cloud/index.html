雲端工程

涵蓋 AWS、GCP、Azure、基礎設施即程式碼、網路、安全性與維運卓越性的完整雲端工程知識。

雲端工程是一門在雲端平台上建構、部署與維運可擴展基礎設施的工程實務。作為一名雲端工程師，你需要負責設計具備韌性的架構、自動化基礎設施、管理安全性，並持續優化成本效率。

本章節提供一條具結構性的學習路徑，帶你從雲端基礎一路進展到跨大型雲端供應商的企業級維運能力。

為什麼是雲端工程？

雲端技術已經徹底改變軟體的建構與維運方式。雲端工程師需要解決關鍵的基礎設施問題：

  • 如何設計能隨需求自動擴展的基礎設施？
  • 如何確保系統在多區域（Multi-region）下仍具備高可用性？
  • 如何在維持開發效率的同時保護雲端資源安全？
  • 如何在不犧牲效能的前提下優化雲端成本？

子領域結構

每個領域都會再細分為 2 至 5 個子領域，以涵蓋更具體的技能範疇。

CE1：雲端供應商基礎

  • AWS 核心服務與架構
  • GCP 基礎與差異比較
  • Azure 服務與生態系
  • Multi-cloud 策略

CE2：基礎設施即程式碼

  • Terraform 基礎與模組化
  • AWS CloudFormation 與 CDK
  • Pulumi 與程式語言型 IaC
  • State 管理與最佳實踐

CE3：網路與安全

  • VPC 設計與 Subnet 劃分
  • Load Balancer（ALB、NLB、GLB）
  • DNS 與 Route 53
  • Network Security Group 與防火牆

CE4：運算與容器

  • EC2 Instance 類型與效能最佳化
  • 容器編排（ECS、EKS、GKE）
  • Serverless 運算（Lambda、Cloud Functions）
  • Kubernetes 基礎

CE5：儲存與託管資料庫

  • Object Storage（S3、GCS、Azure Blob）
  • 託管 SQL 資料庫（RDS、Cloud SQL）
  • NoSQL 服務（DynamoDB、Cosmos DB）
  • Data Lake 架構

CE6：IAM 與 Secrets 管理

  • IAM Policy 與 Role
  • Service Account 與 Federation
  • Secrets Manager 與 Vault
  • Zero-trust 安全模型

CE7：監控與日誌

  • CloudWatch Metrics 與 Dashboard
  • Log Aggregation 與分析
  • 告警與 Incident Management
  • Application Performance Monitoring

CE8：自動化與編排

  • Lambda Function 與 Trigger
  • Step Functions 與 Workflow
  • Event-driven Architecture
  • 基礎設施自動化

CE9：成本與治理

  • Cost Explorer 與 Budget 管理
  • Reserved Instance 與 Savings Plan
  • Resource Tagging 策略
  • FinOps 實踐

CE10：災難復原與高可用性

  • Multi-AZ 與 Multi-region 部署
  • 備份與還原策略
  • RTO 與 RPO 規劃
  • Chaos Engineering

CE11：合規與安全

  • AWS Security Hub 與 GuardDuty
  • 合規框架（SOC2、HIPAA、PCI）
  • 安全稽核與日誌追蹤
  • 雲端滲透測試
