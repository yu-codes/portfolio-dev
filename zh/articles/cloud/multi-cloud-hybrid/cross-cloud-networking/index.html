跨雲網路（Cross-Cloud Networking）是多雲架構中最關鍵的基礎設施層，負責在不同雲端供應商之間建立安全、高效且可靠的網路連線。當企業採用多雲策略時，跨雲網路的設計直接影響應用程式的效能、安全性與可用性。

核心概念

跨雲網路的核心在於解決不同雲端環境之間的連通性問題。每個雲端供應商都有自己的虛擬網路架構——AWS 的 VPC、Azure 的 VNet、GCP 的 VPC Network——這些網路彼此獨立，需要透過特定的技術手段進行互聯。

連線模式

跨雲網路主要有三種連線模式：

  1. 公網互聯（Public Internet）：透過加密隧道（如 IPSec VPN）在公共網路上建立安全通道。成本最低但延遲較高，適合非即時性工作負載。

  2. 專線互聯（Dedicated Interconnect）：透過實體專線（如 AWS Direct Connect、Azure ExpressRoute、GCP Cloud Interconnect）建立私有連線。提供穩定的頻寬與低延遲，但成本較高。

  3. 雲端交換平台（Cloud Exchange）：透過第三方網路交換平台（如 Megaport、Equinix Fabric）實現多雲互聯。提供彈性的連線管理與統一的控制平面。

網路抽象層

為了簡化跨雲網路管理，現代架構通常引入網路抽象層，將底層的雲端網路差異封裝起來。這包括統一的 IP 位址管理（IPAM）、跨雲 DNS 解析、以及一致的網路策略定義。

架構總覽

典型的跨雲網路架構包含以下幾個層次：

傳輸層（Transport Layer）

傳輸層負責建立雲端之間的基本連通性。常見的實作方式包括：

  • Hub-and-Spoke 拓撲：選定一個雲端作為中心節點（Hub），其他雲端透過 Spoke 連接到中心。管理簡單但中心節點可能成為瓶頸。
  • Full-Mesh 拓撲：所有雲端之間都建立直接連線。提供最佳效能但管理複雜度隨節點數量指數增長。
  • Transit Gateway 拓撲：使用雲端原生的 Transit Gateway 服務作為路由中心，結合 Hub-and-Spoke 的簡潔與 Full-Mesh 的效能。

控制層（Control Plane）

控制層負責路由決策與網路策略的分發。關鍵組件包括：

  • BGP 路由交換：透過 Border Gateway Protocol 在不同雲端之間交換路由資訊，實現動態路由收斂。
  • SD-WAN 控制器：提供集中式的網路管理與流量工程能力，支援基於應用的智慧路由。
  • Service Mesh：在應用層實現服務間的跨雲通訊，提供流量管理、可觀測性與安全性。

安全層（Security Layer）

安全層確保跨雲通訊的機密性與完整性：

  • 加密傳輸：所有跨雲流量必須加密，通常使用 IPSec 或 WireGuard 協定。
  • 微分段（Microsegmentation）：在網路層實施零信任原則，限制橫向移動。
  • 統一防火牆策略：跨雲一致的安全群組與網路 ACL 規則。

應用場景

場景一：災難復原（DR）

企業在主要雲端（如 AWS）運行生產工作負載，同時在備援雲端（如 Azure）維護災難復原環境。跨雲網路確保兩個環境之間的資料同步與故障切換能力。關鍵要求包括低 RPO/RTO、自動化故障偵測與切換機制。

場景二：最佳服務選用

不同的雲端供應商在特定服務上有各自的優勢。例如，使用 GCP 的 BigQuery 進行資料分析，同時使用 AWS 的 SageMaker 訓練機器學習模型。跨雲網路使這些服務能夠無縫協作，資料在不同雲端之間高效流動。

場景三：合規與資料主權

某些法規要求資料必須存放在特定地理區域。跨雲網路允許企業在不同區域的雲端之間建立安全通道，同時確保資料流動符合法規要求。透過精細的路由控制，可以確保敏感資料不會離開指定的管轄區域。

場景四：併購整合

當企業進行併購時，被收購方可能使用不同的雲端供應商。跨雲網路提供了一個過渡方案，允許兩個組織的系統在整合期間保持互聯，同時逐步遷移到統一的架構。
