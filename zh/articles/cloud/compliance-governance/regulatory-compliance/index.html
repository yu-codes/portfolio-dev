法規遵循（Regulatory Compliance）是企業在雲端環境中運營時必須滿足的法律與行業標準要求。不同產業、不同地區有各自的合規框架，從金融業的 PCI-DSS 到醫療業的 HIPAA，從通用的 SOC 2 到政府的 FedRAMP。理解這些框架的要求並在雲端架構中實現自動化合規，是現代雲端治理的核心挑戰。

核心概念

合規框架的分類

合規框架可依據其性質分為幾大類：

法律法規（Regulatory）

  • 由政府或監管機構制定，具有法律強制力
  • 違規將面臨罰款、業務限制或刑事責任
  • 例：GDPR、HIPAA、SOX、巴塞爾協定

行業標準（Industry Standard）

  • 由行業組織制定，通常是商業合作的先決條件
  • 不遵守可能導致無法接受特定業務
  • 例：PCI-DSS（支付卡產業）、SWIFT CSP（金融通訊）

自願性框架（Voluntary Framework）

  • 企業自願採用以展示安全能力
  • 通常是客戶選擇供應商的考量因素
  • 例：SOC 2、ISO 27001、CSA STAR

共同責任模型（Shared Responsibility Model）

在雲端環境中，合規責任由雲端供應商與客戶共同承擔：

  • 供應商責任：實體安全、硬體維護、網路基礎設施、虛擬化層安全。供應商通常持有 SOC 2、ISO 27001 等認證以證明其責任範圍的合規性。

  • 客戶責任：資料加密、存取控制、應用安全、作業系統更新、網路配置。客戶必須在其責任範圍內實施合規控制措施。

  • 共享責任：某些控制措施需要雙方合作。例如加密功能由供應商提供，但啟用與金鑰管理是客戶的責任。

合規即程式碼（Compliance as Code）

現代雲端合規趨向自動化，將合規要求轉化為可程式化執行的政策：

  • 政策定義：使用結構化語言定義合規規則（如 Open Policy Agent/Rego、AWS Config Rules）
  • 持續監控：自動化掃描雲端資源配置，偵測不合規狀態
  • 自動修復：發現不合規時自動執行修復操作
  • 證據收集：自動產出稽核所需的合規證據

三道防線（Three Lines of Defense）

第一道防線：預防性控制（Preventive Controls）

在資源建立之前阻止不合規配置：

  • Service Control Policies（SCPs）：限制 AWS 組織內可執行的操作
  • Azure Policy（Deny Effect）：阻止不符合政策的資源建立
  • GCP Organization Policies：在組織層級限制資源配置
  • CI/CD Pipeline 掃描：在部署前檢查 IaC 模板的合規性（如 Checkov、tfsec）

第二道防線：偵測性控制（Detective Controls）

持續監控已部署資源的合規狀態：

  • AWS Config Rules：持續評估 AWS 資源配置
  • Azure Policy（Audit Effect）：標記不合規資源
  • GCP Security Command Center：集中式安全與合規發現
  • 第三方工具：Prisma Cloud、Wiz、Lacework

第三道防線：矯正性控制（Corrective Controls）

發現不合規時自動或手動修復：

  • AWS Config Remediation：自動執行修復 Lambda
  • Azure Policy Remediation Tasks：自動修復不合規資源
  • 自定義自動化：透過事件驅動架構觸發修復工作流

應用場景

場景一：SaaS 公司取得 SOC 2 認證

B2B SaaS 公司需要 SOC 2 Type II 報告以贏得企業客戶信任。實施步驟：定義信任服務準則的控制目標、使用 AWS Config 持續監控安全配置、透過 CloudTrail 收集存取控制證據、部署自動化的漏洞掃描與修復。稽核期間（通常 6-12 個月），系統自動收集所有合規證據，大幅減少稽核準備時間。

場景二：電商平台 PCI-DSS 合規

電商平台處理信用卡支付，必須符合 PCI-DSS 要求。架構設計上將持卡人資料環境（CDE）與其他系統隔離，使用專用 VPC 與嚴格的網路 ACL。所有信用卡資料使用 AES-256 加密，金鑰由 CloudHSM 管理。定期執行滲透測試與漏洞掃描，並透過 WAF 防護 Web 應用。

場景三：醫療系統 HIPAA 合規

醫療科技公司在雲端處理患者健康資訊（PHI）。與雲端供應商簽訂 Business Associate Agreement（BAA），啟用所有儲存服務的加密，限制 PHI 存取僅授權人員。實施完整的稽核日誌記錄所有 PHI 存取事件，並設定 72 小時內的資料外洩通報機制。

場景四：多框架合規整合

金融科技公司同時需要滿足 SOC 2、PCI-DSS 與當地金融監管要求。建立統一的控制措施框架，將不同法規的重疊要求對映到共同的技術控制。例如，「加密靜態資料」這個控制措施同時滿足三個框架的要求。透過合規即程式碼方法，一次實施即可產出多個框架的合規證據。
