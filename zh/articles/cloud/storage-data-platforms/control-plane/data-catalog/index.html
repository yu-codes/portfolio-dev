資料目錄（Data Catalog）是企業資料生態系中的「搜尋引擎」，提供資料資產的集中化索引與搜尋功能。它讓資料使用者能快速找到所需資料、理解其意義，並評估其品質與可信度。

資料目錄架構模式

Push Model（推送模式）：

  • ETL 工具主動推送元資料
  • 即時更新
  • 需要各工具整合

Pull Model（拉取模式）：

  • Crawler 定期掃描資料來源
  • 自動發現新資料
  • 可能有延遲

Hybrid Model（混合模式）：

  • 結合推送與拉取
  • 即時 + 定期全量掃描
  • 最完整但複雜度高

應用場景

企業資料平台

  1. 資料來源：資料庫、API、檔案、串流
  2. 爬蟲掃描：自動發現並建立索引
  3. 目錄服務：搜尋、瀏覽、標註
  4. 治理整合：存取控制、品質監控
  5. 消費端：分析師查找、自助分析

敏感資料合規

  • 自動掃描辨識 PII（個人識別資訊）
  • 分類標籤（個人、敏感、機密）
  • 追蹤敏感資料流向（血緣）
  • 存取審計與報告

自助式資料分析

  • 分析師透過目錄搜尋資料
  • 閱讀業務定義與品質指標
  • 申請存取權限
  • 直接在分析工具中使用
