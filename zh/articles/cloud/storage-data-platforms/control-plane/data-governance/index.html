資料治理（Data Governance）是確保資料在整個生命週期中被正確管理、保護和利用的框架與實踐。它涵蓋政策制定、角色分配、存取控制、品質管理與合規確保等面向。

核心概念

資料治理的支柱

支柱說明關鍵活動
資料品質確保資料準確、完整、及時品質規則、監控、修復
資料安全保護資料免受未授權存取加密、存取控制、稽核
資料隱私保護個人資料權利去識別化、同意管理
資料合規遵守法規要求GDPR、HIPAA、PCI-DSS
資料生命週期管理從建立到銷毀保留政策、歸檔、刪除
資料擁有權明確責任歸屬Data Owner、Data Steward

治理角色定義

角色責任層級
CDO（Chief Data Officer）整體資料策略高階管理
Data Owner特定資料集的業務負責人部門主管
Data Steward日常資料品質與標準維護執行層
Data Custodian技術面的資料管理IT 團隊
Data Consumer資料的使用者所有人
DPO（Data Protection Officer）隱私保護負責人法遵

關鍵比較

治理嚴格度 vs 敏捷度

程度特徵適合風險
過度治理大量審批、流程冗長—創新緩慢
適度治理自動化規則 + 例外流程大多數企業平衡
輕量治理指引為主、自助為主新創/小型合規風險
無治理無規範—最高風險

集中式 vs 聯邦式治理

面向集中式聯邦式（Data Mesh）
控制中央 IT 團隊各領域團隊
標準統一標準互操作性標準
速度較慢較快
適合合規嚴格環境大型分散組織
平台中央資料湖領域資料產品
治理委員會集中決策聯合決策

維度說明量測方式範例

應用場景

GDPR 合規實作

  1. 資料映射：識別所有個人資料的位置
  2. 法律依據：為每種處理活動確定法律基礎
  3. 權利支援：實作存取、刪除、可攜帶權
  4. 同意管理：收集與管理使用者同意
  5. 違規通知：72 小時內通報機制
  6. 影響評估：高風險處理的 DPIA

資料湖治理

  • 分層存取控制（Raw/Curated/Consumption）
  • 自動分類掃描（PII 偵測）
  • 生命週期管理（自動歸檔/刪除）
  • Lake Formation 細粒度授權（表/欄位/行級）

多團隊資料平台

  • Data Mesh：領域驅動的資料擁有權
  • 自助式存取申請與審批
  • 自動化品質檢查門檻
  • 集中化治理標準 + 分散執行
