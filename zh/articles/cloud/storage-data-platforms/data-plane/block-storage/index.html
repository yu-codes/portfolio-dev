區塊儲存（Block Storage）是雲端運算中提供高效能、低延遲持久化儲存的服務。它以固定大小的區塊形式組織資料，主要用作虛擬機的系統磁碟和資料磁碟，也是資料庫等 I/O 密集型工作負載的首選儲存。

核心概念

區塊儲存的特點

特性說明
區塊級存取以固定大小區塊（通常 512B-4KB）讀寫
低延遲微秒到毫秒級存取
高 IOPS可達數萬到數十萬 IOPS
獨佔掛載通常一次僅一個實例掛載（Multi-Attach 例外）
格式化需求需建立檔案系統後使用
網路型透過網路連接（非本地磁碟）

應用場景

資料庫儲存

  • 系統碟：gp3（OS + 二進位檔）
  • 資料碟：io2（資料檔案）
  • 日誌碟：io2（交易日誌，高 IOPS）
  • 備份碟：st1（備份歸檔）

高效能運算

  • 多個 gp3 磁碟 RAID 0 組合
  • 或使用 io2 Block Express 單碟
  • Instance Store 作為暫存空間
  • 結果寫回持久儲存

容器持久儲存

  • Kubernetes PersistentVolume
  • CSI Driver（EBS CSI、PD CSI）
  • StorageClass 定義磁碟類型
  • StatefulSet 自動建立 PVC

AWS 服務實作

EBS（Elastic Block Store）

核心概念

  • 為 EC2 提供持久性區塊儲存（EBS Volume）；儲存在單一 AZ（AZ 韌性）
  • 一次只能掛載至單一 EC2，可卸載並掛載至同 AZ 其他 EC2
  • 一台 EC2 可掛多顆 EBS；透過儲存網路附加（非本地磁碟）
  • 可卸載和重新附加，不與實例生命週期綁定（持久性）
  • Snapshot 備份到 S3，可從快照建立 Volume（跨 AZ 遷移）
  • 按 GB/月 計費（某些情況還有效能費用）

EBS Volume 類型詳細規格

SSD 類型（支援開機磁碟）：

類型IOPS吞吐量容量特點

st1（吞吐優化）500500 MB/s40 MB/s/TB250 MB/s/TB125GB-16TB大數據、日誌、串流

gp2 Credit Bucket 機制：

  • IO Credit Bucket 容量 540 萬 IO Credits
  • 最小 100 IO Credits/秒（任何大小）；超過此限制，每 GB 3 IO Credits/秒（基準效能）
  • 可爆發到 3,000 IOPS（消耗 Credit Bucket）；Bucket 啟動時是滿的
  • 超過 1TB 時基準超過爆發，不使用 Credit 系統

gp3 優勢：

  • 移除 Credit Bucket 架構；基準 3,000 IOPS 和 125 MiB/s 吞吐
  • 額外付費可達 16,000 IOPS 或 1,000 MiB/s；最大吞吐是 gp2 的 4 倍

Nitro 架構

  • 使用 Nitro 架構可達到最高 64,000 EBS IOPS
  • 非 Nitro 架構僅支援 32,000 IOPS

EBS Snapshot

  • 指定時間點的備份，儲存在 S3；增量備份：首次完整，之後只儲存變更區塊
  • 可跨區建立 Volume；計費：按 GB/月（計費使用的資料，非配置的）
  • 新 EBS Volume 立即完整效能；快照還原是延遲載入（漸進取得，請求的區塊立即取得）
  • Fast Snapshot Restore（FSR）：立即還原，每區域最多 50 個快照

EBS Snapshots Archive：

  • 將不常存取的快照移至歸檔層；成本降低 75%；還原時間 24-72 小時

EBS 加密

  1. 儲存資料全程加密（使用 KMS 金鑰）
  2. 與 EC2 傳輸中的資料加密
  3. Snapshot 自動加密
  4. 從加密快照建立的 Volume 自動加密
  5. 從未加密快照建立 Volume 時可選擇加密

加密未加密的 EBS 步驟：

  1. 建立 Snapshot → 2. 複製並加密此快照 → 3. 從加密快照還原為新 Volume

EBS 操作

  • 線上修改：支援線上修改容量、IOPS、類型，不中斷運作
  • Multi-Attach（io1/io2）：可同時掛載至多個 EC2（同一 AZ），需要叢集感知的檔案系統

RAID 設定

RAID 類型說明用途
RAID 0條帶化（併聯）效能提升
RAID 1鏡像容錯備援
  • RAID 0 + EBS 最高可達 260,000 IOPS

Instance Store（暫時性本地儲存）

  • 與 EC2 實體主機綁定的區塊型暫存儲存裝置（Ephemeral）
  • AWS 最高儲存效能，包含在實例價格中
  • 必須在啟動時附加，無法之後新增
  • 停止或終止即清空資料；重開機資料不會消失；實例移動、調整大小或硬體故障時資料遺失

效能規格：

  • D3 實例：4.6 GB/s 吞吐
  • I3 實例：16 GB/s 連續吞吐
  • 比 EBS 更高 IOPS 和吞吐

EBS vs Instance Store 選擇

考量選 EBS選 Instance Store

最高吞吐4,000 MiB/s（Block Express）16 GB/s（I3）
成本按 GB/月含在實例價格中
