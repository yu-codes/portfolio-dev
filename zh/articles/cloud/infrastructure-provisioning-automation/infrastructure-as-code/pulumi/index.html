Pulumi

深入解析 Pulumi 的核心理念、與通用程式語言的整合，以及與其他 IaC 工具的差異

Pulumi 是一款現代化的基礎設施即程式碼平台，其最大特色是允許開發者使用熟悉的通用程式語言（TypeScript、Python、Go、C#、Java）來定義雲端基礎設施，而非學習專用的 DSL。這種設計理念讓開發團隊能運用既有的程式設計技能、IDE 支援、測試框架與套件管理生態來管理基礎設施。

應用場景

TypeScript 基礎設施定義範例

import * as aws from "@pulumi/aws";
import * as pulumi from "@pulumi/pulumi";

// 利用 TypeScript 的型別系統
interface ServiceConfig {
  name: string;
  cpu: number;
  memory: number;
  replicas: number;
}

// 可重用的 Component Resource
class MicroService extends pulumi.ComponentResource {
  public readonly url: pulumi.Output<string>;
  
  constructor(name: string, config: ServiceConfig) {
    super("custom:MicroService", name);
    // ... 資源定義
  }
}

// 使用迴圈建立多個服務
const services: ServiceConfig[] = [
  { name: "api", cpu: 256, memory: 512, replicas: 3 },
  { name: "worker", cpu: 512, memory: 1024, replicas: 2 },
];

services.forEach(svc => new MicroService(svc.name, svc));

Automation API 應用

Pulumi Automation API 允許將 IaC 嵌入到應用程式中，適合建構：

  • 自助式基礎設施平台
  • 多租戶 SaaS 環境佈建
  • 測試環境的動態建立與銷毀
  • CI/CD 管線的程式化整合

遷移策略

從 Terraform 遷移到 Pulumi 的方式：

  1. pulumi import：匯入既有資源到 Pulumi 狀態
  2. tf2pulumi：將 HCL 轉換為 Pulumi 程式碼
  3. 漸進式：新資源用 Pulumi，舊資源逐步遷移
  4. 並行運行：兩套工具管理不同資源集
