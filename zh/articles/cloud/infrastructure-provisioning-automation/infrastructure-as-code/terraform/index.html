Terraform 是由 HashiCorp 開發的開源基礎設施即程式碼（Infrastructure as Code）工具，使用宣告式語言 HCL（HashiCorp Configuration Language）定義雲端基礎設施。它支援多雲環境，透過 Provider 生態系統整合超過 3000 種服務，是目前最廣泛使用的 IaC 工具。

核心概念

基本元素

概念說明用途
Provider雲端服務提供者的 API 介面連接 AWS、GCP、Azure 等
Resource基礎設施的最小管理單位EC2、VPC、S3 等
Data Source查詢既有資源的資訊引用已存在的 VPC ID
Variable參數化配置環境差異、可重用性
Output匯出值供其他模組使用VPC ID、Endpoint URL
Module可重用的資源集合標準化的網路模組
State基礎設施的當前狀態記錄追蹤實際資源與配置的對應

工作流程

terraform init → terraform plan → terraform apply → terraform destroy
     ↓                ↓                ↓                  ↓
  初始化環境      產生執行計畫     套用變更          銷毀資源
  下載 Provider   差異比對        建立/修改/刪除    清除所有資源

安全最佳實踐

  1. 狀態加密：啟用 state file 的加密存儲
  2. 最小權限：Terraform 執行身份使用最小所需權限
  3. 敏感值保護：使用 sensitive = true 標記敏感輸出
  4. 策略即程式碼：使用 Sentinel/OPA 做 plan 前驗證
  5. 審計追蹤：所有 apply 操作記錄在 CI/CD 日誌中
  6. Drift 偵測：定期執行 plan 偵測手動變更

大規模 Terraform 管理

挑戰解決方案

秘密管理整合 Vault、避免寫入 state
