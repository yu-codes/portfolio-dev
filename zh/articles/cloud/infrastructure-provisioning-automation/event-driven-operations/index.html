事件驅動維運（Event-Driven Operations）是一種以事件為核心驅動力的維運模式，將基礎設施和應用的狀態變化轉化為事件，並透過自動化的事件處理管線進行回應。這種模式能大幅縮短事件回應時間，實現即時的自動化修復，是 AIOps 與自治運維的基礎架構。

核心概念

事件驅動維運的要素

要素說明範例
事件來源產生事件的系統CloudWatch、K8s Events
事件匯流排事件的傳輸通道EventBridge、Kafka
事件規則觸發條件定義CPU > 90%、Pod CrashLoop
事件處理器回應事件的邏輯Lambda、Step Functions
事件存儲事件的持久化S3、CloudWatch Logs

事件類型分類

類型說明優先級回應方式
告警事件指標超過閾值高自動修復/通知
變更事件基礎設施狀態變化中記錄/驗證
安全事件潛在安全威脅最高自動隔離/通知
合規事件偏離合規要求高自動修正/報告
資訊事件常規狀態報告低記錄/聚合

事件處理管線

事件產生 → 收集 → 過濾/去重 → 富化 → 關聯分析 
    → 規則匹配 → 動作觸發 → 執行 → 驗證 → 記錄

自動化回應模式比較

模式人工介入風險適用場景範例
全自動修復無中已知問題、低風險重啟服務、擴容
半自動（需審批）審批低中風險操作故障轉移
輔助決策建議極低複雜問題根因分析建議
資訊收集依賴無初始診斷自動收集診斷資訊
升級通知完全無未知問題告警 + 上下文通知

自動修復場景

場景偵測方式自動回應驗證
服務無回應Health Check 失敗重啟 Pod/容器Health Check 恢復
磁碟空間不足指標告警清理日誌/擴容空間恢復
憑證即將過期排程掃描自動更新憑證連線測試
安全群組異常Config Rule 變更自動回復設定合規檢查通過
節點不健康Node Condition遷移 Pod + 替換節點工作負載正常
DDoS 攻擊流量異常偵測啟用 WAF 規則流量正常化

安全事件自動回應

安全事件偵測（GuardDuty/SecurityHub）
    ↓
事件分類（嚴重程度、類型）
    ↓
├── 高風險：自動隔離受影響資源
│   ├── 移除 Security Group 規則
│   ├── 撤銷 IAM 權限
│   └── 觸發取證快照
├── 中風險：通知 + 資訊收集
│   ├── 收集相關日誌
│   └── 通知安全團隊
└── 低風險：記錄 + 週報彙整
