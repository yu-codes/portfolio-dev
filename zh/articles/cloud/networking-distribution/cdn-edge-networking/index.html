CDN 與邊緣網路

CDN（Content Delivery Network，內容傳遞網路）是一組地理分散的伺服器群，透過將內容快取到離使用者最近的邊緣節點，顯著降低延遲並提高載入速度。現代 CDN 已從純粹的靜態快取演進為可執行運算邏輯的邊緣平台。

核心概念

CDN 運作原理

步驟動作說明
1DNS 解析用戶請求被導向最近的邊緣節點
2快取查詢邊緣節點檢查是否有快取
3aCache Hit直接回應（延遲極低）
3bCache Miss向 Origin 回源取得內容
4快取儲存將內容存入邊緣快取
5回應用戶傳遞內容給使用者

關鍵指標

指標說明目標
Cache Hit Ratio快取命中率> 90%
TTFB首位元組時間< 100ms
Origin Shield Hit中間層命中> 80%
Bandwidth Savings頻寬節省> 70%
Error Rate錯誤率< 0.1%

快取策略

策略說明適用內容TTL 建議
Cache Everything全部快取靜態資源1 年
Cache with Revalidation快取但驗證半動態內容1 小時

Bypass Cache不快取動態 API0
Stale-While-Revalidate先回舊的再更新容忍暫時過期視需求

CDN 架構模式

模式一：靜態資源加速

User → CloudFront → S3（Origin）

模式二：全站加速（動靜分離）

User → CDN
  ├── /static/* → S3
  └── /api/* → ALB → Backend

模式三：邊緣運算

User → CDN Edge
  ├── 邊緣邏輯（URL 改寫、A/B 測試）
  └── 必要時 → Origin

應用場景

靜態網站全球部署

  • S3/GCS 作為 Origin
  • CDN 全球快取
  • 自訂域名 + HTTPS
  • Cache-Control: max-age=31536000（資源帶版本號）

SPA + API 架構

  • 靜態資源（HTML/JS/CSS）走 CDN
  • API 請求透過 CDN 動態路由到後端
  • CDN 處理 CORS、壓縮
  • 邊緣處理認證 Token 驗證

串流媒體

  • 影片分段快取（HLS/DASH）
  • 自適應位元率（ABR）
  • Origin Shield 降低回源壓力
  • 全球分發降低緩衝延遲
