私有連線技術讓企業能在公有雲與自有資料中心之間建立安全、可靠的網路通道。從 VPN 加密隧道到專線互連，不同方案在安全性、效能與成本之間提供了多種選擇。

應用場景

混合雲架構

  1. 主要路徑：Direct Connect 10 Gbps（雙線路）
  2. 備援路徑：Site-to-Site VPN（自動故障轉移）
  3. 路由：BGP 宣告地端與雲端路由
  4. 監控：CloudWatch 監控連線狀態

多地點辦公室連雲

  • 總部：Direct Connect（高頻寬）
  • 分公司：Site-to-Site VPN
  • 遠端使用者：Client VPN
  • 統一管理：Transit Gateway Hub

資料遷移

  • 大量歷史資料：Snowball/專線離線傳輸
  • 持續同步：Direct Connect + DataSync
  • 資料庫遷移：DMS over Direct Connect
  • 遷移後：保留 VPN 作為管理通道

AWS 服務實作

Amazon VPC（虛擬私有雲）概述

  • VPC：區域限定虛擬網路，可自訂 CIDR、支援 DNS，預設每區最多建立 5 個
  • 免費元件：VPC、Route Table、NACL、IGW、SG、Subnet、Peering
  • 需付費元件：NAT Gateway、VPC Endpoint、VPN Gateway、Customer Gateway
  • CIDR 最小 /28（16 IPs），最大 /16（65,536 IPs）

核心元件

Subnet（子網）

  • VPC 中的子網段，綁定一個 AZ
  • AWS 每個子網保留 5 個 IP：網路位址、VPC 路由器（base+1）、DNS（base+2）、保留（base+3）、廣播（最後位址）

Route Table（路由表）

  • VPC 有一個 Main Route Table（子網預設使用）
  • 一個子網同時只能關聯一個 Route Table
  • /n 數字越大 = 越精確 = 優先度越高

Internet Gateway (IGW)

  • 讓 Public Subnet 可連上網際網路，區域韌性
  • 一個 VPC 只能有一個 IGW，1 個 IGW = 0 或 1 個 VPC
  • EC2 的作業系統永遠不知道自己的公有 IPv4
  • 停止和啟動 EC2 = 公有 IP 變更

NAT Gateway（託管服務）

  • 讓 Private Subnet 中的實例可連網，外部不能主動連入
  • 運行在公有子網，使用彈性 IP（靜態 IPv4 公有）
  • 不支援 Security Groups，只能用 NACL
  • AZ 韌性，區域韌性需要每個 AZ 各建一個 NATGW
  • 托管服務，可擴展到 45 Gbps，不支援 IPv6，不可兼任 Bastion Host

Egress-Only Internet Gateway

  • IPv6 版僅出站閘道（類似 NAT 但用於 IPv6）
  • Internet Gateway 對 IPv6 允許所有進出；Egress-Only 只允許出站

Stateless vs Stateful 防火牆

NACL（Network ACL）—— Stateless

  • 每個子網都有關聯的 NACL，規則按編號順序處理，匹配後停止
  • * 是隱含拒絕，可明確 ALLOW 和 DENY，需為請求和回應各設規則
  • 每個子網只能有一個 NACL，一個 NACL 可關聯多個子網
  • VPC 建立時有預設 NACL（允許所有），自訂 NACL 預設只有隱含拒絕

Security Groups（SG）—— Stateful

  • 附加到 ENI（彈性網路介面），非實例本身
  • 自動偵測回應流量，允許的請求自動允許回應
  • 無明確拒絕，只有允許或隱含拒絕
  • 支援 IP/CIDR 和邏輯資源（包括其他 SG 和自身）
  • SG 自我參考：相同 SG 的資源可以互相通訊

VPC Peering

  • 兩個 VPC 間直接加密網路連結，支援同/跨區域與同/跨帳號
  • 不具轉送性：A-B 和 B-C peering 不代表 A-C 可以通訊
  • 需更新 Route Table，SG 和 NACL 可過濾流量
  • 同區域 SG 可參考對等 SG

Transit Gateway

  • Hub-and-Spoke 模式串接多個 VPC，可跨帳號跨區域
  • 支援 IP Multicast，簡化複雜的 VPC 互連架構，取代全網格式 VPC Peering

VPC Endpoints

類型架構支援服務費用
Gateway Endpoint路由層級，Prefix ListS3、DynamoDB免費
Interface EndpointENI，使用 PrivateLink大多數 AWS 服務付費

Gateway Endpoints 要點

  • 跨所有 AZ 高可用，區域性（無法跨區域存取）
  • 防止資料外洩：S3 可設定只允許從 Gateway Endpoint 存取

Interface Endpoints 要點

  • 加入到特定子網（ENI），非預設高可用，需每個 AZ 各建一個
  • 只支援 TCP 和 IPv4，透過 Security Groups 控制存取
  • 使用 PrivateLink，提供新的服務端點 DNS

VPC Flow Logs

  • 捕獲元資料（非封包內容），附加到 VPC/子網/ENI
  • Flow Logs 非即時，目的地：S3 或 CloudWatch Logs，可用 Athena 查詢

AWS Site-to-Site VPN 詳解

  • 本地網路與 VPC 間透過網際網路建立 VPN 隧道
  • 包含 Customer Gateway、Virtual Private Gateway、VPN Connection
  • 每條 VPN 包含 2 條隧道以支援 HA

AWS Direct Connect 詳解

  • 提供專線連 AWS，頻寬 1G ～ 100G
  • 可同時存取 AWS 公網與私網資源
  • 建立時間需數日，適合穩定、高頻寬場景

Amazon CloudFront（全球 CDN）

  • AWS CDN 服務，將內容快取至全球 400+ 個邊緣節點（Edge Location）
  • 支援 S3、ALB、EC2 作為 Origin，搭配 OAI/OAC 限制存取
  • 支援地理限制、簽章 URL/Cookie、TTL 設定與 Invalidate
  • 整合 AWS WAF 提供 Layer 7 安全性
  • ACM 憑證（HTTPS）必須在 us-east-1 申請

AWS Global Accelerator

  • 透過 AWS 全球骨幹網路選擇最佳路徑（非 CDN，不快取）
  • 提供兩個 Anycast 靜態 IP，防火牆白名單友善
  • 支援 TCP/UDP 協定，適合即時應用、遊戲
  • 支援端點健康檢查與容錯路由

AWS Transfer Family

  • 完全託管的檔案傳輸服務，支援 SFTP、FTPS、FTP 協定
  • 資料直接存入 S3 或 EFS
  • 與現有身份認證系統整合（AD、LDAP、自訂）
  • 支援 VPC 端點進行私有存取

AWS VPC 最佳實踐

  1. 規劃 CIDR 範圍：避免與其他網路重疊，預留擴展空間
  2. 使用 NAT Gateway：比 NAT Instance 更可靠（45 Gbps），每 AZ 各建一個
  3. 啟用 VPC Flow Logs：用於安全分析和疑難排解
  4. 多 AZ 部署：提高可用性
  5. 優先使用 Gateway Endpoint：S3/DynamoDB 免費且無需修改應用
  6. 搭配 NACL + SG：NACL 可明確拒絕，SG 提供精細控制
  7. VPN 作為 Direct Connect 備援：雙路徑確保高可用
