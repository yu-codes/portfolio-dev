域名系統（Domain Name System，DNS）是網際網路的「電話簿」，將人類可讀的域名轉換為機器可讀的 IP 位址。在雲端架構中，DNS 不僅負責域名解析，更承擔流量管理、服務發現與高可用性切換等關鍵角色。

AWS 服務實作

Amazon Route 53 概述

  • AWS 託管的 DNS 服務，可註冊網域、建立 Hosted Zone
  • 支援公開與私有域名（Private Hosted Zone）
  • 100% 可用性 SLA，全球韌性
  • DNS 位於 VPC Base IP +2 位址（enableDnsSupport/enableDnsHostnames）

R53 主要功能

  • Domain Registrar：網域註冊商，與 TLD 登記處通訊
  • Domain Hosting：網域託管（4 個受管理的 Name Servers）
  • 健康檢查：與記錄分開設定，可用於路由決策

Hosted Zone 類型

類型說明
Public Hosted Zone管理公開網域的 DNS 記錄（可從網際網路查詢）
Private Hosted Zone管理 VPC 內部的私有 DNS 記錄（僅 VPC 內部可查詢）

路由策略詳解

策略說明健康檢查使用場景
Simple單一記錄，隨機回傳多個值不支援基本路由
Weighted依權重分配流量（0 = 停止流量）支援A/B 測試、漸進式部署
Latency導向延遲最低的區域支援全球應用效能優化
Failover主動-被動故障轉移必要災難復原
Geolocation依地理位置路由（國家→大洲→預設）支援在地化內容、授權限制
Geoproximity依地理位置和偏移路由（可調整 + 或 - 偏移）支援進階地理路由
Multivalue Answer多值回應，移除不健康記錄（最多 8 個）支援簡單負載分散

各路由策略重點

  • Simple：不支援健康檢查，隨機回傳所有值；如有多個值客戶端隨機選擇
  • Failover：必須搭配健康檢查，實現 Active-Passive；主要健康回傳主要，失敗回傳次要
  • Weighted：適用於遷移、A/B 測試；如果某記錄健康檢查失敗，會跳過
  • Latency：R53 根據請求來源區域選擇最低延遲記錄
  • Geolocation：匹配順序：國家 → 大洲 → 預設；重疊時最精確優先
  • Geoproximity：正偏移擴大範圍，負偏移縮小範圍；可計算客戶端和記錄之間的距離
  • Multivalue：Simple 的改良版，改善可靠性；移除不健康記錄，最多 8 個

DNS Failover 策略

Active-Active Failover

  • 所有資源大部分時間都可用
  • 使用相同名稱、類型、路由策略的記錄
  • 資源不可用時，R53 偵測到不健康並停止回傳

Active-Passive Failover

  • 主要資源大部分時間可用，次要資源待命
  • 主要全部不健康時，R53 開始回傳次要
  • 使用 Failover 路由策略，必須搭配健康檢查

Route 53 健康檢查詳解

  • 健康檢查與記錄分開設定，但被記錄使用
  • 健康檢查器位於全球各地
  • 每 30 秒檢查（每 10 秒需額外付費）
  • 支援：TCP、HTTP/HTTPS、HTTP/HTTPS + 字串匹配
  • 狀態：Healthy 或 Unhealthy
健康檢查類型說明
Endpoint直接檢查端點健康狀態
Calculated基於其他健康檢查的計算結果（Checks of Checks）
CloudWatch Alarm基於 CloudWatch 警示狀態

健康檢查參數：Protocol（HTTP/HTTPS/TCP）、Interval（10 或 30 秒）、Failure Threshold（連續失敗次數）、String Matching（回應內容檢查）

Route 53 Resolver

Resolver 類型說明
Inbound Endpoint允許本地 DNS 查詢 AWS VPC 中的資源
Outbound Endpoint允許 VPC 中的資源查詢本地 DNS

適用於混合雲架構，實現地端與雲端 DNS 整合（Split-horizon DNS）。

DNSSEC（DNS 安全擴充）

  • 提供 DNS 回應的真實性和完整性驗證
  • 防止 DNS 欺騙（Spoofing）和快取中毒（Cache Poisoning）攻擊
  • 支援 Route 53 公共託管區域

Route 53 Interoperability（互操作性）

R53 可作為三種角色：

  1. R53 兩者皆是：R53 同時作為註冊商和託管服務
  2. R53 僅註冊商：R53 註冊，其他服務（如 Cloudflare）託管
  3. R53 僅託管：其他註冊商（如 GoDaddy），R53 託管

流程：R53 分配 4 個 Name Servers → 建立 Zone 檔案 → 設定網域的 NS 記錄指向 4 個 NS

TTL（Time To Live）

  • 查詢結果在 Resolver 快取的時間（秒）
  • 授權結果：直接從授權伺服器取得
  • 非授權結果：從 Resolver 快取取得

Route 53 最佳實踐

  1. 使用 Alias 記錄：指向 AWS 資源時免費且支援根域名，應為預設選擇
  2. 設定健康檢查：確保流量導向健康的端點
  3. 實施故障轉移：使用 Failover 路由策略搭配健康檢查
  4. 考慮延遲路由：全球用戶使用 Latency 路由改善效能
  5. 使用 Private Hosted Zone：VPC 內部 DNS 解析
  6. 啟用 DNSSEC：加強 DNS 安全性，防止欺騙攻擊
  7. Resolver Endpoints：混合雲 DNS 整合（地端 ↔ 雲端）
  8. Traffic Flow：視覺化路由策略編輯器，適合複雜路由邏輯
