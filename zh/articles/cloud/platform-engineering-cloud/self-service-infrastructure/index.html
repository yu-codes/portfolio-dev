自助式基礎設施（Self-Service Infrastructure）是平台工程的核心交付物，讓開發團隊能夠按需申請、配置與管理基礎設施資源，無需等待中央 IT 團隊的手動介入。這種模式大幅縮短了資源交付時間，同時透過預定義的策略確保安全性與合規性。

核心概念

自助式基礎設施的本質

自助式基礎設施不是「無管理」的自由，而是在策略約束下的自主權。平台團隊定義規則與邊界，開發團隊在這些邊界內自由操作。

層面平台團隊職責開發團隊權限
資源類型定義可用資源目錄從目錄中選擇
規格配置定義允許的規格範圍在範圍內選擇
安全策略設定安全基線自動套用
成本控制定義預算與限額在額度內使用
網路配置定義網路拓撲選擇可用區域

抽象層次

自助服務的抽象層次決定了開發者的認知負擔與靈活性之間的平衡：

抽象等級開發者看到的範例
低（IaaS）VM、VPC、Security GroupTerraform Module
中（PaaS）應用運行環境Heroku-like 介面
高（功能）業務能力“我需要一個資料庫”

請求處理流程

申請請求 → 身份驗證 → 授權檢查 → 策略評估 → 配額確認 
    → 成本估算 → [審批(選配)] → 資源建立 → 配置完成 → 通知開發者

應用場景

資料庫自助服務

開發者申請資料庫的自助流程：

步驟操作自動化內容
1選擇資料庫類型提供 PostgreSQL、MySQL、MongoDB 選項
2選擇規格預定義的 T-shirt sizing（S/M/L/XL）
3選擇環境Dev/Staging/Prod
4確認顯示成本估算、合規檢查結果
5建立自動建立實例、設定備份、配置監控
6交付連線資訊自動注入 Secret Manager

環境即服務

# 環境申請範例
apiVersion: platform.example.com/v1
kind: Environment
metadata:
  name: feature-xyz
  labels:
    team: order-service
    ttl: 7d
spec:
  template: full-stack
  components:
    - type: kubernetes-namespace
    - type: postgresql
      size: small
    - type: redis
      size: small
    - type: s3-bucket
  networking:
    ingress: true
    domain: feature-xyz.dev.example.com

治理與成本控制

治理面向實施方式範例
成本控制預算配額、TTL開發環境自動回收
安全基線預設策略、強制加密所有 DB 必須 TLS
資源標籤強制標籤策略必須包含 owner、cost-center
合規自動合規掃描CIS Benchmark 檢查
最大規格規格上限開發環境最大 4 vCPU

漸進式實施策略

  1. 第一步：將最常見的請求（如建立 K8s namespace）自助化
  2. 第二步：擴展到資料庫、快取等有狀態服務
  3. 第三步：加入環境即服務的能力
  4. 第四步：提供完整的應用堆疊自助建立
  5. 第五步：引入智慧化推薦與自動優化
