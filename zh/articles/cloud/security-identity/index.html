雲端安全與身份管理（Cloud Security & Identity）是確保雲端環境中資料、服務與基礎設施安全的綜合性實踐。在傳統資料中心中，安全邊界以實體網路為界；而在雲端環境中，身份（Identity）成為新的安全邊界。本文將全面介紹雲端安全的核心領域，包括身份存取管理、網路安全、密鑰管理與應用安全。

核心概念

零信任架構（Zero Trust Architecture） 的核心原則是「永不信任，始終驗證」。無論請求來自網路內部或外部，每次存取都必須經過身份驗證、授權檢查與加密傳輸。零信任消除了傳統的「城堡護城河」模型中對內部網路的隱性信任。

身份存取管理（IAM） 是控制「誰可以對什麼資源執行什麼操作」的框架。IAM 包含身份管理（身份的建立、維護與生命週期）、驗證（確認身份的真實性）、授權（確定身份的權限範圍）與稽核（記錄所有存取活動）。

最小權限原則（Principle of Least Privilege） 要求每個身份（人員、服務、應用程式）只擁有完成其工作所需的最少權限，且權限應有時間限制。JIT（Just-In-Time）存取是實現方式之一：權限在需要時才被授予，使用後自動撤銷。

深度防禦（Defense in Depth） 透過多層安全控制提供縱深防護：網路層（防火牆、WAF）、身份層（IAM、MFA）、應用層（輸入驗證、加密）、資料層（加密、存取控制）、監控層（日誌、威脅偵測）。任何單一層被突破都不會導致完全淪陷。

密鑰管理（Secrets Management） 安全地儲存與分發敏感資訊（API 金鑰、密碼、憑證），確保密鑰不會出現在程式碼庫、日誌或環境變數中。HashiCorp Vault、AWS Secrets Manager 與 Azure Key Vault 是常見方案，支援自動輪替與存取稽核。

共享責任模型（Shared Responsibility Model） 明確定義雲端供應商與客戶各自的安全職責。供應商負責「雲的安全」（實體基礎設施、虛擬化層）；客戶負責「雲中的安全」（作業系統、應用程式、資料、存取管理）。

架構總覽

雲端安全架構的分層設計：

身份層 — 集中式身份提供者（IdP）如 Okta、Azure AD 管理所有人員與服務身份。透過 SAML/OIDC 實現單一登入（SSO），聯邦身份（Federation）讓組織使用現有目錄服務存取雲端資源。服務對服務通訊使用短期憑證（如 IAM Role、Workload Identity）。

網路層 — VPC/VNet 提供網路隔離，安全群組與網路 ACL 控制流量。Private Link/Service Endpoint 讓流量不離開雲端骨幹網路。Transit Gateway 或 Hub-and-Spoke 拓撲管理多 VPC 通訊。WAF 與 DDoS 防護保護面向網際網路的端點。

資料層 — 靜態加密（Encryption at Rest）使用 KMS 管理的金鑰；傳輸加密（Encryption in Transit）強制使用 TLS 1.2+。資料分類（Data Classification）根據敏感度分級並套用對應的保護措施。資料遺失防護（DLP）偵測與阻止敏感資料外洩。

應用層 — Container 安全（映像掃描、Runtime Protection）、API 安全（OAuth 2.0、速率限制）、程式碼安全（SAST/DAST、依賴漏洞掃描）。OWASP Top 10 作為基本防護基線。

偵測與回應層 — SIEM 整合雲端日誌進行威脅偵測，SOAR 自動化安全回應。Cloud Security Posture Management（CSPM）持續掃描配置偏差。

應用場景

零信任網路轉型 — 企業從 VPN 存取模式轉向零信任。導入 BeyondCorp 模型：每個請求都經過身份驗證（裝置憑證 + 使用者身份）、動態授權（根據裝置健康狀態、使用者風險分數）與加密傳輸。取消全域 VPN，改用應用程式級別的存取代理。

多雲 IAM 統一 — 組織同時使用 AWS、Azure 與 GCP。透過集中式 IdP（Okta）實現統一身份，各雲端帳戶透過 OIDC Federation 信任此 IdP。定義統一的角色命名規範與權限邊界，透過 IaC 同步管理各雲端的 IAM 策略。

密鑰零洩漏策略 — 導入 Vault 作為集中式密鑰管理。所有應用程式透過 Vault Agent 自動注入密鑰，不再使用環境變數。密鑰自動輪替、存取日誌全面記錄。程式碼掃描工具阻止含有密鑰的 Commit 被推送。

合規自動化 — 使用 CSPM 工具持續掃描雲端環境配置，自動對照 CIS Benchmark、SOC 2 與 PCI-DSS 要求。違規項目自動產生修復工單或直接透過 IaC 修正。合規報告自動產生，減少人工稽核準備時間 80%。

AWS 服務實作

AWS Shield（DDoS 防護）

版本說明保護範圍
Shield Standard免費CloudFront / Route 53
Shield Advanced付費EC2、ALB、CloudFront、Route 53

Shield Advanced 額外功能

  • 24/7 DDoS 回應團隊（DRT）支援
  • 成本保護（DDoS 攻擊期間的額外費用補償）
  • 進階報告和診斷

AWS WAF（Web 應用防火牆）

  • 保護 Layer 7（HTTP）流量
  • 可阻擋 SQL Injection / XSS 等常見攻擊
  • 可部署在 CloudFront、ALB、API Gateway、AppSync
類型說明
IP 規則允許或阻擋特定 IP
地理位置規則依國家/地區過濾
速率限制規則限制請求頻率
AWS 管理規則預設安全規則集
自訂規則自定義過濾條件

AWS Firewall Manager

  • 集中式防火牆管理服務，跨帳號設定 WAF / Shield Advanced / Route 53 DNS 防火牆
  • 適用於合規需求（如封鎖特定國家 IP）
  • 需要 AWS Organizations 支援

AWS GuardDuty

  • 威視偵測服務，基於機器學習與異常行為判斷
  • 資料來源：VPC Flow Logs / DNS Logs / CloudTrail
  • 可偵測加密貨幣無幣搏活動等攻擊
偵測類型說明
未授權存取異常登入與 API 呼叫
加密貨幣挖矿計算資源滚用
惡意 IP 通訊與已知惡意來源通訊
資料外洩大量資料從 S3 流出
帳號洩露異常身份使用模式

Amazon Inspector

  • EC2 安全性評估服務（需要安裝 agent）
  • 覦蓋項目：Network Reachability（檢查是否可被不預期存取）、Host Assessment（檢查漏洞與錯誤設定）
類型說明
網路可達性檢查網路設定安全性
主機評估檢查 OS 漏洞和設定
CVE 評估對應已知漏洞資料庫
CIS 基準對應安全設定最佳實踐

Amazon Macie

  • 偵測與保護 S3 中的 敏感資料（如 PII） 的託管服務
  • 可主動識別資料類型並發出警報
可偵測的敏感資料類型說明
個人識別資訊（PII）姓名、電話、證件號碼
金融資訊信用卡號、銀行帳號
健康資訊（PHI）病歷、蠍隆診斷
認證資訊API 金鑰、密碼

AWS Config

  • 多區帳號的 資源設定監控與稽核服務
  • 可整合 SNS 通知變更事件
  • 結合 CloudTrail 提供設定歷史紀錄
  • 適用於法規遵循需求（如 PCI-DSS、HIPAA）

Config Rules：

  • 評估資源設定是否符合規範
  • 支援 AWS 管理規則和自訂規則
  • 可自動修復不合規資源

AWS Security Hub

  • 集中式安全態勢管理服務，彙整來自多個安全服務的發現
  • 跨帳號和跨區域檢視
合規標準說明

Security Hub 主要功能：

  • 自動化發現關聯：彙整來自 GuardDuty、Inspector、Macie、IAM Access Analyzer 的發現
  • 安全分數：量化安全態勢的合規狀態
  • 自訂洞察：建立自訂查詢識別特定安全問題
  • 自動化修復：整傐 Systems Manager Automation 和 Lambda

AWS IAM Access Analyzer

  • 識別與外部實體共享的資源
  • 分析 IAM 政策以發現過度權限
  • 支援 S3 存儲桶、IAM 角色、KMS 金鑰等資源
  • 跨帳號分析支援 AWS Organizations

AWS 安全最佳實踐

  1. 啟用 GuardDuty：持續監控威視
  2. 使用 Config Rules：確保資源合規
  3. 定期執行 Inspector：掃描漏洞
  4. 保護敏感資料：使用 Macie 掃描 S3
  5. 分層防護：同時使用 Shield 和 WAF
  6. 使用 Security Hub：集中管理安全發現
  7. 啟用 IAM Access Analyzer：識別意外的存取權限
