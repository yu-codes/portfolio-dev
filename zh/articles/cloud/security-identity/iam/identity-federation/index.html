身份聯邦

身份聯邦（Identity Federation）允許使用者透過外部身份提供者（IdP）的認證來存取雲端資源，無需在每個雲端平台維護獨立的帳號。這種機制實現了集中式身份管理、單一登入（SSO），並大幅簡化了多雲環境中的存取管理。

核心概念

身份聯邦的關鍵角色

角色說明範例
身份提供者（IdP）驗證使用者身份的系統Okta、Azure AD、Google Workspace
服務提供者（SP）提供資源存取的系統AWS、GCP、Azure 服務
信賴關係（Trust）IdP 與 SP 之間的信任配置SAML Metadata、OIDC Discovery
斷言/令牌（Token）身份驗證的證明SAML Assertion、JWT
屬性映射IdP 屬性對應到 SP 權限群組 → IAM 角色

聯邦類型

類型說明適用場景
企業聯邦企業 IdP 登入雲端員工存取雲端控制台
工作負載聯邦服務間的身份互信K8s Pod 存取雲端 API
社交聯邦社交帳號登入消費者應用
B2B 聯邦合作夥伴組織間互信跨組織協作

架構總覽

SAML 聯邦流程

使用者 → 存取雲端控制台 → 重新導向到 IdP
    → IdP 驗證身份（用戶名/密碼 + MFA）
    → IdP 產生 SAML Assertion
    → 重新導向回 SP（雲端）
    → SP 驗證 Assertion
    → 授予臨時安全憑證
    → 存取雲端資源

屬性型存取控制（ABAC）與聯邦

透過 IdP 傳遞的屬性（部門、職位、專案）自動對應到雲端資源的存取權限：

IdP 屬性雲端映射存取範圍
department=engineering開發環境角色Dev/Staging 資源
role=dba資料庫管理角色RDS/DynamoDB
project=alpha專案標籤匹配Alpha 標籤資源
level=senior生產環境存取生產帳號角色

安全最佳實踐

  1. 強制 MFA：聯邦登入時要求多因素認證
  2. 最短會話：設定最短有效的會話時間
  3. 監控異常：偵測異常的聯邦登入模式
  4. 信賴最小化：僅信賴必要的 IdP
  5. 權限映射審查：定期審查群組到角色的映射
  6. 撤銷機制：確保能即時撤銷聯邦存取
