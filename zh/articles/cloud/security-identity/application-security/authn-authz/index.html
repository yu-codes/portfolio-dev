認證與授權

深入解析認證（Authentication）與授權（Authorization）的核心差異、協議標準與雲端實踐

認證（Authentication, AuthN）與授權（Authorization, AuthZ）是應用程式安全的兩大支柱。認證回答「你是誰？」，授權回答「你能做什麼？」。在雲端微服務架構中，正確實現認證與授權機制對於保護資料與服務至關重要。

核心概念

認證 vs 授權

面向認證（AuthN）授權（AuthZ）
核心問題你是誰？你能做什麼？
驗證對象身份（Identity）權限（Permission）
執行時機先於授權認證之後
失敗結果401 Unauthorized403 Forbidden
相關標準OIDC、SAML、WebAuthnOAuth 2.0、RBAC、ABAC
實現方式密碼、生物辨識、MFA角色、政策、屬性

認證因素

因素類型範例安全等級
知識因素你知道什麼密碼、PIN低
持有因素你擁有什麼手機、安全金鑰中
生物因素你是什麼指紋、臉部高
位置因素你在哪裡IP、GPS輔助
行為因素你怎麼做打字模式輔助

Token 架構

ID Token (OIDC)           Access Token (OAuth 2.0)
├── 使用者身份資訊          ├── 授權範圍（Scope）
├── 認證時間               ├── 有效期
├── 認證方式               ├── 發行者
└── 用途：前端身份確認       └── 用途：API 存取授權

Refresh Token
├── 長期有效
├── 用於更新 Access Token
└── 安全存儲要求最高

無密碼總是WebAuthn體驗好

安全最佳實踐

  1. 永不自建認證：使用經過驗證的認證服務
  2. 強制 HTTPS：所有認證流量必須加密
  3. 短期 Token：Access Token 有效期 5-60 分鐘
  4. 安全存儲：前端使用 HttpOnly Secure Cookie
  5. 集中授權：使用統一的授權引擎
  6. 審計記錄：記錄所有認證與授權事件
  7. 多因素認證：高敏感操作強制 MFA
  8. 最小範圍：Token Scope 最小化
