渲染管線優化

渲染管線優化是確保網頁在各種裝置上達到穩定 60fps（每幀 16.67ms 預算）的系統性方法。從 JavaScript 執行到最終像素顯示，每個環節都存在潛在的效能瓶頸。本文涵蓋完整的渲染管線優化策略，幫助開發者打造絲滑流暢的使用者體驗。

核心概念

幀預算（Frame Budget）

瀏覽器以 60fps 的速率更新畫面，意味著每一幀的處理時間僅有約 16.67ms。扣除瀏覽器的內部開銷，留給開發者的有效時間約為 10-12ms。

每幀的處理流程：

Input → JavaScript → Style → Layout → Paint → Composite
         (JS)        (計算)   (佈局)   (繪製)   (合成)

若任何階段超出時間預算，該幀將被丟棄（掉幀），使用者將感受到明顯的卡頓。

像素管線（Pixel Pipeline）

完整的像素管線包含五個階段，但並非每次更新都需經歷全部階段：

  1. JS / CSS Animations / Web Animations API：觸發視覺變化的來源
  2. Style：計算哪些 CSS 規則套用到哪些元素
  3. Layout：計算元素的幾何資訊
  4. Paint：繪製像素到圖層
  5. Composite：合成圖層到螢幕

三種渲染路徑

  • 完整路徑：JS → Style → Layout → Paint → Composite（修改幾何屬性）
  • 跳過 Layout：JS → Style → Paint → Composite（修改外觀屬性如 color）
  • 僅 Composite：JS → Style → Composite（修改 transform / opacity）

requestAnimationFrame (rAF)

requestAnimationFrame 是瀏覽器提供的幀同步 API，確保 JavaScript 在正確的時機執行：每幀的開始處。這避免了 setTimeout / setInterval 可能錯過幀或在幀中間執行的問題。

function update() {
  // 在幀開始時執行動畫邏輯
  element.style.transform = `translateX(${x}px)`;
  requestAnimationFrame(update);
}
requestAnimationFrame(update);

├── Virtual DOM / Incremental DOM
├── 時間切片 (Time Slicing)
└── Web Worker 離線處理

Level 3: 合成層優化
├── will-change 提升
├── contain 屬性
└── content-visibility

Level 2: 繪製層優化
├── 減少繪製區域
├── 圖層管理
└── 簡化繪製複雜度

Level 1: 佈局層優化
├── 避免強制同步佈局
├── 讀寫分離
└── DOM 操作批次化

應用場景

場景一：長列表虛擬捲動

大量 DOM 元素是渲染效能的天敵。虛擬捲動僅渲染可視區域的元素：

class VirtualScroller {
  constructor(container, items, itemHeight) {
    this.container = container;
    this.items = items;
    this.itemHeight = itemHeight;
    this.visibleCount = Math.ceil(container.clientHeight / itemHeight) + 2;
    
    container.addEventListener('scroll', () => {
      requestAnimationFrame(() => this.render());
    });
  }

  render() {
    const scrollTop = this.container.scrollTop;
    const startIdx = Math.floor(scrollTop / this.itemHeight);
    const endIdx = startIdx + this.visibleCount;
    
    // 僅更新可視範圍內的 DOM
    this.updateVisibleItems(startIdx, endIdx);
    
    // 使用 transform 定位，避免觸發佈局
    this.content.style.transform = `translateY(${startIdx * this.itemHeight}px)`;
  }
}

場景二：使用 CSS contain 優化卡片列表

/* 告訴瀏覽器每張卡片是獨立的佈局與繪製單元 */
.card {
  contain: layout style paint;
  /* 卡片內部的變更不會影響外部佈局 */
}

/* 離屏內容延遲渲染 */
.card:not(.visible) {
  content-visibility: auto;
  contain-intrinsic-size: 0 300px;
}

場景三：高效能動畫系統

// 使用 Web Animations API 實現高效能動畫
function createSmoothAnimation(element) {
  const animation = element.animate([
    { transform: 'translateX(0)', opacity: 1 },
    { transform: 'translateX(100px)', opacity: 0.5 },
    { transform: 'translateX(0)', opacity: 1 }
  ], {
    duration: 2000,
    iterations: Infinity,
    easing: 'cubic-bezier(0.4, 0, 0.2, 1)',
    composite: 'replace' // 避免合成開銷累積
  });

  // 使用 IntersectionObserver 自動暫停離屏動畫
  const observer = new IntersectionObserver(([entry]) => {
    if (entry.isIntersecting) {
      animation.play();
    } else {
      animation.pause();
    }
  });
  observer.observe(element);
  
  return animation;
}
