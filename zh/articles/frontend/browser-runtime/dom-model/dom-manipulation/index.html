DOM（Document Object Model）是瀏覽器將 HTML 文件解析後產生的樹狀結構表示，提供程式化的介面讓 JavaScript 能夠動態讀取與修改頁面內容、結構與樣式。掌握 DOM 操作是前端開發的基礎技能，也是理解現代框架底層運作原理的關鍵。

核心概念

DOM 樹狀結構

瀏覽器將 HTML 解析為節點（Node）組成的樹狀結構：

document
 └── html (Element)
      ├── head (Element)
      │    ├── title (Element)
      │    │    └── "頁面標題" (Text)
      │    └── meta (Element)
      └── body (Element)
           ├── h1 (Element)
           │    └── "Hello" (Text)
           └── p (Element)
                └── "World" (Text)

節點類型

節點類型nodeType說明範例

元素選取方法

現代 DOM 操作主要使用以下選取方法：

// CSS 選擇器（推薦）
document.querySelector('.card');          // 第一個匹配
document.querySelectorAll('.card');       // 所有匹配（NodeList）

// 傳統方法
document.getElementById('app');           // 單一元素
document.getElementsByClassName('card');  // HTMLCollection（即時更新）
document.getElementsByTagName('div');     // HTMLCollection（即時更新）

// 相對選取
element.closest('.parent');              // 向上尋找最近匹配
element.matches('.active');              // 檢查是否匹配選擇器

NodeList vs HTMLCollection

// NodeList（靜態快照）- querySelectorAll 回傳
const nodeList = document.querySelectorAll('.item');
nodeList.forEach(item => { /* ... */ });  // 支援 forEach

// HTMLCollection（即時集合）- getElementsBy* 回傳
const collection = document.getElementsByClassName('item');
// 不支援 forEach，需轉換
Array.from(collection).forEach(item => { /* ... */ });

重繪與重排

DOM 操作會觸發瀏覽器的渲染更新：

  • 重排（Reflow/Layout）：元素幾何屬性改變，需重新計算佈局

    • 觸發：寬高、位置、顯示、字型大小改變
    • 成本：高（影響子元素與後續元素）

  • 重繪（Repaint）：僅外觀改變，不影響佈局

    • 觸發：顏色、背景、陰影、可見度改變
    • 成本：中等

  • 合成（Composite）：僅合成層變動

    • 觸發：transform、opacity
    • 成本：低（GPU 加速）

應用場景

場景一：高效批次 DOM 更新

使用 DocumentFragment 避免多次 reflow：

// 不佳：每次迴圈觸發 reflow
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  list.appendChild(li);  // 觸發 1000 次 reflow
}

// 優化：使用 DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);  // 不觸發 reflow
}
list.appendChild(fragment);  // 僅觸發 1 次 reflow

場景二：安全的動態內容插入

// 危險：XSS 攻擊向量
element.innerHTML = userInput;  // 永遠不要這樣做！

// 安全：使用 textContent
element.textContent = userInput;

// 需要結構時：使用 DOM API 建立
function createCard(data) {
  const card = document.createElement('div');
  card.className = 'card';

  const title = document.createElement('h3');
  title.textContent = data.title;  // 安全

  const content = document.createElement('p');
  content.textContent = data.content;  // 安全

  card.append(title, content);
  return card;
}

場景三：讀寫分離優化

避免強制同步佈局（Forced Synchronous Layout）：

// 不佳：讀寫交錯觸發多次 reflow
items.forEach(item => {
  const height = item.offsetHeight;  // 讀（觸發 layout）
  item.style.height = height * 2 + 'px';  // 寫
});

// 優化：先讀後寫
const heights = items.map(item => item.offsetHeight);  // 批次讀
items.forEach((item, i) => {
  item.style.height = heights[i] * 2 + 'px';  // 批次寫
});

場景四：MutationObserver 監聽 DOM 變化

const observer = new MutationObserver((mutations) => {
  mutations.forEach(mutation => {
    if (mutation.type === 'childList') {
      console.log('子節點變化:', mutation.addedNodes, mutation.removedNodes);
    }
    if (mutation.type === 'attributes') {
      console.log('屬性變化:', mutation.attributeName);
    }
  });
});

observer.observe(targetNode, {
  childList: true,
  attributes: true,
  subtree: true
});

// 停止觀察
observer.disconnect();

場景五：虛擬滾動概念

處理大量資料列表時，僅渲染可見區域的 DOM：

function renderVisibleItems(scrollTop, containerHeight, itemHeight, items) {
  const startIndex = Math.floor(scrollTop / itemHeight);
  const endIndex = Math.min(
    startIndex + Math.ceil(containerHeight / itemHeight) + 1,
    items.length
  );

  const fragment = document.createDocumentFragment();
  for (let i = startIndex; i < endIndex; i++) {
    const el = document.createElement('div');
    el.style.transform = `translateY(${i * itemHeight}px)`;
    el.textContent = items[i];
    fragment.appendChild(el);
  }

  container.innerHTML = '';
  container.appendChild(fragment);
}
