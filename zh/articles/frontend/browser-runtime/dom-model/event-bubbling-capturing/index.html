事件冒泡與捕獲

瀏覽器的事件傳播機制分為捕獲（Capturing）與冒泡（Bubbling）兩個階段。理解事件如何在 DOM 樹中流動，是實作事件委派、阻止事件傳播、處理複雜互動邏輯的基礎。本文深入解析事件傳播的三個階段、相關 API 與實務應用模式。

核心概念

事件傳播三階段

當 DOM 上的事件被觸發時，事件會經歷三個傳播階段：

  1. 捕獲階段（Capture Phase）：事件從 window → document → html → body → … 向下傳播到目標元素
  2. 目標階段（Target Phase）：事件到達觸發事件的元素本身
  3. 冒泡階段（Bubble Phase）：事件從目標元素向上冒泡回 window
         window  ←─── 冒泡 ───┐
           │                   │
        document               │
           │                   │
         <html>                │
           │                   │
         <body>                │
           │                   │
         <div>                 │
           │                   │
  捕獲 ──►<button> ─── 目標 ──►┘

addEventListener 的第三參數

// 預設：冒泡階段監聽（useCapture = false）
element.addEventListener('click', handler);
element.addEventListener('click', handler, false);

// 捕獲階段監聽
element.addEventListener('click', handler, true);

// 使用 options 物件（推薦）
element.addEventListener('click', handler, {
  capture: false,    // 是否在捕獲階段觸發
  once: true,        // 只觸發一次後自動移除
  passive: true,     // 不會呼叫 preventDefault()
  signal: controller.signal  // 用 AbortController 移除
});

事件物件關鍵屬性

element.addEventListener('click', (event) => {
  // 目標相關
  event.target;         // 實際觸發事件的元素（事件源）
  event.currentTarget;  // 綁定事件監聽器的元素（等同 this）

  // 傳播控制
  event.stopPropagation();         // 停止事件繼續傳播
  event.stopImmediatePropagation(); // 停止傳播 + 阻止同元素其他監聽器
  event.preventDefault();           // 阻止預設行為

  // 階段資訊
  event.eventPhase;    // 1:捕獲, 2:目標, 3:冒泡
  event.bubbles;       // 是否會冒泡
  event.cancelable;    // 是否可取消預設行為
  event.composed;      // 是否能穿越 Shadow DOM
});

關鍵比較

stopPropagation vs stopImmediatePropagation

方法阻止傳播阻止同元素其他監聽器使用時機
stopPropagation是否阻止事件到達父元素
stopImmediatePropagation是是完全阻止後續所有處理
// stopPropagation：同元素的其他 handler 仍會觸發
button.addEventListener('click', (e) => {
  e.stopPropagation();
  console.log('handler 1');  // 觸發
});
button.addEventListener('click', (e) => {
  console.log('handler 2');  // 仍觸發！
});

// stopImmediatePropagation：後續 handler 不觸發
button.addEventListener('click', (e) => {
  e.stopImmediatePropagation();
  console.log('handler 1');  // 觸發
});
button.addEventListener('click', (e) => {
  console.log('handler 2');  // 不觸發
});

preventDefault vs stopPropagation vs return false

方法阻止預設行為阻止傳播適用場景
preventDefault()是否阻止連結跳轉、表單提交
stopPropagation()否是防止事件向上冒泡
return false (jQuery)是是jQuery 中等同兩者
return false (原生)僅 on* 屬性否不建議使用

捕獲 vs 冒泡使用場景

階段常見用途範例
捕獲全局攔截、優先處理表單驗證攔截、無障礙焦點管理
冒泡事件委派、通用處理列表點擊委派、全局錯誤處理

應用場景

場景一：理解事件傳播順序

const parent = document.querySelector('#parent');
const child = document.querySelector('#child');

// 捕獲階段
parent.addEventListener('click', () => {
  console.log('1. Parent 捕獲');
}, true);

child.addEventListener('click', () => {
  console.log('2. Child 捕獲');
}, true);

// 冒泡階段
child.addEventListener('click', () => {
  console.log('3. Child 冒泡');
}, false);

parent.addEventListener('click', () => {
  console.log('4. Parent 冒泡');
}, false);

// 點擊 child 的輸出順序：
// 1. Parent 捕獲
// 2. Child 捕獲
// 3. Child 冒泡
// 4. Parent 冒泡

場景二：阻止事件傳播

// 場景：下拉選單點擊不關閉
document.addEventListener('click', () => {
  closeAllDropdowns();
});

dropdown.addEventListener('click', (e) => {
  e.stopPropagation();  // 阻止冒泡到 document
});

場景三：使用捕獲階段做全局攔截

// 在捕獲階段攔截所有 focus 事件
document.addEventListener('focus', (e) => {
  // 全局焦點追蹤
  analytics.track('focus', {
    element: e.target.tagName,
    id: e.target.id
  });
}, true);  // 必須使用捕獲，因為 focus 不冒泡

場景四：passive 事件監聽器提升滾動效能

// 告知瀏覽器不會呼叫 preventDefault
// 瀏覽器可立即執行滾動，不用等待 handler 完成
document.addEventListener('touchstart', handler, { passive: true });
document.addEventListener('wheel', handler, { passive: true });

// 若確實需要阻止預設行為，不可使用 passive
document.addEventListener('touchmove', (e) => {
  if (shouldPreventScroll) {
    e.preventDefault();  // passive: true 時此行無效
  }
}, { passive: false });

場景五：使用 AbortController 管理事件監聽器

const controller = new AbortController();

// 綁定多個事件，統一管理
element.addEventListener('click', handleClick, { signal: controller.signal });
element.addEventListener('keydown', handleKey, { signal: controller.signal });
element.addEventListener('mouseover', handleHover, { signal: controller.signal });

// 一次移除所有事件監聽器
controller.abort();
