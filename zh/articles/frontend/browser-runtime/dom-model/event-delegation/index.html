事件委派（Event Delegation）是利用事件冒泡機制，將子元素的事件監聽器統一綁定在父元素上的設計模式。這種模式不僅能大幅減少記憶體使用，還能自動處理動態新增的元素，是前端開發中最重要的效能優化技巧之一。

核心概念

事件委派原理

事件委派的運作基於兩個核心機制：

  1. 事件冒泡：子元素的事件會向上冒泡到父元素
  2. event.target：可以精確識別實際觸發事件的元素
// 不使用事件委派：為每個按鈕綁定監聽器
document.querySelectorAll('.btn').forEach(btn => {
  btn.addEventListener('click', handleClick);  // N 個監聽器
});

// 使用事件委派：只在父元素綁定一個監聽器
document.querySelector('.btn-container').addEventListener('click', (e) => {
  if (e.target.matches('.btn')) {
    handleClick(e);  // 1 個監聽器處理所有按鈕
  }
});

為何需要事件委派

考慮一個有 1000 個項目的列表：

  • 無委派：1000 個事件監聽器 → 高記憶體消耗、綁定耗時
  • 有委派：1 個事件監聯器 → 低記憶體消耗、即時生效

更重要的是，動態新增的元素不需要額外綁定事件：

// 問題：後來新增的 .item 元素沒有事件監聽器
list.innerHTML += '<li class="item">New Item</li>';

// 事件委派：自動涵蓋新增的元素
list.addEventListener('click', (e) => {
  if (e.target.matches('.item')) {
    // 所有 .item（包括未來新增的）都會被處理
  }
});

關鍵方法：matches() 與 closest()

// matches()：檢查元素是否匹配選擇器
e.target.matches('.btn');
e.target.matches('[data-action]');
e.target.matches('.card .title');

// closest()：向上尋找最近的匹配祖先（含自身）
// 當點擊的是子元素（如 icon）而非目標元素本身時特別有用
const card = e.target.closest('.card');
if (card) {
  const id = card.dataset.id;
  handleCardClick(id);
}

closest() 解決巢狀元素問題

<button class="btn">
  <svg class="icon">...</svg>
  <span class="text">Click Me</span>
</button>

點擊 <svg> 時，e.target 是 svg 而非 button：

// 錯誤：點擊 icon 時不會觸發
container.addEventListener('click', (e) => {
  if (e.target.matches('.btn')) { /* ... */ }
});

// 正確：使用 closest 向上尋找
container.addEventListener('click', (e) => {
  const btn = e.target.closest('.btn');
  if (btn) { /* ... */ }
});

架構總覽

事件委派設計模式

┌─────────────────────────────────────┐
│         Container（委派根）           │ ← 綁定 1 個事件監聽器
│                                      │
│  ┌──────┐ ┌──────┐ ┌──────┐        │
│  │ Item │ │ Item │ │ Item │  ...    │ ← N 個子元素
│  └──────┘ └──────┘ └──────┘        │
│                                      │
│  ┌──────┐                           │
│  │ New  │  ← 動態新增，自動被處理    │
│  └──────┘                           │
└─────────────────────────────────────┘

事件流：
Item click → 冒泡 → Container handler → 
  e.target.closest('.item') → 執行對應邏輯

通用事件委派實作架構

class EventDelegator {
  constructor(root) {
    this.root = root;
    this.handlers = new Map();
  }

  on(eventType, selector, handler) {
    if (!this.handlers.has(eventType)) {
      this.handlers.set(eventType, []);
      this.root.addEventListener(eventType, (e) => {
        this.dispatch(eventType, e);
      });
    }
    this.handlers.get(eventType).push({ selector, handler });
  }

  dispatch(eventType, event) {
    const handlers = this.handlers.get(eventType) || [];
    for (const { selector, handler } of handlers) {
      const target = event.target.closest(selector);
      if (target && this.root.contains(target)) {
        handler.call(target, event, target);
      }
    }
  }

  off(eventType, selector) {
    const handlers = this.handlers.get(eventType) || [];
    const index = handlers.findIndex(h => h.selector === selector);
    if (index > -1) handlers.splice(index, 1);
  }
}

事件委派與框架的關係

原生 DOM 事件
    │
    ▼
事件委派模式
    │
    ├── jQuery .on(event, selector, handler)
    │
    ├── React Synthetic Event System
    │      └── 事件統一委派到 root container
    │
    ├── Vue v-on 指令
    │      └── 編譯時決定是否使用委派
    │
    └── Svelte
           └── 編譯時生成直接綁定（不使用委派）

關鍵比較

直接綁定 vs 事件委派

比較項目直接綁定事件委派
監聽器數量N 個（每元素一個）1 個（父元素）
記憶體使用較高較低
動態元素需手動綁定自動涵蓋
初始化速度較慢（綁定 N 次）快（綁定 1 次）
事件精確性精確需要選擇器匹配
除錯難度低稍高
不冒泡事件正常運作無法使用

委派層級選擇

委派位置優點缺點適用場景
直接父元素精確、效能好範圍小單一列表
區塊容器涵蓋多個群組選擇器稍複雜功能區塊
document.body全局覆蓋每次點擊都觸發全局功能（tooltip）
document涵蓋所有效能考量很少使用

matches() vs closest() 使用時機

方法使用時機處理巢狀範例
matches()目標是葉子節點不處理純文字按鈕
closest()目標有子元素向上尋找含 icon 的按鈕
組合使用複雜結構精確匹配卡片內不同操作

應用場景

場景一：動態列表管理

// Todo List 事件委派
const todoList = document.querySelector('#todo-list');

todoList.addEventListener('click', (e) => {
  const deleteBtn = e.target.closest('[data-action="delete"]');
  const toggleBtn = e.target.closest('[data-action="toggle"]');
  const editBtn = e.target.closest('[data-action="edit"]');

  if (deleteBtn) {
    const item = deleteBtn.closest('.todo-item');
    item.remove();
    return;
  }

  if (toggleBtn) {
    const item = toggleBtn.closest('.todo-item');
    item.classList.toggle('completed');
    return;
  }

  if (editBtn) {
    const item = editBtn.closest('.todo-item');
    enableEditMode(item);
  }
});

場景二：資料表格互動

const table = document.querySelector('#data-table');

table.addEventListener('click', (e) => {
  // 排序標頭
  const th = e.target.closest('th[data-sort]');
  if (th) {
    const field = th.dataset.sort;
    const direction = th.dataset.direction === 'asc' ? 'desc' : 'asc';
    th.dataset.direction = direction;
    sortTable(field, direction);
    return;
  }

  // 行選取
  const tr = e.target.closest('tbody tr');
  if (tr) {
    tr.classList.toggle('selected');
    updateSelectedCount();
  }
});

場景三：表單驗證委派

const form = document.querySelector('#registration-form');

// 使用 focusout（blur 的冒泡版本）做即時驗證
form.addEventListener('focusout', (e) => {
  const input = e.target.closest('input[data-validate]');
  if (!input) return;

  const rules = input.dataset.validate.split(',');
  const errors = validate(input.value, rules);

  const errorEl = input.nextElementSibling;
  if (errors.length) {
    input.classList.add('invalid');
    errorEl.textContent = errors[0];
  } else {
    input.classList.remove('invalid');
    errorEl.textContent = '';
  }
});

場景四：限制委派範圍防止誤觸發

// 確保匹配的元素在委派容器內
container.addEventListener('click', (e) => {
  const btn = e.target.closest('.action-btn');

  // 重要：確認找到的元素在容器內
  if (btn && container.contains(btn)) {
    handleAction(btn.dataset.action);
  }
});

場景五：組合事件類型委派

// 同一容器處理多種事件
const app = document.querySelector('#app');

// 使用 Map 管理不同事件的委派
const delegations = {
  click: {
    '.nav-link': (e, el) => navigate(el.href),
    '.modal-close': (e, el) => closeModal(el.closest('.modal')),
    '.toggle-btn': (e, el) => el.classList.toggle('active')
  },
  input: {
    '.search-input': debounce((e, el) => search(el.value), 300),
    '.filter-select': (e, el) => applyFilter(el.value)
  }
};

Object.entries(delegations).forEach(([event, handlers]) => {
  app.addEventListener(event, (e) => {
    for (const [selector, handler] of Object.entries(handlers)) {
      const target = e.target.closest(selector);
      if (target && app.contains(target)) {
        handler(e, target);
        break;
      }
    }
  });
});
