WebSocket 即時通訊

WebSocket 是一種在單一 TCP 連線上提供全雙工通訊的協定，使得伺服器可以主動推送資料給客戶端，無需客戶端反覆輪詢。這項技術徹底改變了即時網頁應用的開發方式，從聊天室到股票報價，再到線上遊戲，WebSocket 無處不在。

核心概念

協定升級機制

WebSocket 連線始於一個標準的 HTTP 請求，透過 Upgrade 標頭將協定從 HTTP 升級為 WebSocket。這個過程稱為「握手」（Handshake）：

  1. 客戶端發送帶有 Upgrade: websocket 和 Connection: Upgrade 標頭的 HTTP 請求
  2. 伺服器回應 101 Switching Protocols 狀態碼
  3. 連線升級完成，雙方可以開始雙向傳輸資料

訊息幀（Frame）

WebSocket 使用幀作為資料傳輸的基本單位。每個幀包含：

  • FIN 位元：標記是否為訊息的最後一個幀
  • Opcode：定義幀的類型（文字、二進位、關閉、Ping、Pong）
  • Mask：客戶端到伺服器的幀必須遮罩處理
  • Payload：實際傳輸的資料內容

連線生命週期

WebSocket 連線具有明確的生命週期管理：

  • 開啟：透過握手建立連線
  • 傳輸：雙方可隨時發送訊息
  • 心跳：透過 Ping/Pong 幀維持連線活躍
  • 關閉：任一方可發起關閉握手，優雅地結束連線

伺服器端架構

典型的 WebSocket 伺服器架構包含：

  • 連線管理器：追蹤所有活躍的 WebSocket 連線
  • 訊息路由器：根據訊息類型分發到對應的處理器
  • 房間/頻道管理：支援群組廣播和訂閱機制
  • 心跳監控：偵測斷線並清理失效連線

擴展架構

在大規模部署中，常見的擴展模式：

  • 水平擴展：搭配 Redis Pub/Sub 或 Kafka 實現跨節點訊息同步
  • 負載均衡：使用 Sticky Session 確保 WebSocket 連線固定到同一節點
  • 連線閘道：專門的 WebSocket 閘道處理連線管理，業務邏輯分離到後端服務

WebSocket vs Socket.IO

Socket.IO 是建立在 WebSocket 之上的抽象層，提供：

  • 自動降級（WebSocket → HTTP Long Polling）
  • 自動重連機制
  • 房間和命名空間
  • 事件驅動的 API
  • 二進位串流支援

然而 Socket.IO 並非純 WebSocket，它有自己的協定層，無法與標準 WebSocket 伺服器互通。

應用場景

即時聊天系統

WebSocket 是聊天應用的首選技術：

  • 訊息即時送達，延遲極低
  • 支援已讀回執和打字指示器
  • 適合群組聊天的廣播機制

即時資料儀表板

金融市場報價、系統監控面板等需要持續更新的場景：

  • 股票價格即時推送
  • 伺服器效能指標即時監控
  • IoT 設備狀態即時回報

協同編輯

多人同時編輯文件的應用：

  • Google Docs 風格的即時協作
  • 程式碼協同編輯（如 VS Code Live Share）
  • 白板和設計工具的即時同步

線上遊戲

對延遲要求極高的互動場景：

  • 多人線上遊戲的狀態同步
  • 即時對戰遊戲的指令傳輸
  • 遊戲大廳的配對系統

通知系統

需要即時推送通知的應用：

  • 社群媒體即時通知
  • 電商訂單狀態更新
  • 告警系統的即時警報
