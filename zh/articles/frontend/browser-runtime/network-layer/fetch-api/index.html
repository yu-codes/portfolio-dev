Fetch API

Fetch API 是現代瀏覽器提供的網路請求介面，基於 Promise 設計，取代了傳統的 XMLHttpRequest。它提供了更簡潔、更強大的方式來處理 HTTP 請求與回應，支援串流處理、請求取消、以及與 Service Worker 的無縫整合。

核心概念

基本語法

const response = await fetch(url, options);
const data = await response.json();

Fetch 返回一個 Promise，resolve 為 Response 物件。注意：HTTP 錯誤狀態（如 404、500）不會導致 Promise reject，僅在網路故障或請求被阻止時才會 reject。

Request 物件

Request 物件封裝了請求的所有資訊：

const request = new Request('https://api.example.com/data', {
  method: 'POST',
  headers: new Headers({
    'Content-Type': 'application/json',
    'Authorization': 'Bearer token'
  }),
  body: JSON.stringify({ key: 'value' }),
  mode: 'cors',
  credentials: 'include',
  cache: 'no-cache',
  signal: controller.signal
});

常用選項：

  • method：HTTP 方法（GET、POST、PUT、DELETE 等）
  • headers：請求標頭，可用 Headers 物件或普通物件
  • body：請求主體（GET/HEAD 不可使用）
  • mode：跨域模式（cors、no-cors、same-origin）
  • credentials：Cookie 策略（omit、same-origin、include）
  • cache：快取策略（default、no-store、reload、no-cache 等）
  • signal：AbortSignal，用於取消請求

Response 物件

Response 物件代表伺服器的回應：

const response = await fetch(url);

// 狀態資訊
response.status;      // HTTP 狀態碼 (200, 404, etc.)
response.ok;          // true if status 200-299
response.statusText;  // 狀態文字
response.headers;     // Headers 物件

// 讀取回應主體（僅能讀取一次）
await response.json();       // 解析為 JSON
await response.text();       // 解析為文字
await response.blob();       // 解析為 Blob
await response.arrayBuffer(); // 解析為 ArrayBuffer
await response.formData();   // 解析為 FormData

AbortController - 請求取消

const controller = new AbortController();

// 設定超時
const timeoutId = setTimeout(() => controller.abort(), 5000);

try {
  const response = await fetch(url, { signal: controller.signal });
  clearTimeout(timeoutId);
  return await response.json();
} catch (err) {
  if (err.name === 'AbortError') {
    console.log('請求已取消或超時');
  }
  throw err;
}

串流處理（Streaming）

Fetch API 支援透過 ReadableStream 逐步讀取回應，適合大檔案下載或即時資料處理：

const response = await fetch(largeFileUrl);
const reader = response.body.getReader();
const contentLength = +response.headers.get('Content-Length');

let received = 0;
const chunks = [];

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  chunks.push(value);
  received += value.length;
  console.log(`Progress: ${(received / contentLength * 100).toFixed(1)}%`);
}

與 Service Worker 的整合

Fetch API 的請求會觸發 Service Worker 的 fetch 事件，允許實現離線快取、請求代理等進階功能：

// service-worker.js
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then(cached => {
      return cached || fetch(event.request);
    })
  );
});

應用場景

場景一：封裝通用請求函式

class HttpClient {
  constructor(baseURL, defaultHeaders = {}) {
    this.baseURL = baseURL;
    this.defaultHeaders = defaultHeaders;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseURL}${endpoint}`;
    const controller = new AbortController();
    const timeout = options.timeout || 10000;
    const timeoutId = setTimeout(() => controller.abort(), timeout);

    try {
      const response = await fetch(url, {
        ...options,
        headers: { ...this.defaultHeaders, ...options.headers },
        signal: controller.signal
      });

      if (!response.ok) {
        throw new HttpError(response.status, response.statusText);
      }

      const contentType = response.headers.get('Content-Type');
      if (contentType?.includes('application/json')) {
        return await response.json();
      }
      return await response.text();
    } finally {
      clearTimeout(timeoutId);
    }
  }

  get(endpoint, options) {
    return this.request(endpoint, { ...options, method: 'GET' });
  }

  post(endpoint, body, options) {
    return this.request(endpoint, {
      ...options,
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(body)
    });
  }
}

場景二：並行請求與錯誤處理

// 並行請求，全部成功
const [users, posts, comments] = await Promise.all([
  fetch('/api/users').then(r => r.json()),
  fetch('/api/posts').then(r => r.json()),
  fetch('/api/comments').then(r => r.json())
]);

// 並行請求，部分失敗不影響整體
const results = await Promise.allSettled([
  fetch('/api/service-a').then(r => r.json()),
  fetch('/api/service-b').then(r => r.json()),
  fetch('/api/service-c').then(r => r.json())
]);

const successful = results
  .filter(r => r.status === 'fulfilled')
  .map(r => r.value);

場景三：帶重試機制的請求

async function fetchWithRetry(url, options = {}, maxRetries = 3) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch(url, options);
      if (response.ok) return response;
      if (response.status < 500) throw new Error(`Client error: ${response.status}`);
    } catch (err) {
      if (attempt === maxRetries) throw err;
    }
    // 指數退避
    await new Promise(r => setTimeout(r, Math.pow(2, attempt) * 1000));
  }
}

場景四：檔案上傳與進度追蹤

async function uploadFile(file, onProgress) {
  // Fetch API 不原生支援上傳進度，需搭配可監控的方案
  const formData = new FormData();
  formData.append('file', file);

  // 方案：使用分塊上傳模擬進度
  const chunkSize = 1024 * 1024; // 1MB
  const totalChunks = Math.ceil(file.size / chunkSize);

  for (let i = 0; i < totalChunks; i++) {
    const chunk = file.slice(i * chunkSize, (i + 1) * chunkSize);
    const chunkForm = new FormData();
    chunkForm.append('chunk', chunk);
    chunkForm.append('index', i);
    chunkForm.append('total', totalChunks);

    await fetch('/api/upload/chunk', { method: 'POST', body: chunkForm });
    onProgress?.((i + 1) / totalChunks * 100);
  }
}
