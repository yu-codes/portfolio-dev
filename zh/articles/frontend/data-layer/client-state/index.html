客戶端狀態（Client State）是指完全存在於瀏覽器端、不需要與伺服器同步的應用狀態。正確區分客戶端狀態與伺服器狀態，並為每種狀態選擇適當的管理策略，是建構清晰前端架構的關鍵。

核心概念

客戶端狀態的分類

類型範例生命週期共享範圍
UI 狀態Modal 開關、Tab 選擇、Sidebar 展開組件存在期間通常區域
表單狀態輸入值、驗證錯誤、髒標記表單存在期間表單組件樹
導覽狀態當前路由、歷史紀錄應用生命週期全域
使用者偏好主題、語言、佈局偏好持久化全域
暫存狀態草稿、未提交的變更使用者決定區域或全域
衍生狀態過濾後的列表、計算值依賴存在時與來源相同

客戶端 vs 伺服器狀態

面向客戶端狀態伺服器狀態
所有權前端完全擁有伺服器為唯一真相來源
同步需求無需要保持與伺服器一致
過期可能不會過期隨時可能過期
管理工具useState/Zustand/ContextReact Query/SWR
失效問題無核心挑戰
多人共享否是

狀態管理決策樹

這份資料需要與伺服器同步嗎？
├── 是 → 使用 Server State 工具（React Query/SWR）
└── 否 → Client State
    ├── 只有一個組件使用？
    │   └── 是 → useState / 區域狀態
    ├── 父子組件需要共享？
    │   └── 是 → 提升狀態 / Props
    ├── 不相關的組件需要共享？
    │   └── 是 → 全域狀態（Zustand/Context）
    └── 需要跨頁面/Session 持久？
        └── 是 → localStorage + 狀態庫

架構總覽

客戶端狀態的管理層級

層級工具適用
組件內useState, useReducer, ref只有該組件使用
組件樹Props 提升、Context父子/近鄰共享
功能模組Zustand store, Pinia store功能內多組件共享
全域Global store, Context跨模組共享
持久化localStorage + rehydrate跨 Session
URLRouter state, query params可分享的狀態

狀態存放位置選擇

存放位置觸及範圍持久性效能影響適用
useState單一組件組件生命週期最小UI toggle
ContextProvider 子樹Provider 生命週期可能導致過度渲染主題/語言
Zustand任意組件應用生命週期精確訂閱全域 UI 狀態
URL params任意組件瀏覽器歷史序列化成本篩選/分頁
localStorage任意組件永久I/O 成本偏好設定
sessionStorage任意組件Tab 生命週期I/O 成本暫時草稿

常見的狀態組合模式

App State
├── Auth State (全域)
│   └── user, token, permissions
├── UI State (混合)
│   ├── theme (全域 + 持久化)
│   ├── sidebar (全域)
│   └── modal (區域/全域)
├── Navigation State (URL)
│   └── route, params, query
├── Form State (區域)
│   └── values, errors, touched
└── Server State (React Query)
    └── API 資料, 快取

關鍵比較

客戶端狀態管理工具選型

需求推薦工具原因
簡單 UI toggleuseState最輕量
跨組件主題/語言Context (低頻更新)框架內建
全域 UI 狀態Zustand輕量 + 精確訂閱
複雜表單React Hook Form效能 + 功能完整
URL 狀態Router / nuqs可分享 + 書籤
持久化偏好Zustand persist middleware自動持久化
撤銷/重做Zustand + immer middleware不可變更新

URL State vs Store State

面向URL StateStore State
可分享是（複製 URL）否
書籤是否
瀏覽器歷史是（前進/後退）否
容量有限（URL 長度）幾乎無限
型別序列化為字串任意型別
效能URL 變化觸發導覽精確訂閱
適用篩選、分頁、排序UI 狀態、暫存資料

應用場景

UI 狀態管理

UI 狀態範圍管理方式
Modal 開關觸發組件useState（簡單）或 全域 store（需跨組件）
Sidebar 展開LayoutContext 或 Zustand
通知隊列全域Zustand + 自動移除
拖拽狀態拖拽區域useState + ref
焦點管理表單/導覽DOM ref
動畫狀態組件useState / useSpring

表單狀態策略

策略工具適用場景
受控 (每欄位 useState)React 原生1-3 欄位的簡單表單
非受控 (ref)useRef效能優先的大型表單
表單庫（受控）Formik, VeeValidate需要即時驗證
表單庫（非受控）React Hook Form效能 + 複雜驗證
Schema 驗證Zod + React Hook Form型別安全驗證

持久化策略

資料儲存同步方式
主題偏好localStorage載入時讀取
語言選擇localStorage + Cookie載入時讀取 + SSR
表單草稿sessionStorage離開前儲存
折疊/展開狀態localStorageZustand persist
最近搜尋localStorage手動管理（有上限）
閱讀進度IndexedDB大量資料持久化

常見錯誤

錯誤問題正確做法
把伺服器資料放入全域 store快取失效問題使用 React Query
所有狀態都全域化不必要的耦合最小範圍原則
用 Context 管理高頻更新效能差使用 Zustand
URL 狀態 vs Store 混用不一致選定一個作為真相來源
