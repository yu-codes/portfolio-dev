API 整合層是前端應用與後端服務之間的橋樑。良好的 API 層設計能隔離後端變更的影響、統一錯誤處理、簡化快取策略，並提供一致的開發體驗。這一層的品質直接影響應用的可維護性與可擴展性。

設計建議

建議原因
將 API 層與 UI 層分離更容易測試和替換
使用 Code-gen減少手動型別定義的維護成本
統一錯誤格式前端只需處理一種錯誤結構
實作請求取消避免競態條件和浪費資源
定義重試策略提升應用的容錯能力
