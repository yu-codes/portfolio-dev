Vite

深入解析 Vite 的設計哲學與核心機制，涵蓋原生 ESM 開發架構、依賴預打包、HMR 實作原理與生產建構策略

Vite（法語中「快速」之意）是由尤雨溪於 2020 年創建的下一代前端建構工具。它利用瀏覽器原生 ES Modules 支援，在開發階段實現按需編譯與即時模組熱替換（HMR），徹底解決了 Webpack 等傳統打包工具在大型專案中冷啟動慢、HMR 延遲高的問題。Vite 已成為 Vue、React、Svelte 等主流框架的官方推薦建構工具。

核心概念

原生 ESM 開發伺服器：Vite 在開發階段不進行打包。當瀏覽器透過 <script type="module"> 發出模組請求時，Vite 攔截該請求，即時轉譯對應的原始碼（TypeScript、JSX、Vue SFC 等）並回傳。這意味著無論專案多大，啟動速度幾乎恆定——因為只處理當前頁面實際請求的模組。

依賴預打包（Dependency Pre-Bundling）：第三方套件（node_modules）通常是 CommonJS 格式且有大量子模組。Vite 使用 esbuild 在首次啟動時將這些依賴預打包為單一 ESM 檔案，解決兩個問題：(1) 將 CJS/UMD 轉換為 ESM；(2) 減少 HTTP 請求數量（如 lodash-es 有 600+ 個子模組）。

模組熱替換（HMR）：Vite 的 HMR 基於原生 ESM 的模組圖。當檔案修改時，Vite 僅需精確失效該模組及其直接依賴鏈，透過 WebSocket 通知瀏覽器重新請求受影響的模組。相比 Webpack 需重新打包整個 chunk，Vite 的 HMR 速度與專案規模無關。

生產建構：開發模式使用原生 ESM 但生產環境仍需打包（HTTP/2 下大量小檔案仍不如打包高效）。Vite 選擇 Rollup 作為生產打包器，因其成熟的 Tree Shaking、Code Splitting 與外掛生態。Vite 5+ 也正在整合 Rolldown（Rust 版 Rollup），未來將統一開發與生產的建構引擎。

外掛系統：Vite 外掛相容 Rollup 外掛介面，並擴展了開發伺服器專屬的鉤子（如 configureServer、transformIndexHtml、handleHotUpdate）。一套外掛可同時服務開發與生產兩種模式。

架構總覽

Vite 核心架構
├── 開發伺服器（Dev Server）
│   ├── HTTP 伺服器（Connect / Koa）
│   ├── 模組圖（Module Graph）
│   │   ├── URL → 模組映射
│   │   ├── 依賴追蹤
│   │   └── HMR 邊界偵測
│   ├── 請求攔截與轉譯
│   │   ├── .ts → esbuild 轉譯
│   │   ├── .vue → @vitejs/plugin-vue 編譯
│   │   ├── .jsx → esbuild / Babel 轉譯
│   │   └── .css → PostCSS / CSS Modules
│   └── WebSocket（HMR 通訊）
│
├── 依賴預打包（Pre-Bundle）
│   ├── 掃描入口依賴（esbuild scan）
│   ├── CJS → ESM 轉換
│   ├── 多模組合併（flat bundling）
│   └── 快取管理（node_modules/.vite）
│
├── 生產建構（Build）
│   ├── Rollup（程式碼分割、Tree Shaking）
│   ├── esbuild（程式碼壓縮）
│   ├── CSS 提取與壓縮
│   └── 資源處理（圖片、字型、WASM）
│
└── 外掛系統
    ├── Rollup 相容鉤子（resolveId, load, transform）
    ├── Vite 專屬鉤子（configureServer, handleHotUpdate）
    └── 框架整合（@vitejs/plugin-vue, @vitejs/plugin-react）

應用場景

Vue / React / Svelte 新專案：所有主流框架的官方 CLI 或建構模板現已預設使用 Vite。npm create vite@latest 提供所有框架的啟動模板，搭配 TypeScript、ESLint 等選項。從零開始新專案時，Vite 是當前最佳選擇。

大型企業專案遷移：對於既有 Webpack 專案，Vite 的開發體驗改善顯著。遷移重點在於：(1) 將 Webpack loader 替換為對應的 Vite 外掛；(2) 處理 CommonJS 依賴的相容性；(3) 調整環境變數（process.env → import.meta.env）。社群提供了 webpack-to-vite 等遷移工具輔助。

函式庫開發：Vite 的 Library Mode 可用於打包 npm 函式庫。透過 build.lib 配置，可同時輸出 ESM 與 CJS 格式，搭配自動產生的型別宣告。對於更進階的函式庫需求，tsup（基於 esbuild）或 unbuild（Vite 團隊維護）是更專門的選擇。

Monorepo 環境：Vite 在 Monorepo 中表現優異。搭配 pnpm workspace，各子套件可共用 Vite 設定（透過 extends），並利用 Vite 的依賴預打包機制處理內部套件間的相互引用。Turborepo 或 Nx 可進一步提供建構快取與任務編排。

靜態網站與文件：VitePress（Vite + Vue 驅動的靜態網站生成器）是 Vite 應用的典型案例。它在開發時享受 Vite 的即時 HMR，建構時生成靜態 HTML，兼具開發體驗與部署效率。
