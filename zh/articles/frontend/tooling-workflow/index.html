工具鏈

建構工具與工作流程

前端工具與工作流（Tooling & Workflow）是現代前端開發的基礎設施層。從原始碼到生產環境的每一步——程式碼撰寫、品質檢查、模組打包、開發預覽、部署發布——都依賴一系列精心組合的工具鏈。選擇合適的工具並建立高效的工作流程，能顯著提升團隊的開發速度與程式碼品質。

核心概念

前端工具鏈涵蓋三大類別：

建構工具（Build Tools / Bundlers）：負責將原始碼（JSX、TypeScript、CSS Modules 等）轉換為瀏覽器可執行的標準格式。核心工作包括模組解析、程式碼轉譯、Tree Shaking、Code Splitting、資源最佳化。代表工具：Webpack、Vite、esbuild、Rollup、Turbopack。

程式碼品質工具（Linters & Formatters）：自動化檢查程式碼風格一致性與潛在錯誤。Linter（ESLint、Biome）分析程式碼邏輯與模式；Formatter（Prettier、Biome）統一格式化風格。兩者互補，確保團隊程式碼一致。

開發伺服器（Dev Servers）：提供快速的本地開發體驗，包含 Hot Module Replacement（HMR）、Source Map、Proxy 等功能。現代 Dev Server（Vite、Turbopack）利用原生 ESM 實現近乎即時的模組更新。

除了上述三大類，工具鏈還包括：

  • 任務執行器：npm scripts、Turborepo（Monorepo 任務編排）
  • 版本管理：Changesets、semantic-release
  • Monorepo 工具：pnpm workspaces、Nx、Turborepo
  • 型別檢查：TypeScript Compiler（tsc）

架構總覽

現代前端工具鏈的典型工作流程：

原始碼 → Linter/Formatter → TypeScript → Bundler → 最佳化 → 部署
  │         (ESLint)        (tsc)      (Vite)    (Terser)   (CDN)
  │                                      │
  └── Dev Server (HMR) ←────────────────┘

開發階段工作流：

  1. 程式碼儲存 → ESLint + Prettier 自動修正（透過 IDE 整合或 lint-staged）
  2. TypeScript 即時型別檢查（IDE 內建）
  3. Vite Dev Server 提供 HMR，修改即時反映
  4. Git Commit 觸發 Husky Pre-commit Hook → lint-staged 檢查

CI/CD 階段工作流：

  1. pnpm install → 安裝依賴（使用 lockfile 確保確定性）
  2. tsc --noEmit → 型別檢查
  3. eslint . → 程式碼品質檢查
  4. vitest run → 執行測試
  5. vite build → 生產建構
  6. Bundle Analyzer → 檢查體積預算
  7. Deploy → 部署至平台

Monorepo 架構下，Turborepo 或 Nx 提供任務快取與平行執行：

apps/
  web/          → Next.js 主站
  admin/        → React Admin
packages/
  ui/           → 共用元件庫
  utils/        → 共用工具函式
  config/       → 共用 ESLint/TS 設定

工具鏈整合趨勢：

  • All-in-One：Biome（Lint + Format）、Bun（Runtime + Bundler + Package Manager）
  • Framework-native：Next.js 內建 Turbopack、Nuxt 內建 Vite
  • Rust-based：效能驅動的新一代工具（SWC、Biome、Rolldown）

應用場景

場景一：新專案初始化。使用 pnpm create vite 建立專案骨架，加入 ESLint（@antfu/eslint-config）+ Prettier + TypeScript。設定 Husky + lint-staged 確保每次 commit 都通過品質檢查。整個設定過程 < 10 分鐘。

場景二：Legacy 專案遷移。Webpack 4 專案逐步遷移至 Vite：先確認所有依賴支持 ESM，替換 Webpack-specific 的 API（如 require.context），配置 Vite 別名對齊原有路徑。Dev Server 啟動時間從 30 秒降至 1 秒。

場景三：Monorepo 建構最佳化。使用 Turborepo 管理多個前端應用與共用套件。透過 Remote Caching，CI 中未變更的套件直接使用快取，建構時間從 15 分鐘降至 3 分鐘。

場景四：自訂 ESLint 規則。針對團隊特有的程式碼規範（如禁止特定 API 使用、強制命名慣例）撰寫自訂 ESLint 插件，透過 AST 分析自動偵測違規並提供 Auto-fix。
