框架生態系

前端框架與渲染模型

目錄

  • 核心概念
  • 架構總覽

前端框架生態系是現代 Web 開發的基石。從 2013 年 React 引入虛擬 DOM 的革命性概念，到如今 Svelte 的編譯時最佳化、Solid 的細粒度響應式、Qwik 的可恢復性（Resumability），前端框架的演進從未停歇。理解框架生態系的全貌，有助於在技術選型時做出明智的決策。

核心概念

元件化（Component-Based Architecture）：所有現代前端框架的核心理念。將 UI 拆解為可重用的元件，每個元件封裝自身的結構（HTML）、樣式（CSS）與邏輯（JavaScript）。元件間透過 props 向下傳遞資料、透過事件向上通訊。

響應式系統（Reactivity System）：框架最核心的能力——當資料改變時，自動更新對應的 DOM。不同框架採用不同的響應式策略：React 使用虛擬 DOM 差異比較、Vue 使用 Proxy-based 的依賴追蹤、Svelte 在編譯時分析依賴關係、Solid 使用 Signal-based 的細粒度追蹤。

渲染模型（Rendering Models）：決定 HTML 在何時、何處生成。CSR（客戶端渲染）在瀏覽器中生成；SSR（伺服器端渲染）在伺服器生成後送至客戶端；SSG（靜態生成）在建構時預先生成；ISR（增量靜態再生成）則結合 SSG 與動態更新。

Hydration（注水）：SSR 頁面在瀏覽器中「復活」的過程。伺服器送來的是靜態 HTML，hydration 將事件監聽器附加到 DOM 上，使頁面具備互動能力。這個過程的效能成本是近年框架創新的主要驅動力。

架構總覽

前端框架生態系
├── 元件框架 (Component Frameworks)
│   ├── React（虛擬 DOM、JSX）
│   ├── Vue（Proxy 響應式、SFC）
│   ├── Svelte（編譯時、無虛擬 DOM）
│   ├── Solid（Signal、細粒度）
│   ├── Angular（完整框架、依賴注入）
│   └── Qwik（Resumability）
├── 元框架 (Meta-Frameworks)
│   ├── Next.js（React 生態）
│   ├── Nuxt（Vue 生態）
│   ├── SvelteKit（Svelte 生態）
│   ├── Astro（多框架、內容優先）
│   └── Remix（React、Web 標準）
└── 渲染模型 (Rendering Models)
    ├── CSR - Client-Side Rendering
    ├── SSR - Server-Side Rendering
    ├── SSG - Static Site Generation
    ├── ISR - Incremental Static Regeneration
    ├── Streaming SSR
    └── Partial Hydration / Islands

渲染模型選擇指南：

  • CSR：管理後台、複雜互動應用（登入後才能使用）
  • SSR：需要 SEO 且資料頻繁更新的頁面
  • SSG：部落格、文件站、行銷頁面
  • ISR：電商產品頁（需 SEO 但資料會變動）

應用場景

內容型網站（部落格、文件）：Astro 或 Next.js (SSG) 是首選。Astro 的 Islands Architecture 讓頁面預設零 JavaScript，僅在需要互動的元件載入框架程式碼，達到極致的載入效能。

大型 SaaS 應用：Next.js App Router 搭配 React Server Components 能在伺服器端渲染大部分 UI，減少客戶端 JavaScript bundle。搭配 Streaming SSR 實現漸進式載入，使用者不需等待所有資料就能看到頁面框架。

跨平台應用：React + React Native 實現 Web 與行動端的程式碼共享。Vue 搭配 Capacitor 或 Ionic 也能達到類似效果。Svelte 則透過 Tauri 支援桌面應用。

微互動落地頁：Svelte 的編譯時最佳化使其產出極小的 bundle，適合對載入速度要求極高的行銷頁面。搭配動畫庫如 Motion One，可實現流暢的使用者體驗。

企業級應用：Angular 的完整框架特性（依賴注入、模組系統、RxJS 整合）使其適合大型企業團隊。嚴格的架構約束降低了不同開發者寫出差異極大的程式碼的可能性。
