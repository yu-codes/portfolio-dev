狀態與生命週期

全面解析 React 元件的狀態管理機制與生命週期模型，涵蓋 Class 與 Hooks 兩種範式的對照與遷移策略

React 元件的狀態（State）與生命週期（Lifecycle）是理解 React 運作方式的基石。從 Class 元件的明確生命週期方法，到 Hooks 的宣告式副作用管理，React 的狀態模型經歷了深刻演進。掌握兩種範式的核心差異，有助於維護既有程式碼並設計新架構。

React 18 引入自動批次（Automatic Batching），統一了不同場景的行為：

應用場景

表單狀態管理

表單是狀態管理的經典場景。受控元件將每個欄位值綁定至 state，確保 React 為唯一真相來源。對於大型表單，使用 useReducer 統一管理欄位狀態、驗證結果與提交狀態，避免散落的 useState 造成維護困難。

動畫與過渡狀態

動畫涉及高頻狀態更新，需特別注意效能。使用 useRef 儲存動畫幀 ID，搭配 requestAnimationFrame 驅動更新。React 18 的 useTransition 可將動畫相關更新標記為低優先權，避免阻塞使用者輸入。

資料同步與快取

當本地狀態需與伺服器保持同步，需處理樂觀更新、衝突解決與快取失效。這類場景適合使用專門的 Server State 管理工具（如 TanStack Query），將快取策略、重試邏輯與背景同步封裝為宣告式 API。
