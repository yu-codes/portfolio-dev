JSX 語法

深入解析 JSX 語法的核心概念、編譯原理與最佳實踐，涵蓋表達式嵌入、條件渲染與列表處理等進階技巧

JSX（JavaScript XML）是 React 生態系統中最具辨識度的語法擴展，它允許開發者在 JavaScript 中直接撰寫類似 HTML 的標記語言。JSX 並非模板引擎，而是 JavaScript 的語法糖，最終會被編譯為標準的函式呼叫。理解 JSX 的運作原理，是掌握 React 開發的第一步。

核心概念

JSX 的本質

JSX 表達式在編譯後會轉換為 React.createElement() 呼叫（React 17+ 使用新的 JSX Transform）：

JSX 寫法編譯結果
<div className="box">Hello</div>React.createElement('div', {className: 'box'}, 'Hello')
<MyComponent title="hi" />React.createElement(MyComponent, {title: 'hi'})
<>{children}</>React.createElement(React.Fragment, null, children)

表達式嵌入規則

JSX 中使用花括號 {} 嵌入 JavaScript 表達式，但需注意以下限制：

類型可否嵌入說明
字串、數字✅直接渲染
布林值、null、undefined✅不渲染任何內容
陣列✅依序渲染各元素
物件❌需轉換為字串或取值
函式❌需呼叫後取回傳值
陳述句（if/for）❌需改用表達式形式

屬性命名轉換

由於 JSX 最終編譯為 JavaScript，HTML 屬性需遵循 camelCase 命名：

HTML 屬性JSX 屬性原因
classclassNameclass 為 JS 保留字
forhtmlForfor 為 JS 保留字
tabindextabIndexcamelCase 慣例
onclickonClick合成事件系統
style="color:red"style={{color:'red'}}需物件格式

架構總覽

JSX 編譯流程

JSX 的處理經歷三個主要階段：

  1. 解析階段（Parsing）：Babel 或 TypeScript 編譯器將 JSX 語法解析為 AST（抽象語法樹）
  2. 轉換階段（Transform）：將 JSX 節點轉換為函式呼叫節點
  3. 生成階段（Generation）：輸出標準 JavaScript 程式碼

React 17 引入的新 JSX Transform 帶來了重要變化：

特性傳統模式新 JSX Transform
引入方式需手動 import React自動注入
輸出函式React.createElementjsx / jsxs
Bundle 大小較大略小
向後相容-完全相容

條件渲染模式

JSX 中無法使用 if 陳述句，需採用表達式替代方案：

模式語法適用場景
三元運算子{cond ? <A/> : <B/>}二選一渲染
短路求值{cond && <A/>}條件顯示
立即執行函式{(() => { if... })()}複雜邏輯
提取變數在 return 前處理多重條件
元件抽取獨立條件元件可複用邏輯

關鍵比較

JSX vs 模板語法

比較維度JSX (React)Template (Vue/Angular)
語法形式JavaScript 表達式自定義模板語法
學習曲線需熟悉 JS類 HTML 較直覺
型別檢查原生支援 TypeScript需額外工具
IDE 支援完整 JS 工具鏈需專用插件
靈活度極高（圖靈完備）受模板限制
編譯優化有限（runtime）可做靜態分析優化
除錯體驗標準 JS 除錯需 source map

應用場景

動態表單生成

JSX 的表達式能力使動態表單生成極為自然。透過陣列映射可動態渲染表單欄位，結合條件渲染處理不同欄位類型（文字、選擇、日期等），並以遞迴元件處理巢狀結構。

設計系統元件

JSX 的組合特性非常適合建構設計系統：透過 children prop 實現插槽機制、利用展開運算子傳遞屬性集合、以 render props 或 compound components 實現進階組合模式。

伺服器元件整合

React Server Components 中，JSX 語法保持一致，但執行環境從客戶端轉移至伺服器。開發者需區分 'use client' 與 'use server' 標記，理解序列化邊界對 JSX 的限制。
