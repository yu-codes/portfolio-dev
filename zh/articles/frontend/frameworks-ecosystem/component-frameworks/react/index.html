React

React 生態系

React 是由 Meta（前 Facebook）開發並維護的 JavaScript UI 函式庫，自 2013 年開源以來一直是前端開發的領導者。React 的核心哲學是「UI 是狀態的函式」——給定相同的輸入（props 與 state），元件總是渲染相同的輸出。這種函式式的思維模型，搭配虛擬 DOM、單向資料流與豐富的生態系，使 React 成為構建複雜使用者介面的首選工具。

核心概念

JSX（JavaScript XML）：React 最具辨識度的特徵。JSX 允許在 JavaScript 中直接撰寫類似 HTML 的語法，經過 Babel 或 SWC 編譯後轉化為 React.createElement() 呼叫。JSX 不是模板——它擁有 JavaScript 的完整表達力，可以使用條件表達式、map、函式呼叫等任何 JavaScript 語法。

Hooks（鉤子）：React 16.8 引入的革命性 API。Hooks 讓函式元件能夠使用狀態與副作用，取代了類別元件的生命週期方法。核心 Hooks 包括：

  • useState：管理元件本地狀態
  • useEffect：處理副作用（API 呼叫、訂閱、DOM 操作）
  • useContext：存取上下文（Context）中的值
  • useMemo / useCallback：記憶化計算結果與回呼函式
  • useRef：存取 DOM 元素或保存跨渲染的可變值
  • useReducer：複雜狀態邏輯的管理

Server Components（伺服器元件）：React 18+ 的重大創新。伺服器元件在伺服器端執行，不會被包含在客戶端 bundle 中。它們可以直接存取資料庫、讀取檔案系統，且零客戶端 JavaScript 成本。搭配 Next.js App Router，伺服器元件已成為 React 應用的預設。

Concurrent Features（並行特性）：React 18 引入的並行渲染能力。useTransition 讓開發者標記低優先級的狀態更新，React 可以中斷這些更新以回應更高優先級的互動（如使用者輸入），確保 UI 始終保持回應。

應用場景

大型 SaaS 產品：React 搭配 Next.js App Router 是建構複雜 SaaS 產品的黃金組合。Server Components 減少客戶端 JavaScript、Streaming SSR 提升首次載入體驗、搭配 TanStack Query 管理伺服器狀態，可輕鬆處理數百個頁面與複雜的資料流。

設計系統與元件庫：React 是最多設計系統選擇的框架。Radix UI 提供無樣式的可存取元件原語、shadcn/ui 提供可複製貼上的元件、Material UI 與 Ant Design 提供完整的企業級元件套件。

跨平台開發：React Native 讓 React 開發者能用相同的心智模型開發 iOS 與 Android 應用。搭配 Expo，可實現一套程式碼同時部署 Web、iOS、Android。

即時協作工具：React 搭配 WebSocket 與樂觀更新，適合建構如 Notion、Figma 等即時協作工具。搭配 CRDT 函式庫（如 Yjs）可實現多人同時編輯。

資料密集型儀表板：搭配 D3.js 或 Recharts 建構資料視覺化儀表板。TanStack Virtual 處理大量資料表格的虛擬滾動，確保即使渲染數萬列資料也能保持流暢。
