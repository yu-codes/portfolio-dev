元件框架

React、Vue 等元件框架

元件框架（Component Frameworks）是現代前端開發的核心工具。它們提供了一套元件化的抽象模型，讓開發者能以宣告式的方式構建使用者介面。從 React 的函式式元件到 Vue 的單檔案元件，從 Svelte 的編譯時魔法到 Solid 的細粒度信號，每種框架都代表了對「如何高效構建 UI」這個問題的不同回答。

核心概念

元件生命週期（Component Lifecycle）：每個元件從建立到銷毀都經歷一系列階段。React 使用 Effect Hook 管理副作用；Vue 提供 onMounted、onUpdated、onUnmounted 等組合式生命週期鉤子；Svelte 使用 onMount、onDestroy 等簡潔的回呼。理解生命週期是避免記憶體洩漏與不必要渲染的關鍵。

狀態管理機制（State Management）：元件框架的核心差異在於如何追蹤與響應狀態變化。React 的 useState 觸發整個元件重新渲染；Vue 的 ref/reactive 透過 Proxy 精確追蹤依賴；Svelte 的 $state 在編譯時分析並產生最小化的 DOM 更新程式碼；Solid 的 createSignal 實現真正的細粒度更新。

模板語法 vs JSX：框架在「如何描述 UI」上分為兩派。React 和 Solid 使用 JSX（JavaScript XML），將 HTML 嵌入 JavaScript 中；Vue 和 Svelte 使用模板語法，將 JavaScript 嵌入增強版的 HTML 中。JSX 提供完整的 JavaScript 表達力；模板語法則更易於靜態分析與編譯最佳化。

虛擬 DOM vs 直接 DOM 操作：React 和 Vue 使用虛擬 DOM 作為中間層，透過 diff 演算法計算最小 DOM 更新；Svelte 在編譯時確定哪些 DOM 節點會被更新，直接生成命令式的更新程式碼，省去執行時的 diff 成本；Solid 則透過信號追蹤，僅更新訂閱了該信號的具體 DOM 節點。

應用場景

大型企業應用：React 的龐大生態系與豐富的人才池使其成為企業首選。搭配 TypeScript、React Query、Zustand 等工具，可建構複雜的商業應用。Meta、Airbnb、Netflix 等公司都以 React 為主力。

快速原型開發：Vue 的低學習曲線與優秀的開發體驗使其適合快速迭代。單檔案元件（SFC）將模板、腳本、樣式集中管理，開發者無需在多個檔案間切換。搭配 Nuxt 3 可快速搭建全端應用。

效能敏感的應用：Svelte 和 Solid 的極小 bundle size 與高效的 DOM 更新，適合對效能要求嚴苛的場景，如嵌入式 widget、低端設備支援、動畫密集的行銷頁面。

漸進式遷移：Vue 最初的設計就是可漸進式採用的。可以從一個小的互動元件開始，逐步擴展到整個頁面，最終遷移至 SPA。這對於有大量 jQuery 遺留程式碼的專案特別友善。

互動式資料視覺化：Svelte 的 D3 整合非常自然，因為它不使用虛擬 DOM，可以直接操作 SVG 元素。Solid 的細粒度響應式也非常適合需要高頻更新的即時圖表。
