響應式模型

深入比較前端框架的響應式程式設計模型，涵蓋 Signal、Observable、Proxy 與 Compiler-based 等不同實作方式的原理與取捨

響應式模型（Reactivity Model）是前端框架的核心引擎，決定了框架如何偵測資料變化並高效更新使用者介面。從 Angular 的 Dirty Checking 到 Solid 的 Signal，從 Vue 的 Proxy 攔截到 Svelte 的編譯時分析，每種模型都代表了不同的設計取捨。

Signal 模型深入

Signal 是目前趨勢最明顯的響應式原語：

概念角色行為
Signal可變響應式值讀取時追蹤、寫入時通知
Computed/Memo衍生值惰性求值、自動快取
Effect副作用依賴變化時重新執行
Batch批次更新合併多次修改為一次更新

應用場景

Signal 的統一趨勢

2023-2024 年，Signal 模型被多個框架採納：Angular 16+、Preact Signals、Solid.js、Qwik。TC39 也提出了 Signal 標準提案。這種趨勢反映了業界對細粒度、可預測更新的共識。

混合模型實踐

現代框架趨向混合模型：Vue 使用 Proxy 追蹤 + 元件級更新；Angular 從 Zone.js 遷移至 Signal；React 考慮 Compiler（React Forget）減少手動 memo。沒有單一模型適合所有場景。

效能關鍵場景選擇

對於大型資料表格（10,000+ 行）、即時協作編輯器、遊戲 UI 等高頻更新場景，Signal 或編譯器模型的 O(1) 精準更新優勢顯著。VDOM diff 的 O(n) 特性在這些場景中成為瓶頸。
