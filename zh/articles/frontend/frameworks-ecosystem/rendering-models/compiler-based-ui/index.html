編譯器驅動 UI

探索編譯器驅動的 UI 框架如何透過靜態分析消除執行時開銷，涵蓋 Svelte、Solid、React Compiler 與 Vue Vapor 的設計理念

編譯器驅動 UI（Compiler-based UI）代表了前端框架的新趨勢：將執行時的工作前移至編譯時，透過靜態分析產生最佳化的更新指令，消除 Virtual DOM diff 或執行時響應式系統的開銷。Svelte 是先驅，而 React Compiler 和 Vue Vapor Mode 則代表主流框架對這一方向的擁抱。

React Compiler（React Forget）

React Compiler 的目標不同於 Svelte：

面向SvelteReact Compiler
目標消除 VDOM自動記憶化
輸入.svelte 自訂語法標準 JSX
產出原生 DOM 操作優化的 React 程式碼
相容性全新語法完全向後相容
自動化memo 概念不存在自動加入 useMemo/useCallback
狀態模型編譯器管理仍使用 Hooks

效能良好更佳

應用場景

效能極致優化

嵌入式裝置、低階手機或網路受限環境中，編譯器驅動框架的優勢最為顯著。零/極小 Runtime 意味著更少的下載量和解析時間，直接 DOM 操作避免了 VDOM 的 CPU 開銷。

Widget 與嵌入式元件

第三方嵌入式元件（如聊天窗口、評論系統）需要極小的程式碼體積。Svelte 編譯後的元件不帶框架包袱，每個元件獨立打包體積極小，適合作為可嵌入的微型應用。

漸進式採納

React Compiler 和 Vue Vapor 的設計允許漸進式採納：不改變開發者已知的 API，僅在建構時自動最佳化。團隊無需學習新語法或遷移程式碼，升級工具鏈即可獲得效能提升。
