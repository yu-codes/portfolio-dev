Virtual DOM

深入剖析 Virtual DOM 的設計原理、Diff 演算法、效能特徵與現代框架中的演進方向

Virtual DOM（虛擬 DOM）是 React 引入的核心概念，透過在記憶體中維護一份輕量的 DOM 樹表示，比較前後差異後才對真實 DOM 進行最小化更新。這個抽象層帶來了宣告式 UI 程式設計的可能，但也引入了固有的效能開銷。理解 VDOM 的運作方式與限制，有助於做出正確的框架選擇。

跨平台綁定瀏覽器可渲染至任意目標
心智負擔需追蹤 DOM 狀態重新描述整個 UI

Diff 演算法假設

完整樹比較為 O(n³)，React 透過兩個假設將其優化至 O(n)：

假設說明效果
類型假設不同類型元素產生不同樹類型變化即重建子樹
Key 假設key 標識同層元素身份同 key 視為同一節點
同層比較僅比較同一層級的節點不跨層移動

架構總覽

Reconciliation 流程

  1. 渲染階段：呼叫 render/函式元件產生新 VDOM 樹
  2. 比較階段：新舊 VDOM 逐節點比較
  3. 收集階段：記錄所有需要的 DOM 操作
  4. 提交階段：批次執行 DOM 變更

Diff 策略細節

情況處理方式效能
相同類型、相同 props不做任何操作O(1)
相同類型、不同 props僅更新變化的屬性O(props)
不同類型卸載舊子樹、掛載新子樹O(subtree)
列表有 key透過 key 匹配移動、增刪O(n)
列表無 key逐位置比較O(n) 但不精確

應用場景

跨平台渲染

VDOM 的最大架構優勢在於渲染目標的抽象。React Native 將 VDOM 映射至原生 UI 元件、react-three-fiber 映射至 3D 場景、Ink 映射至終端 UI。這種抽象層讓同一套元件邏輯適用於多個平台。

複雜互動介面

對於拖放排序、可撤銷編輯器、協作白板等複雜 UI，VDOM 的宣告式模型大幅降低開發複雜度。開發者只需描述「目標狀態」，框架負責計算最小 DOM 變更路徑。

服務端渲染

VDOM 可在非瀏覽器環境（Node.js）執行，將 VDOM 序列化為 HTML 字串。這是 SSR 的基礎。React 的 renderToString / renderToPipeableStream 即基於此能力實現伺服器渲染。
