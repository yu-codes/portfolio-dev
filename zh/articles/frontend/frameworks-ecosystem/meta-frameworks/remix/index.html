Remix

Remix 是一個以 Web 基礎設施為核心的全端 React 框架，強調利用 HTTP、HTML Form、瀏覽器原生行為等 Web 標準來建構應用。其設計哲學是「擁抱平台」而非對抗平台，透過巢狀路由、漸進增強與伺服器優先的資料載入，提供卓越的使用者體驗與開發者體驗。

架構總覽

資料流模型

Remix 的資料流遵循簡潔的單向模式：

  1. Loader：使用者導航時，伺服器載入資料
  2. Component：使用 useLoaderData() 渲染 UI
  3. Action：使用者提交表單，伺服器處理變更
  4. Revalidation：Action 完成後自動重新執行相關 Loader
階段自動行為開發者職責
導航平行呼叫所有 loader撰寫 loader 函式
渲染串流式 HTML 回應撰寫元件
變更呼叫 action + 重驗證撰寫 action 函式
錯誤顯示最近 ErrorBoundary定義錯誤 UI

應用場景

資料密集型應用

Remix 的巢狀路由平行載入特性，特別適合儀表板類應用。每個面板作為獨立路由段，可平行取得資料、獨立顯示載入狀態與錯誤，避免單一失敗影響整個頁面。

表單密集型應用

CRM、ERP 等後台系統充滿表單操作。Remix 的 Form + Action 模式搭配漸進增強，確保核心功能在任何環境下都能運作。搭配 useActionData 處理驗證錯誤回饋，useFetcher 實現即時自動儲存。

高可靠性需求

由於 Remix 應用在 JavaScript 失效時仍可運作（表單提交、頁面導航），適合對可靠性要求極高的場景：政府服務、金融交易、醫療系統等。漸進增強確保基本功能不依賴客戶端 JavaScript。
