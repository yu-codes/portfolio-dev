CSS 佈局與樣式系統

CSS（Cascading Style Sheets）是現代網頁開發中不可或缺的三大基石之一，負責控制網頁的視覺呈現與互動體驗。從簡單的顏色與字型設定，到複雜的響應式佈局與動畫效果，CSS 為開發者提供了一套強大且靈活的樣式描述語言。隨著 CSS3 及後續規範的演進，它已經從單純的「美化工具」轉變為一個功能完善的佈局與設計系統。

核心概念

層疊與繼承

CSS 的名稱中「Cascading」一詞揭示了其最核心的機制——層疊。當多個樣式規則套用到同一元素時，瀏覽器會根據特定的優先順序決定最終呈現的樣式。這個優先順序由三個因素決定：來源重要性（Origin Importance）、選擇器特異性（Specificity）、以及宣告順序（Source Order）。

繼承機制則讓某些屬性（如 font-family、color）自動從父元素傳遞到子元素，大幅減少了重複宣告的需要。理解哪些屬性會繼承、哪些不會，是撰寫高效 CSS 的基礎。

盒模型（Box Model）

每個 HTML 元素在 CSS 中都被視為一個矩形盒子，由內容區域（Content）、內距（Padding）、邊框（Border）與外距（Margin）組成。box-sizing 屬性決定了寬高的計算方式：content-box（預設）只計算內容區域，而 border-box 則包含 padding 與 border，後者在實務開發中更為直覺且常用。

選擇器系統

CSS 選擇器是連接 HTML 結構與樣式規則的橋樑。從基本的元素選擇器、類別選擇器、ID 選擇器，到進階的屬性選擇器、偽類（:hover、:nth-child()）與偽元素（::before、::after），選擇器的組合能力讓開發者精確地指定樣式的作用範圍。

CSS 自定義屬性

CSS 變數（Custom Properties）以 -- 前綴定義，透過 var() 函數引用，實現了原生的主題切換與設計系統。與預處理器變數不同，CSS 變數具有即時性與繼承性，可在執行時期動態修改。

架構總覽

現代 CSS 的架構可以從以下幾個維度來理解：

CSS 技術架構
├── 佈局系統（Layout）
│   ├── Flexbox — 一維彈性佈局
│   ├── Grid — 二維格線佈局
│   └── 定位系統 — absolute / relative / fixed / sticky
├── 視覺呈現（Visual）
│   ├── 色彩與背景
│   ├── 文字排版（Typography）
│   └── 陰影與濾鏡
├── 響應式設計（Responsive）
│   ├── Media Queries
│   ├── Container Queries
│   └── 流體排版（Fluid Typography）
├── 動態效果（Motion）
│   ├── Transitions
│   ├── Animations（@keyframes）
│   └── Scroll-driven Animations
└── 架構方法論
    ├── BEM 命名規範
    ├── ITCSS 分層架構
    └── CSS Modules / CSS-in-JS

佈局系統是 CSS 中最為核心的部分。Flexbox 擅長處理單一方向的元素排列與對齊，Grid 則提供二維的格線控制能力。兩者並非互斥，而是互補——在實際專案中經常搭配使用。

應用場景

企業級設計系統：透過 CSS 自定義屬性建立一套完整的設計 Token 系統，包含色彩、間距、字型大小等變數，確保品牌一致性並支援主題切換。

響應式網頁應用：結合 Media Queries 與 Container Queries，根據視窗大小與容器尺寸調整佈局，確保在桌面、平板與手機上都有良好的使用體驗。

複雜資料儀表板：使用 CSS Grid 建立多區塊的儀表板佈局，搭配 grid-template-areas 語法使版面結構一目了然，並透過 Media Queries 在小螢幕上重新排列區塊。

高效能動畫：利用 transform 與 opacity 觸發 GPU 加速的合成動畫，避免觸發 Layout 或 Paint，實現 60fps 的流暢視覺效果。

無障礙設計：運用 prefers-reduced-motion、prefers-color-scheme 等媒體查詢，尊重使用者的系統偏好設定，並確保焦點指示器（focus indicator）清晰可見。
