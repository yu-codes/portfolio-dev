CSS 定位系統控制元素在頁面中的精確位置，從基本的文件流定位到脫離文件流的絕對定位與固定定位。理解定位機制與堆疊上下文是處理複雜 UI 佈局的關鍵。

核心概念

position 屬性值

值是否脫離文件流定位參考常見用途
static否無（預設值）正常文件流
relative否自身原始位置微調位置、建立定位參考
absolute是最近的非 static 祖先彈出選單、徽章
fixed是視窗（viewport）固定導覽列、回到頂部按鈕
sticky部分捲動容器表格標頭、側邊導覽

偏移屬性

定位元素透過 top、right、bottom、left 控制偏移：

.positioned {
  position: absolute;
  top: 20px;
  left: 50%;
  transform: translateX(-50%); /* 置中技巧 */
}

包含區塊（Containing Block）

absolute 定位的參考點取決於包含區塊：

  1. 向上尋找最近的 position 值不為 static 的祖先元素
  2. 若找不到，以初始包含區塊（通常是 <html>）為參考
  3. 注意：transform、filter、will-change 也會建立新的包含區塊
.parent {
  position: relative; /* 建立定位參考 */
}

.child {
  position: absolute;
  top: 0;
  right: 0; /* 定位到父元素的右上角 */
}

架構總覽

堆疊上下文（Stacking Context）

堆疊上下文決定了重疊元素的前後順序，是 CSS 中最常被誤解的概念之一。

建立堆疊上下文的條件

  • 根元素 <html>
  • position 為 absolute 或 relative 且 z-index 不為 auto
  • position 為 fixed 或 sticky
  • opacity 小於 1
  • transform 不為 none
  • filter 不為 none
  • isolation: isolate
  • will-change 指定了上述任一屬性
  • contain: layout 或 contain: paint

z-index 運作規則

/* z-index 僅在同一堆疊上下文內比較 */
.parent-a { position: relative; z-index: 1; }
.child-a  { position: absolute; z-index: 9999; } /* 無法超越 parent-b */

.parent-b { position: relative; z-index: 2; }
.child-b  { position: absolute; z-index: 1; }   /* 仍在 parent-a 之上 */

堆疊順序（由後到前）

  1. 堆疊上下文的背景與邊框
  2. 負 z-index 的定位元素
  3. 非定位的區塊元素（正常流）
  4. 非定位的浮動元素
  5. 非定位的行內元素
  6. z-index: 0 或 auto 的定位元素
  7. 正 z-index 的定位元素

Sticky 定位詳解

.sticky-header {
  position: sticky;
  top: 0;        /* 觸發黏性的閾值 */
  z-index: 100;
  background: white;
}

Sticky 的限制條件：

  • 必須指定至少一個方向的偏移值（top/right/bottom/left）
  • 父容器不能設置 overflow: hidden 或 overflow: auto
  • 黏性效果僅在父容器範圍內有效
  • 父容器的高度必須大於 sticky 元素

定位置中方法彙整

/* 方法一：absolute + transform */
.center-1 {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* 方法二：absolute + inset + margin */
.center-2 {
  position: absolute;
  inset: 0;
  margin: auto;
  width: 200px;
  height: 200px;
}

/* 方法三：absolute + calc */
.center-3 {
  position: absolute;
  top: calc(50% - 100px);  /* 需已知高度 */
  left: calc(50% - 150px); /* 需已知寬度 */
}

關鍵比較

relative vs absolute vs fixed

特性relativeabsolutefixed
文件流保留原始空間脫離脫離
參考基準原始位置定位祖先視窗
捲動行為隨頁面捲動隨定位祖先固定不動
常見用途微調/作為參考彈出層/裝飾固定 UI

fixed vs sticky

特性fixedsticky
定位方式始終固定捲動到閾值後固定
參考viewport捲動容器
範圍全域父容器內
文件流完全脫離部分保留
transform 影響會改變行為不受影響

inset 屬性

CSS Logical Properties 引入的 inset 是 top/right/bottom/left 的縮寫：

/* 傳統寫法 */
.element { top: 0; right: 0; bottom: 0; left: 0; }

/* inset 縮寫 */
.element { inset: 0; }              /* 四邊皆 0 */
.element { inset: 10px 20px; }      /* 上下 10px，左右 20px */
.element { inset: 1rem 2rem 3rem; } /* 上 1rem，左右 2rem，下 3rem */

應用場景

場景一：下拉選單

.dropdown {
  position: relative;
}

.dropdown-menu {
  position: absolute;
  top: 100%;
  left: 0;
  z-index: 1000;
  min-width: 200px;
  display: none;
}

.dropdown:hover .dropdown-menu,
.dropdown:focus-within .dropdown-menu {
  display: block;
}

場景二：固定側邊工具列

.toolbar {
  position: fixed;
  right: 1rem;
  bottom: 1rem;
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
  z-index: 999;
}

場景三：表格固定表頭

.table-wrapper {
  max-height: 400px;
  overflow-y: auto;
}

.table-wrapper th {
  position: sticky;
  top: 0;
  background: white;
  box-shadow: 0 1px 0 #ddd;
}

場景四：全螢幕覆蓋層（Overlay）

.overlay {
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
  z-index: 9999;
  display: flex;
  justify-content: center;
  align-items: center;
}
