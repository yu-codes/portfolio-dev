佈局系統

CSS 佈局方法

CSS 佈局系統是前端開發的核心能力之一，它決定了網頁元素如何在二維平面上排列與定位。從早期的 Table 佈局、Float 佈局，到現代的 Flexbox 與 Grid，CSS 佈局技術經歷了巨大的演進。如今，開發者擁有多種強大且靈活的佈局工具，能夠輕鬆實現從簡單的居中對齊到複雜的響應式版面設計。

核心概念

正常流（Normal Flow）

在沒有任何佈局宣告的情況下，瀏覽器按照「正常流」排列元素。區塊級元素（Block）垂直堆疊，行內元素（Inline）水平排列直到換行。理解正常流是掌握所有佈局技術的基礎，因為其他佈局方式本質上都是對正常流的修改或脫離。

格式化上下文（Formatting Context）

CSS 佈局的另一個關鍵概念是格式化上下文。Block Formatting Context（BFC）建立了一個獨立的佈局區域，內部元素的佈局不會影響外部。觸發 BFC 的常見方式包括：設定 overflow 為非 visible 值、使用 display: flow-root、或將元素設為 Flex/Grid 容器。

Flexbox 彈性佈局

Flexbox 是一個一維佈局模型，專為在單一方向（水平或垂直）上分配空間和對齊內容而設計。它引入了「主軸」（Main Axis）和「交叉軸」（Cross Axis）的概念：

.container {
  display: flex;
  flex-direction: row;      /* 主軸方向 */
  justify-content: center;  /* 主軸對齊 */
  align-items: center;      /* 交叉軸對齊 */
  gap: 1rem;                /* 元素間距 */
}

.item {
  flex: 1 1 auto;  /* flex-grow | flex-shrink | flex-basis */
}

flex 屬性的三個值分別控制：當有剩餘空間時的擴展比例（grow）、空間不足時的收縮比例（shrink）、以及元素的初始尺寸（basis）。

Grid 格線佈局

CSS Grid 是一個二維佈局系統，能夠同時控制行與列。它特別適合定義頁面級別的版面結構：

.layout {
  display: grid;
  grid-template-columns: 250px 1fr 300px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header  header"
    "sidebar content aside"
    "footer  footer  footer";
  gap: 1.5rem;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.aside   { grid-area: aside; }
.footer  { grid-area: footer; }

grid-template-areas 語法讓佈局結構視覺化，極大提升了程式碼的可讀性。搭配 fr 單位可以實現靈活的比例分配。

定位系統（Positioning）

CSS 定位機制透過 position 屬性提供了脫離正常流的佈局能力：

  • relative：相對於元素自身的原始位置偏移，不影響其他元素
  • absolute：相對於最近的定位祖先元素定位，完全脫離正常流
  • fixed：相對於視窗定位，不隨頁面捲動
  • sticky：在捲動到特定閾值前表現為 relative，之後變為 fixed

架構總覽

CSS 佈局系統架構
├── 正常流（Normal Flow）
│   ├── Block Layout
│   ├── Inline Layout
│   └── Block Formatting Context (BFC)
├── Flexbox
│   ├── 容器屬性（flex-direction, justify-content, align-items）
│   ├── 項目屬性（flex-grow, flex-shrink, flex-basis）
│   └── 對齊與排序（order, align-self）
├── Grid
│   ├── 顯式格線（grid-template-*）
│   ├── 隱式格線（grid-auto-*）
│   ├── 區域命名（grid-template-areas）
│   └── 子格線（subgrid）
├── 定位系統
│   ├── relative / absolute
│   ├── fixed / sticky
│   └── 堆疊上下文（z-index）
└── 多欄佈局（Multi-column）
    ├── column-count
    └── column-gap

何時選擇 Flexbox vs Grid

  • 一維排列（單行或單列的元素分佈）→ Flexbox
  • 二維排列（需要同時控制行列對齊）→ Grid
  • 內容大小未知，需要彈性分配空間 → Flexbox
  • 版面結構固定，需要精確的軌道控制 → Grid
  • 元素需要重疊 → Grid 或 Position
  • 實際開發中：兩者經常搭配使用——Grid 負責頁面大結構，Flexbox 負責組件內部排列

應用場景

響應式導航列：使用 Flexbox 實現水平導航列，透過 justify-content: space-between 分配 Logo 與選單的位置，並以 flex-wrap: wrap 在小螢幕上自動換行。

儀表板佈局：以 CSS Grid 定義整體版面結構，搭配 grid-template-areas 命名各區域，再利用 Media Queries 在不同斷點重新安排區域順序。

卡片網格系統：使用 grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)) 建立自適應卡片網格，無需額外的 Media Queries 即可自動調整欄數。

固定側邊欄：結合 position: sticky 與 top: 0 實現捲動時固定的側邊欄導航，當使用者捲動主要內容時，導航始終可見。

疊層 UI 元素：利用 position: absolute 與 z-index 實現下拉選單、工具提示（Tooltip）與 Modal 對話框等需要重疊的 UI 元素。
