隨著專案規模增長，CSS 的維護性與可擴展性成為關鍵挑戰。CSS 架構方法論提供命名規範、組織策略與模組化方案，讓樣式程式碼在團隊協作中保持一致性與可預測性。

核心概念

CSS 架構面臨的問題

  1. 全域作用域：所有選擇器都是全域的，容易產生衝突
  2. 特異度戰爭：!important 氾濫，選擇器越寫越長
  3. 死代碼累積：難以判斷哪些樣式仍在使用
  4. 副作用：修改一處樣式可能影響其他頁面
  5. 重複程式碼：缺乏有效的複用機制

主要方法論概覽

  • BEM：命名規範，強調區塊-元素-修飾符結構
  • OOCSS：物件導向 CSS，分離結構與外觀
  • SMACSS：分類組織（Base/Layout/Module/State/Theme）
  • ITCSS：倒三角架構，依特異度分層
  • Atomic CSS / Utility-First：原子化工具類別

架構總覽

BEM（Block Element Modifier）

BEM 是最廣泛採用的命名規範：

/* Block：獨立的元件 */
.card { }

/* Element：區塊的組成部分，以雙底線連接 */
.card__header { }
.card__body { }
.card__footer { }

/* Modifier：區塊或元素的變體，以雙連字號連接 */
.card--featured { }
.card__header--compact { }

<article class="card card--featured">
  <header class="card__header card__header--compact">
    <h2 class="card__title">標題</h2>
  </header>
  <div class="card__body">
    <p class="card__text">內容</p>
  </div>
  <footer class="card__footer">
    <button class="card__button">閱讀更多</button>
  </footer>
</article>

ITCSS 分層架構

Settings    → 變數、設定（$colors, $spacing）
Tools       → Mixin、函數
Generic     → Reset、Normalize
Elements    → 裸標籤樣式（h1, a, p）
Objects     → 佈局相關的通用模式
Components  → 具體的 UI 元件
Utilities   → 覆蓋與輔助類別（!important 允許）

CSS Modules

透過建置工具實現樣式的局部作用域：

/* Button.module.css */
.button {
  padding: 0.5rem 1rem;
  border-radius: 4px;
}

.primary {
  background: #007bff;
  color: white;
}

.large {
  padding: 0.75rem 1.5rem;
  font-size: 1.125rem;
}

import styles from './Button.module.css';

function Button({ variant, size, children }) {
  return (
    <button className={`${styles.button} ${styles[variant]} ${styles[size]}`}>
      {children}
    </button>
  );
}

命名規範比較

/* BEM */
.search-form__input--disabled { }

/* camelCase（CSS Modules） */
.searchFormInput { }

/* Utility */
.opacity-50.cursor-not-allowed.bg-gray-100 { }

/* SMACSS */
.is-disabled .l-search .input { }

應用場景

場景一：元件庫開發

使用 BEM + ITCSS 建立可移植的 UI 元件庫：

// components/_button.scss
.btn {
  display: inline-flex;
  align-items: center;
  padding: var(--btn-padding-y) var(--btn-padding-x);
  border-radius: var(--btn-radius);
  font-weight: 500;
  transition: all 0.2s;

  &--primary { background: var(--color-primary); color: white; }
  &--secondary { background: var(--color-secondary); color: white; }
  &--outline { border: 2px solid currentColor; background: transparent; }
  &--sm { --btn-padding-y: 0.25rem; --btn-padding-x: 0.5rem; }
  &--lg { --btn-padding-y: 0.75rem; --btn-padding-x: 1.5rem; }

  &__icon { margin-right: 0.5rem; }
  &__text { white-space: nowrap; }
}

場景二：大型 SPA 專案

使用 CSS Modules 確保元件樣式隔離，避免全域汙染。

場景三：快速原型開發

Tailwind CSS 適合快速迭代 UI，無需在 HTML 與 CSS 檔之間切換。

場景四：漸進式重構

既有專案可先引入 ITCSS 分層結構，再逐步將元件遷移至 BEM 或 CSS Modules。
