響應式網頁設計（Responsive Web Design，RWD）使網站能夠根據不同裝置的螢幕尺寸、解析度與方向，自動調整佈局與內容呈現。本文探討 Mobile-First 策略、媒體查詢技術與現代響應式設計的最佳實踐。

核心概念

響應式設計三大支柱

  1. 流動式佈局（Fluid Layout）：使用百分比、fr、vw 等相對單位取代固定像素
  2. 彈性圖片（Flexible Images）：圖片與媒體隨容器縮放
  3. 媒體查詢（Media Queries）：針對不同條件套用不同樣式

Viewport Meta 標籤

<meta name="viewport" content="width=device-width, initial-scale=1.0">

這是響應式設計的必要設定，告訴瀏覽器使用裝置實際寬度而非預設的 980px。

Mobile-First 策略

先為最小螢幕撰寫基礎樣式，再透過 min-width 媒體查詢逐步增強：

/* 基礎樣式（手機） */
.container {
  padding: 1rem;
}

.grid {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

/* 平板以上 */
@media (min-width: 768px) {
  .grid {
    flex-direction: row;
    flex-wrap: wrap;
  }
  .grid-item {
    flex: 1 1 calc(50% - 0.5rem);
  }
}

/* 桌機以上 */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
  .grid-item {
    flex: 1 1 calc(33.333% - 0.67rem);
  }
}

常見斷點（Breakpoints）

名稱寬度裝置類型
xs< 576px手機直式
sm≥ 576px手機橫式
md≥ 768px平板
lg≥ 1024px小型桌機
xl≥ 1280px桌機
2xl≥ 1536px大螢幕

架構總覽

媒體查詢語法

/* 寬度範圍 */
@media (min-width: 768px) and (max-width: 1023px) { }

/* 新語法（Level 4） */
@media (768px <= width < 1024px) { }

/* 方向 */
@media (orientation: landscape) { }

/* 高解析度螢幕 */
@media (min-resolution: 2dppx) { }
@media (-webkit-min-device-pixel-ratio: 2) { }

/* 偏好設定 */
@media (prefers-color-scheme: dark) { }
@media (prefers-reduced-motion: reduce) { }
@media (prefers-contrast: high) { }

/* 互動能力 */
@media (hover: hover) { }   /* 支援 hover 的裝置 */
@media (pointer: coarse) { } /* 觸控裝置 */

響應式排版

/* clamp() 實現流暢排版 */
h1 {
  font-size: clamp(1.5rem, 4vw + 0.5rem, 3rem);
}

p {
  font-size: clamp(1rem, 2vw + 0.5rem, 1.25rem);
  line-height: 1.6;
  max-width: 65ch; /* 最佳閱讀寬度 */
}

/* 容器查詢單位（新） */
.card-title {
  font-size: clamp(1rem, 3cqi, 1.5rem);
}

響應式圖片

<!-- srcset + sizes -->
<img src="image-800.jpg"
     srcset="image-400.jpg 400w,
             image-800.jpg 800w,
             image-1200.jpg 1200w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1000px) 50vw,
            33vw"
     alt="響應式圖片範例">

<!-- picture 元素 -->
<picture>
  <source media="(min-width: 1024px)" srcset="hero-desktop.webp">
  <source media="(min-width: 768px)" srcset="hero-tablet.webp">
  <img src="hero-mobile.webp" alt="Hero Image">
</picture>

Container Queries（容器查詢）

.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
}

@container card (min-width: 600px) {
  .card-title {
    font-size: 1.5rem;
  }
}

關鍵比較

Mobile-First vs Desktop-First

面向Mobile-FirstDesktop-First
基礎樣式手機桌機
媒體查詢min-width（向上增強）max-width（向下適配）
CSS 大小較精簡可能有冗餘覆蓋
效能手機載入較快手機可能載入多餘樣式
設計思維內容優先功能優先

媒體查詢 vs 容器查詢

面向Media QueryContainer Query
依據視窗大小容器大小
元件重用性低（依賴頁面上下文）高（自適應）
巢套全域局部
瀏覽器支援優秀現代瀏覽器
適用場景頁面佈局元件級響應

相對單位比較

單位基準適用場景
em父元素字體大小元件內部間距
rem根元素字體大小全域間距與字體
vw/vh視窗寬/高全螢幕佈局
%父元素尺寸流動佈局
ch“0” 字元寬度文字寬度限制
cqi容器行內尺寸容器查詢內

應用場景

場景一：響應式導覽（漢堡選單）

.nav-links {
  display: none;
}

.hamburger {
  display: block;
}

@media (min-width: 768px) {
  .nav-links {
    display: flex;
    gap: 1.5rem;
  }
  .hamburger {
    display: none;
  }
}

場景二：響應式表格

小螢幕將表格轉為卡片式：

@media (max-width: 600px) {
  table, thead, tbody, tr, th, td {
    display: block;
  }
  thead { display: none; }
  td::before {
    content: attr(data-label);
    font-weight: bold;
    display: block;
  }
}

場景三：深色模式

:root {
  --bg: #ffffff;
  --text: #1a1a1a;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1a1a1a;
    --text: #f0f0f0;
  }
}

場景四：減少動態效果

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
