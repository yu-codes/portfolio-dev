CSS 盒模型（Box Model）是網頁佈局的基礎概念，每個 HTML 元素都被視為一個矩形盒子，由內容區域、內距、邊框與外距四層構成。深入理解盒模型的運作機制是精準控制佈局的前提。

核心概念

盒模型四層結構

由內而外依序為：

  1. Content（內容區域）：元素的實際內容（文字、圖片等）
  2. Padding（內距）：內容與邊框之間的空間
  3. Border（邊框）：包圍 padding 的邊界線
  4. Margin（外距）：元素與其他元素之間的空間
.box {
  width: 300px;
  padding: 20px;
  border: 5px solid #333;
  margin: 15px;
}

box-sizing 屬性

box-sizing 決定 width 和 height 的計算方式：

  • content-box（預設）：width/height 僅計算內容區域

    • 實際佔用寬度 = width + padding-left + padding-right + border-left + border-right
    • 範例：300px + 40px + 10px = 350px

  • border-box：width/height 包含 padding 和 border

    • 實際佔用寬度 = width（padding 和 border 內含）
    • 範例：300px（內容自動縮減為 250px）
/* 全域設定 border-box（現代最佳實踐） */
*, *::before, *::after {
  box-sizing: border-box;
}

Margin 折疊（Margin Collapsing）

相鄰的垂直外距會合併為較大的那一個，而非相加：

.paragraph-1 { margin-bottom: 30px; }
.paragraph-2 { margin-top: 20px; }
/* 實際間距為 30px，不是 50px */

折疊發生的條件

  1. 相鄰兄弟元素：上方元素的 margin-bottom 與下方元素的 margin-top
  2. 父子元素：父元素的 margin-top 與第一個子元素的 margin-top（無 border/padding 阻隔時）
  3. 空元素：自身的 margin-top 與 margin-bottom

阻止折疊的方法

  • 父元素加上 overflow: hidden（或 auto）
  • 父元素加上 border 或 padding
  • 父元素設為 display: flex 或 display: grid
  • 使用 gap 取代 margin（在 flex/grid 容器中）

架構總覽

內距（Padding）詳解

/* 四邊不同值（上/右/下/左，順時針） */
padding: 10px 20px 15px 25px;

/* 上下 / 左右 */
padding: 10px 20px;

/* 四邊相同 */
padding: 10px;

/* 邏輯屬性（支援書寫方向） */
padding-block: 10px 20px;    /* block-start / block-end */
padding-inline: 15px 25px;   /* inline-start / inline-end */

外距（Margin）詳解

/* 自動水平置中 */
.centered {
  width: 800px;
  margin: 0 auto;
}

/* 負值 margin */
.pull-up {
  margin-top: -20px; /* 向上拉動 */
}

.bleed {
  margin-left: -1rem;
  margin-right: -1rem; /* 突破父容器邊界 */
}

邊框（Border）詳解

/* 基本邊框 */
border: 2px solid #333;

/* 各邊獨立 */
border-top: 3px dashed red;
border-left: none;

/* 圓角 */
border-radius: 8px;
border-radius: 50%; /* 圓形 */
border-radius: 10px 0 10px 0; /* 對角圓角 */

/* border-image */
border-image: linear-gradient(45deg, #f06, #9f6) 1;

Outline vs Border

.element {
  border: 2px solid blue;   /* 佔據空間，影響佈局 */
  outline: 2px solid red;   /* 不佔空間，不影響佈局 */
  outline-offset: 4px;      /* 可設定偏移 */
}

Display 與盒模型的關係

Displaywidth/heightmarginpadding換行
block有效四邊有效四邊有效獨占一行
inline無效僅水平有效四邊可見但垂直不推擠不換行
inline-block有效四邊有效四邊有效不換行

關鍵比較

content-box vs border-box

面向content-boxborder-box
width 計算僅內容內容 + padding + border
加 padding 時總寬增大內容區縮小
百分比寬度容易溢出可預測
預設使用瀏覽器預設現代開發推薦
直覺性低高

面向PaddingMargin
背景色範圍包含不包含
點擊區域包含不包含
折疊行為不折疊垂直折疊
負值不可可
auto 值不可可（置中）
用途內部空間元素間距

寬度計算方式

/* content-box 計算 */
.content-box {
  box-sizing: content-box;
  width: 300px;
  padding: 20px;
  border: 5px solid;
  /* 實際佔用：300 + 20*2 + 5*2 = 350px */
}

/* border-box 計算 */
.border-box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 5px solid;
  /* 實際佔用：300px */
  /* 內容區域：300 - 20*2 - 5*2 = 250px */
}

應用場景

場景一：等寬分欄佈局

border-box 讓百分比佈局不會因 padding 溢出：

*, *::before, *::after {
  box-sizing: border-box;
}

.column {
  width: 33.333%;
  padding: 1rem;
  float: left;
  /* 無需擔心 padding 導致總寬超過 100% */
}

場景二：負 Margin 的全幅區塊

在有限容器中實現全幅背景：

.container { max-width: 800px; margin: 0 auto; padding: 0 2rem; }

.full-bleed {
  margin-left: calc(-50vw + 50%);
  margin-right: calc(-50vw + 50%);
  padding-left: calc(50vw - 50%);
  padding-right: calc(50vw - 50%);
  background: #f5f5f5;
}

場景三：按鈕的 hit area 優化

使用 padding 增大可點擊區域，提升觸控體驗：

.icon-button {
  padding: 12px;
  /* 圖示僅 24px，但點擊區域為 48px（符合觸控最小要求） */
}

/* 或使用偽元素擴大 hit area */
.link {
  position: relative;
}
.link::after {
  content: '';
  position: absolute;
  inset: -8px;
}

場景四：避免 Margin Collapsing

/* 使用 flex 的 gap 取代 margin */
.article-list {
  display: flex;
  flex-direction: column;
  gap: 1.5rem; /* 不會折疊，間距可預測 */
}

/* 或使用 padding 隔離 */
.parent {
  padding-top: 1px; /* 阻止父子 margin 折疊 */
}
