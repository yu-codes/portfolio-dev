Web 基礎

HTML、CSS 與瀏覽器基礎

Web 基礎是所有前端開發的根基。無論框架如何演進——從 jQuery 到 React、從 Webpack 到 Vite——最終送到瀏覽器的仍然是 HTML、CSS 與 JavaScript。深入理解這三大支柱以及瀏覽器的運作機制，是區分「會用框架」與「真正理解前端」的分水嶺。許多進階問題的根源，往往是對 Web 基礎的理解不夠深入。

核心概念

HTML — 結構與語義：HTML 不只是排版工具，它是文件的語義結構。正確使用 <article>、<nav>、<main>、<aside> 等語義標籤，不僅幫助搜尋引擎理解頁面內容（SEO），也讓螢幕閱讀器能正確導航（Accessibility）。現代 HTML 還包含豐富的原生能力——<dialog> 取代自製 Modal、<details> 實現免 JS 的折疊面板、<input type="date"> 提供平台原生的日期選擇器。

CSS — 樣式與佈局：CSS 的核心機制包括層疊（Cascade）、繼承（Inheritance）、特異性（Specificity）與盒模型（Box Model）。現代 CSS 已遠超「設定顏色與字型」的範疇——Flexbox 與 Grid 提供強大的佈局能力、Custom Properties 實現動態主題切換、Container Queries 讓組件根據自身容器大小響應而非視窗大小、@layer 提供精確的樣式優先權控制。

瀏覽器運作機制：瀏覽器從接收 HTML 到顯示像素，經歷解析（Parse）→ 建構 DOM/CSSOM → 佈局（Layout）→ 繪製（Paint）→ 合成（Composite）的流水線。理解這個流程是效能最佳化的基礎——知道哪些 CSS 屬性只觸發 Composite（如 transform、opacity），哪些會觸發完整的 Layout 重排（如 width、height）。

HTTP 與網路層：Web 應用建構在 HTTP 協定之上。理解 HTTP/1.1、HTTP/2、HTTP/3 的差異，HTTPS 的加密機制，以及快取策略（Cache-Control、ETag、Service Worker），是前端效能最佳化與安全設計的基礎。

應用場景

SEO 最佳化：搜尋引擎爬蟲依賴 HTML 結構理解頁面內容。正確的 heading 層級（h1→h2→h3）、結構化資料（JSON-LD）、語義化標籤的使用，直接影響搜尋排名。<meta> 標籤提供 Open Graph 與 Twitter Card 資訊，確保社群分享時的預覽呈現。

Core Web Vitals 效能最佳化：Google 的 LCP（最大內容繪製）、FID/INP（互動延遲）、CLS（累積佈局位移）三大指標直接來自瀏覽器渲染機制。理解 Critical Rendering Path 才能有效最佳化——內聯關鍵 CSS 減少 LCP、延遲非關鍵 JS 改善 INP、為圖片設定尺寸避免 CLS。

漸進式增強（Progressive Enhancement）：以 HTML 核心功能為基礎，CSS 增添視覺體驗，JavaScript 加入互動增強。這確保在低階裝置或 JS 失效時，使用者仍能取得核心內容。<noscript> 提供降級方案，CSS @supports 檢測特性支援度。

離線體驗（PWA）：Service Worker 運作在瀏覽器層面，攔截網路請求實現離線快取。理解 HTTP 快取層級（Memory Cache → Disk Cache → Service Worker → Network）是設計快取策略的前提。配合 Cache API 與 IndexedDB，能構建完整的離線優先體驗。

跨瀏覽器相容性：不同瀏覽器引擎（Blink、WebKit、Gecko）對 Web 標準的實作進度不一。CSS 的 @supports、JavaScript 的 Feature Detection、以及 polyfill 策略，都建立在對瀏覽器基礎的深入理解之上。Baseline 標準幫助團隊判斷哪些功能已被廣泛支援。
