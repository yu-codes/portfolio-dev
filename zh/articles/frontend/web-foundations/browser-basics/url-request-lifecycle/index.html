當使用者在瀏覽器地址列輸入 URL 並按下 Enter 後，背後會經歷一系列複雜的處理步驟。從 URL 解析、DNS 查詢、TCP 連線、TLS 握手到最終的頁面渲染，理解這個完整的請求生命週期，是前端開發者進行效能優化與問題診斷的關鍵基礎。

核心概念

URL 結構解析

URL（Uniform Resource Locator）是網路資源的標準定址方式：

https://www.example.com:443/path/page?key=value&lang=zh#section
  │         │           │      │          │              │
scheme    host        port   path      query          fragment

各組成部分的意義：

  • Scheme（協定）：決定通訊方式（http、https、ftp、ws）
  • Host（主機）：可以是域名或 IP 位址
  • Port（埠號）：HTTP 預設 80，HTTPS 預設 443
  • Path（路徑）：伺服器上的資源位置
  • Query String（查詢字串）：以 ? 開頭的鍵值對參數
  • Fragment（片段）：以 # 開頭，僅在客戶端使用，不會傳送至伺服器

DNS 解析機制

DNS（Domain Name System）將域名轉換為 IP 位址，解析過程遵循層級查詢：

  1. 瀏覽器快取：檢查最近解析過的記錄
  2. 作業系統快取：檢查 hosts 檔案與 OS 層級快取
  3. 路由器快取：本地路由器的 DNS 快取
  4. ISP DNS 伺服器：網路服務提供商的遞迴解析器
  5. 根名稱伺服器：全球 13 組根伺服器
  6. 頂級域名伺服器：如 .com、.org 的 TLD 伺服器
  7. 權威名稱伺服器：最終回應域名對應的 IP

DNS 記錄類型

類型用途範例
AIPv4 位址對應example.com → 93.184.216.34
AAAAIPv6 位址對應example.com → 2606:2800:220:1:…
CNAME別名記錄www.example.com → example.com
MX郵件伺服器example.com → mail.example.com
NS名稱伺服器example.com → ns1.example.com
TXT文字記錄用於 SPF、DKIM 驗證

關鍵渲染路徑（Critical Rendering Path）中的阻塞資源：

  • CSS：阻塞渲染（Render Blocking）
  • JavaScript：阻塞解析（Parser Blocking），除非使用 async/defer

策略語法載入時機執行時機阻塞解析

策略用途語法

應用場景

場景一：效能優化 - 減少 DNS 查詢

<head>
  <!-- 預先解析第三方域名 -->
  <link rel="dns-prefetch" href="//fonts.googleapis.com">
  <link rel="dns-prefetch" href="//cdn.example.com">

  <!-- 預先建立完整連線 -->
  <link rel="preconnect" href="https://api.example.com" crossorigin>
</head>

場景二：關鍵資源預載

<!-- 預載關鍵字型 -->
<link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>

<!-- 預載關鍵 CSS -->
<link rel="preload" href="/css/critical.css" as="style">

<!-- 預取下一頁資源 -->
<link rel="prefetch" href="/next-page.html">

場景三：HTTP 快取策略

# 不變的靜態資源（含 hash 的檔名）
Cache-Control: public, max-age=31536000, immutable

# 可能更新的資源
Cache-Control: no-cache
ETag: "abc123"

# 不快取
Cache-Control: no-store

場景四：Navigation Timing API 效能測量

// 取得詳細的載入效能指標
const timing = performance.getEntriesByType('navigation')[0];

console.log({
  dns: timing.domainLookupEnd - timing.domainLookupStart,
  tcp: timing.connectEnd - timing.connectStart,
  tls: timing.secureConnectionStart > 0
    ? timing.connectEnd - timing.secureConnectionStart : 0,
  ttfb: timing.responseStart - timing.requestStart,
  download: timing.responseEnd - timing.responseStart,
  domParse: timing.domInteractive - timing.responseEnd,
  totalLoad: timing.loadEventEnd - timing.startTime
});

場景五：Service Worker 離線快取

// sw.js - 攔截請求實現離線存取
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  );
});
