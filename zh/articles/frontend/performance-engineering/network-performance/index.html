網路效能

深入解析前端網路效能優化策略，涵蓋資源載入優先權、快取策略、預連接技術與 HTTP 協定最佳化

網路效能是前端載入速度的關鍵瓶頸——無論程式碼多精簡，仍需透過網路傳輸到使用者裝置。從 DNS 解析到 TLS 握手，從資源優先權到快取策略，每個環節都是優化機會。理解瀏覽器的資源載入行為與 HTTP 協定特性，是做出正確優化決策的基礎。

應用場景

關鍵路徑最佳化

識別並最佳化載入關鍵路徑：preload LCP 圖片（通常是 Hero Banner）、preconnect 到 API 域名、inline 關鍵 CSS、defer 非關鍵 JS。確保 LCP 所需的所有資源以最高優先權、最短路徑到達瀏覽器。

離線快取策略

Service Worker 實作多層快取：Cache First 用於靜態資源（JS/CSS/圖片）確保離線可用；Network First 用於 API 請求確保資料新鮮；Stale-While-Revalidate 用於不常變的 API 回應平衡速度與新鮮度。

全球化部署

面向全球使用者的應用：靜態資源部署至多區域 CDN、API 使用就近的 Edge Worker、字型使用 font-display: swap 避免 FOIT、第三方腳本（分析、廣告）使用 defer/async 不阻塞首屏。地區差異可達 5-10 倍延遲改善。
