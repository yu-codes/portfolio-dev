懶載入

深入解析前端懶載入技術的實作策略，涵蓋圖片、元件、路由與資料的延遲載入模式與效能最佳化

懶載入（Lazy Loading）是延遲非關鍵資源載入的技術，直到使用者實際需要時才請求。從圖片懶載入到元件延遲水合，這個概念貫穿前端效能優化的各個層面。正確實施懶載入可大幅減少初始載入量、節省頻寬並改善所有 Web Vitals 指標。

核心概念

懶載入目標

資源類型懶載入方式觸發條件效益
圖片loading=“lazy” / IO進入視口節省頻寬
影片preload=“none”使用者播放大量節省
元件動態 import需要渲染時減少初始 JS
路由路由分割導航時減少初始 JS
資料無限捲動捲動觸發減少初始請求
第三方腳本defer/async/lazy閒置時不阻塞渲染
字型font-display: swap需要時避免 FOIT

原生懶載入

屬性元素值行為
loading="lazy"<img>lazy進入視口才載入
loading="lazy"<iframe>lazy進入視口才載入
loading="eager"<img>eager立即載入（預設）
fetchpriority="low"<img>low低優先權
fetchpriority="high"<img>highLCP 圖片
decoding="async"<img>async非同步解碼

關鍵比較

懶載入 vs 預載入

面向懶載入預載入
目的延遲非必要資源提前載入即將需要的
時機需要時才載入閒置時提前載入
效果減少初始載入減少後續等待
風險使用時需等待浪費頻寬
組合先懶載入，搭配預載入預測使用者下一步

應用場景

圖片密集型網站

電商產品列表可能有 50-100 張圖片。使用原生 loading="lazy" 搭配佔位符（固定寬高比 aspect-ratio）避免 CLS。首屏 4-6 張圖片使用 eager + preload，其餘全部懶載入。可節省 70%+ 的初始頁面大小。

重量級編輯器

富文本編輯器（如 Monaco、TipTap）體積龐大。在使用者點擊「編輯」按鈕時才動態載入編輯器模組，載入期間顯示靜態預覽或骨架屏。使用者不編輯時完全不載入相關 JS，節省 200KB+ 初始 Bundle。

社群 Feed 流

社群媒體的無限捲動 Feed：使用 IntersectionObserver 偵測接近底部時載入下一批、同時懶載入每則貼文的圖片/影片、配合虛擬列表回收已離開視口的 DOM 節點。確保即使 Feed 有千則內容，DOM 節點數仍維持常數。
