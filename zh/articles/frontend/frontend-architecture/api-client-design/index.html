API Client 是前端與後端通訊的核心抽象層，負責請求發送、回應處理、錯誤管理與快取策略。良好的 API Client 設計能統一通訊邏輯、簡化元件程式碼、提升型別安全與開發效率。從基礎的 fetch 封裝到進階的 Server State 管理，API Client 的設計深度反映了應用的成熟度。

應用場景

企業級 API 架構

大型應用的 API Client 需支援：多環境切換（dev/staging/prod）、請求簽名、加密傳輸、分散式追蹤 ID 注入、限流控制。以 Factory 模式建立不同配置的 client 實例，透過 IoC 容器管理依賴。

樂觀更新模式

列表刪除、按讚等操作適合樂觀更新：先在 UI 反映變更，再發送 API。失敗時自動回退（rollback）。TanStack Query 的 onMutate + onError + onSettled 提供了完整的樂觀更新生命週期。

離線優先應用

結合 IndexedDB 儲存離線資料、Background Sync API 排程離線請求、Service Worker 攔截網路請求提供快取回應。API Client 層透明處理線上/離線狀態切換，元件層無需關心網路狀態。
