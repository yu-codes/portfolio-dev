組件模型

UI 元件設計模式

組件模型是現代前端框架的基石，它將使用者介面拆解為獨立、可重用且可組合的建構單元。每個組件封裝了特定的 UI 邏輯與視覺呈現，透過定義明確的輸入（Props）與輸出（Events）介面實現組件間的通訊。理解組件模型的設計哲學，是寫出可維護、可擴展前端程式碼的關鍵。

核心概念

組件的本質：組件是一個將輸入（資料）映射為輸出（UI）的函式。在 React 中，它就是一個接收 props 並回傳 JSX 的函式；在 Vue 中，它是包含 template、script、style 的單檔案元件（SFC）。無論哪種實作方式，核心理念一致——給定相同的輸入，應產生相同的 UI 輸出。

組件的分類：

  • 展示型組件（Presentational）：負責「東西長什麼樣」。接收 props 並渲染 UI，不包含業務邏輯或副作用。例如 <Button>、<Card>、<Avatar>。
  • 容器型組件（Container）：負責「東西如何運作」。處理資料獲取、狀態管理與業務邏輯，將結果透過 props 傳遞給展示型組件。
  • 佈局型組件（Layout）：定義頁面結構與空間配置，透過 Slots 或 children 接收內容。例如 <PageLayout>、<Grid>、<Stack>。

組件的生命週期：每個組件都經歷建立（Mount）→ 更新（Update）→ 銷毀（Unmount）的完整生命週期。框架提供相對應的鉤子讓開發者在特定時機執行邏輯，如初始化資料獲取、清理訂閱等。

應用場景

設計系統（Design System）：組件模型是設計系統的技術實現。每個 UI 元素（Button、Input、Modal）對應一個組件，透過 Props 暴露可配置的 variants（primary/secondary）、sizes（sm/md/lg）與 states（disabled/loading）。Slots 則用於允許消費者注入自訂內容。

表單建構器：表單組件是組件模型的經典應用。一個 <FormField> 組件透過 Render Props 或 Scoped Slots 將驗證狀態暴露給自訂 UI，實現「邏輯共用、外觀各異」的效果。Container 組件負責整體表單狀態與提交邏輯。

資料表格：複雜表格需要大量的組件組合——<Table>、<TableHead>、<TableRow>、<TableCell>。Column 定義透過 Render Props 模式允許每一欄自訂渲染邏輯，而排序、篩選等功能則由 HOC 或 Hooks 注入。

動態佈局：Dashboard 類應用中，佈局本身就是一個接受 widget 組件作為 children 的容器。每個 widget 是獨立的組件實例，佈局組件負責 Grid 排列與拖拉互動，widget 組件只關心自身的資料展示。

微前端整合：在微前端架構中，每個子應用暴露為一個 Web Component 或框架組件。組件模型確保子應用之間的隔離——各自管理狀態與生命週期，只透過明確定義的 Props/Events 介面通訊。
