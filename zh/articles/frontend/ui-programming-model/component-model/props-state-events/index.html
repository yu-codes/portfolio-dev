Props、State 與事件

全面解析前端組件的資料傳遞機制：Props 輸入、State 狀態管理與事件通訊模型

Props、State 與事件是前端組件模型的三大支柱。Props 提供外部輸入，State 管理內部狀態，事件則實現組件間的通訊。理解這三者的運作機制與最佳實踐，是設計良好組件架構的基礎。

核心概念

三者的角色定位

概念方向可變性所有權類比
Props父 → 子唯讀（對接收方）父組件函式參數
State組件內部可變（透過 API）組件自身區域變數
Events子 → 父——回呼函式 / 訊號

Props 設計原則

原則說明好處
唯讀性子組件不應修改 props資料流可預測
單向資料流只從父流向子除錯容易
型別宣告明確定義接受的型別開發期檢查
預設值提供合理的預設API 更友好
最小化只傳遞必要的資料降低耦合
穩定參照避免每次渲染建立新物件/函式效能最佳化

State 分類

類型描述範例管理方式
UI State介面互動狀態開關、hover、focus區域 state
Form State表單輸入值文字欄位、勾選區域 state / 表單庫
Server State遠端資料API 回應React Query/SWR
URL State路由參數頁碼、篩選Router
Global State全域共享使用者、主題Context/Store

Props 設計最佳實踐

實踐說明
扁平化避免深層巢狀的 prop 物件
Boolean prop 語意使用正向命名（isVisible 而非 isHidden）
列舉限制使用聯合型別限定可選值
回呼命名使用 onXxx 命名慣例
避免過多 props超過 5-7 個考慮重構
文件化添加 JSDoc 或 Storybook

效能最佳化

問題原因解決方案
