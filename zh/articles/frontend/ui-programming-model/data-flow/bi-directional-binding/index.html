雙向綁定（Two-Way Binding）是一種讓 UI 元素與資料模型自動保持同步的機制。當資料變更時 UI 自動更新，當使用者操作 UI 時資料也自動更新。這種模式在表單處理中特別有用，但也帶來了可追蹤性的挑戰。

核心概念

雙向綁定的本質

方向流動觸發方式
Model → View資料更新驅動 UI 更新響應式系統/setState
View → Model使用者操作更新資料事件監聽/onChange

雙向綁定本質上是「單向綁定 + 事件處理」的語法糖：

// 雙向綁定（語法糖）
<input v-model="name">

// 等價的單向資料流展開
<input :value="name" @input="name = $event.target.value">

關鍵比較

單向綁定 vs 雙向綁定

面向單向綁定雙向綁定
資料流State → View（單方向）State ↔ View
程式碼量較多（手動處理事件）較少（語法糖）
可追蹤性高（明確知道何時何處更新）中（自動同步可能隱藏邏輯）
除錯容易較難追蹤更新源
效能控制細粒度（可 debounce）框架預設行為
表單處理較繁瑣簡潔直覺
複雜邏輯明確的控制流可能需要 watcher/effect
代表ReactVue v-model, Angular ngModel

效能考量

問題原因解決方案
頻繁更新每次按鍵都觸發debounce / .lazy
不必要的渲染父組件重新渲染memo / computed
大量表單欄位所有欄位一起更新分割為子組件
深層物件監聽deep watch 開銷使用扁平化結構

何時不用雙向綁定

場景原因替代方案
需要嚴格審計雙向綁定隱藏了更新源明確的 action dispatch
複雜的狀態轉換自動同步不適合狀態機 + 明確事件
跨組件同步雙向綁定只適合父子全域 store
效能關鍵區域需要精確控制更新頻率手動控制
