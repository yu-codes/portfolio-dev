JavaScript 核心

JavaScript 語言核心機制

JavaScript 的核心語言機制——閉包與作用域、原型與類別、ES Modules——構成了這門語言的骨幹。深入理解這些機制不僅能避免常見陷阱，更能讓開發者寫出優雅、高效且可維護的程式碼。

核心概念

閉包（Closure） 是 JavaScript 最強大也最容易被誤解的特性之一。當一個函式「記住」了它被定義時的詞法環境（Lexical Environment），即使該函式在其定義範圍之外被調用，仍能存取那些變數——這就是閉包。

function createCounter(initial) {
  let count = initial;
  return {
    increment: () => ++count,
    getCount: () => count
  };
}

閉包的應用無處不在：模組模式、事件處理器、柯里化（Currying）、記憶化（Memoization）。理解閉包也意味著理解 JavaScript 的記憶體管理——被閉包引用的變數不會被垃圾回收。

作用域（Scope） 在 JavaScript 中遵循詞法作用域（Lexical Scoping）規則。let 和 const 提供區塊作用域，而 var 則是函式作用域。暫時性死區（Temporal Dead Zone, TDZ）是 let/const 宣告前的不可存取區域，這是 ES6 引入的重要安全機制。

原型鏈（Prototype Chain） 是 JavaScript 物件導向的根基。每個物件都有一個內部連結 [[Prototype]]，當存取一個屬性時，引擎會沿著原型鏈向上查找。ES6 的 class 語法是原型繼承的語法糖，底層機制並未改變。

ES Modules 是 JavaScript 的官方模組系統。它提供靜態的 import/export 語法，使得打包工具能在編譯時分析依賴圖、執行 Tree Shaking。與 CommonJS 的 require() 不同，ES Modules 的綁定是「活的」（live bindings）——匯出值的變更會反映到所有匯入處。

架構總覽

JavaScript 核心機制的理解可以從「執行上下文」的角度統一觀察：

執行上下文（Execution Context） 是 JavaScript 引擎執行程式碼的基本單位。每當進入一個函式、模組或全域環境，引擎便建立一個新的執行上下文，包含：

  1. 變數環境（Variable Environment）：儲存 var 宣告與函式宣告（hoisting 的目標）
  2. 詞法環境（Lexical Environment）：儲存 let/const 宣告，形成作用域鏈
  3. this 繫結：根據調用方式決定 this 的值

原型系統的層次：

instance → Constructor.prototype → Object.prototype → null

當使用 class 語法時：

class Animal { speak() {} }
class Dog extends Animal { bark() {} }

// 實際的原型鏈：
// dog → Dog.prototype → Animal.prototype → Object.prototype → null

模組圖（Module Graph）： ES Modules 形成一個有向無環圖（DAG）。引擎分三階段處理模組：

  1. 建構（Construction）：解析模組並建立模組記錄
  2. 實例化（Instantiation）：分配記憶體空間給匯出值（但尚未賦值）
  3. 求值（Evaluation）：執行模組程式碼，填入實際值

應用場景

資料封裝與私有狀態：閉包提供了在 ES6 Private Fields 之前唯一可靠的私有變數機制。在函式庫開發中，利用閉包隱藏內部實作細節是常見模式。

高階函式與函數式程式設計：柯里化、部分應用（Partial Application）、函式組合（Compose）都建立在閉包之上。像 lodash/fp 或 ramda 這類函式庫大量運用這些技巧。

模組化設計：ES Modules 的靜態特性讓 IDE 能提供精確的自動完成、跳轉定義、重構支援。搭配 TypeScript 的 paths 配置，可以建立清晰的模組別名與層次化的匯入規則。

效能優化：理解原型鏈的查找成本，可以在效能敏感的場景中做出正確決策——例如將頻繁存取的方法定義在原型上而非實例上，或是使用 Object.create(null) 建立無原型的純淨字典。
