作用域與閉包

全面解析 JavaScript 作用域鏈、詞彙作用域與閉包的運作原理及實務應用

作用域決定了變數的可見性與生命週期，而閉包則是 JavaScript 最強大的特性之一，它讓函式能夠「記住」其建立時的環境。這兩個概念是理解模組化、資料封裝與許多設計模式的基礎。

核心概念

作用域類型

JavaScript 中有三種主要作用域：

作用域類型建立方式生命週期範例
全域作用域最外層程式碼程式結束前持續存在window 上的屬性
函式作用域function 宣告函式執行期間var 宣告的變數
區塊作用域{} 區塊區塊執行期間let/const 宣告的變數

詞彙作用域（Lexical Scope）

JavaScript 採用靜態（詞彙）作用域，意味著作用域在程式碼撰寫時就已決定，而非執行時：

全域作用域
├── 函式 A 的作用域
│   ├── 函式 B 的作用域（可存取 A 和全域的變數）
│   └── 變數 x
└── 變數 y

作用域鏈的查找規則：從當前作用域開始，逐層向外查找，直到全域作用域。若找不到則拋出 ReferenceError。

閉包（Closure）

閉包是函式與其詞彙環境的組合。當內部函式引用了外部函式的變數時，即使外部函式已執行完畢，這些變數仍會被保留。

閉包的三個必要條件：

  1. 存在巢狀函式
  2. 內部函式引用外部函式的變數
  3. 內部函式在外部函式作用域之外被執行

架構總覽

作用域鏈解析機制

步驟動作說明
1查找當前作用域在目前執行上下文的變數環境中尋找
2沿作用域鏈向上透過 [[OuterEnv]] 參照外層環境
3到達全域作用域最終的查找終點
4未找到拋出 ReferenceError（嚴格模式）

閉包的記憶體模型

函式物件
├── [[Code]]: 函式程式碼
├── [[Scope]]: 作用域鏈參照
└── [[Environment]]: 詞彙環境快照
    ├── 自由變數 a → 值
    ├── 自由變數 b → 值
    └── [[OuterEnv]] → 外層環境

引擎只會保留閉包實際引用的變數（在大多數現代引擎中），而非整個外部作用域的所有變數。

應用場景

模組模式（Module Pattern）

閉包是實現資訊隱藏的經典方式：

const counter = (() => {
  let count = 0; // 私有變數
  return {
    increment: () => ++count,
    getCount: () => count,
  };
})();

函式工廠

利用閉包建立客製化函式：

function multiplier(factor) {
  return (number) => number * factor;
}
const double = multiplier(2);
const triple = multiplier(3);

記憶化（Memoization）

閉包保存快取結果：

function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (!cache.has(key)) cache.set(key, fn(...args));
    return cache.get(key);
  };
}

事件處理與回呼

閉包讓回呼函式能存取定義時的上下文資料，常見於事件監聽器與非同步操作中。

常見陷阱

陷阱原因解決方案
迴圈中的 var共享同一變數使用 let 或 IIFE
記憶體洩漏閉包持有大物件參照及時解除參照、使用 WeakRef
意外共享狀態多個閉包共享外部變數使用獨立的封裝
