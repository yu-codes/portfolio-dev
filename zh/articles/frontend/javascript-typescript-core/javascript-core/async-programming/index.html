JavaScript 的非同步程式設計是前端開發中最重要的概念之一。由於 JavaScript 是單執行緒語言，非同步機制讓它能夠在等待 I/O 操作時不阻塞主執行緒，確保使用者介面的流暢性。

Promise 狀態機

Promise 有三種不可逆的狀態：

         ┌─── fulfilled（已實現）→ 觸發 .then(onFulfilled)
pending ─┤
         └─── rejected（已拒絕）→ 觸發 .then(_, onRejected) 或 .catch()
狀態說明可轉移至
pending初始狀態fulfilled 或 rejected
fulfilled操作成功完成無（最終狀態）
rejected操作失敗無（最終狀態）

async/await 本質

async 函式本質上是回傳 Promise 的函式，await 則是 .then() 的語法糖：

語法等價行為
async function f() {}回傳值自動包裝為 Promise
await promise暫停執行直到 Promise 解決
await 非 Promise 值等價於 await Promise.resolve(value)
try/catch等價於 .catch()

應用場景

並行控制

限制同時進行的非同步操作數量，避免壓垮伺服器或瀏覽器：

async function pooledExecution(tasks, concurrency) {
  const results = [];
  const executing = new Set();
  for (const task of tasks) {
    const p = task().then(r => { executing.delete(p); return r; });
    executing.add(p);
    if (executing.size >= concurrency) {
      await Promise.race(executing);
    }
  }
  return Promise.all(results);
}

重試機制

  • 設定最大重試次數
  • 每次重試間隔倍增（指數退避）
