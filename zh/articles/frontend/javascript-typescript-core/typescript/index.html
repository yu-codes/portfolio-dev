TypeScript 型別系統

TypeScript 的型別系統是一套圖靈完備的型別層語言，遠比表面上的「加個型別標註」來得深邃。從基礎的型別註解到進階的條件型別與型別推斷，TypeScript 讓開發者能在編譯時期捕捉錯誤、表達複雜的資料契約、並享受卓越的 IDE 支援。

核心概念

結構型別系統（Structural Typing） 是 TypeScript 與 Java/C# 最根本的差異。TypeScript 不關心型別的「名字」，只關心「結構」：

interface Point { x: number; y: number; }
interface Coordinate { x: number; y: number; }

// Point 與 Coordinate 完全相容，因為結構相同
const p: Point = { x: 1, y: 2 };
const c: Coordinate = p; // 合法

這種設計讓 TypeScript 能無縫相容 JavaScript 的鴨子型別（Duck Typing）慣例。

型別收窄（Type Narrowing） 是 TypeScript 控制流分析的核心能力。透過條件檢查、typeof、instanceof、自定義 Type Guard，TypeScript 能在不同的程式碼分支中自動收窄變數的型別：

function process(value: string | number) {
  if (typeof value === 'string') {
    // 此處 value 自動收窄為 string
    return value.toUpperCase();
  }
  // 此處 value 自動收窄為 number
  return value.toFixed(2);
}

泛型（Generics） 讓型別成為「參數化」的。就像函式接受值作為參數，泛型讓型別接受其他型別作為參數，實現真正的型別抽象與複用：

function identity<T>(value: T): T { return value; }
// T 會被推斷為傳入值的型別
const result = identity(42); // result: number

條件型別（Conditional Types） 與 infer 關鍵字讓 TypeScript 的型別系統具備了「程式設計」能力——可以根據型別的結構進行分支判斷與模式匹配。

架構總覽

TypeScript 型別系統的知識架構：

第一層：基礎型別

  • 原始型別：string、number、boolean、null、undefined、symbol、bigint
  • 物件型別：interface、type alias、陣列、元組（Tuple）
  • 特殊型別：any、unknown、never、void

第二層：型別運算

  • 聯合型別（Union）：A | B
  • 交叉型別（Intersection）：A & B
  • 字面值型別（Literal Types）："hello" | 42 | true
  • 模板字面值型別（Template Literal Types）：`${string}Handler`

第三層：泛型與約束

  • 泛型函式、泛型介面、泛型類別
  • 約束（Constraints）：<T extends SomeType>
  • 預設型別參數：<T = DefaultType>
  • 變異數（Variance）：協變、逆變、不變

第四層：進階型別程式設計

  • 條件型別：T extends U ? X : Y
  • 映射型別（Mapped Types）：{ [K in keyof T]: ... }
  • 遞迴型別（Recursive Types）
  • infer 型別推斷

第五層：型別系統實戰

  • Discriminated Unions + Exhaustive Checking
  • Builder Pattern 與 Fluent API 型別設計
  • 型別安全的事件系統
  • 型別層級的狀態機

應用場景

API 回應型別化：使用泛型搭配 Zod 或 io-ts 建立執行時驗證與編譯時型別的統一方案。API 回應不再是 any，而是經過驗證的強型別資料。

型別安全的狀態管理：利用 Discriminated Unions 定義應用狀態，搭配 Exhaustive Checking 確保所有狀態分支都被處理。遺漏的分支會在編譯時報錯而非在執行時爆炸。

函式庫 API 設計：透過泛型推斷與條件型別，讓函式庫的使用者無需手動標註型別就能獲得完整的型別提示。如 TanStack Query 的 useQuery 能根據 queryFn 的回傳值自動推斷 data 的型別。

大型專案漸進式強化：從 strict: false 開始，逐步啟用嚴格選項。搭配 @ts-expect-error 標記已知問題，利用 type-coverage 工具追蹤型別覆蓋率的提升進度。
