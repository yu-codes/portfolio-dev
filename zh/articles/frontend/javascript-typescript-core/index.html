JavaScript/TypeScript

JS/TS 核心語言特性

JavaScript 與 TypeScript 是現代前端開發的基礎語言。JavaScript 作為瀏覽器的原生語言，其獨特的原型繼承、閉包機制與事件驅動模型形塑了整個 Web 開發範式。TypeScript 則在 JavaScript 之上加入靜態型別系統，為大型專案帶來編譯時期的安全保障與卓越的開發者體驗。

核心概念

JavaScript 的語言本質：JavaScript 是一門多範式語言——它同時支援物件導向（透過原型鏈）、函數式（一級函式、閉包）與事件驅動程式設計。理解這三種範式如何在 JavaScript 中交互作用，是精通這門語言的關鍵。

單執行緒非同步模型：JavaScript 採用單執行緒搭配事件迴圈（Event Loop）的執行模型。這意味著所有程式碼在同一個執行緒上運行，但透過回呼佇列（Callback Queue）與微任務佇列（Microtask Queue）實現非阻塞的非同步操作。這種模型避免了多執行緒的競態條件問題，但也要求開發者理解非同步操作的排程順序。

TypeScript 的設計哲學：TypeScript 是 JavaScript 的「嚴格超集」——任何合法的 JavaScript 程式碼都是合法的 TypeScript。它的型別系統是「漸進式」的，允許團隊逐步增加型別覆蓋率。更重要的是，TypeScript 的型別在編譯後完全抹除，不會影響執行時效能。

型別系統的結構化特性：TypeScript 採用結構型別系統（Structural Typing），而非名義型別系統（Nominal Typing）。兩個型別只要結構相容就視為相同，這與 Java/C# 的名義型別截然不同，也是許多開發者初次使用 TypeScript 時的困惑來源。

架構總覽

JavaScript/TypeScript 的知識體系可以從三個核心領域來理解：

語言核心（Language Core）：

  • 作用域與閉包：詞法作用域、閉包的記憶體影響、模組模式
  • 原型與類別：原型鏈繼承、ES6 Class 語法糖、Mixin 模式
  • ES Modules：靜態匯入/匯出、動態 import()、循環依賴處理

執行模型（Runtime Model）：

  • 事件迴圈：Call Stack、Task Queue、Microtask Queue 的協作
  • Promise 與 async/await：非同步流程控制、錯誤處理、並行策略
  • Web Workers：真正的多執行緒計算、SharedArrayBuffer、Atomics

型別系統（Type System）：

  • 基礎型別與進階型別：Union、Intersection、Literal Types、Template Literal Types
  • 泛型與工具型別：Generic Constraints、Conditional Types、Mapped Types、infer
  • 型別收窄：Type Guards、Discriminated Unions、Exhaustive Checking

這三個領域彼此關聯但各有獨立的知識深度。語言核心是理解 JavaScript 行為的基礎；執行模型決定了非同步程式碼的正確撰寫方式；型別系統則提供了編譯時期的安全網。

應用場景

全端 TypeScript 開發：使用 TypeScript 貫穿前後端，搭配 tRPC 或 GraphQL Code Generator，可以實現從 API 定義到前端消費的端對端型別安全。一處修改，全鏈路的型別錯誤立即顯現。

高效能計算場景：當前端需要處理大量計算（如影像處理、3D 渲染、資料分析）時，Web Workers 提供了真正的平行計算能力。搭配 SharedArrayBuffer 可以實現零拷貝的跨執行緒資料共享。

函式庫開發：開發供他人使用的 JavaScript 函式庫時，TypeScript 的型別宣告檔（.d.ts）不僅是文件，更是 API 契約。良好的泛型設計能讓消費端享受完整的型別推斷。

漸進式型別化遷移：大型 JavaScript 專案可以逐步引入 TypeScript。從 JSDoc 型別註解開始，到 allowJs 混合模式，再到完全的 strict mode，每一步都能立即獲得回報。
