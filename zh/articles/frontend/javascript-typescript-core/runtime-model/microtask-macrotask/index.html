微任務與巨任務

詳解 JavaScript 微任務與巨任務的區別、優先級機制與實務排程策略

微任務（Microtask）與巨任務（Macrotask）是 JavaScript 事件迴圈中兩種不同優先級的非同步任務佇列。理解它們的差異與執行順序，是預測和控制非同步程式碼行為的關鍵。

核心概念

任務分類

分類常見來源優先級執行策略
同步任務一般程式碼最高立即執行
微任務Promise.then, queueMicrotask高當前巨任務結束後全部清空
巨任務setTimeout, 事件回呼低每輪事件迴圈取一個

微任務來源

API環境說明
Promise.then/catch/finally通用Promise 狀態改變時排入
queueMicrotask()通用直接排入微任務佇列
MutationObserver瀏覽器DOM 變更觀察
process.nextTickNode.js比微任務優先級更高
await 後續程式碼通用本質是 Promise.then

巨任務來源

API環境說明
setTimeout / setInterval通用計時器回呼
setImmediateNode.jsCheck 階段執行
requestAnimationFrame瀏覽器渲染前執行（特殊類別）
I/O 回呼Node.js檔案、網路 I/O
UI 事件瀏覽器click, scroll 等
MessageChannel通用通道訊息
postMessage瀏覽器跨窗口/Worker 通訊

執行順序模型

│            巨任務 (Task) 執行               │
│  ┌──────────────────────────────────────┐  │
│  │        同步程式碼逐行執行              │  │
│  │  (可能產生新的微任務或巨任務排程)      │  │
│  └──────────────────────────────────────┘  │
└────────────────────┬───────────────────────┘
                     │ 同步程式碼執行完畢
                     ▼
┌────────────────────────────────────────────┐
│         微任務佇列全部清空                  │
│  ┌─ microtask 1 ─┐                        │
│  ├─ microtask 2 ─┤  ← 執行中產生的新微     │
│  ├─ microtask 3 ─┤    任務也會在此輪清空   │
│  └─ microtask N ─┘                        │
└────────────────────┬───────────────────────┘
                     │ 所有微任務清空
                     ▼

Node.js 中的優先級

在 Node.js 中，任務優先級更為細緻：

process.nextTick（最高）
  ↓
微任務（Promise.then 等）
  ↓
巨任務（各階段：Timers → Poll → Check → Close）
層級佇列清空策略
1nextTick queue遞歸清空（可能餓死其他佇列）
2microtask queue遞歸清空
3timer queue取到期的回呼
4poll queueI/O 回呼
5check queuesetImmediate
6close queue關閉事件

關鍵比較

微任務 vs 巨任務執行特性

面向微任務巨任務
每輪執行數量全部清空通常一個
執行時機當前巨任務後下一輪迴圈
阻塞渲染是（如過多）不會（每個之間可渲染）
遞歸排程可能導致無限迴圈安全（每輪一個）
用途需要立即反應的操作延遲、定期執行
堆疊追蹤連續在同一個巨任務上下文獨立的堆疊

各排程 API 的選擇指南

需求推薦 API原因
盡快執行，渲染前queueMicrotask優先級最高的非同步
DOM 更新後反應MutationObserver微任務，DOM 變更後立即
下一幀渲染requestAnimationFrame與瀏覽器渲染同步
延遲執行setTimeout至少等一輪巨任務
讓出主執行緒scheduler.yield()新 API，保留優先級
低優先級工作requestIdleCallback瀏覽器閒置時

queueMicrotask vs Promise.resolve().then

面向queueMicrotask(fn)Promise.resolve().then(fn)
語意明確排入微任務透過 Promise 間接排入
效能略好（少一層包裝）多一個 Promise 物件
錯誤處理拋出即為未捕獲例外成為 rejected Promise
可讀性意圖更清晰較為 hacky

應用場景

批次 DOM 更新

利用微任務將多次 DOM 操作合併：

let pending = false;
let updates = [];

function scheduleUpdate(update) {
  updates.push(update);
  if (!pending) {
    pending = true;
    queueMicrotask(() => {
      const batch = updates.slice();
      updates = [];
      pending = false;
      applyBatch(batch);
    });
  }
}

避免微任務飢餓

無限產生微任務會阻塞渲染和巨任務：

情境風險解法
遞歸 Promise 鏈永不讓出主執行緒設定上限，改用 setTimeout
MutationObserver 觸發修改無限循環使用旗標防止重入
大量微任務排程UI 凍結分批使用巨任務
