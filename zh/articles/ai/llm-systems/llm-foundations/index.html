大型語言模型（Large Language Model, LLM）是當代人工智慧最具影響力的技術突破之一。從 GPT 到 LLaMA，從 BERT 到 Claude，這些擁有數十億至數兆參數的模型展現了驚人的語言理解與生成能力。本文將從基礎理論出發，全面探討 LLM 的核心原理、訓練流程、架構演進與工程實踐。

核心概念

什麼是語言模型

語言模型的本質是一個機率分布——給定前文，預測下一個 token 的機率：

$$P(x_t | x_1, x_2, …, x_{t-1})$$

這個看似簡單的目標（Next Token Prediction），在足夠大的資料與模型規模下，湧現出了理解、推理、創作等複雜能力。

規模定律（Scaling Laws）

OpenAI 在 2020 年發現的規模定律揭示了模型效能與三個因素的冪律關係：

因素說明影響
參數量（N）模型的可學習參數數量越大 → 損失越低
資料量（D）訓練 token 數量越多 → 損失越低
計算量（C）FLOPs（浮點運算次數）越大 → 損失越低

Chinchilla Scaling Law（DeepMind, 2022）進一步指出：最佳配置下，參數量與 token 數應按 1:20 的比例增長。例如 70B 模型需要約 1.4T tokens 訓練。

湧現能力（Emergent Abilities）

當模型規模超過某個臨界點時，會突然展現出小模型不具備的能力：

  • In-Context Learning：透過 Prompt 中的示例學習新任務
  • Chain-of-Thought Reasoning：逐步推理解決複雜問題
  • Instruction Following：理解並遵循自然語言指令
  • Code Generation：生成可執行的程式碼

這些能力並非顯式訓練，而是規模帶來的副產物。

架構總覽

Transformer 架構核心

現代 LLM 均基於 Transformer 架構，其核心機制包括：

Self-Attention（自注意力）： 讓序列中每個位置都能關注其他所有位置，捕捉長距離依賴。計算複雜度為 $O(n^2)$，其中 n 為序列長度。

Multi-Head Attention： 使用多個注意力頭（Head），每個頭學習不同的關注模式（語法、語義、位置等）。

Feed-Forward Network（FFN）： 每個位置獨立的全連接層，負責非線性轉換與知識儲存。研究顯示 FFN 層是模型儲存事實知識的主要位置。

三種 Transformer 變體

架構代表模型訓練目標適用任務
Encoder-onlyBERT, RoBERTaMasked LM理解任務（分類、NER）
Decoder-onlyGPT, LLaMACausal LM生成任務（對話、寫作）
Encoder-DecoderT5, BARTSeq2Seq條件生成（翻譯、摘要）

現代 LLM 幾乎全採用 Decoder-only 架構，因其在大規模下表現最佳且訓練效率高。

LLM 訓練三階段

[Stage 1: Pre-training（預訓練）]
    目標：學習語言知識與世界知識
    資料：數兆 token 的網路文本
    方法：Next Token Prediction
    成本：數百萬美元 GPU 時間
        ↓
[Stage 2: SFT（監督式微調）]
    目標：學習遵循指令、對話格式
    資料：高品質的指令-回答對（10K-100K）
    方法：Supervised Fine-Tuning
    成本：數千美元
        ↓
[Stage 3: RLHF/DPO（對齊訓練）]
    目標：符合人類偏好、拒絕有害請求
    資料：人類偏好比較資料
    方法：PPO / DPO / RLAIF
    成本：數萬美元

位置編碼演進

由於 Self-Attention 本身不包含位置資訊，需額外注入：

方法代表模型特點
絕對位置（Sinusoidal）原始 Transformer固定、不可學習
學習式絕對位置GPT-2, BERT可學習但長度固定
RoPE（旋轉位置編碼）LLaMA, Qwen支援外推、效能優異
ALiBiBLOOM線性偏置、簡潔

RoPE 已成為主流選擇，透過旋轉矩陣將位置資訊融入 Attention 計算，並可透過 NTK-aware 等方法擴展到更長序列。

開源 vs 閉源模型

維度開源（LLaMA, Mistral）閉源（GPT-4, Claude）
成本部署成本，無 API 費按 token 計費
隱私資料不外傳需信任供應商
客製化可微調、修改僅 Prompt/API
最新能力通常落後 3-6 月最前沿
維護負擔自行負責供應商處理
社群生態活躍（HF、vLLM）限於 API 生態
