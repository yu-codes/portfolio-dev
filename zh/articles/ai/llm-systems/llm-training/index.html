LLM 訓練

大型語言模型訓練技術

大型語言模型（LLM）的訓練是一項涉及海量資料處理、分散式運算與多階段優化的系統工程。從 GPT-3 的 1750 億參數到 GPT-4 的推測萬億級規模，LLM 的訓練流程已演變為結合預訓練、監督式微調與人類偏好對齊的完整技術棧。本文將系統性介紹 LLM 訓練的全貌。

核心概念

訓練三階段範式

現代 LLM 的訓練遵循三階段範式，每個階段有不同的目標與方法：

第一階段：預訓練（Pre-training）

預訓練是 LLM 獲取通用語言能力的基礎階段。模型在大規模語料庫上透過自監督學習（通常是 Next Token Prediction）學習語言的統計規律、世界知識與推理能力。

  • 資料規模：通常需要數兆（Trillion）個 token 的訓練語料
  • 計算成本：GPT-4 級別的模型需要數千張 GPU 訓練數月
  • 目標函數：Causal Language Modeling（自回歸語言模型）

第二階段：監督式微調（Supervised Fine-Tuning, SFT）

預訓練後的模型雖然具備語言能力，但無法有效遵循指令。SFT 階段使用高品質的「指令-回應」配對資料，教導模型理解並執行人類指令。

  • 資料特徵：人工撰寫或篩選的高品質對話資料
  • 資料規模：通常 10K-100K 筆高品質範例
  • 關鍵技術：Instruction Tuning、Chat Template 設計

第三階段：人類偏好對齊（Alignment）

透過 RLHF 或 DPO 等技術，將模型的輸出與人類偏好對齊，使其產生更有幫助、更安全的回答。

預訓練資料工程

高品質的訓練資料是 LLM 成功的關鍵，資料工程涵蓋：

階段工作內容工具/方法
資料收集網路爬取、書籍、程式碼Common Crawl、GitHub
資料清洗去重、過濾低品質內容MinHash、Perplexity Filter
資料混合不同來源的比例調配Domain Weighting
Tokenization文字轉換為 token 序列BPE、SentencePiece

訓練基礎設施

LLM 訓練需要大規模的分散式計算基礎設施：

  • 硬體：NVIDIA A100/H100 GPU 叢集，搭配高速 NVLink 與 InfiniBand 網路
  • 框架：Megatron-LM、DeepSpeed、FSDP
  • 並行策略：資料並行、張量並行、流水線並行的組合

分散式訓練策略

大型模型無法放入單一 GPU，需要多種並行策略的組合：

  • Data Parallelism（DP）：將資料分散到多個 GPU，每個 GPU 持有完整模型副本
  • Tensor Parallelism（TP）：將模型的張量運算切分到多個 GPU
  • Pipeline Parallelism（PP）：將模型的不同層分配到不同 GPU
  • ZeRO Optimization：分散優化器狀態、梯度與參數以節省記憶體

應用場景

場景一：企業級 LLM 訓練

大型科技公司從零訓練基礎模型，需要完整的資料工程管道、大規模 GPU 叢集與專業團隊。典型案例包括 OpenAI 的 GPT 系列、Google 的 Gemini 與 Meta 的 LLaMA。

場景二：領域特化模型

醫療、法律、金融等領域可透過 Continual Pre-training 在領域語料上繼續訓練，再經 SFT 適配特定任務。例如 Bloomberg 的 BloombergGPT 專注金融領域。

場景三：輕量化微調部署

中小企業透過 LoRA/QLoRA 等 PEFT 方法，以有限的計算資源快速將開源模型適配到自身業務，實現成本效益的最佳平衡。

場景四：研究與實驗

研究人員在較小規模上驗證新架構或訓練策略，如 Scaling Laws 實驗、新對齊算法的驗證等。
