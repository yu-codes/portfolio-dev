AI 工程

涵蓋從數學基礎到正式環境機器學習系統的完整 AI／ML 工程知識，包括資料工程、模型訓練、MLOps 與推論部署。

AI 工程連接了資料科學研究與正式環境機器學習系統之間的落差。作為一名 AI 工程師，你需要負責建構可擴展的 ML Pipeline、高效率訓練模型，並將模型部署到真實世界應用中提供服務。

本章節提供一條具結構性的學習路徑，帶你從數學基礎一路進展到可操作大規模 ML 系統的工程能力。

為什麼是 AI 工程？

機器學習正在改變每一個產業，但要將 ML 實驗真正轉化為可上線的正式系統，需要一套獨特且完整的工程能力：

  • 如何建構能處理 TB 級訓練資料的資料管線？
  • 如何在多 GPU 環境中高效率訓練模型？
  • 如何部署每秒可提供數百萬次預測的模型服務？
  • 如何監控模型表現並在正式環境中偵測模型漂移（Drift）？

子領域結構

每個領域都會再細分為 2 至 5 個子領域，以涵蓋更具體的技能範疇。

AI1：數學與 ML 基礎

  • ML 所需的線性代數（向量、矩陣、矩陣分解）
  • 微積分與最佳化（梯度、反向傳播）
  • 機率與統計（分布、貝氏推論）
  • 核心 ML 演算法（回歸、分類、分群）

AI2：資料工程

  • 資料管線設計（Batch vs Streaming）
  • ETL 最佳實踐與工具（Airflow、Spark）
  • 資料品質與驗證
  • 資料版本控制與血緣追蹤（Lineage）

AI3：特徵工程

  • 數值與類別特徵處理
  • 文字與影像特徵擷取
  • 特徵選擇與降維技術
  • 自動化特徵工程

AI4：建模與訓練

  • 傳統 ML 模型（Tree、Ensemble、SVM）
  • 神經網路架構（CNN、RNN、Transformer）
  • 超參數最佳化
  • Transfer Learning 與 Fine-tuning

AI5：深度學習框架

  • PyTorch 基礎與進階模式
  • TensorFlow／Keras 生態系
  • JAX 與函數式 ML
  • 自訂 Layer 與 Training Loop

AI6：評估與指標

  • 分類與回歸評估指標
  • 排名與推薦系統指標
  • Cross-validation 策略
  • 實驗追蹤（MLflow、W&B）

AI7：MLOps

  • ML Pipeline 編排
  • 模型版本控制與 Registry
  • ML 專用 CI/CD
  • Feature Store

AI8：推論與服務化

  • 模型 Serving 框架（TorchServe、TF Serving）
  • 模型最佳化（Quantization、Pruning）
  • Batch 與即時推論
  • GPU Serving 與水平擴展

AI9：可解釋性與公平性

  • SHAP、LIME 與可解釋性工具
  • 偏差偵測與緩解
  • 模型文件與 Model Card
  • Responsible AI 實踐

AI10：分散式訓練

  • Data Parallelism 與 Model Parallelism
  • 分散式訓練框架（Horovod、DeepSpeed）
  • 大型語言模型訓練
  • 雲端 ML 訓練服務

AI11：產品化

  • ML 模型 A/B Testing
  • 模型監控與 Drift Detection
  • Feedback Loop 與持續學習
  • ML 系統架構模式
