類別型與時序特徵工程

特徵工程是機器學習中最具創造性的環節之一。類別型變數和時間序列資料各有其獨特的挑戰：類別型變數需要轉換為模型可理解的數值表示，而時間序列資料則蘊含豐富的時間動態資訊等待被提取。本文將探討這兩大類特徵工程的核心技術。

核心概念

類別型變數編碼

類別型變數（Categorical Variables）是取有限離散值的變數，如顏色（紅/藍/綠）、國家、職業等。機器學習模型通常需要數值輸入，因此需要適當的編碼策略。

基礎編碼方法：

  • 標籤編碼（Label Encoding）：將類別映射為整數（0, 1, 2, …）。簡單但引入了不存在的序數關係，僅適合樹模型
  • 獨熱編碼（One-Hot Encoding）：每個類別創建一個二元特徵。不引入虛假的序數關係，但高基數（high cardinality）類別會導致維度爆炸
  • 序數編碼（Ordinal Encoding）：適用於有自然順序的類別（如教育程度：小學 < 國中 < 高中 < 大學）

進階編碼方法：

  • 目標編碼（Target Encoding）：用該類別對應的目標變數平均值取代。資訊量大但容易過擬合，需搭配正則化（如加入全局均值的平滑）
  • 頻率編碼（Frequency Encoding）：用類別出現的頻率取代。簡單且不會過擬合，但不同類別可能有相同頻率
  • 二進制編碼（Binary Encoding）：先標籤編碼再轉為二進制表示，維度為 log₂(類別數)
  • WOE（Weight of Evidence）編碼：金融風控中常用，基於好壞樣本比例的對數

處理高基數類別：

  • 合併低頻類別為「其他」
  • 使用 Hash Encoding 將類別映射到固定維度
  • 學習式嵌入（Entity Embedding）：透過神經網路學習類別的分散表示

時間序列特徵提取

時間序列資料的特徵工程需要從時間維度中提取有意義的資訊。

日期時間特徵：

  • 週期性特徵：小時、星期幾、月份、季節、是否假日
  • 循環編碼：使用 sin/cos 變換保留週期性（如小時：sin(2π×hour/24), cos(2π×hour/24)）
  • 時間距離：距離特定事件（如上次購買、註冊日期）的天數

滾動窗口特徵：

  • 統計量：滾動平均、標準差、最大值、最小值、中位數
  • 窗口大小選擇：短期窗口（7天）捕捉近期趨勢，長期窗口（90天）捕捉季節性
  • 指數加權移動平均（EWMA）：近期資料權重更大

滯後特徵（Lag Features）：

  • 用過去 t-1, t-2, …, t-k 的值作為特徵
  • 差分特徵：當前值與前一期的差值，捕捉變化趨勢
  • 自相關特徵：不同滯後期數的自相關係數

頻域特徵：

  • 傅立葉變換提取頻率成分
  • 小波變換捕捉不同尺度的模式
  • 功率譜密度分析

時間序列的特殊考量

  • 資料洩漏（Data Leakage）：絕對不能使用未來資訊作為特徵。交叉驗證必須使用時間序列分割（TimeSeriesSplit）
  • 非穩態性（Non-stationarity）：趨勢和季節性會導致統計特性隨時間改變，需先進行差分或去趨勢
  • 不等間距：感測器資料可能不等時間間隔，需要重採樣或專門處理

架構總覽

特徵工程的系統性流程：

  1. 類別型處理：

    • 識別類別型特徵（包括數值偽裝的類別，如郵遞區號）
    • 分析基數（低/中/高）和有無自然序
    • 選擇適當編碼方法
    • 處理訓練集未見的新類別（Unknown Category 策略）

  2. 時序特徵處理：

    • 解析日期時間欄位
    • 建立多尺度滾動統計量
    • 提取滯後與差分特徵
    • 確保無未來資訊洩漏

  3. 交互特徵：

    • 類別 × 時間的交互（如「週末 × 促銷商品」的銷售特徵）
    • 多類別的組合特徵

應用場景

場景一：電商推薦系統的類別處理

商品類別（數千種）使用 Target Encoding + 平滑；品牌（數萬種）使用 Entity Embedding 在深度模型中學習；城市使用 Frequency Encoding。使用者瀏覽序列則提取最近 N 次瀏覽的類別分佈作為特徵。

場景二：零售銷售預測

結合日期時間特徵（週幾、是否假期、發薪日附近）、滾動銷售統計（7天/28天均值）、同期比較（去年同月銷售）、促銷事件距離等。對商品類別使用 Target Encoding（以歷史平均銷量為目標）。

場景三：設備預測性維護

從感測器時序資料中提取滾動統計量（振動標準差增加可能表示軸承磨損）、頻域特徵（異常頻率成分）、趨勢斜率。設備型號和製造商使用標籤編碼（樹模型友好）。

場景四：金融信用評分

申請者職業（高基數）使用 WOE 編碼；教育程度使用序數編碼；歷史還款記錄提取滾動逾期次數、連續準時期數等時序特徵。這些特徵組合為 scorecard 模型的輸入。
