分類與迴歸評估指標

模型評估是機器學習工作流程中至關重要的環節。選擇正確的評估指標不僅影響模型選擇的決策，更直接關係到模型在實際應用中的表現。本文將系統性地介紹分類與迴歸問題中常用的評估指標，分析各指標的適用情境與限制。

核心概念

分類評估的基礎

分類問題的評估建立在混淆矩陣（Confusion Matrix）之上：

  • True Positive (TP)：正確預測為正類
  • True Negative (TN)：正確預測為負類
  • False Positive (FP)：錯誤預測為正類（型一錯誤）
  • False Negative (FN)：錯誤預測為負類（型二錯誤）

從混淆矩陣衍生出的核心指標：

  • Accuracy（準確率）：(TP + TN) / (TP + TN + FP + FN)
  • Precision（精確率）：TP / (TP + FP)，預測為正的樣本中實際為正的比例
  • Recall（召回率）：TP / (TP + FN)，實際為正的樣本中被正確預測的比例
  • F1 Score：Precision 與 Recall 的調和平均數

迴歸評估的基礎

迴歸問題評估預測值與真實值之間的差異：

  • 殘差（Residual）：預測值與真實值的差異
  • 偏差（Bias）：模型預測的系統性偏移
  • 變異數（Variance）：模型預測的波動程度

應用場景

場景一：信用卡詐欺偵測

詐欺交易通常佔比不到 1%，此時：

  • 不適合使用 Accuracy：即使全預測為非詐欺，準確率也可達 99%
  • 優先使用 AUC-PR：在正類稀少時比 AUC-ROC 更具鑑別力
  • 關注 Recall：確保大部分詐欺交易被偵測到
  • 監控 Precision：避免過多正常交易被誤判，影響用戶體驗

場景二：房價預測

連續值預測問題中：

  • 使用 RMSE 作為主要指標：與房價同量綱，直覺可理解
  • 搭配 MAE 降低異常值影響：豪宅價格不應過度影響整體評估
  • 使用 MAPE 進行跨區域比較：不同地區房價基數不同
  • 檢查 R² 確認模型解釋力：評估模型是否捕捉到價格變動的主要因素

場景三：多標籤文本分類

新聞分類等多標籤場景：

  • 使用 Micro-F1 評估整體表現：考慮所有類別的 TP/FP/FN 總和
  • 使用 Macro-F1 確保各類別公平：避免高頻類別主導評估結果
  • 使用 Hamming Loss 衡量標籤級別的錯誤率

場景四：時間序列預測

預測未來銷售量等場景：

  • 使用 RMSE 懲罰大幅偏差的預測
  • 使用 sMAPE 進行對稱的相對誤差評估
  • 使用 WAPE 對高銷量產品給予更大權重
  • 分時段評估指標，識別模型在不同時間段的表現差異

延伸閱讀

推薦資源

  • 論文：「The Relationship Between Precision-Recall and ROC Curves」— Davis & Goadrich (2006)
  • 論文：「Beyond Accuracy: Behavioral Testing of NLP Models」— Ribeiro et al. (2020)
  • 工具：scikit-learn 的 metrics 模組提供完整的評估指標實作
  • 視覺化：使用 Yellowbrick 庫進行模型評估視覺化

進階主題

  • 校準曲線（Calibration Curve）：評估模型機率預測的可靠性
  • 成本敏感評估（Cost-Sensitive Evaluation）：根據業務成本自定義評估指標
  • 統計顯著性檢定：使用 McNemar’s test 或 bootstrap 確認模型差異的顯著性
  • 線上評估指標：A/B 測試中的轉換率、CTR 等業務導向指標
