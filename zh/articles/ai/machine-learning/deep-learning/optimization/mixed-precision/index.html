混合精度訓練

混合精度訓練（Mixed Precision Training）是一種透過同時使用低精度（FP16/BF16）與高精度（FP32）浮點數進行訓練的技術。它能在幾乎不損失模型精度的情況下，大幅降低記憶體使用量並加速計算，是現代深度學習訓練的標準配備。

核心概念

浮點數格式

深度學習中常見的數值格式：

格式位元數指數位尾數位動態範圍精度
FP3232823±3.4×10³⁸高
FP1616510±6.5×10⁴中
BF161687±3.4×10³⁸低

為什麼使用混合精度

  • 記憶體節省：FP16 佔用空間為 FP32 的一半，可訓練更大模型或使用更大 Batch Size
  • 計算加速：現代 GPU 的 Tensor Core 對 FP16/BF16 有專門加速，吞吐量為 FP32 的 2-8 倍
  • 頻寬節省：減少記憶體存取帶來的延遲

混合精度的挑戰

直接使用 FP16 訓練會遇到兩大問題：

  1. 數值溢出/下溢：FP16 動態範圍小（最小正數約 6×10⁻⁸），梯度容易歸零
  2. 精度損失：累加大量小數值時捨入誤差累積，影響收斂

架構總覽

標準混合精度訓練流程

混合精度訓練的核心設計原則是「計算用低精度，累積用高精度」：

1. 維護 FP32 主權重（Master Weights）
2. 前向傳播：將權重轉為 FP16 計算
3. 計算 Loss 並進行 Loss Scaling
4. 反向傳播：使用 FP16 計算梯度
5. Unscale 梯度回 FP32
6. 使用 FP32 梯度更新 FP32 主權重

Loss Scaling 機制

Loss Scaling 是解決 FP16 梯度下溢的關鍵技術：

靜態 Loss Scaling：

  • 將 Loss 乘以固定倍數（如 1024）
  • 梯度相應放大，避免下溢為零
  • 更新前除以相同倍數還原

動態 Loss Scaling（推薦）：

  • 從較大 Scale Factor 開始（如 2¹⁵）
  • 若出現 Inf/NaN，跳過更新並減半 Scale
  • 若連續 N 步正常，加倍 Scale
  • 自動找到最佳 Scaling 範圍

AMP（Automatic Mixed Precision）

PyTorch AMP 提供簡潔的 API：

from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()
    
    with autocast():  # 自動選擇 FP16/FP32
        output = model(data)
        loss = criterion(output, target)
    
    scaler.scale(loss).backward()  # Scaled 反向傳播
    scaler.step(optimizer)          # Unscale + 更新
    scaler.update()                 # 調整 Scale Factor

運算精度分配策略

AMP 自動決定各運算的精度：

運算類型使用精度原因
矩陣乘法（Linear, Conv）FP16Tensor Core 加速
SoftmaxFP32數值穩定性
Layer NormalizationFP32累加需高精度
Loss 計算FP32避免溢出
EmbeddingFP16查表不受精度影響

關鍵比較

FP16 vs BF16

特性FP16BF16
動態範圍小（需 Loss Scaling）大（與 FP32 相同）
精度10 位尾數7 位尾數
Loss Scaling必要通常不需要
硬體支援V100+A100+、TPU
收斂穩定性需小心調整通常直接可用
典型加速比2-3×2-3×

建議：若硬體支援 BF16，優先使用 BF16，可避免 Loss Scaling 的複雜性。

應用場景

大規模預訓練

  • GPT、LLaMA 等大模型訓練標配 BF16 混合精度
  • 搭配 FSDP/ZeRO 進一步壓縮記憶體
  • H100 GPU 的 FP8 支援可再提升 2× 吞吐量

電腦視覺訓練

  • ResNet、ViT 等模型使用 FP16 AMP 可獲得接近 2× 加速
  • Batch Normalization 層自動保持 FP32

微調大型模型

  • QLoRA：4-bit 量化基礎模型 + BF16 LoRA 適配器
  • 顯著降低微調所需的 GPU 記憶體

推理加速

  • TensorRT 支援 FP16/INT8 推理
  • 混合精度推理在精度損失極小的情況下提供 2-4× 加速

邊緣裝置部署

  • 移動端 GPU 對 FP16 有良好支援
  • 配合量化技術可在低功耗裝置上運行大模型
