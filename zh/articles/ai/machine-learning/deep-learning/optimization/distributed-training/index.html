分散式訓練概念

隨著深度學習模型規模的急劇增長，單一 GPU 已無法滿足大規模模型的訓練需求。分散式訓練技術透過多台機器或多張 GPU 協同工作，大幅縮短訓練時間並突破記憶體限制。本文將深入探討資料平行、模型平行以及混合平行等核心策略。

核心概念

為什麼需要分散式訓練

現代深度學習模型（如 GPT-4、LLaMA 等）參數量動輒數十億甚至數兆，單一加速器面臨以下瓶頸：

  • 記憶體限制：模型參數、梯度與優化器狀態無法全部載入單張 GPU
  • 計算時間：數據量龐大時，單機訓練可能需要數週甚至數月
  • 吞吐量需求：生產環境要求快速迭代與實驗

基本術語

術語說明
Worker參與訓練的計算節點（GPU/TPU）
Parameter Server集中管理參數更新的伺服器架構
All-Reduce所有 Worker 同步梯度的通訊操作
Ring All-Reduce環形拓撲的高效梯度聚合方式
Gradient Accumulation累積多步梯度再更新，模擬大 Batch

同步 vs 非同步訓練

  • 同步訓練（Synchronous）：所有 Worker 完成前向與反向傳播後，統一聚合梯度再更新參數。一致性高但受最慢 Worker 影響。
  • 非同步訓練（Asynchronous）：各 Worker 獨立計算並更新參數，吞吐量高但可能出現 stale gradient 問題。

架構總覽

資料平行（Data Parallelism）

資料平行是最常見的分散式策略，核心思路為：

  1. 每個 Worker 持有完整模型副本
  2. 訓練資料被切分為多份，各 Worker 處理不同的 mini-batch
  3. 計算梯度後透過 All-Reduce 聚合
  4. 所有 Worker 使用相同的聚合梯度更新模型

優點：實作簡單、擴展性好
缺點：每個 Worker 需完整模型，記憶體仍為瓶頸

常見框架實現：

  • PyTorch DistributedDataParallel (DDP)
  • Horovod
  • TensorFlow MirroredStrategy

模型平行（Model Parallelism）

當模型過大無法放入單張 GPU 時，將模型拆分到多個裝置：

張量平行（Tensor Parallelism）：

  • 將單層的權重矩陣切分到多張 GPU
  • 如 Megatron-LM 將 Attention 和 MLP 層拆分
  • 通訊頻繁但記憶體效率高

流水線平行（Pipeline Parallelism）：

  • 將模型按層分配到不同 GPU
  • 使用 micro-batch 減少氣泡時間（bubble time）
  • GPipe、PipeDream 為代表性實現

混合平行策略

實際大規模訓練通常結合多種策略：

3D 平行 = 資料平行 × 張量平行 × 流水線平行

例如訓練 GPT-3 時：

  • 8 路張量平行（單節點內 8 張 GPU）
  • 64 路流水線平行（跨節點）
  • 資料平行複製多組以上結構

維度資料平行張量平行流水線平行

記憶體效率低（完整模型副本）高中

ZeRO 優化器

DeepSpeed 的 ZeRO（Zero Redundancy Optimizer）結合了資料平行與模型平行的優勢：

  • ZeRO Stage 1：切分優化器狀態
  • ZeRO Stage 2：切分優化器狀態 + 梯度
  • ZeRO Stage 3：切分優化器狀態 + 梯度 + 參數

ZeRO-3 可實現接近模型平行的記憶體效率，同時保持資料平行的簡潔性。

FSDP（Fully Sharded Data Parallel）

PyTorch 原生的 FSDP 類似 ZeRO-3：

  • 將模型參數、梯度、優化器狀態完全分片
  • 前向/反向傳播時動態收集所需參數
  • 計算完成後立即釋放，降低峰值記憶體

應用場景

大型語言模型預訓練

  • 使用 3D 平行 + ZeRO 訓練百億至兆級參數模型
  • 典型配置：數百至數千張 A100/H100 GPU
  • 訓練時間從數月縮短至數週

電腦視覺大模型

  • ViT-Giant、Stable Diffusion 等模型的訓練
  • 資料平行為主，配合梯度累積模擬超大 Batch Size

多模態模型

  • CLIP、Flamingo 等視覺-語言模型
  • 不同模態的 encoder 可分配到不同 GPU 組

科學計算與仿真

  • 氣象預測、蛋白質結構預測
  • AlphaFold 訓練使用 128 張 TPUv3

企業級微調

  • 使用 FSDP 或 DeepSpeed ZeRO 在有限 GPU 上微調大模型
  • 4-8 張 GPU 即可微調 70B 參數模型（配合 QLoRA）
