GPU 與多 GPU 訓練

深度學習的突破性進展離不開 GPU 的強大算力。隨著模型規模從百萬參數成長到千億參數，單一 GPU 已無法滿足訓練需求，多 GPU 乃至多節點的分散式訓練成為必要。本文將系統性地介紹 GPU 加速訓練的原理、多 GPU 並行策略，以及實務中的最佳實踐。

核心概念

GPU 加速的原理

GPU 相較於 CPU 的核心優勢在於大規模並行計算：

  • SIMT 架構：Single Instruction Multiple Threads，數千個核心同時執行相同操作
  • 高記憶體頻寬：GPU 的 HBM（High Bandwidth Memory）提供數 TB/s 的頻寬
  • 專用硬體：Tensor Core 專為矩陣乘法優化，提供數倍的 TFLOPS 提升
  • 批次處理：深度學習的 mini-batch 天然適合 GPU 的並行計算模式

GPU 記憶體構成

訓練時 GPU 記憶體的主要佔用：

  • 模型參數：權重與偏差（FP32 時每個參數 4 bytes）
  • 梯度：與參數大小相同
  • 優化器狀態：Adam 需要 2 倍參數大小（一階和二階動量）
  • 啟動值（Activations）：前向傳播的中間結果，用於反向傳播
  • 臨時緩衝區：通訊緩衝、計算暫存等

並行策略概述

  • 資料並行（Data Parallelism）：每個 GPU 持有完整模型，處理不同資料批次
  • 模型並行（Model Parallelism）：將模型分割到多個 GPU 上
  • 管線並行（Pipeline Parallelism）：將模型按層分割，形成流水線
  • 張量並行（Tensor Parallelism）：將單一層的運算分割到多個 GPU

應用場景

場景一：單 GPU 最大化效能

在單張 RTX 4090（24GB）上訓練中型模型：

  • 啟用 混合精度（AMP）：torch.cuda.amp.autocast(dtype=torch.bfloat16)
  • 使用 梯度累積：等效 batch size 放大 4-8 倍而不增加記憶體
  • 啟用 梯度檢查點：用 30% 的額外計算換取 60% 的記憶體節省
  • 使用 Flash Attention：減少注意力計算的記憶體佔用從 O(n²) 到 O(n)
  • 優化 DataLoader：num_workers=4, pin_memory=True, prefetch_factor=2

場景二：多 GPU 訓練中型模型

使用 4x A100 80GB 訓練 7B 參數模型：

  • 使用 FSDP（ZeRO-3）：分片參數、梯度與優化器狀態
  • 設定 sharding_strategy=FULL_SHARD：記憶體效率最大化
  • 配合 activation checkpointing：進一步減少啟動值記憶體
  • 使用 BF16 混合精度：A100 的 BF16 Tensor Core 效能最佳
  • 通訊使用 NCCL 後端搭配 NVLink 高速互聯

場景三：千億參數模型訓練

使用數百張 GPU 訓練 175B+ 參數模型：

  • 採用 3D Parallelism：
    • Tensor Parallel = 8（節點內 NVLink 連接的 GPU）
    • Pipeline Parallel = 4-8（跨節點）
    • Data Parallel = 剩餘 GPU 數量
  • 使用 Megatron-LM 或 DeepSpeed 框架
  • 實作 Interleaved Pipeline Schedule 減少管線氣泡
  • 設定 梯度累積步數 確保全域 batch size 足夠大
  • 啟用 Elastic Training 容錯機制應對節點故障

場景四：推理優化與部署

將訓練好的大模型高效部署：

  • 使用 量化：INT8/INT4 量化減少模型大小與推理延遲
  • 使用 KV-Cache 優化自回歸生成的記憶體使用
  • 實作 Continuous Batching：動態批處理提升 GPU 利用率
  • 使用 vLLM 或 TensorRT-LLM 推理引擎
  • 設定 Tensor Parallel 將大模型分佈到多張推理 GPU
