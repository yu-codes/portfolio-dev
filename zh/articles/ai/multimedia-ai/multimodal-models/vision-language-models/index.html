視覺語言模型

視覺語言模型（Vision-Language Models, VLM）是連接視覺與語言兩大模態的 AI 系統，能夠同時理解影像內容與文字語意。從 CLIP 的對比學習到 GPT-4V 的多模態推理，VLM 正在重新定義機器對視覺世界的理解方式。

核心概念

多模態對齊

VLM 的核心挑戰是將視覺與語言兩個異質模態映射到共享的語意空間：

  • 對比式對齊：CLIP 風格，透過對比學習拉近匹配的圖文對
  • 生成式對齊：透過圖像描述生成（captioning）隱式學習對齊
  • 融合式對齊：使用交叉注意力（Cross-Attention）深度融合兩模態

視覺編碼器

將影像轉換為模型可處理的視覺 token：

$$V = \text{VisualEncoder}(I) \in \mathbb{R}^{N \times d}$$

常用架構包括 ViT、CLIP ViT、SigLIP 等。視覺 token 數量影響計算成本與細節保留。

模態融合策略

  • 早期融合（Early Fusion）：在底層就混合視覺與文字 token
  • 晚期融合（Late Fusion）：各自編碼後在高層融合
  • 交叉注意力融合：文字 token 透過 cross-attention 查詢視覺特徵
  • 投影式融合：視覺特徵經線性投影後作為 LLM 的輸入前綴

指令跟隨能力

現代 VLM 透過視覺指令微調（Visual Instruction Tuning）獲得遵循多模態指令的能力，能回答關於影像的開放式問題、執行推理、生成描述等。

架構總覽

CLIP（2021）

OpenAI 的對比式語言-影像預訓練模型，在 4 億圖文對上訓練：

  • 雙編碼器架構：ViT（視覺）+ Transformer（文字）
  • 對比學習目標：最大化匹配對的餘弦相似度
  • 零樣本遷移：無需微調即可進行開放詞彙分類
  • 應用基礎：為後續 VLM 提供視覺編碼器

BLIP / BLIP-2（2022-2023）

BLIP：統一理解與生成的視覺語言模型，使用 CapFilt 策略清洗網路爬取的圖文資料。

BLIP-2：引入 Q-Former 作為視覺與語言之間的橋接模組，以少量可訓練參數連接凍結的視覺編碼器與 LLM，大幅降低訓練成本。

LLaVA（2023）

Visual Instruction Tuning 的先驅工作：

  1. 使用 CLIP ViT 編碼影像
  2. 線性投影層將視覺 token 映射至 LLM 空間
  3. 與文字 token 拼接後輸入 LLaMA/Vicuna
  4. 兩階段訓練：預訓練對齊 + 指令微調

LLaVA-1.5、LLaVA-NeXT 持續改進，在多個 VLM 基準上表現優異。

GPT-4V / GPT-4o（2023-2024）

OpenAI 的多模態大模型，展現前所未有的視覺理解與推理能力：

  • 複雜場景的詳細描述與推理
  • 圖表、文件、程式碼的理解
  • 空間推理與計數
  • 多輪視覺對話

Gemini（2023-2024）

Google 的原生多模態模型，從頭開始聯合訓練視覺與語言，而非後期融合：

  • 支援影像、影片、音頻、文字的統一處理
  • 長上下文窗口支援長影片理解
  • Gemini 1.5 Pro 支援高達 100 萬 token

開源生態

  • InternVL 2：上海 AI Lab 的開源 VLM，多項基準 SOTA
  • Qwen-VL：阿里的多模態模型系列
  • CogVLM：清華的視覺語言模型
  • Phi-3 Vision：微軟的小型多模態模型

編碼器式 vs 解碼器式

編碼器式（CLIP）適合檢索、分類等判別任務；解碼器式（LLaVA）適合生成、問答等開放式任務。現代趨勢是結合兩者優勢。

凍結 vs 端到端訓練

凍結視覺編碼器（BLIP-2）節省計算但限制視覺表示的適應性；端到端訓練視覺編碼器（InternVL）效果更好但成本更高。

應用場景

視覺問答（VQA）

回答關於影像的自然語言問題，從簡單的物件識別到複雜的推理與常識判斷，是 VLM 最核心的應用。

影像描述生成

為影像自動生成自然語言描述，用於無障礙輔助（為視障者描述圖片）、社群媒體內容標注等。

文件與圖表理解

解析學術論文中的圖表、商業簡報的視覺內容、表格資料的擷取等，協助知識工作者處理多模態文件。

視覺化程式設計

根據 UI 設計稿自動生成前端程式碼、根據手繪草圖產生視覺化圖表等，連接視覺設計與程式實現。

機器人與具身智能

將視覺場景理解與語言指令結合，使機器人能理解「把紅色杯子放到桌上」等自然語言指令，並在視覺環境中執行。
