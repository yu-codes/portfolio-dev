硬體加速

GPU/TPU 硬體加速

硬體加速是 AI 運算效能的物理基礎。深度學習的核心運算——大規模矩陣乘法和卷積——具有高度平行性，這使得專為平行處理設計的加速器能比通用 CPU 快數十到數百倍。從 NVIDIA 的 CUDA GPU 到 Google 的 TPU，再到各家新創的專用 ASIC，硬體加速技術的演進直接推動了 AI 能力的飛躍。

核心概念

GPU 架構原理：現代 GPU 由數千個運算核心（CUDA Cores / Streaming Multiprocessors）組成，設計哲學是「用大量簡單核心實現高度平行」。NVIDIA H100 擁有 132 個 SM，每個 SM 包含 128 個 CUDA Cores 和 4 個 Tensor Cores。Tensor Cores 是專為矩陣乘加運算（MMA）設計的硬體單元，可在一個時脈週期內完成 4×4 或更大的矩陣運算。

TPU 架構原理：Google 的 Tensor Processing Unit 採用脈動陣列（Systolic Array）架構，資料在處理單元間以固定節奏流動，避免了 GPU 中的指令排程開銷。TPU v4 的脈動陣列為 128×128，特別適合大批次的矩陣乘法。TPU 透過 ICI（Inter-Chip Interconnect）組成 Pod，提供超大規模的互聯頻寬。

記憶體層級：AI 加速器的效能通常受記憶體頻寬限制（Memory-Bound）。HBM（High Bandwidth Memory）提供數 TB/s 的頻寬，但容量有限。理解記憶體層級（Registers → Shared Memory/L1 → L2 → HBM）對核心優化至關重要。FlashAttention 正是利用 SRAM（Shared Memory）避免反覆讀寫 HBM 而實現的突破。

混合精度訓練：利用較低精度（FP16/BF16）進行運算，在維持數值穩定性的同時獲得速度提升和記憶體節省。BF16 保留了 FP32 的指數範圍（8 bits exponent），因此更不容易溢出（overflow），已成為大模型訓練的標準精度。

架構總覽

現代 AI 加速器的記憶體與運算架構：

NVIDIA H100 SXM 架構：
┌─────────────────────────────────────────┐
│  132 Streaming Multiprocessors (SMs)     │
│  ┌─────────────────────────────────┐    │
│  │ 128 FP32 CUDA Cores per SM      │    │
│  │ 4 Fourth-Gen Tensor Cores per SM │    │
│  │ 256KB Register File per SM       │    │
│  │ 256KB L1/Shared Memory per SM    │    │
│  └─────────────────────────────────┘    │
│  50MB L2 Cache                           │
│  80GB HBM3 @ 3.35 TB/s                  │
│  NVLink 4.0: 900 GB/s (bi-directional)  │
│  PCIe Gen5: 128 GB/s                    │
└─────────────────────────────────────────┘

運算強度分析（Arithmetic Intensity）：衡量每 Byte 記憶體存取所對應的浮點運算次數（FLOPs/Byte）。當運算強度低於硬體的「屋頂線」（Roofline）時，工作負載為記憶體頻寬受限；反之為運算受限。大多數 Transformer 的 Attention 操作為記憶體受限，而 FFN 層為運算受限。

編譯器優化：XLA（Accelerated Linear Algebra）、TVM 和 Triton（OpenAI）等編譯器將高階計算圖轉換為高效的硬體指令，自動進行算子融合（Operator Fusion）、記憶體規劃和排程優化。

精度格式比較：FP32（32 bit）是基準；TF32（19 bit，NVIDIA 獨有）在 Tensor Core 上自動使用；BF16（16 bit）是訓練標配；FP8（8 bit，H100 支援）用於進一步加速訓練和推論；INT8/INT4 主要用於推論量化。

CUDA vs ROCm vs OneAPI：CUDA 生態最成熟，絕大多數 AI 框架和函式庫都有 CUDA 實作；ROCm 是 AMD 的開源替代，相容性持續改善但仍有差距；OneAPI 是 Intel 的跨架構方案，主要支援 Gaudi 和 Arc GPU。

應用場景

大模型訓練叢集：Meta 的 LLaMA 3 訓練使用 16,384 張 H100 GPU，配合自訂的 RDMA 網路，實現超過 90% 的 MFU（Model FLOPs Utilization）。這需要精密的通訊-運算重疊技術和故障恢復機制。

推論加速：TensorRT 將模型編譯為 H100 優化的 engine，利用 FP8 Tensor Core 和 kernel auto-tuning 實現比原生 PyTorch 快 3-5 倍的推論速度。

科學計算：分子動力學模擬（如 GROMACS）和氣候模型利用 GPU 的平行能力加速數值計算，某些場景可達 100 倍以上加速比。

邊緣 AI 加速：NVIDIA Jetson（Orin）、Apple Neural Engine 和 Qualcomm Hexagon DSP 為手機和 IoT 裝置提供功耗受限下的 AI 加速。
