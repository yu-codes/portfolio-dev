GPU 與 TPU 加速運算

GPU（Graphics Processing Unit）與 TPU（Tensor Processing Unit）是當前深度學習加速運算的兩大核心硬體平台。理解其架構差異、適用場景與程式設計模型，對於設計高效的 AI 基礎設施至關重要。

核心概念

GPU 架構基礎：現代 GPU 包含數千個 CUDA 核心，採用 SIMT（Single Instruction, Multiple Threads）執行模型。NVIDIA 的 GPU 架構經歷了 Pascal → Volta → Ampere → Hopper → Blackwell 的演進，每代引入新的計算能力。

Tensor Core：自 Volta 架構起，NVIDIA 引入專門的 Tensor Core 用於矩陣乘累加（MMA）運算。Hopper 架構的第四代 Tensor Core 支援 FP8 精度，單 GPU 可達數百 TFLOPS 的張量運算能力。

TPU 架構基礎：Google 的 TPU 採用脈動陣列（Systolic Array）架構，專為矩陣運算優化。TPU v4 包含兩個 TensorCore，每個含有 128x128 的 MXU（Matrix Multiply Unit），配備高頻寬記憶體（HBM）。

記憶體層次結構：GPU 擁有暫存器 → 共享記憶體 → L1/L2 快取 → HBM 的多層記憶體；TPU 則簡化為暫存器 → SRAM → HBM 的結構。記憶體頻寬常是效能瓶頸。

互連技術：NVIDIA NVLink 提供 GPU 間高速連接（900 GB/s per GPU on Hopper），NVSwitch 實現全連接拓撲。Google 的 ICI（Inter-Chip Interconnect）連接 TPU 晶片形成 Pod。

CUDA 程式設計模型：CUDA 將運算組織為 Grid → Block → Thread 的層次結構。開發者需管理記憶體搬移（Host↔Device）、核心啟動配置、共享記憶體使用等。cuDNN、cuBLAS 等程式庫封裝了常見深度學習運算的高效實現。

XLA 編譯器：TPU 使用 XLA（Accelerated Linear Algebra）編譯器將高階運算圖轉換為底層指令。JAX 框架透過 XLA 提供統一的 GPU/TPU 程式設計介面。

核心融合（Kernel Fusion）：將多個逐元素運算合併為單一核心，減少記憶體存取次數。FlashAttention 即是一個成功的融合核心範例。

效能功耗比：TPU 在特定工作負載（如大型 Transformer）上具有較佳的效能功耗比，而 GPU 在靈活性和泛用性上勝出。

應用場景

大規模模型訓練：GPT-4 使用數千張 A100/H100 GPU 訓練；Google 的 PaLM 和 Gemini 系列在 TPU v4 Pod 上訓練。硬體選擇往往取決於雲端供應商和團隊技術棧。

即時推理服務：GPU 的 TensorRT 推理引擎可將延遲優化至毫秒級。TPU 在 Google 內部服務（Search、YouTube）中大量用於推理。

科學計算與模擬：GPU 的 FP64 能力使其適用於分子動力學、天氣預報等需要雙精度計算的場景。TPU 則在特定科學計算（如 AlphaFold）中表現優異。

邊緣部署：NVIDIA Jetson 系列將 GPU 帶入邊緣場景；Google Edge TPU 則用於手機和 IoT 裝置上的輕量推理。

多模態工作負載：影片處理、3D 渲染與 AI 的混合工作負載更適合 GPU，因其具備完整的圖形處理管線。
