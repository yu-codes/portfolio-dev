資料平行訓練

資料平行訓練（Data Parallelism）是分散式深度學習中最常見且最直觀的平行化策略。其核心思想是將訓練資料分割至多個計算裝置，每個裝置持有完整的模型副本，並在各自的資料子集上進行前向與反向傳播，最後透過梯度聚合來同步模型參數。

核心概念

資料平行訓練的基本原理可分為以下幾個步驟：

資料分割（Data Partitioning）：將一個大型 mini-batch 均勻分配到 N 個 GPU 上，每個 GPU 處理 batch_size / N 筆樣本。這種方式能夠線性擴展訓練吞吐量。

模型複製（Model Replication）：每個計算裝置持有一份完整的模型參數副本。在訓練開始時，所有裝置的模型參數保持一致。

梯度同步（Gradient Synchronization）：在反向傳播完成後，各裝置需要交換並聚合梯度。最常用的方法是 AllReduce 操作，將所有裝置的梯度取平均後分發回每個裝置。

參數更新（Parameter Update）：每個裝置使用聚合後的梯度獨立更新本地模型參數，由於梯度相同且初始參數一致，更新後的模型仍保持同步。

PyTorch 的 DistributedDataParallel（DDP）是目前最主流的實現方式。相較於舊版的 DataParallel（DP），DDP 使用多進程而非多執行緒，避免了 Python GIL 的限制，且支援多機多卡的擴展。

通訊後端選擇：NCCL（NVIDIA Collective Communications Library）是 GPU 間通訊的首選後端，Gloo 則適用於 CPU 訓練場景。

Ring-AllReduce 演算法：將 N 個裝置排列成環形拓撲，每個裝置分別將梯度分段傳遞給下一個節點，經過 2(N-1) 步完成梯度聚合，通訊量與節點數無關。

Gradient Bucketing：DDP 將參數梯度分組打包，在反向傳播過程中重疊計算與通訊，有效隱藏通訊延遲。

同步與非同步模式：同步 SGD 確保所有裝置使用相同梯度更新，收斂性有理論保證；非同步 SGD 允許裝置獨立更新，吞吐量更高但可能影響收斂品質。

線性擴展性：理論上 N 張 GPU 可獲得 N 倍加速，但實際因通訊開銷，擴展效率通常在 85%-95% 之間。批次大小增加時需配合學習率調整策略（如 Linear Scaling Rule）。

大批次訓練挑戰：當總批次大小過大時，可能導致泛化能力下降。LARS/LAMB 優化器專門設計來解決大批次訓練的收斂問題。

應用場景

影像分類大規模訓練：ImageNet 訓練是資料平行的經典應用。透過 256 張 GPU 和大批次策略，可在數分鐘內完成 ResNet-50 的訓練。

自然語言預訓練：BERT、GPT 等模型的預訓練階段大量使用資料平行。當模型能完整放入單張 GPU 記憶體時，資料平行是最高效的選擇。

推薦系統嵌入訓練：大規模推薦模型（如 DLRM）的稠密層適合用資料平行處理，而稀疏嵌入層則可能需要結合模型平行。

聯邦學習場景：資料平行的概念可延伸至聯邦學習，各客戶端在本地資料上訓練後聚合模型更新，保護資料隱私。

超參數搜索：多個工作節點可同時嘗試不同超參數組合，加速模型調優過程。
