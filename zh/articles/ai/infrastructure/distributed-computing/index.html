分散式運算

分散式訓練運算

分散式運算是現代大規模 AI 模型訓練的基石技術。當單一 GPU 的記憶體和運算能力無法滿足模型訓練需求時，我們需要將工作負載分散到多個運算節點上協同完成。分散式運算的核心挑戰在於如何在保持數學等價性的同時，最大化硬體利用率並最小化通訊開銷。

核心概念

分散式訓練的本質是將「大問題拆解為小問題」，但拆解方式直接影響效率和可行性：

資料平行（Data Parallelism）：最基本也最常用的分散式策略。每個 GPU 持有完整的模型副本，但處理不同的數據批次。前向傳播獨立進行，反向傳播後透過 AllReduce 操作同步梯度。優點是實作簡單、擴展性好；缺點是每張 GPU 都需容納完整模型，對超大模型不適用。

模型平行（Model Parallelism）：當模型太大無法放入單一 GPU 時，將模型拆分到多張 GPU 上。分為張量平行（Tensor Parallelism）和管線平行（Pipeline Parallelism）兩種主要方式。張量平行將單層的矩陣運算拆分到多 GPU；管線平行將模型的不同層分配到不同 GPU。

ZeRO 優化：DeepSpeed 提出的 Zero Redundancy Optimizer 技術，透過分片儲存優化器狀態（Stage 1）、梯度（Stage 2）和模型參數（Stage 3），在保持資料平行簡潔性的同時大幅降低記憶體需求。ZeRO-3 可將記憶體需求降低至原本的 1/N（N 為 GPU 數量）。

混合平行策略：實際的大模型訓練通常組合多種平行策略。例如 Megatron-LM 的 3D 平行：張量平行用於節點內（利用 NVLink 高頻寬），管線平行用於節點間，資料平行用於進一步擴展。

架構總覽

分散式訓練的通訊模式決定了架構設計：

AllReduce 模式（資料平行）：
GPU 0 ──┐
GPU 1 ──┤── AllReduce ──→ 同步梯度 ──→ 更新參數
GPU 2 ──┤
GPU 3 ──┘

管線平行模式：
Input → [Layer 0-3 on GPU 0] → [Layer 4-7 on GPU 1] → ... → Output
                ↑                        ↑
            Micro-batch 1           Micro-batch 2 (交錯執行)

張量平行模式（以 MLP 為例）：
Input → [Linear_col_0 on GPU 0] → [Linear_row_0 on GPU 0] → AllReduce → Output
      → [Linear_col_1 on GPU 1] → [Linear_row_1 on GPU 1] ↗

通訊原語：AllReduce、AllGather、ReduceScatter、Broadcast 和 P2P Send/Recv 構成分散式訓練的通訊基礎。NCCL（NVIDIA Collective Communication Library）是 GPU 間集合通訊的事實標準，針對不同拓撲自動選擇最優演算法（Ring、Tree、Direct）。

梯度壓縮：為降低通訊量，可採用梯度壓縮技術如 Top-K Sparsification、Random-K 或低位元量化。PowerSGD 透過低秩近似壓縮梯度，可將通訊量降低 10-100 倍。

關鍵比較

平行策略記憶體效率運算效率實作複雜度適用場景
資料平行 (DDP)低（全複製）高低模型可放入單 GPU
ZeRO Stage 3高中高中大模型、記憶體受限
張量平行中高（節點內）高Transformer 層內切分
管線平行中中（有氣泡）高超深模型跨節點
3D 平行最高高最高千億參數以上模型

通訊後端比較：NCCL 適合 NVIDIA GPU 同構環境；Gloo 支持 CPU 和跨平台；MPI（如 OpenMPI）提供最豐富的通訊原語但配置較複雜。

管線排程策略：GPipe 的同步排程有明確的氣泡（bubble）比例為 (p-1)/m（p 為管線深度，m 為 micro-batch 數）；PipeDream 的異步排程降低氣泡但引入權重版本不一致問題；Interleaved Pipeline（Megatron-LM）透過虛擬階段交錯進一步減少氣泡。

應用場景

大型語言模型訓練：LLaMA-70B 訓練使用 2048 張 A100 GPU，採用 FSDP（ZeRO-3 的 PyTorch 原生實作）搭配張量平行。訓練吞吐量達到每張 GPU 約 380 tokens/s。

科學計算模擬：AlphaFold 的 MSA 和 Evoformer 模組需要跨序列維度的分散式運算，使用自訂的 all-to-all 通訊模式。

推薦系統訓練：Embedding Table 可能佔用數百 GB 記憶體，需要專用的表平行（Table Parallelism）策略，將不同的 Embedding Table 分配到不同 GPU。

分散式強化學習：PPO 訓練中 Actor 和 Learner 分離，需要高效的經驗回放（Experience Replay）通訊機制和非同步更新策略。

延伸閱讀

本章節深入探討以下子主題：

  • 資料平行（Data Parallel）：DDP、FSDP 和 ZeRO 的實作細節、效能調優和常見陷阱
  • 模型平行（Model Parallel）：張量平行的矩陣切分策略、管線平行的排程演算法，以及 Sequence Parallelism 等進階技術

掌握分散式運算是邁向大模型時代的必備技能，理解各策略的權衡取捨能幫助工程師為特定模型和硬體環境選擇最優的訓練配置。
