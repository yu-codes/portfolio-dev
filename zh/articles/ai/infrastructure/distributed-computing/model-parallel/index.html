模型平行訓練

模型平行訓練（Model Parallelism）是當單一模型無法完整放入單張加速器記憶體時所採用的分散式策略。隨著大型語言模型參數量從數十億攀升至數兆，模型平行已成為訓練超大規模模型不可或缺的技術。

核心概念

模型平行的基本思想是將模型的不同部分分配到不同的計算裝置上，主要分為兩大類：

張量平行（Tensor Parallelism, TP）：將單一層的運算拆分到多個裝置上。例如，一個大型矩陣乘法可沿列或行方向分割，各裝置計算部分結果後再合併。Megatron-LM 是張量平行的代表性實現。

流水線平行（Pipeline Parallelism, PP）：將模型按層切分成若干階段（stages），每個階段放置在不同裝置上。資料依序通過各階段，類似工業流水線。GPipe 和 PipeDream 是經典的流水線平行框架。

混合平行（Hybrid Parallelism）：實際大規模訓練通常結合資料平行、張量平行和流水線平行。例如 Megatron-DeepSpeed 採用 3D 平行策略，在不同維度施加不同的平行化手段。

氣泡問題（Bubble Problem）：流水線平行中，由於各階段需等待前一階段的輸出，會產生計算空閒時間（氣泡）。微批次（micro-batching）技術可有效減少氣泡比例。

架構總覽

張量平行（層內切分）:
┌──────────────────────────────┐
│ Layer N                       │
│ ┌──────┐ ┌──────┐ ┌──────┐  │
│ │GPU 0 │ │GPU 1 │ │GPU 2 │  │
│ │Col 0 │ │Col 1 │ │Col 2 │  │
│ └──────┘ └──────┘ └──────┘  │
│      AllReduce / AllGather    │
└──────────────────────────────┘

流水線平行（層間切分）:
Stage 0    → Stage 1    → Stage 2    → Stage 3
[Layer 0-7]  [Layer 8-15] [Layer 16-23] [Layer 24-31]
  GPU 0        GPU 1        GPU 2         GPU 3

Megatron-LM 張量平行實現：對 Transformer 的 Self-Attention 和 MLP 層進行列平行與行平行切分。Attention 的 Q、K、V 投影沿注意力頭維度分割，MLP 的第一層沿列分割、第二層沿行分割，每層只需兩次 AllReduce 通訊。

GPipe 微批次排程：將 mini-batch 切分為 M 個 micro-batch，流水線充分利用後，氣泡比例降至 (P-1)/(M+P-1)，其中 P 為流水線階段數。

1F1B 排程策略：PipeDream 提出的「一前一後」策略，交錯執行前向與反向傳播，減少記憶體峰值佔用同時維持流水線效率。

記憶體優化：激活值重計算（Activation Recomputation）可用記憶體換計算，在流水線平行中尤為重要，避免儲存所有微批次的中間激活值。

擴展性分析：張量平行適合在高頻寬互連（如 NVLink）的節點內使用，通常限制在 8 張 GPU 以內；流水線平行適合跨節點擴展，通訊需求相對較低。

應用場景

大型語言模型訓練：GPT-3（175B）、PaLM（540B）、LLaMA-2（70B）等模型的訓練都依賴模型平行。典型配置如 TP=8（節點內）+ PP=8（跨節點）+ DP=64（資料平行）。

超長序列處理：處理超過 100K token 的長文本時，序列平行可分攤激活值記憶體。Ring Attention 和 Ulysses 等技術專門解決長序列的平行化問題。

多模態大模型：視覺-語言模型（如 GPT-4V）的視覺編碼器和語言模型可分配到不同流水線階段，利用流水線平行處理異質架構。

模型推理服務：張量平行也廣泛用於推理階段，將大模型分散到多張 GPU 以滿足延遲要求。vLLM 和 TensorRT-LLM 都支援推理時的張量平行。

科學計算大模型：蛋白質結構預測（AlphaFold）、氣候模擬等科學模型同樣受益於模型平行策略。
