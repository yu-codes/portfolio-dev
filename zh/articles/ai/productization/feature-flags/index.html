Feature Flags

Feature Flags（功能旗標）是一種軟體開發技術，允許團隊在不重新部署的情況下動態開啟或關閉功能。在 AI 產品中，Feature Flags 是實現漸進式發布、即時回滾和目標投放的關鍵基礎設施，與 A/B 測試和線上實驗平台深度整合。

核心概念

Feature Flag 定義：Feature Flag 本質上是程式碼中的條件分支，透過外部配置決定執行路徑。從簡單的布林開關到複雜的多變量控制，Flag 的形式多樣。

Flag 類型分類：

  • Release Flags：控制新功能的發布節奏，短生命週期
  • Experiment Flags：支撐 A/B 測試的流量分配，中等生命週期
  • Ops Flags：運維層面的控制開關（如降級、限流），長生命週期
  • Permission Flags：基於用戶屬性的功能權限控制

漸進式發布（Gradual Rollout）：從 1% → 5% → 25% → 50% → 100% 逐步擴大新功能的覆蓋範圍。每個階段監控關鍵指標，發現問題可立即回滾至 0%。

目標規則（Targeting Rules）：基於用戶屬性（地區、裝置、訂閱層級、用戶分群）決定是否啟用功能。支援複雜的布林邏輯組合。

Flag 生命週期管理：Feature Flags 若不及時清理會造成技術債。需要建立清理流程：Flag 達到 100% 發布後，移除條件分支並刪除 Flag 配置。

SDK 架構設計：客戶端 SDK 在本地快取 Flag 配置，透過串流連線即時接收更新。評估邏輯在客戶端本地執行，確保零延遲影響。

配置傳播機制：

  • 輪詢模式：定期拉取最新配置（簡單但有延遲）
  • 串流模式：透過 SSE/WebSocket 即時推送變更
  • CDN 模式：配置檔透過 CDN 分發，適合超大規模

審計與合規：所有 Flag 變更需記錄操作人、時間、變更內容。關鍵 Flag 的修改需經過審批流程。滿足 SOC2、GDPR 等合規要求。

緊急控制（Kill Switch）：預設建立關鍵功能的緊急關閉開關。當新 AI 模型出現品質問題或資源過載時，一鍵回滾至舊版本。

應用場景

AI 模型漸進上線：新訓練的推薦模型先對 1% 內部員工開放，確認指標正常後逐步擴大至所有用戶。結合自動化指標監控，異常時自動回滾。

多模型路由：透過 Feature Flag 控制不同用戶群使用不同 LLM（如 GPT-4 vs Claude vs 本地模型），實現成本和品質的動態平衡。

功能降級策略：當 AI 服務過載或異常時，透過 Ops Flag 自動降級為規則系統或快取結果，確保用戶體驗不完全中斷。

付費功能控制：進階 AI 功能（如高品質模型、更長上下文）透過 Permission Flag 控制，僅對付費用戶開放。

合規與地區限制：某些 AI 功能可能因法規限制無法在特定地區提供（如生成式 AI 在某些司法管轄區的限制），透過地區目標規則控制。
