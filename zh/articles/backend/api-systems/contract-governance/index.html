契約治理

API 契約演化與治理

契約治理（Contract Governance）是確保 API 在整個生命週期中保持品質、一致性與相容性的系統性實踐。當系統規模擴大、團隊數量增加，若缺乏有效的契約治理機制，API 的演進將變得混亂且充滿風險——一個看似無害的欄位修改可能導致數十個下游服務崩潰。本章節將探討如何透過 Schema 驗證、相容性檢查與生命週期管理來治理 API 契約。

核心概念

API 契約（API Contract） 是服務提供者與消費者之間的正式協議，定義了介面的輸入、輸出、行為與保證。契約的核心組成包括：

  • Schema 定義：資料結構、型別、必填欄位、驗證規則
  • 行為規範：端點語義、錯誤處理方式、冪等性保證
  • 非功能性約束：SLA（延遲、可用性）、Rate Limit、資料保留政策
  • 演進規則：什麼變更是允許的、如何通知消費者

契約優先（Contract-First） 的開發方式要求在實作前先定義 API 規格，這帶來多項好處：

  • 前後端可平行開發，減少等待時間
  • 消費者能提前基於 Mock 進行整合開發
  • 規格成為 Single Source of Truth
  • 自動化驗證確保實作符合契約

消費者驅動契約（Consumer-Driven Contracts, CDC） 則從消費者角度出發——每個消費者定義它所依賴的最小契約子集，提供者確保滿足所有消費者的契約。這種方式在微服務架構中特別有價值。

向後相容性（Backward Compatibility） 是契約治理的核心挑戰。一個向後相容的變更不會破壞既有消費者的功能：

  • ✅ 新增可選欄位
  • ✅ 新增新端點
  • ✅ 放寬輸入驗證（接受更多格式）
  • ❌ 移除或重新命名欄位
  • ❌ 變更欄位型別
  • ❌ 增加必填欄位

架構總覽

一個完善的契約治理體系包含以下組件：

API 規格 → Linting 檢查 → Breaking Change 偵測 → CI/CD Gate
   ↓              ↓                ↓                    ↓
Schema Registry  Style Guide   Changelog 產生      部署/阻擋
   ↓
Mock 服務 ← 消費者 → Contract Test → 提供者

Schema Registry：集中儲存與版本控制所有 API 規格。類似於程式碼的 Git 倉庫，但專門管理 API Schema。代表工具如 Apicurio Registry、Confluent Schema Registry。

API Linting：透過規則引擎自動檢查 API 規格的風格一致性、最佳實踐遵循度。Spectral 是最常用的 OpenAPI Linting 工具。

Breaking Change Detection：比對新舊版本規格，自動識別不相容的變更。工具如 openapi-diff、optic 可整合到 CI Pipeline 中。

契約測試框架：Pact 是消費者驅動契約測試的代表工具，支援多種語言，透過 Pact Broker 集中管理契約與驗證結果。

應用場景

大型微服務生態系統：數百個服務之間存在複雜的依賴關係。透過 Schema Registry + CDC 測試 + CI Breaking Change Gate 建立全自動化的契約治理流水線，確保任何服務的 API 變更不會意外破壞其他服務。

開放 API 平台營運：API 提供者對外部開發者有向後相容的承諾。使用 Sunset Header 通知棄用、提供至少 6 個月的遷移窗口、維護 Changelog 與 Migration Guide。

Schema Evolution in Event-Driven Systems：Kafka 等訊息系統中的 Schema 演進需要特別謹慎。Confluent Schema Registry 支援 BACKWARD、FORWARD、FULL 相容模式，確保生產者與消費者的 Schema 版本相容。

API 標準化治理：大型組織制定統一的 API Style Guide（如 Zalando、Google），透過 Spectral 自定義規則集在 CI 中自動執行風格檢查。

子主題導覽

本契約治理系列深入探討三大核心主題：

  • Schema 驗證（Schema Validation）：JSON Schema、OpenAPI Schema 驗證機制、Spectral Linting 規則撰寫與自動化整合
  • 向後相容性（Backward Compatibility）：相容性規則定義、Breaking Change 偵測工具、消費者驅動契約測試（Pact）
  • API 生命週期（API Lifecycle）：從設計到棄用的完整流程管理、版本策略、Sunset 政策與遷移指南
