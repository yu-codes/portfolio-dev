RBAC 與 ABAC 權限模型

深入比較 RBAC 與 ABAC 兩種主流權限控制模型的設計原理、實作策略與適用場景

存取控制（Access Control）是資訊安全的核心機制。RBAC（基於角色的存取控制）與 ABAC（基於屬性的存取控制）是兩種最主流的權限模型。本文深入比較這兩種模型的設計理念、實作方式與適用場景。

核心概念

存取控制模型演進

模型全稱核心思想年代
DACDiscretionary Access Control資源擁有者決定1970s
MACMandatory Access Control系統強制策略1970s
RBACRole-Based Access Control基於角色分配權限1992
ABACAttribute-Based Access Control基於屬性動態決策2005
ReBACRelationship-Based Access Control基於關係2019

RBAC 核心概念

概念描述範例
使用者 (User)系統使用者Alice, Bob
角色 (Role)職責的抽象Admin, Editor, Viewer
權限 (Permission)對資源的操作read:article, write:article
角色分配使用者 → 角色Alice → Admin
權限分配角色 → 權限Admin → read:, write:

ABAC 核心概念

概念描述範例
主體屬性 (Subject)使用者的特性department=engineering, level=senior
資源屬性 (Resource)資源的特性type=document, classification=secret
動作 (Action)要執行的操作read, write, delete
環境屬性 (Environment)情境條件time=working_hours, ip=internal
策略 (Policy)決策規則IF subject.dept == resource.dept THEN allow

架構總覽

RBAC 層級模型

層級名稱特點
RBAC0核心使用者-角色-權限基本映射
RBAC1層級角色繼承（角色層級）
RBAC2約束互斥角色、基數約束
RBAC3統一合併 RBAC1 + RBAC2

角色繼承範例：

角色繼承自額外權限
Super AdminAdminsystem:manage
AdminEditoruser:manage
EditorViewerarticle:write, article:delete
Viewer-article:read

ABAC 策略語言 (XACML 架構)

組件功能說明
PEP (Policy Enforcement Point)執行決策攔截請求、執行允許/拒絕
PDP (Policy Decision Point)做出決策評估策略、回傳決策
PIP (Policy Information Point)提供屬性查詢使用者/資源屬性
PAP (Policy Administration Point)管理策略建立與維護策略

權限檢查的實作模式

模式描述效能靈活性
硬編碼程式碼中直接判斷高極低
設定檔外部設定定義規則高中
資料庫查詢動態查詢權限表中高
策略引擎專用引擎評估中極高
服務化獨立權限服務較低（網路）極高

特性RBACABAC

RBAC 實作方案比較

方案描述優點缺點
簡單角色使用者直接對應角色最簡單不夠靈活
角色+權限角色是權限的集合靈活管理角色組合
層級角色角色有繼承關係減少重複繼承可能複雜
動態角色根據上下文切換角色靈活實作複雜
資源角色每個資源有獨立角色精確控制角色爆炸

應用場景

根據場景選擇模型

場景推薦模型理由
企業內部管理系統RBAC角色明確、簡單
多租戶 SaaSRBAC + 租戶隔離租戶間角色獨立
醫療系統ABAC需考慮時間、地點、關係
雲端資源管理ABAC (IAM)資源屬性多變
社交平台ReBAC基於好友關係
電商平台RBAC + 少量 ABAC角色為主、偶需屬性判斷
金融系統ABAC多因素決策、合規需求

常見權限設計模式

模式描述範例
資源:動作permission = resource:actionarticle:read, user:delete
層級資源上層權限涵蓋下層org:manage > team:manage > project:manage
萬用字元匹配多種資源/動作:read, article:
否定權限明確拒絕!user:delete
條件權限帶條件的權限article:edit WHERE author == self

最佳實踐

實踐說明
最小權限原則只給予完成任務所需的最少權限
職責分離敏感操作需要多個角色配合
定期審查定期檢查權限分配是否合理
預設拒絕未明確允許的一律拒絕
集中管理權限定義集中管理
可審計記錄所有權限變更與存取
測試覆蓋權限規則需有測試
