串流協定

串流協定允許伺服器與客戶端之間持續傳輸資料，而非傳統的一問一答模式。在即時通訊、大型資料傳輸與事件驅動架構中，串流協定扮演關鍵角色。本文系統性地比較各種串流技術的設計理念與適用場景。

核心概念

串流模式分類

模式描述範例
伺服器推送 (Server Push)伺服器主動推送資料給客戶端即時通知、股價更新
客戶端串流 (Client Stream)客戶端持續發送資料檔案上傳、音訊串流
雙向串流 (Bidirectional)雙方同時發送接收即時聊天、視訊通話
事件串流 (Event Stream)持續的事件流日誌、變更通知

技術方向底層協定瀏覽器支援

Server-Sent Events (SSE) 深入

特性描述
格式text/event-stream
事件欄位event, data, id, retry
重連機制自動重連 + Last-Event-ID
保活註解行 (: keepalive)

SSE 事件格式：

欄位用途範例
event事件類型event: message
data事件資料data: {“text”: “hello”}

retry重連間隔 (ms)retry: 5000

實際選擇決策

需求推薦技術理由

大規模串流的挑戰

挑戰描述解決方案
連接管理百萬級長連接連接伺服器叢集

負載均衡長連接難均衡一致性雜湊

有序性保證訊息亂序序號 + 排序
監控連接狀態追蹤連接健康檢查
