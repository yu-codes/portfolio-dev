機器人流量佔全球網路流量的相當比例，其中惡意機器人用於爬蟲抓取、帳號竊取、庫存搶購、垃圾訊息等攻擊行為。有效的反機器人策略需要在不影響正常使用者體驗的前提下，準確識別並攔截自動化攻擊。

核心概念

機器人類型分類

類別目的特徵危害
爬蟲機器人抓取網站內容高頻存取、特定路徑智慧財產權侵害
暴力破解猜測帳號密碼大量登入嘗試帳號竊取
搶購機器人搶購限量商品極快速下單庫存不公平分配
垃圾機器人發送垃圾內容大量註冊、發文平台品質下降
撞庫攻擊使用洩漏密碼嘗試分散式、多帳號大規模帳號入侵
API 濫用未授權使用 API超出限額、異常模式資源消耗、資料洩漏

機器人偵測信號

信號類別具體指標正常 vs 機器人
速率請求頻率人類有思考時間
行為滑鼠/觸控軌跡機器人缺少自然動作
瀏覽器指紋JS 環境、Canvas機器人環境異常
網路特徵IP 信譽、ASN資料中心 IP 可疑
協議特徵TLS 指紋、HTTP 標頭自動化工具特徵
認知能力CAPTCHA 回應機器人難以通過

偵測準確度指標

指標說明目標
True Positive正確識別機器人越高越好
False Positive誤判正常使用者為機器人越低越好
True Negative正確放行正常使用者越高越好
False Negative漏放機器人越低越好

決策模型

根據風險分數採取不同動作：

風險分數判定動作
0-30正常直接放行
30-60可疑無感驗證（Cookie 挑戰）
60-80高度可疑CAPTCHA 挑戰
80-100確定為機器人直接攔截

被動偵測 vs 主動挑戰

維度被動偵測主動挑戰
使用者感知無有（需互動）
準確度中-高高
對高級機器人可能失效較有效
實作複雜度高中
延遲影響無增加等待時間
無障礙影響無可能影響部分使用者

應用場景

防止撞庫攻擊

撞庫（Credential Stuffing）的防禦組合：

  1. 速率限制：限制單一 IP 的登入嘗試次數
  2. 設備指紋：識別異常的登入環境
  3. 漸進式挑戰：失敗次數增加時提高驗證難度
  4. 密碼洩漏檢查：比對已知洩漏密碼庫
  5. 異常偵測：監控全域登入失敗率突增
  6. 通知機制：可疑登入時通知帳號擁有者

電商搶購場景防護

限量商品搶購的防護策略：

階段策略目的
預備期裝置指紋建立、行為基線識別正常使用者
搶購前排隊系統、PoW 挑戰過濾機器人
搶購中行為驗證、速率限制確保公平
搶購後訂單審核、異常取消清除機器人訂單
