資料結構

常用資料結構

資料結構是組織與儲存資料的方式，決定了資料如何被存取、修改與操作。選擇適當的資料結構能讓演算法的效率產生數量級的差異——同一問題使用陣列可能需要 O(n) 時間，換成雜湊表就能達到 O(1)。深入理解資料結構的特性與權衡，是寫出高效能程式碼的關鍵。

核心概念

資料結構可依據儲存方式分為兩大類：

線性資料結構（Linear）：元素按照順序排列，每個元素最多有一個前驅和一個後繼。包括陣列（Array）、鏈結串列（Linked List）、堆疊（Stack）、佇列（Queue）和雙端佇列（Deque）。線性結構的優勢在於概念簡單且遍歷直觀。

非線性資料結構（Non-linear）：元素之間存在層級或網狀關係。包括樹（Tree）、圖（Graph）、堆積（Heap）和字典樹（Trie）。非線性結構適合表達複雜的層級關係與多對多連結。

每種資料結構都需要考慮以下操作的效率：存取（Access） 讀取特定位置的元素；搜尋（Search） 找到特定值的位置；插入（Insertion） 新增元素；刪除（Deletion） 移除元素。不同資料結構在這些操作上的效能各有優劣，沒有「萬能」的結構。

抽象資料型別（ADT） 是資料結構的邏輯接口，定義了支援的操作集合而不規定實作方式。例如「Map」這個 ADT 可以用雜湊表、平衡二元搜尋樹或跳表來實現，各有不同的效能特徵。

架構總覽

陣列與鏈結串列是最基礎的資料結構。陣列提供 O(1) 隨機存取但插入刪除為 O(n)；鏈結串列提供 O(1) 頭部插入刪除但隨機存取為 O(n)。動態陣列（如 Java ArrayList、Python list）透過攤銷擴容策略兼顧了兩者的優點。在現代硬體上，由於 CPU 快取的局部性原理，陣列的實際效能往往優於理論分析。

樹與圖是表達層級與網狀關係的核心結構。二元搜尋樹（BST）支援 O(log n) 的搜尋、插入、刪除，但最差情況退化為 O(n)。平衡樹（AVL、Red-Black Tree）透過旋轉操作保證 O(log n) 的最差效能。B-Tree 及其變體 B+Tree 是資料庫索引的基石，針對磁碟 I/O 進行最佳化。圖則以鄰接矩陣或鄰接表表示，支援 BFS、DFS 等遍歷演算法。

雜湊表（Hash Table） 透過雜湊函數將鍵映射到陣列索引，實現平均 O(1) 的查詢、插入、刪除。碰撞處理策略包括鏈結法（Chaining）和開放定址法（Open Addressing）。雜湊表是現代軟體中使用最頻繁的資料結構之一，從程式語言的字典到資料庫的索引、從快取系統到分散式一致性雜湊，無處不在。

選擇資料結構時需要綜合考量：操作頻率（讀多寫少 vs 寫多讀少）、資料規模、是否需要有序性、記憶體限制、以及並行存取需求。

應用場景

作業系統核心：程序調度使用優先佇列（堆積）、檔案系統目錄使用 B-Tree、虛擬記憶體頁表使用多層雜湊表、網路協定棧使用環形緩衝區（Ring Buffer）。

資料庫引擎：B+Tree 作為磁碟索引提供 O(log n) 查詢；LSM-Tree 最佳化寫入密集場景；Skip List 在 Redis 的有序集合中替代平衡樹；Bloom Filter 快速判斷元素是否存在於大型資料集。

編譯器與直譯器：抽象語法樹（AST）表示程式結構；符號表使用雜湊表進行變數查找；語法分析使用堆疊進行括號匹配與運算式求值。

網路系統：路由表使用 Trie（前綴樹）進行最長前綴匹配；連接池使用佇列管理可用連接；負載均衡器使用一致性雜湊環分配請求。

遊戲開發：場景圖使用樹結構組織遊戲物件；碰撞偵測使用四叉樹（Quadtree）或八叉樹（Octree）進行空間分割；尋路演算法在圖結構上執行 A* 搜尋。
