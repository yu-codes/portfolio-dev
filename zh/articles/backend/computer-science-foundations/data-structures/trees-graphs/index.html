樹與圖

樹與圖是非線性資料結構的核心，廣泛應用於資料庫索引、檔案系統、網路路由與社群網路等領域。本文深入探討各種樹結構的設計取捨與圖的表示方法，以及它們在實際系統中的應用。

核心概念

樹的基本分類

樹類型特點典型應用
二元樹 (Binary Tree)每節點最多兩個子節點表達式解析
二元搜尋樹 (BST)左 < 根 < 右有序集合
AVL 樹自平衡，高度差 ≤ 1頻繁搜尋
紅黑樹自平衡，較寬鬆Java TreeMap, Linux
B-Tree多路平衡，適合磁碟資料庫索引
B+ Tree資料只在葉節點檔案系統、資料庫
Trie前綴樹自動完成、路由
堆積 (Heap)完全二元樹，堆積性質優先佇列

二元樹遍歷方式

遍歷方式順序應用場景實作
前序 (Pre-order)根 → 左 → 右複製樹、序列化遞迴/堆疊
中序 (In-order)左 → 根 → 右取得排序結果 (BST)遞迴/堆疊
後序 (Post-order)左 → 右 → 根刪除樹、計算表達式遞迴/堆疊
層序 (Level-order)逐層從左到右BFS、最短路徑佇列

圖的基本概念

概念描述範例
頂點 (Vertex)圖中的節點使用者、城市
邊 (Edge)頂點間的連線關係、道路
有向圖邊有方向關注關係
無向圖邊無方向友誼關係
加權圖邊有權重距離、成本
DAG有向無環圖任務依賴

特性AVL紅黑樹B-TreeB+ Tree

B-Tree vs B+ Tree

特性B-TreeB+ Tree
資料位置所有節點僅葉節點
葉節點鏈結無有（形成鏈結串列）
範圍查詢較慢快（循序掃描）
空間利用較低較高
磁碟讀取可能更少（命中內部節點）穩定（總是到葉節點）
實際應用MongoDBMySQL, PostgreSQL

關鍵比較

圖遍歷演算法

演算法策略資料結構應用
BFS (廣度優先)逐層擴展佇列最短路徑（無權）
DFS (深度優先)盡可能深入堆疊/遞迴連通分量、拓撲排序

BFS vs DFS 比較：

特性BFSDFS
空間複雜度O(V)（最寬層）O(V)（最深路徑）
最短路徑保證是（無權圖）否
完整性是是（有限圖）
適合稀疏圖是是
適合深圖否（記憶體）是

最短路徑演算法

演算法適用情況複雜度限制
BFS無權圖O(V+E)邊權相同
Dijkstra非負權重O((V+E)log V)不可有負權
Bellman-Ford可有負權O(VE)可偵測負環
Floyd-Warshall全源最短路O(V³)適合小圖
A*有啟發函式依啟發函式需要啟發函式

應用場景

樹結構的實際應用

應用使用的樹原因
資料庫索引B+ Tree磁碟友善，範圍查詢快
記憶體有序集合紅黑樹平衡性好，插刪快
IP 路由查找Trie最長前綴匹配
文字自動完成Trie前綴搜尋
優先排程堆積O(1) 取最值

編譯器 AST多元樹語法結構表示

圖結構的實際應用

應用圖類型演算法
社群網路無向圖BFS (六度分隔)
地圖導航加權有向圖Dijkstra / A*
任務排程DAG拓撲排序
網頁排名有向圖PageRank
推薦系統二部圖協同過濾
網路流量流量網路最大流演算法
