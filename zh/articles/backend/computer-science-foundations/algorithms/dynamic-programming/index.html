動態規劃

動態規劃（Dynamic Programming, DP）是解決具有重疊子問題與最佳子結構性質問題的強大技術。它將複雜問題分解為較小的子問題，並儲存子問題的解以避免重複計算。本文系統性地介紹動態規劃的設計方法論與常見模式。

核心概念

動態規劃的兩個必要條件

條件描述判斷方式
最佳子結構 (Optimal Substructure)最佳解包含子問題的最佳解問題可以用子問題的解組合而成
重疊子問題 (Overlapping Subproblems)相同子問題被重複計算遞迴樹中有重複節點

動態規劃 vs 其他策略

策略條件特點典型問題
暴力搜尋無嘗試所有可能小規模問題
分治法子問題不重疊獨立解決子問題合併排序
貪婪法貪婪選擇性質每步最佳最小生成樹
動態規劃最佳子結構 + 重疊子問題記住子問題解最短路徑
回溯法約束滿足嘗試 + 剪枝N 皇后

自頂向下 vs 自底向上

方式描述優點缺點
自頂向下 (Memoization)遞迴 + 記憶化只計算需要的子問題遞迴開銷、堆疊溢出
自底向上 (Tabulation)迭代填表無遞迴開銷可能計算不需要的子問題

架構總覽

動態規劃解題框架

步驟動作說明
1定義狀態明確 dp[i] 代表什麼
2找出狀態轉移方程dp[i] 與 dp[j] (j < i) 的關係
3確定邊界條件初始值（base case）
4確定計算順序確保子問題先被解決
5空間最佳化（可選）滾動陣列等技巧

常見 DP 模式

模式狀態定義範例問題
線性 DPdp[i] = 前 i 個元素的最佳解最長遞增子序列
區間 DPdp[i][j] = 區間 [i,j] 的最佳解矩陣鏈乘法
背包 DPdp[i][w] = 前 i 物品容量 w0/1 背包問題
樹形 DPdp[node] = 以 node 為根的最佳解樹的最大獨立集
狀態壓縮 DPdp[mask] = 集合 mask 的最佳解旅行商問題
數位 DPdp[pos][…] = 前 pos 位數字計數問題

經典問題的狀態轉移方程

問題狀態轉移方程
費氏數列dp[i] = 第 i 個數dp[i] = dp[i-1] + dp[i-2]
爬樓梯dp[i] = 到第 i 階的方法數dp[i] = dp[i-1] + dp[i-2]
最長共同子序列dp[i][j] = s1[:i] 與 s2[:j] 的 LCSif s1[i]==s2[j]: dp[i-1][j-1]+1 else max(dp[i-1][j], dp[i][j-1])
0/1 背包dp[i][w] = 前 i 物品容量 w 最大值dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi]+vi)
編輯距離dp[i][j] = s1[:i] 到 s2[:j] 的操作數min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1]+cost)
最長遞增子序列dp[i] = 以 i 結尾的 LIS 長度dp[i] = max(dp[j]+1) for j < i if a[j] < a[i]

關鍵比較

背包問題家族

變體特點狀態轉移複雜度
0/1 背包每物品最多選一次dp[i][w] = max(不選, 選)O(nW)
完全背包每物品可選無限次正序遍歷容量O(nW)
多重背包每物品有數量限制二進制拆分O(nW log k)
分組背包每組最多選一個組內互斥選取O(nKW)
混合背包混合以上類型分別處理依組合而定

空間最佳化技巧

技巧原始空間最佳化後適用場景
滾動陣列O(n²)O(n)只依賴前一行
一維陣列O(n×W)O(W)0/1 背包（逆序）
狀態壓縮O(2ⁿ × n)O(2ⁿ)集合 DP
只記錄兩行O(n×m)O(min(n,m))LCS、編輯距離

DP 與貪婪的比較

特性動態規劃貪婪法
保證最佳解✅ 是❌ 不一定
時間複雜度通常較高通常較低
適用條件最佳子結構貪婪選擇性質
實作複雜度較高較低
典型問題最短路徑（一般圖）最短路徑（無負權）
正確性驗證狀態轉移的正確性需要數學證明

常見陷阱與除錯

陷阱描述解決方式
狀態定義不清dp 代表什麼含糊寫明確的狀態定義
邊界錯誤初始條件設錯手動驗算小例子
轉移遺漏未考慮所有情況列舉所有轉移來源
計算順序依賴的子問題尚未計算畫依賴圖確認順序
溢出問題數值過大使用長整數或取模
