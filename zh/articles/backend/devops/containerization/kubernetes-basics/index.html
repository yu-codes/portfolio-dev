Kubernetes 基礎

Kubernetes 容器編排平台的核心概念，涵蓋架構組件、工作負載管理與服務發現機制

Kubernetes（K8s）是業界標準的容器編排平台，自動化容器化應用的部署、擴展與管理。它提供了宣告式的配置模型和強大的自我修復能力，讓工程團隊能夠可靠地運行大規模的分散式系統。

核心概念

K8s 架構組件

組件角色功能
API Server控制面板入口處理所有 REST 請求
etcd分散式儲存存放叢集所有狀態
Scheduler調度器決定 Pod 放在哪個節點
Controller Manager控制器管理確保實際狀態符合期望
kubelet節點代理管理節點上的 Pod
kube-proxy網路代理實現 Service 的網路規則

核心資源物件

資源用途層級
Pod最小部署單元（1+ 容器）基礎
Deployment管理 Pod 的副本和更新工作負載
Service穩定的網路端點網路
ConfigMap非敏感配置資料配置
Secret敏感配置資料配置
Ingress外部流量入口路由網路
PersistentVolume持久化儲存儲存
Namespace資源隔離組織

宣告式 vs 命令式

方式範例優點適用場景
宣告式kubectl apply -f deploy.yaml可重複、可版控生產環境
命令式kubectl create deployment nginx快速、直觀開發除錯

Pod 生命週期

狀態說明後續動作
Pending已建立，等待調度或映像拉取等待資源
Running至少一個容器運行中正常運作
Succeeded所有容器成功完成（Job）清理
Failed容器以非零退出碼結束根據重啟策略處理
Unknown無法取得 Pod 狀態節點通訊問題

關鍵比較

工作負載類型

類型用途副本管理適用場景
Deployment無狀態應用可擴縮、滾動更新Web 服務、API
StatefulSet有狀態應用穩定的網路/儲存標識資料庫、快取叢集
DaemonSet每個節點一份自動在新節點部署日誌收集、監控代理
Job一次性任務確保完成批次處理、遷移
CronJob定時任務排程執行定期報表、清理

Service 類型

類型存取範圍用途適用場景
ClusterIP叢集內部服務間通訊微服務互相呼叫
NodePort節點 IP + 固定端口開發/除錯快速暴露服務
LoadBalancer外部負載均衡對外服務雲端環境
ExternalNameDNS 映射存取外部服務遷移/抽象化

資源請求與限制

設定意義效果
requests.cpu最低保證的 CPU調度依據
requests.memory最低保證的記憶體調度依據
limits.cpuCPU 使用上限超出會被節流
limits.memory記憶體使用上限超出會被 OOMKill

應用場景

水平自動擴展（HPA）

HPA 根據指標自動調整 Pod 副本數：

指標類型範例適用場景
CPU 使用率平均 CPU > 70% 時擴展CPU 密集型服務
記憶體使用率平均 Memory > 80% 時擴展記憶體密集型
自定義指標佇列深度 > 100基於業務指標
外部指標RPS > 1000基於流量

健康檢查配置

探測器用途失敗結果
livenessProbe容器是否存活重啟容器
readinessProbe是否可接收流量從 Service 移除
startupProbe是否完成啟動啟動前不檢查其他探測

配置參數：

參數說明建議值
initialDelaySeconds首次檢查的延遲依應用啟動時間
periodSeconds檢查間隔10-30 秒
timeoutSeconds逾時時間1-5 秒
failureThreshold連續失敗次數後觸發3 次
successThreshold連續成功次數後恢復1 次

K8s 的網路模型

K8s 網路的基本要求：

  • 所有 Pod 可互相通訊（無 NAT）
  • 節點可與所有 Pod 通訊
  • Pod 看到的自身 IP 與其他 Pod 看到的相同
網路插件（CNI）特色適用場景
Calico網路策略豐富、BGP大規模、需策略
CiliumeBPF 高效能效能敏感、可觀測
Flannel簡單、輕量小型叢集
