Terraform 基礎

Terraform 是 HashiCorp 開發的開源基礎設施即程式碼工具，使用宣告式的 HCL（HashiCorp Configuration Language）定義雲端和本地資源。它支援多雲環境，擁有龐大的 Provider 生態系統，是目前最廣泛使用的 IaC 工具。

核心概念

Terraform 工作流程

命令作用說明
terraform init初始化下載 Provider、設定後端
terraform plan預覽變更顯示將要執行的操作
terraform apply套用變更實際建立/修改資源
terraform destroy銷毀資源刪除所有管理的資源
terraform import匯入資源將現有資源納入管理
terraform state狀態管理查看和操作狀態

資源的生命週期

操作符號說明
Create+建立新資源
Update in-place~原地更新（不需重建）
Destroy and recreate-/+需刪除重建
Destroy-刪除資源

HCL 核心語法元素

元素用途範例
resource定義基礎設施資源resource "aws_instance" "web" {}
data查詢既有資源data "aws_ami" "latest" {}
variable輸入變數variable "region" {}
output輸出值output "ip" { value = ... }
locals本地計算值locals { name = "..." }
module模組引用module "vpc" { source = "..." }
providerProvider 配置provider "aws" { region = "..." }

關鍵比較

模組來源選擇

來源優點缺點適用
Terraform Registry社群維護、開箱即用可能不符合需求快速起步
內部模組完全客製需自行維護組織標準化
Git Repository版本控制需管理存取權限跨團隊共享
Local path開發方便不適合共享開發階段

常見反模式

反模式問題正確做法
巨大的 state一個 state 管理太多資源按層級/服務分割
硬編碼值不可重用使用變數和 data source
無模組化重複程式碼抽取共用模組
忽略 plan直接 apply先 plan 再 apply
狀態手動修改不一致風險使用 terraform state 命令
Provider 版本不固定可能升級破壞使用版本約束

應用場景

團隊協作的工作流程

流程步驟動作工具
1. 開發在分支上修改配置IDE + terraform fmt
2. 驗證本地 plan 確認terraform plan
3. 提交建立 Pull RequestGit
4. CI 檢查自動化驗證tflint + plan + checkov
5. 審查團隊成員 review plan 輸出GitHub/GitLab
6. 套用合併後自動 applyCI/CD pipeline

安全的重構策略

重新組織 Terraform 配置而不影響基礎設施：

操作方法風險
重命名資源moved block無（Terraform 處理）
移入模組moved block + 模組封裝低
分割 stateterraform state mv中（需小心操作）
匯入現有資源import block低

Terraform 最佳實踐摘要

類別實踐
程式碼使用 terraform fmt、固定 Provider 版本
狀態遠端後端 + 加密 + 鎖定
變數驗證規則、合理的預設值
模組版本標籤、完整的 README
CI/CD自動 plan、人工審批 apply
安全不在程式碼中存放密鑰
