Overview

Infra-Toolbox is a modular infrastructure service toolkit where each service is independently packaged as a Docker Compose stack. Designed for backend systems, development environments, and operational tooling, it supports both Windows (Docker Desktop + Git Bash/WSL) and Linux environments with cross-platform POSIX-compatible shell scripts.

Directory Structure

infra-toolbox/
├── .github/workflows/ci.yml      # GitHub Actions CI/CD
├── claude-code/                   # Claude Code development guide
├── filebrowser/                   # Web file manager
├── geoserver/                     # GeoServer map services
├── minio/                         # MinIO S3-compatible storage
├── minio-backup/                  # MinIO backup service
├── minio-backup-shell/            # Shell-based MinIO backup
├── postgres/                      # PostgreSQL 14 (WAL enabled)
├── postgres-backup-logical/       # Logical backup (pg_dump)
├── postgres-backup-physical/      # Physical backup (WAL/PITR)
├── postgres-backup-shell/         # Shell-based PostgreSQL backup
├── redis-backup/                  # Redis backup service
├── skills/                        # AI Agent Skills framework
├── system-checks/                 # System health checks
├── system-monitor/                # Resource monitoring (CPU/RAM/Storage)
└── tests/                         # Test framework
    └── run_tests.sh

Each service follows a standard structure:

service_name/
├── docker-compose.yml    # Docker configuration
├── .env.example          # Environment variable template
├── scripts/              # Automation scripts (backup/restore)
└── README.md             # Service documentation

Key Features

  • Fully Modular: Each service independently deployable via docker compose up -d
  • Comprehensive Backup: Logical (pg_dump), Physical (WAL/PITR), encrypted, with automated drill tests
  • Cross-Platform: POSIX shell scripts compatible with Windows/Linux, automatic MSYS_NO_PATHCONV handling
  • CI/CD Pipeline: 9-stage GitHub Actions — unit tests, Docker validation, ShellCheck, integration tests, backup drills, encryption tests
  • Shared Network: All services communicate via infra-toolbox-network
  • Port Allocation: Organized by category (10001-10009 monitoring, 10010-10019 storage, 10020-10029 file management)
  • AI Skills Framework: Universal Agent Skills format compatible with Claude, ChatGPT, Cursor, etc.
