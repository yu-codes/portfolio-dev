Pages-Hub

Centralized GitHub Pages deployment hub — managing multiple static site projects with unified routing, CI/CD pipelines, and subdomain management.

GitHub Pages CI/CD Static Sites GitHub Actions
View on GitHub

Overview

Pages-Hub is the centralized orchestration layer for managing multiple GitHub Pages static site projects under a unified domain. It handles routing, CI/CD pipeline coordination, and deployment lifecycle for all sub-projects published under yu-codes.github.io, providing a single control plane for multi-project static hosting.

Directory Structure

pages-hub/
├── .github/
│   └── workflows/
│       ├── deploy-portfolio.yml    # Portfolio site deployment
│       ├── deploy-docs.yml         # Documentation site deployment
│       ├── sync-subprojects.yml    # Cross-repo synchronization
│       └── cleanup.yml             # Stale deployment cleanup
├── config/
│   ├── routes.json                 # URL routing configuration
│   ├── projects.json               # Registered sub-projects

│   └── rollback.sh                 # Rollback mechanism
├── templates/
│   ├── 404.html                    # Shared 404 page
│   └── redirect.html               # URL redirect template
├── public/                          # Aggregated deployment output
├── README.md

Key Features

  • Multi-Project Orchestration: Register and manage multiple static sites from a single hub with independent deployment lifecycles
  • Unified Routing: Clean URL paths (/portfolio/, /docs/, etc.) routed to respective sub-project builds via static file structure
  • CI/CD Pipeline Coordination: GitHub Actions workflows that trigger on sub-project changes, with cross-repo dispatch for automated rebuilds
  • Deployment Validation: Pre-deploy checks verifying build output integrity, broken links, and asset availability before publishing
  • Rollback Mechanism: Quick rollback to previous deployment via Git history, ensuring zero-downtime recovery
  • Stale Cleanup: Automated cleanup of orphaned deployments and outdated build artifacts
