LLM 應用

大型語言模型應用開發

LLM 應用開發是將大型語言模型的能力轉化為解決實際問題的工程實踐。這不僅是簡單地呼叫 API，而是涉及 Prompt 設計、上下文管理、外部知識整合、工具呼叫編排以及安全防護的完整系統工程。隨著 LLM 能力的快速提升，應用開發模式也在從簡單的問答對話演進到自主完成複雜任務的 AI Agent。

核心概念

Prompt Engineering：Prompt 是人類與 LLM 溝通的介面。有效的 Prompt 設計能顯著提升模型輸出品質而無需微調模型。核心技術包括：

  • Few-shot Learning：在 Prompt 中提供範例，引導模型理解期望的輸出格式和風格
  • Chain-of-Thought（CoT）：要求模型逐步推理，顯著提升複雜邏輯和數學問題的準確率
  • System Prompt：定義模型的角色、能力邊界和行為規範，是 LLM 應用的基礎配置
  • Structured Output：透過 JSON Schema 或函數定義約束模型輸出結構化數據

Retrieval-Augmented Generation（RAG）：LLM 的訓練數據有截止日期，且無法包含所有領域知識。RAG 在推論時動態檢索相關文件並注入上下文，讓模型能回答訓練數據之外的問題。這是企業 LLM 應用最重要的模式之一。

Tool Calling（函數呼叫）：LLM 本質上只能生成文本，無法執行動作。透過 Tool Calling，模型可以呼叫外部 API、查詢數據庫、執行程式碼等。模型學會「何時該用哪個工具」是 AI Agent 的基礎能力。

AI Agent：將 LLM 作為推理和決策引擎，結合記憶（Memory）、規劃（Planning）和行動（Action）能力，自主完成複雜多步驟任務。Agent 代表了 LLM 應用的最高形態。

架構總覽

典型的 LLM 應用架構分層設計：

使用者介面層
    ↓
應用編排層 (LangChain / LlamaIndex / Semantic Kernel)
    ├─ Prompt 管理 & 模板
    ├─ Chain / Graph 編排
    ├─ Memory 管理 (短期 / 長期)
    └─ Tool Routing
    ↓
整合層
    ├─ Vector Store (Pinecone / Weaviate / pgvector)
    ├─ Document Loaders & Chunking
    ├─ External APIs & Tools
    └─ Code Execution Sandbox
    ↓
LLM 服務層 (OpenAI API / Anthropic / 本地部署)

RAG 系統流程：文件入庫（Chunking → Embedding → 存入向量資料庫）→ 檢索（Query Embedding → 相似度搜尋 → Reranking）→ 生成（檢索結果 + 原始問題 → LLM 回答）。進階 RAG 技術包括 Hypothetical Document Embedding（HyDE）、Self-RAG 和 Corrective RAG。

Agent 架構：ReAct 模式（Reasoning + Acting 交替進行）、Plan-and-Execute（先規劃再執行）、Multi-Agent（多個專業 Agent 協作）。LangGraph 提供基於狀態圖的 Agent 編排，支持條件分支、循環和人類介入。

關鍵比較

模式適用場景複雜度可控性延遲
直接 Prompting簡單 QA、翻譯低高低
RAG知識密集型問答中中中
Tool Calling需要外部動作中高中中高
Single Agent多步驟自動化高較低高
Multi-Agent複雜協作任務最高最低最高

RAG vs Fine-tuning：RAG 適合知識經常更新、需要引用來源的場景；Fine-tuning 適合需要改變模型行為風格或學習特定格式的場景。兩者可以結合使用。

編排框架比較：LangChain 生態最大但抽象較重；LlamaIndex 專精於 RAG 和數據整合；Semantic Kernel 適合 .NET 生態；Haystack 適合生產環境的文件處理；直接使用 SDK 最靈活但需自行處理更多細節。

向量資料庫比較：Pinecone 全託管無運維負擔；Weaviate 提供混合搜尋（向量 + 關鍵字）；pgvector 適合已有 PostgreSQL 的團隊；ChromaDB 適合原型快速開發。

應用場景

企業智慧客服：結合 RAG 檢索企業知識庫，搭配 Tool Calling 查詢訂單系統、發起退款等操作。需要嚴格的幻覺控制和權限管理。

程式碼助理：理解代碼庫上下文，提供程式碼補全、重構建議和 Bug 修復。需要大上下文視窗和精確的程式碼理解能力。Cursor 和 GitHub Copilot 是典型代表。

數據分析 Agent：使用者以自然語言描述分析需求，Agent 自動生成 SQL/Python 程式碼、執行查詢、產生可視化報告。需要 Code Interpreter 和錯誤恢復機制。

文件處理自動化：自動提取合約關鍵條款、生成摘要、比對文件差異。結合 OCR 和 Layout Parser 處理非結構化文件。

研究助理：自動搜尋相關論文、整理文獻綜述、提出研究問題。需要學術資料庫 API 整合和高品質摘要能力。

延伸閱讀

本章節深入探討以下子主題：

  • Prompt Engineering：系統性的 Prompt 設計方法論、進階技術（Self-Consistency、Tree-of-Thought）和 Prompt 管理最佳實踐
  • Tool Calling & Agents：函數呼叫的實作細節、Agent 架構模式（ReAct、Plan-and-Execute）和多 Agent 協作框架
  • RAG 系統：從基礎 RAG 到進階 RAG（HyDE、Self-RAG、Graph RAG）的完整技術棧，包含 Chunking 策略、Embedding 選擇和 Reranking 優化

LLM 應用開發是一個兼具創意和工程嚴謹性的領域。掌握核心模式和最佳實踐，能幫助開發者構建出既強大又可靠的 AI 應用。
