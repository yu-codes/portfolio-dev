Tool Calling 與 AI Agent

Tool Calling（工具呼叫）賦予 LLM 與外部系統互動的能力，使其從純文字生成器進化為能夠執行操作的智能代理。AI Agent 則是建構在 Tool Calling 之上的自主系統，能夠規劃、推理並執行多步驟任務。本文將深入探討從基礎的 Function Calling 到完整 Agent 框架的設計與實作。

核心概念

什麼是 Tool Calling

Tool Calling 讓 LLM 能夠：

  • 辨識何時需要外部工具（而非直接回答）
  • 生成結構化的工具呼叫請求（通常為 JSON）
  • 整合工具回傳結果產生最終回答

基本流程：

使用者問題 → LLM 判斷需要工具 → 生成呼叫參數
→ 執行工具 → 回傳結果 → LLM 整合回答

Function Calling 格式

以 OpenAI 的格式為例，定義可用工具：

{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "取得指定城市的天氣資訊",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {"type": "string", "description": "城市名稱"},
        "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
      },
      "required": ["city"]
    }
  }
}

AI Agent 的核心能力

能力說明
感知（Perception）接收並理解使用者輸入與環境資訊
規劃（Planning）將複雜任務分解為子步驟
推理（Reasoning）判斷下一步行動
行動（Action）呼叫工具或執行操作
記憶（Memory）維護對話歷史與長期知識
反思（Reflection）評估結果並調整策略

架構總覽

ReAct 模式（Reasoning + Acting）

ReAct 是最經典的 Agent 設計模式，交替進行推理與行動：

Thought: 使用者詢問台北明天的天氣，我需要查詢天氣 API
Action: get_weather(city="Taipei", date="tomorrow")
Observation: 明天台北天氣晴，氣溫 28°C
Thought: 我已獲得天氣資訊，可以回答使用者
Answer: 明天台北天氣晴朗，預計氣溫為 28°C。

多 Agent 協作架構

複雜任務可由多個專業 Agent 協作完成：

Orchestrator Agent（協調者）
    ├── Research Agent（資料蒐集）
    ├── Code Agent（程式開發）
    ├── Review Agent（品質檢查）
    └── Deploy Agent（部署執行）

協作模式：

  • 階層式：主 Agent 分配任務給子 Agent
  • 對等式：Agent 之間直接通訊協商
  • 流水線式：按順序傳遞處理結果

Tool Calling 實作架構

使用者輸入
    ↓
[LLM 推理層]
    - 理解意圖
    - 選擇工具
    - 生成參數
    ↓
[工具執行層]
    - 參數驗證
    - 權限檢查
    - 執行工具
    - 錯誤處理
    ↓
[結果整合層]
    - 解析工具輸出
    - 判斷是否需要更多工具呼叫
    - 生成最終回答
    ↓
回應使用者

應用場景

開發者工具

  • GitHub Copilot Agent：理解程式碼庫、執行修改、運行測試
  • Cursor / Windsurf：IDE 內的 AI 編碼代理
  • Devin：自主完成軟體開發任務

企業自動化

  • 客戶服務：查詢訂單、處理退款、預約排程
  • 資料分析：查詢資料庫、生成報表、執行計算
  • 文件處理：解析文件、擷取資訊、填寫表單

研究與創作

  • 深度研究：多步驟資料蒐集與綜合分析
  • 內容創作：搜尋資料、撰寫文章、生成圖片
  • 論文輔助：文獻檢索、摘要整理、引用管理

個人助理

  • 行程管理：整合日曆、Email、待辦事項
  • 智能家居：控制 IoT 裝置
  • 購物決策：比價、查評價、下訂單

MCP（Model Context Protocol）

Anthropic 提出的開放協定，標準化工具接入：

  • 統一的工具描述格式
  • 跨平台工具共享
  • 安全的權限管理
