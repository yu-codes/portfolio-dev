RLHF 人類回饋強化學習

RLHF（Reinforcement Learning from Human Feedback）是讓大型語言模型與人類偏好對齊的核心技術。透過人類標註者的偏好回饋訓練獎勵模型，再用強化學習優化語言模型的輸出，使其產生更有幫助、更安全且更符合人類期望的回答。本文將深入探討 RLHF 流程及其替代方案 DPO。

核心概念

對齊問題（Alignment Problem）

預訓練的 LLM 僅學會預測下一個 token，但這不等同於：

  • 有幫助（Helpful）：實際回答使用者的問題
  • 無害（Harmless）：不產生有害或冒犯性內容
  • 誠實（Honest）：承認不確定性，不捏造事實

RLHF 的目標就是讓模型在這三個維度上與人類偏好對齊。

RLHF 三階段流程

第一階段：監督微調（SFT）
    預訓練模型 → 高品質對話資料微調 → SFT 模型

第二階段：獎勵模型訓練（Reward Model）
    SFT 模型生成多個回答 → 人類排序偏好 → 訓練 RM

第三階段：強化學習優化（PPO/RLHF）
    SFT 模型 + RM → PPO 訓練 → 對齊後的模型

關鍵術語

術語說明
Reward Model (RM)學習人類偏好的評分模型
PPOProximal Policy Optimization，常用 RL 演算法
KL Divergence約束模型不偏離 SFT 基準太遠
Reference ModelSFT 模型，作為 KL 約束的參考
Preference Data人類標註的偏好對（chosen vs rejected）
Bradley-Terry Model將偏好轉為機率的數學模型

架構總覽

獎勵模型訓練

資料收集：

  1. 給定 prompt，讓 SFT 模型生成多個回答（通常 2-4 個）
  2. 人類標註者按品質排序這些回答
  3. 形成偏好對：(prompt, chosen, rejected)

訓練目標： $$ \mathcal{L}{RM} = -\mathbb{E}{(x, y_w, y_l) \sim D}\left[\log \sigma(r_\theta(x, y_w) - r_\theta(x, y_l))\right] $$

其中 $y_w$ 是偏好的回答，$y_l$ 是不偏好的回答，$r_\theta$ 是獎勵模型的分數。

PPO 訓練流程

迴圈：
    1. 從 prompt 資料集取樣
    2. 使用當前策略（LLM）生成回答
    3. 獎勵模型評分
    4. 計算 KL 懲罰：reward_final = r(x,y) - β·KL(π||π_ref)
    5. 使用 PPO 更新策略
    6. 重複

KL 約束的作用：

  • 防止模型「破解」獎勵模型（reward hacking）
  • 保持語言流暢性和多樣性
  • β 值控制約束強度

DPO（Direct Preference Optimization）

DPO 跳過獎勵模型訓練，直接從偏好資料優化策略：

核心洞察：最優的 RL 解可以用封閉形式表達為偏好資料的函數。

訓練目標： $$ \mathcal{L}{DPO} = -\mathbb{E}{(x, y_w, y_l)}\left[\log \sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right] $$

優勢：

  • 不需訓練獎勵模型
  • 不需 RL 訓練基礎設施
  • 訓練更穩定、更簡單
  • 效果與 RLHF 相當

其他對齊方法

ORPO（Odds Ratio Preference Optimization）：

  • 結合 SFT 和偏好學習為一個步驟
  • 不需參考模型
  • 訓練更簡單高效

KTO（Kahneman-Tversky Optimization）：

  • 只需要回答是「好」或「壞」的二元標記
  • 不需要成對偏好資料
  • 降低資料收集成本

IPO（Identity Preference Optimization）：

  • 修正 DPO 的過度擬合問題
  • 更嚴格的理論保證

對齊稅（Alignment Tax）

對齊訓練可能導致某些能力下降：

能力影響原因
程式碼生成輕微下降格式偏好改變
創意寫作可能受限安全約束
事實性提升鼓勵坦承不確定
指令遵循大幅提升SFT + RLHF 核心目標
數學推理輕微下降非主要優化目標

應用場景

ChatGPT 風格對話模型

  • OpenAI 的 InstructGPT 開創了 RLHF 應用
  • GPT-4 使用多輪 RLHF 提升對話品質
  • Claude 使用 Constitutional AI + RLHF

開源模型對齊

  • LLaMA 2 Chat：使用 RLHF 訓練對話版本
  • Zephyr：使用 DPO 對齊 Mistral
  • Tulu：系統化研究不同對齊方法

安全對齊

  • 訓練模型拒絕有害請求
  • 減少偏見和歧視性輸出
  • 處理邊界案例（jailbreak 防護）

特定領域對齊

  • 醫療模型：偏好安全保守的建議
  • 教育模型：偏好引導式而非直接給答案
  • 客服模型：偏好禮貌、結構化的回應

線上對齊

  • 使用使用者隱式回饋（點讚/點踩）持續改進
  • A/B 測試不同對齊策略
  • 逐步收緊安全邊界
