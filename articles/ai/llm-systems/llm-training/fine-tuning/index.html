模型微調技術

模型微調（Fine-tuning）是將預訓練的大型語言模型適配到特定任務或領域的關鍵技術。隨著模型規模增長，全量微調的成本變得極高，因此參數高效微調（PEFT）方法應運而生。本文將深入探討 LoRA、QLoRA 等主流微調技術的原理與實踐。

核心概念

為什麼需要微調

預訓練模型雖然擁有廣泛的語言知識，但在以下場景需要微調：

  • 領域適配：醫療、法律、金融等專業領域的術語與知識
  • 行為對齊：讓模型遵循特定的回應風格與格式
  • 任務特化：針對特定任務（分類、摘要、翻譯）最佳化
  • 知識注入：將企業內部知識融入模型

微調類型光譜

類型修改範圍記憶體需求效果
全量微調（Full Fine-tuning）所有參數極高最強
LoRA低秩適配器低接近全量
QLoRA量化 + LoRA極低接近 LoRA
Prompt Tuning軟提示極低中等
Adapter Tuning適配器層低良好
Prefix Tuning前綴向量極低中等

訓練資料格式

微調資料通常使用指令格式（Instruction Format）：

{
  "instruction": "請將以下文章摘要為三個重點",
  "input": "（原文內容）",
  "output": "1. 第一點\n2. 第二點\n3. 第三點"
}

對話格式（Chat Format）：

{
  "messages": [
    {"role": "system", "content": "你是一位專業的醫療顧問"},
    {"role": "user", "content": "什麼是高血壓？"},
    {"role": "assistant", "content": "高血壓是指..."}
  ]
}

架構總覽

LoRA（Low-Rank Adaptation）

LoRA 的核心洞察：微調時的權重變化矩陣是低秩的，可用兩個小矩陣的乘積來近似。

原理：

  • 原始權重 W₀ 保持凍結（不更新）
  • 新增低秩分解：ΔW = BA，其中 B ∈ R^(d×r)，A ∈ R^(r×k)
  • r（秩）遠小於 d 和 k（通常 r = 8~64）
  • 前向傳播：h = W₀x + BAx

優勢：

  • 可訓練參數量降低 10,000 倍以上
  • 記憶體需求大幅減少
  • 推理時可將 LoRA 權重合併回原模型，無額外延遲
  • 可切換不同 LoRA 適配器用於不同任務

關鍵超參數：

參數說明典型值
rank (r)低秩維度8-64
alpha縮放因子16-128
target_modules套用 LoRA 的層q_proj, v_proj, k_proj, o_proj
dropoutLoRA Dropout0.05-0.1

QLoRA（Quantized LoRA）

QLoRA 在 LoRA 基礎上加入量化技術，進一步壓縮記憶體：

  1. 4-bit NormalFloat（NF4）量化：將基礎模型量化為 4-bit
  2. 雙重量化（Double Quantization）：量化常數也被量化
  3. 分頁優化器（Paged Optimizers）：使用統一記憶體避免 OOM
  4. LoRA 適配器保持 BF16 精度訓練

效果：65B 參數模型可在單張 48GB GPU 上微調。

DoRA（Weight-Decomposed Low-Rank Adaptation）

分解權重為幅度（magnitude）與方向（direction），分別適配：

  • 方向用 LoRA 更新
  • 幅度用獨立的可訓練向量
  • 在部分任務上優於標準 LoRA

全量微調流程

預訓練模型
    ↓
準備資料（清洗、格式化、切分）
    ↓
設定訓練配置（學習率、Batch Size、Epochs）
    ↓
訓練（通常 1-5 Epochs）
    ↓
評估（Loss、任務指標、人工評估）
    ↓
部署（合併權重或載入 Adapter）

應用場景

企業領域模型

  • 金融：財報分析、風險評估模型
  • 醫療：病歷摘要、診斷輔助
  • 法律：合約審查、法規問答

程式碼模型微調

  • 在企業內部程式碼庫上微調 Code LLaMA
  • 特定框架/語言的程式碼補全最佳化
  • 程式碼審查風格統一

多語言適配

  • 在特定語言語料上微調提升小語種能力
  • 保持英文能力的同時增強中文/日文等

對話風格定制

  • 客服機器人的品牌語氣
  • 教育場景的引導式回答風格
  • 角色扮演與創意寫作

知識蒸餾

  • 使用大模型（GPT-4）生成訓練資料
  • 微調小模型達到接近大模型的效果
  • 降低推理成本同時維持品質
