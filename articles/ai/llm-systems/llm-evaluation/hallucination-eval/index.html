幻覺檢測與評估

幻覺（Hallucination）是大型語言模型最關鍵的可靠性問題之一，指模型生成看似合理但實際不正確或無根據的內容。在高風險應用場景中（醫療、法律、金融），幻覺可能造成嚴重後果。本文將探討幻覺的分類、檢測方法與緩解策略。

核心概念

幻覺的定義與分類

按事實性分類：

類型說明範例
事實性幻覺（Factual）生成與事實不符的內容「愛因斯坦是物理學家，1879 年出生於法國」
忠實性幻覺（Faithfulness）生成與輸入來源不符的內容摘要中加入原文沒有的資訊
編造（Fabrication）完全虛構不存在的實體引用不存在的論文

按嚴重程度分類：

程度說明影響
輕微（Minor）細節錯誤（日期、數字）可容忍
中度（Moderate）邏輯錯誤或過度推論需修正
嚴重（Critical）完全虛構或危險誤導不可接受

幻覺產生的原因

  1. 訓練資料問題：噪音、過時、矛盾的資料
  2. 解碼策略：高 temperature 增加創造性但也增加幻覺
  3. 知識邊界模糊：模型不知道自己不知道什麼
  4. pattern completion 傾向：模型傾向生成「看起來對」的內容
  5. 長文本退化：長序列生成中逐漸偏離事實

評估維度

維度說明評估方式
Faithfulness（忠實度）輸出是否忠於輸入來源對比輸入與輸出
Factuality（事實性）輸出是否符合世界知識與知識庫比對
Groundedness（有據性）每個聲明是否有來源支持逐句驗證
Consistency（一致性）多次回答是否一致重複查詢比較

架構總覽

自動檢測方法

基於 NLI 的方法（Natural Language Inference）：

  1. 將生成的文本拆分為句子/聲明
  2. 以來源文件為前提，各聲明為假設
  3. NLI 模型判斷「蘊含」（支持）或「矛盾」（幻覺）
來源：「台灣位於亞洲東部，人口約 2,300 萬」
聲明：「台灣人口超過 5,000 萬」→ 矛盾 → 幻覺
聲明：「台灣是亞洲國家」→ 蘊含 → 正確

基於 QA 的方法（QAFactEval）：

  1. 從生成文本中抽取事實性問題
  2. 使用來源文件回答這些問題
  3. 比較回答與原生成內容的一致性

基於 LLM 的方法（LLM-as-Judge）：

  • 使用 GPT-4 等強模型評估輸出的事實性
  • 提供來源文件和生成文本，要求判斷忠實度
  • 可擴展到多維度評估

基於不確定性的方法：

  • 分析模型的 token 機率分布
  • 高熵位置可能代表不確定（潛在幻覺）
  • 語義等價的多次生成比較一致性

應用場景

RAG 系統品質監控

  • 持續監控回答的 Faithfulness Score
  • 檢測何時模型「添油加醋」超出檢索內容
  • 自動標記需人工審查的低信心回答

醫療 AI 安全

  • 確保醫療建議基於可靠來源
  • 對高風險聲明（用藥、診斷）特別嚴格
  • 強制引用臨床指引

法律文件審查

  • 驗證合約分析結果的準確性
  • 確保法規引用真實存在
  • 標記不確定的法律解釋

新聞與內容生成

  • 事實查核自動化
  • 檢測 AI 生成文章中的虛構資訊
  • 確保引用的統計數據有來源
