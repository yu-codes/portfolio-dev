混合精度運算

Table of Contents

  • 架構總覽

  • 應用場景
  • 延伸閱讀

混合精度訓練（Mixed Precision Training）是一種同時使用低精度（FP16/BF16）和全精度（FP32）進行模型訓練的技術。它能在幾乎不損失模型品質的前提下，大幅提升訓練速度並減少記憶體佔用，是現代深度學習訓練的標準配置。

核心概念

數值格式基礎：不同浮點格式在表示範圍和精度上有所取捨：

  • FP32（32-bit）：1 位符號 + 8 位指數 + 23 位尾數，動態範圍大且精度高
  • FP16（16-bit）：1 位符號 + 5 位指數 + 10 位尾數，範圍有限（±65504）
  • BF16（Brain Float 16）：1 位符號 + 8 位指數 + 7 位尾數，範圍同 FP32 但精度較低
  • FP8（E4M3/E5M2）：最新一代格式，進一步壓縮至 8-bit

混合精度的「混合」含義：並非所有運算都用低精度。前向傳播和反向傳播的矩陣運算使用 FP16/BF16 加速，而權重的主副本（master weights）和某些對精度敏感的運算仍保持 FP32。

損失縮放（Loss Scaling）：FP16 的動態範圍有限，梯度值常落在可表示範圍之外（下溢）。損失縮放將 loss 乘以一個大的縮放因子，使梯度值移入 FP16 可表示範圍，更新時再除以該因子。

動態損失縮放：自動調整縮放因子——若一定步數內無溢位則增大因子，遇到溢位則減小因子並跳過該步更新。PyTorch 的 GradScaler 實現了此機制。

Tensor Core 加速：NVIDIA Tensor Core 對 FP16/BF16 矩陣運算提供硬體加速，吞吐量是 FP32 CUDA Core 的 8-16 倍。使用混合精度即可自動觸發 Tensor Core 運算。

記憶體節省分析：模型參數從 FP32 轉為 FP16 儲存，記憶體減半。激活值同樣使用 FP16，進一步節省記憶體。但需額外保存 FP32 master weights，實際節省約 37.5%（考慮 optimizer states）。

AMP（Automatic Mixed Precision）：PyTorch 的 torch.cuda.amp 模組自動決定哪些運算使用低精度、哪些保持全精度。autocast 上下文管理器處理型別轉換，使用者無需手動管理。

BF16 vs FP16：BF16 保持了 FP32 的動態範圍，幾乎不需要損失縮放，使用更簡便。但精度較低（7 vs 10 位尾數），在某些累加運算中可能需要注意。現代 LLM 訓練普遍偏好 BF16。

應用場景

標準模型訓練加速：對於 ResNet、BERT 等主流模型，混合精度訓練可帶來 1.5-3 倍速度提升，且精度損失可忽略。這是最低成本的訓練加速手段。

大型語言模型預訓練：GPT-3、LLaMA 等模型的預訓練全部使用 BF16 混合精度。在數千億 token 的訓練過程中，BF16 的數值穩定性經過充分驗證。

記憶體受限場景：當模型大小接近 GPU 記憶體上限時，混合精度可釋放足夠記憶體來增大批次大小或處理更長序列。

推理優化：推理階段可更積極地使用低精度（FP16 甚至 INT8/FP8），因為不需要梯度計算。TensorRT 和 vLLM 都支援多種精度的推理。

邊緣裝置部署：在記憶體和算力受限的邊緣裝置上，低精度運算是必要條件。量化感知訓練（QAT）可視為混合精度的延伸。
