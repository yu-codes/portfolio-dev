Horovod 與 DeepSpeed

Table of Contents

  • 應用場景
  • 延伸閱讀

Horovod 與 DeepSpeed 是兩個廣泛使用的分散式訓練框架，分別代表了不同時期和不同設計哲學的解決方案。Horovod 以簡潔的 API 和高效的通訊著稱，DeepSpeed 則以極致的記憶體優化和超大模型支援聞名。

核心概念

Horovod 設計哲學：由 Uber 開發，目標是讓分散式訓練「只需修改幾行程式碼」。基於 MPI 的通訊模型，使用 Ring-AllReduce 進行梯度同步，支援 TensorFlow、PyTorch 和 MXNet。

Horovod 核心 API：

  • hvd.init()：初始化分散式環境
  • hvd.DistributedOptimizer()：包裝優化器以自動同步梯度
  • hvd.broadcast_parameters()：廣播初始參數確保一致性
  • 透過 horovodrun 或 mpirun 啟動多進程訓練

DeepSpeed 設計哲學：由 Microsoft 開發，專注於突破記憶體牆限制，使兆級參數模型的訓練成為可能。透過 ZeRO（Zero Redundancy Optimizer）系列技術，在不犧牲通訊效率的前提下大幅減少記憶體佔用。

ZeRO 三階段優化：

  • ZeRO Stage 1：分片 Optimizer States（記憶體減少 4x）
  • ZeRO Stage 2：分片 Optimizer States + Gradients（記憶體減少 8x）
  • ZeRO Stage 3：分片 Optimizer States + Gradients + Parameters（記憶體減少線性於裝置數）

ZeRO-Offload / ZeRO-Infinity：將部分資料卸載到 CPU 記憶體或 NVMe SSD，進一步擴展可訓練模型規模。

DeepSpeed 配置系統：透過 JSON 配置檔管理所有分散式訓練設定，包括 ZeRO 階段、批次大小、學習率排程、混合精度等。這種聲明式配置大幅簡化了超大規模訓練的管理。

通訊優化：Horovod 的 Tensor Fusion 將小張量合併後再通訊；DeepSpeed 的通訊重疊（Communication Overlap）將參數收集與前向計算重疊執行。

梯度累積：兩者都支援梯度累積以模擬大批次訓練。DeepSpeed 的 gradient accumulation 與 ZeRO 深度整合，確保記憶體效率。

選擇建議：如果模型能放入單張 GPU 且只需資料平行，Horovod 或原生 DDP 已足夠；如果模型超過單卡記憶體或需要極致效能優化，DeepSpeed 或 FSDP 是更好的選擇。

應用場景

企業級 NLP 模型訓練：多數企業的 BERT/GPT 微調任務使用 Horovod 或 DDP 即可滿足需求，部署簡單且效能穩定。

超大語言模型預訓練：DeepSpeed 的 ZeRO-3 + Offload 使得在有限 GPU 資源上訓練百億參數模型成為可能。搭配 Megatron-DeepSpeed 可擴展至兆級。

研究實驗快速迭代：Horovod 的低侵入性 API 適合研究者快速將單 GPU 實驗擴展到多 GPU，無需大幅修改程式碼。

成本敏感的訓練場景：DeepSpeed ZeRO-Offload 可利用 CPU 記憶體擴展可訓練規模，減少對昂貴 GPU 記憶體的需求。

端到端訓練平台：DeepSpeed 整合了訓練、壓縮（剪枝/量化/蒸餾）和推理的完整管線，適合建構一站式 MLOps 平台。
