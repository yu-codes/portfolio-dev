訓練系統框架是將分散式運算理論轉化為可操作工具的軟體層。這些框架封裝了複雜的通訊邏輯、記憶體管理和排程策略，讓研究者和工程師能專注於模型本身而非底層系統細節。從早期的 Horovod 到如今的 DeepSpeed 和 Megatron-LM，訓練框架的演進反映了模型規模和訓練需求的劇烈變化。

核心概念

訓練框架的角色：訓練框架位於 AI 運算堆疊的中間層，向下對接通訊庫（NCCL、MPI）和硬體抽象（CUDA），向上為使用者提供簡潔的 API。好的訓練框架應該做到：最小化程式碼改動量、自動處理分散式複雜性、提供靈活的策略組合，並具備良好的故障恢復能力。

記憶體優化策略：大模型訓練中，GPU 記憶體被以下部分佔用：模型參數（Φ）、梯度（Φ）、優化器狀態（如 Adam 為 2Φ）和活化值（取決於批次和序列長度）。訓練框架透過各種技術降低記憶體需求：

  • ZeRO（Zero Redundancy Optimizer）：將優化器狀態、梯度和參數分片到多 GPU，消除冗餘儲存
  • Activation Checkpointing（梯度檢查點）：只保存部分活化值，需要時重新計算，以時間換空間
  • Offloading：將暫時不需要的張量卸載到 CPU 記憶體或 NVMe SSD

通訊優化：框架需要解決通訊與運算的重疊（Overlap）問題。透過將梯度同步與反向傳播重疊執行，可以隱藏大部分通訊延遲。進階技術包括梯度壓縮、稀疏通訊和階層式 AllReduce。

容錯與恢復：大規模訓練可能持續數週，硬體故障不可避免。現代框架支持非同步 checkpoint 儲存、彈性訓練（Elastic Training）和自動重啟機制。

├─────────────────────────────────────────────────┤

Megatron-LM 3D 平行：
Data Parallel (across node groups)
  └─ Pipeline Parallel (across nodes)
       └─ Tensor Parallel (within node, NVLink)

DeepSpeed 核心元件：ZeRO Optimizer（記憶體優化）、DeepSpeed-Chat（RLHF 訓練管線）、Inference Engine（推論加速）、Compression Library（模型壓縮）。透過 JSON 配置檔控制策略選擇，無需修改訓練程式碼。

PyTorch FSDP：Fully Sharded Data Parallel 是 PyTorch 原生的 ZeRO-3 實作。與 PyTorch 2.0 的 torch.compile 和 DTensor 深度整合，提供更好的可組合性。使用 auto_wrap_policy 控制分片粒度。

效能調優關鍵：ZeRO Stage 選擇取決於記憶體壓力——Stage 1 最低通訊開銷但節省最少，Stage 3 節省最多但通訊量大；Gradient Accumulation Steps 可在不增加通訊頻率的情況下增大有效批次大小；通訊-運算重疊需要足夠的運算粒度來隱藏通訊。

應用場景

從零預訓練 LLM：使用 Megatron-DeepSpeed（兩者結合）訓練百億到千億參數模型。典型配置：ZeRO-1 + 張量平行 8 路（節點內）+ 管線平行 4-8 路（跨節點）+ 資料平行擴展。

大模型微調：使用 DeepSpeed ZeRO-3 + LoRA 在有限 GPU（如 4-8 張 A100）上微調 70B 模型。ZeRO-Offload 可進一步將部分計算卸載到 CPU，適合資源受限場景。

RLHF 訓練：DeepSpeed-Chat 提供完整的 RLHF 管線（SFT → Reward Model → PPO），自動管理多模型的記憶體分配和通訊。

研究快速迭代：Horovod 和 PyTorch DDP 適合模型規模中等（< 10B）的研究場景，配置簡單，幾行程式碼即可完成分散式化。
