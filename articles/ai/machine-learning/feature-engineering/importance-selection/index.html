特徵重要性與選擇

在機器學習專案中，並非所有特徵都對預測目標有同等貢獻。識別重要特徵並移除不相關或冗餘的特徵，不僅能提升模型效能，還能降低計算成本、減少過擬合風險並提高模型可解釋性。本文將系統性地介紹特徵重要性的評估方法與特徵選擇策略。

核心概念

特徵重要性的評估方法

模型內建重要性：

  • 樹模型特徵重要性：基於分裂時的資訊增益或基尼不純度減少。隨機森林和 XGBoost 都提供 feature_importances_ 屬性。注意：對高基數類別特徵有偏好
  • 線性模型係數：標準化後的迴歸係數絕對值反映特徵重要性。前提是特徵需做標準化處理
  • 注意力權重：深度學習模型中 Attention 機制可視為一種動態的特徵重要性

模型無關方法（Model-Agnostic）：

  • Permutation Importance：隨機打亂某特徵的值，觀察模型效能下降程度。下降越多表示該特徵越重要。優點是不依賴特定模型，缺點是相關特徵會互相影響
  • SHAP（SHapley Additive exPlanations）：基於博弈論的 Shapley 值，為每個預測分配各特徵的貢獻。理論上最公平的歸因方法，但計算成本高
  • LIME（Local Interpretable Model-agnostic Explanations）：在局部用簡單模型擬合黑箱模型的行為，得到局部特徵重要性

統計方法：

  • 相關係數：Pearson（線性相關）、Spearman（單調相關）、Kendall（序數相關）
  • 互資訊（Mutual Information）：衡量特徵與目標之間的非線性依賴關係
  • 卡方檢定：類別特徵與類別目標之間的獨立性檢定
  • ANOVA F-test：數值特徵與類別目標之間的關聯性

特徵選擇方法

過濾法（Filter Methods）： 獨立於模型，僅基於特徵與目標的統計關係進行篩選。

  • 方差閾值：移除方差接近零的特徵（幾乎是常數）
  • 相關性篩選：保留與目標相關性高的特徵，移除特徵間高度相關的冗餘特徵
  • 互資訊篩選：選擇與目標互資訊最高的 Top-K 特徵
  • 優點：計算快速、不易過擬合
  • 缺點：忽略特徵間的交互效果

包裝法（Wrapper Methods）： 使用特定模型的效能作為特徵子集的評估準則。

  • 前向選擇（Forward Selection）：從空集開始，每次加入使效能提升最大的特徵
  • 後向消除（Backward Elimination）：從全部特徵開始，每次移除影響最小的特徵
  • 遞迴特徵消除（RFE）：iteratively 訓練模型並移除最不重要的特徵
  • 優點：考慮特徵交互，找到的子集對特定模型最優
  • 缺點：計算量大（需要反覆訓練模型），可能過擬合於特定模型

嵌入法（Embedded Methods）： 在模型訓練過程中同時進行特徵選擇。

  • L1 正則化（Lasso）：將不重要特徵的係數壓縮為零，自動實現特徵選擇
  • 樹模型 + 重要性閾值：訓練樹模型後，保留重要性超過閾值的特徵
  • Elastic Net：結合 L1 和 L2 正則化，既能選擇特徵又能處理相關特徵群組
  • 優點：效率高於包裝法，考慮特徵交互
  • 缺點：依賴特定模型假設

降維 vs 特徵選擇

特徵選擇保留原始特徵的子集，而降維則建立新的特徵組合：

  • PCA：找到方差最大的正交方向，新特徵是原始特徵的線性組合
  • LDA：監督式降維，最大化類別間距離與類別內距離的比值
  • 自編碼器：非線性降維，學習資料的壓縮表示

架構總覽

特徵選擇的系統性工作流程：

  1. 初步篩選：

    • 移除零方差和近零方差特徵
    • 移除缺失率過高（>80%）的特徵
    • 移除與業務邏輯不相關的特徵

  2. 統計分析：

    • 計算單變量統計量（相關係數、互資訊、卡方值）
    • 分析特徵間的多重共線性（VIF > 10 需警惕）
    • 視覺化特徵分佈與目標的關係

  3. 模型驅動選擇：

    • 訓練基準模型（隨機森林/XGBoost）獲取特徵重要性
    • 使用 Permutation Importance 進行驗證
    • RFE 或 Boruta 演算法進行穩健選擇

  4. 驗證與迭代：

    • 比較使用全部特徵 vs 選擇後特徵的交叉驗證表現
    • 檢查是否遺漏了重要特徵（domain knowledge）
    • 確認選擇結果在不同資料子集上穩定

Tree Importance vs Permutation Importance：

  • Tree Importance：訓練時計算，快速但偏好高基數特徵、受相關特徵影響
  • Permutation Importance：測試集上計算，更公正但需注意相關特徵的互相掩蓋效應

SHAP vs LIME：

  • SHAP：全局一致的歸因，理論保證，但精確計算為指數複雜度
  • LIME：局部近似，計算較快，但不同的擾動策略可能得到不同結果

應用場景

場景一：基因體學中的生物標記發現

基因表達資料通常有數萬個特徵但僅數百個樣本（p » n）。使用 Elastic Net 結合 Boruta 演算法篩選出顯著的基因標記。過濾法先移除低方差基因，再用包裝法精選候選標記。

場景二：金融信用評分模型

監管要求模型具有可解釋性。使用 SHAP 分析各特徵的貢獻，篩選出 15-20 個最具預測力且業務上可解釋的特徵。移除相關性 > 0.9 的冗餘特徵避免多重共線性問題。

場景三：工業品質預測

製造過程中可能有數百個感測器特徵。先用方差篩選和相關性分析移除冗餘，再用 XGBoost 特徵重要性選擇 Top-50，最後用 RFE 精簡至 20 個最關鍵的製程參數。

場景四：NLP 文本分類

TF-IDF 向量可能有數十萬維度。使用卡方檢定或互資訊選擇最具區分力的 5000-10000 個詞彙特徵。或使用 L1 正則化的邏輯回歸自動將大部分詞彙的權重壓縮為零。
