分散式訓練與精度優化

深度學習模型的訓練優化是一項系統工程，涵蓋從單一 GPU 的記憶體管理到跨節點分散式訓練的全面技術。隨著模型規模從億級參數增長到千億甚至萬億級，訓練優化技術成為決定能否成功訓練的關鍵因素。本文將系統性介紹分散式訓練策略、混合精度訓練、記憶體優化與梯度累積等核心技術。

核心概念

記憶體瓶頸分析

訓練深度模型時，GPU 記憶體被以下組件佔用：

組件說明典型佔比
模型參數權重矩陣~25%
梯度反向傳播計算的梯度~25%
優化器狀態Adam 的 m、v 動量~50%
激活值前向傳播的中間結果視 batch size

以 FP32 訓練一個 7B 參數模型為例：

  • 參數：7B × 4 bytes = 28 GB
  • 梯度：28 GB
  • Adam 優化器：56 GB（m + v 各 28 GB）
  • 總計：~112 GB（不含激活值）

這遠超單張 GPU（A100 80GB）的容量，因此需要各種優化技術。

混合精度訓練（Mixed Precision Training）

混合精度訓練使用半精度（FP16/BF16）與全精度（FP32）的組合，在不損失模型品質的前提下大幅降低記憶體使用與加速計算：

核心策略：

  • 前向/反向傳播：使用 FP16/BF16 計算，速度提升 2-3 倍
  • 主權重（Master Weights）：維持 FP32 副本確保數值穩定
  • 損失縮放（Loss Scaling）：放大損失值避免梯度下溢

FP16 vs BF16：

精度指數位尾數位動態範圍精度
FP32823大高
FP16510小（易溢位）中
BF1687大（同 FP32）較低

BF16 因其與 FP32 相同的動態範圍，在 LLM 訓練中更為穩定，是目前的主流選擇。

梯度累積（Gradient Accumulation）

當 GPU 記憶體無法容納理想的 batch size 時，梯度累積提供了等效的解決方案：

  1. 使用小 batch 進行前向/反向傳播
  2. 累積多個 micro-batch 的梯度
  3. 累積完成後才執行參數更新

等效 batch size = micro_batch_size × accumulation_steps × num_GPUs

梯度檢查點（Gradient Checkpointing）

又稱 Activation Checkpointing，以時間換空間的策略：

  • 前向傳播時只保留部分層的激活值（檢查點）
  • 反向傳播時重新計算被丟棄的激活值
  • 記憶體節省約 60-70%，計算時間增加約 30%

ZeRO 三階段

DeepSpeed 的 ZeRO（Zero Redundancy Optimizer）透過三個漸進階段消除資料並行中的記憶體冗餘：

階段分散內容記憶體節省通訊開銷
ZeRO-1優化器狀態4×與 DDP 相同
ZeRO-2+ 梯度8×略增
ZeRO-3+ 參數線性擴展較高

FSDP（Fully Sharded Data Parallel）

PyTorch 原生的分散式訓練方案，概念類似 ZeRO-3：

  • 將模型參數、梯度與優化器狀態完全分片到各 GPU
  • 計算時按需收集（All-Gather）完整參數
  • 反向傳播後分散（Reduce-Scatter）梯度
  • 與 PyTorch 生態系無縫整合

應用場景

場景一：單 GPU 最大化利用

研究人員或個人開發者在有限硬體上訓練模型。結合混合精度、梯度累積與梯度檢查點，可在單張消費級 GPU 上微調 7B 模型。

場景二：多 GPU 叢集訓練

團隊在多節點 GPU 叢集上預訓練中大型模型。使用 DeepSpeed ZeRO-3 或 FSDP，搭配高速 InfiniBand 網路，實現接近線性的擴展效率。

場景三：超大規模 LLM 預訓練

如 GPT-4、LLaMA-3 級別的千億參數模型訓練，需要結合資料並行、張量並行與流水線並行的 3D 並行策略，搭配 ZeRO 記憶體優化。

場景四：推論效能優化

訓練完成後的推論階段也需優化：量化（INT8/INT4）、KV Cache 優化、Speculative Decoding 等技術大幅降低推論延遲與成本。
