損失函數與優化器設計

在深度學習的實踐中，損失函數的設計與優化器的選擇直接影響模型的訓練效率和最終表現。進階的損失函數能引導模型學習更好的表示，而適當的優化器設計則能在複雜的損失地形中有效導航。本文將深入探討超越基礎的進階損失函數設計與現代優化器的運作原理。

核心概念

進階損失函數設計

處理類別不平衡的損失函數：

  • Weighted Cross-Entropy：對少數類別賦予更大的損失權重。簡單直接但需要手動設定權重

  • Focal Loss：γ 參數動態降低易分類樣本的損失權重，讓模型專注於困難樣本。在目標檢測（RetinaNet）中大獲成功

    FL(p) = -α(1-p)^γ · log(p)

    當 γ=0 退化為標準 Cross-Entropy，γ=2 是常用設定

  • Dice Loss / IoU Loss：直接優化與評估指標（Dice coefficient / IoU）相關的損失，常用於影像分割任務

度量學習損失（Metric Learning Losses）：

  • Contrastive Loss：拉近正樣本對、推遠負樣本對的距離。需要成對的訓練資料

  • Triplet Loss：同時考慮 anchor-positive-negative 三元組，要求正樣本距離比負樣本距離小一個 margin

    L = max(0, d(a,p) - d(a,n) + margin)

  • InfoNCE / NT-Xent：對比學習的標準損失（SimCLR、CLIP），在 batch 內構造正負樣本對

  • ArcFace / CosFace：在 angular space 中加入 margin，用於人臉辨識等高精度需求

知識蒸餾損失：

  • KL Divergence：讓學生模型的輸出分佈逼近教師模型的 soft predictions

  • Temperature Scaling：提高溫度使 softmax 輸出更平滑，揭示類別間的相對關係

    L_distill = KL(σ(z_s/T) || σ(z_t/T)) × T²

多任務學習損失：

  • 固定權重加總：L = w₁L₁ + w₂L₂ + … 簡單但需手動調整
  • Uncertainty Weighting：根據每個任務的同方差不確定性自動學習損失權重
  • GradNorm：根據各任務的梯度大小動態平衡損失權重

現代優化器設計

Adam 家族：

  • Adam（Adaptive Moment Estimation）： 結合一階動量（梯度的指數移動平均）和二階動量（梯度平方的指數移動平均）。為每個參數自適應地調整學習率。

    m_t = β₁m_{t-1} + (1-β₁)g_t v_t = β₂v_{t-1} + (1-β₂)g_t² θ_t = θ_{t-1} - η · m̂_t / (√v̂_t + ε)

  • AdamW： 將權重衰減（weight decay）從梯度更新中解耦。在 Adam 中，L2 正則化的梯度會被自適應學習率縮放，導致正則化效果不一致。AdamW 直接對參數做衰減，是目前大型模型訓練的標準選擇。

  • AdaFactor： 記憶體高效的 Adam 變體，使用行和列的統計量近似完整的二階動量矩陣。適合超大模型（如 T5）的訓練。

SGD 的復興：

  • SGD + Momentum + Weight Decay： 在 ImageNet 訓練中，精心調整的 SGD 常常與 Adam 表現相當甚至更好。泛化能力通常優於 Adam（收斂到更平坦的最小值）。

  • SGDW：類比 AdamW，將權重衰減與梯度更新解耦的 SGD 版本。

新興優化器：

  • LAMB（Layer-wise Adaptive Moments for Batch training）： 為大 batch 訓練設計，透過逐層自適應信賴比率（trust ratio）穩定訓練。允許使用更大的 batch size 而不損失精度。

  • LION（EvoLved Sign Momentum）： 由 Google 用演化演算法搜尋出的優化器。只使用梯度的符號（sign）進行更新，記憶體比 Adam 少 50%。

  • Sophia： 使用二階資訊（Hessian 對角線的估計）進行自適應學習率調整，在 LLM 預訓練中展現出比 Adam 更快的收斂速度。

學習率排程

常用排程策略：

  • Step Decay：每隔固定 epoch 將學習率乘以衰減因子
  • Cosine Annealing：學習率按餘弦曲線從高到低，支持 restart
  • Linear Warmup + Cosine Decay：現代標準做法，先線性增加再餘弦衰減
  • One-Cycle Policy：先升後降的三角形排程，SuperConvergence 的關鍵
  • Polynomial Decay：以多項式函數衰減，BERT 預訓練中常用

架構總覽

損失函數與優化器的選擇框架：

  1. 問題分析：

    • 識別任務類型與評估指標
    • 分析資料分佈（是否不平衡、是否有噪音標籤）
    • 確定是否有多任務或輔助損失需求

  2. 損失函數設計：

    • 主損失：與目標直接相關的監督信號
    • 輔助損失：正則化項、一致性約束、蒸餾損失
    • 損失權重：固定或自適應

  3. 優化器選擇：

    • 預設選擇：AdamW（通用性最好）
    • 電腦視覺：SGD + Momentum（可能泛化更好）
    • 大型語言模型：AdamW / Sophia
    • 記憶體受限：AdaFactor / LION

  4. 學習率配置：

    • 確定峰值學習率（LR Finder）
    • 選擇 warmup 步數（通常佔總步數的 5-10%）
    • 選擇衰減策略（cosine 最通用）

應用場景

場景一：大型語言模型預訓練

GPT/LLaMA 的訓練配置：

  • 損失：Autoregressive Cross-Entropy（next token prediction）
  • 優化器：AdamW（β₁=0.9, β₂=0.95, ε=1e-8）
  • 學習率：Linear Warmup（2000 steps）+ Cosine Decay
  • Weight Decay：0.1（排除 bias 和 LayerNorm 參數）
  • Gradient Clipping：max_norm = 1.0

場景二：目標檢測系統

YOLO / Faster R-CNN 的損失組合：

  • 分類損失：Focal Loss（γ=2, α=0.25）解決前景/背景不平衡
  • 定位損失：CIoU Loss 同時考慮重疊面積、中心距離和長寬比
  • 置信度損失：Binary Cross-Entropy
  • 多尺度特徵的損失加權平衡

場景三：人臉辨識系統

使用 ArcFace Loss 訓練嵌入模型：

  • 在 angular space 加入 additive margin（m=0.5）
  • 特徵縮放因子 s=64
  • 配合 SGD + Momentum（lr=0.1, momentum=0.9）
  • Step Decay 學習率排程
  • 訓練後的嵌入向量用於 1:1 驗證和 1:N 辨識

場景四：自監督對比學習

SimCLR / MoCo 的訓練配置：

  • 損失：InfoNCE with temperature τ=0.07
  • 大 batch size（4096+）提供足夠的負樣本
  • 優化器：LARS / LAMB（支持大 batch 穩定訓練）
  • Cosine LR Schedule without restart
  • 投影頭（Projection Head）的設計影響表示品質
