模型選擇策略

面對一個新的機器學習問題，選擇合適的模型是決定專案成敗的關鍵決策。沒有一個模型能在所有問題上表現最好（No Free Lunch Theorem），因此理解不同模型的特性、適用場景以及系統化的評估方法至關重要。

核心概念

偏差-方差權衡

模型的泛化誤差可以分解為三個部分：

  • 偏差（Bias）：模型假設帶來的系統性誤差。高偏差意味著模型太簡單，無法捕捉資料中的模式（欠擬合）
  • 方差（Variance）：模型對訓練資料波動的敏感度。高方差意味著模型太複雜，過度擬合訓練集的噪音
  • 不可約誤差（Irreducible Error）：資料本身的隨機噪音，無法透過任何模型消除

模型選擇的核心目標就是找到偏差與方差的最佳平衡點。

模型複雜度的維度

影響模型選擇的關鍵因素：

資料特性：

  • 樣本數量：小資料偏好簡單模型，大資料能支撐複雜模型
  • 特徵維度：高維稀疏資料適合線性模型，低維稠密資料可用非線性模型
  • 資料類型：結構化表格資料 → 樹模型；序列資料 → RNN/Transformer；影像 → CNN
  • 雜訊程度：高噪音資料需要更強的正則化或更簡單的模型

問題需求：

  • 預測精度：是否需要追求最高精度（Kaggle 競賽）還是「夠好就行」
  • 可解釋性：金融、醫療、法律領域常要求模型可解釋
  • 推論速度：即時推薦需要毫秒級響應，批次預測可容忍更長時間
  • 訓練成本：GPU 預算、訓練時間限制
  • 維護成本：模型更新頻率、技術債務

評估框架

交叉驗證（Cross-Validation）：

  • K-Fold CV：將資料分為 K 份，輪流作為驗證集。標準選擇 K=5 或 K=10
  • Stratified K-Fold：保持每折中目標變數的分佈一致，適合不平衡資料
  • Time Series Split：時間序列資料的擴展窗口或滑動窗口驗證
  • Leave-One-Out CV：極端情況，每次只留一個樣本驗證，計算量大但偏差低

評估指標的選擇：

  • 分類：Accuracy、Precision、Recall、F1、AUC-ROC、AUC-PR（不平衡時優先）
  • 回歸：MSE、RMSE、MAE、R²、MAPE
  • 排序：NDCG、MAP、MRR
  • 實際業務：利潤、成本節省、客戶滿意度（將 ML 指標轉化為業務指標）

超參數調整

搜尋策略：

  • 網格搜尋（Grid Search）：窮舉所有組合，適合參數空間小的情況
  • 隨機搜尋（Random Search）：隨機抽樣參數組合，在高維空間中比網格搜尋更有效
  • 貝氏優化（Bayesian Optimization）：用高斯過程建模搜尋空間，智能選擇下一個評估點。Optuna、Hyperopt 是流行的實作
  • Hyperband / ASHA：早停策略，快速淘汰表現差的配置，節省計算資源

架構總覽

系統化的模型選擇流程：

  1. 問題定義：

    • 明確任務類型（分類/回歸/排序/生成）
    • 確定評估指標和業務約束
    • 定義成功標準（baseline + target）

  2. 快速基線（Baseline）：

    • 建立最簡單的合理模型（如均值預測、邏輯回歸）
    • 作為後續改進的比較基準
    • 幫助判斷問題的難度

  3. 候選模型篩選：

    • 根據資料特性和約束條件圈定候選集
    • 結構化資料：LR → RF → XGBoost → 深度學習
    • 通常不超過 3-5 個候選模型

  4. 初步評估：

    • 使用預設超參數進行交叉驗證
    • 比較各模型的表現和訓練時間
    • 淘汰明顯不適合的模型

  5. 精細調參：

    • 對 Top 2-3 模型進行超參數搜尋
    • 使用貝氏優化或 Hyperband 提高效率
    • 注意過度調參的風險（在驗證集上過擬合）

  6. 最終評估：

    • 在完全獨立的測試集上評估最終模型
    • 分析錯誤案例（Error Analysis）
    • 評估模型的穩定性和公平性

關鍵比較

小樣本（<1000）線性模型 / SVM低方差，不易過擬合

影像/語音CNN / Transformer天然處理空間/序列結構
序列/文本Transformer / LSTM捕捉長程依賴

樹模型 vs 神經網路在表格資料上的對決：

  • 2022 年多項研究證實：在中小型表格資料上，調參後的 XGBoost 通常優於或持平深度學習
  • 深度學習在表格資料上的優勢在於：能與其他模態（文本、圖片）端到端聯合訓練
  • TabNet、FT-Transformer 等專門的表格深度模型正在縮小差距

單一模型 vs 集成模型：

  • 單一模型：簡單、可解釋、部署容易
  • Stacking/Blending：精度通常更高（Kaggle 常勝策略），但複雜度大增
  • 實務中，單一 XGBoost 或 LightGBM 通常是最佳性價比選擇

應用場景

場景一：電商推薦系統

分階段使用不同模型：粗排用簡單的雙塔模型（速度快、處理百萬級候選）、精排用深度交叉網路（精度高、處理千級候選）、重排用業務規則和多樣性調整。每個階段的模型選擇需平衡精度與延遲。

場景二：醫療輔助診斷

可解釋性是硬性要求。使用 XGBoost 進行預測，配合 SHAP 解釋每個診斷決策。或使用 Generalized Additive Model（GAM）提供天然的可解釋性。模型不僅要準確，還要讓醫生能理解和信任。

場景三：自動駕駛感知系統

需要在嚴格的延遲限制內完成推論。選擇輕量化 CNN（MobileNet、EfficientNet-Lite）配合模型剪枝和量化。在精度和推論速度之間進行 Pareto 優化。

場景四：Kaggle 競賽

追求極致精度，不考慮推論成本和可解釋性。標準策略：訓練多種不同架構的模型（XGBoost、LightGBM、CatBoost、NN），然後通過 Stacking 或加權平均進行集成。多樣性是集成有效的關鍵。
