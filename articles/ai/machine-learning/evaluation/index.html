模型效能評估指標與方法

模型評估是機器學習開發流程中至關重要的環節，它決定了模型是否達到上線標準、是否存在偏差，以及在不同場景下的表現是否穩定。選擇正確的評估指標與方法，直接影響模型的商業價值與可靠度。

核心概念

評估的目的

模型評估不僅僅是「計算準確率」，而是回答以下問題：

  • 泛化能力：模型在未見過的資料上表現如何？是否過擬合？
  • 業務對齊：模型指標是否反映實際業務需求？高 AUC 是否等於高商業價值？
  • 公平性：模型對不同群體是否存在系統性偏差？
  • 穩定性：多次訓練的結果是否一致？不同資料切分是否產生顯著差異？

評估策略

Hold-out 驗證：簡單快速，但受資料切分隨機性影響。適合資料量充足時的快速迭代。

K-Fold 交叉驗證：更穩健的效能估計，每筆資料都作為驗證集一次。計算成本為 K 倍，但能有效估計模型的變異性。

時間序列切分：按時間順序切分，避免未來資訊洩漏。使用滑動視窗或擴展視窗策略。

分層抽樣：確保每個 fold 中各類別的比例與全體資料一致，特別適用於不平衡資料集。

架構總覽

完整的模型評估架構包含以下層次：

資料層：定義訓練集、驗證集與測試集的切分策略，確保無資料洩漏。時間序列需按時間排序，推薦系統需按使用者分割。

指標層：根據業務需求選擇主要指標（用於模型選擇）與輔助指標（用於多維度了解模型行為）。主要指標應與業務 KPI 直接相關。

分析層：錯誤分析（Error Analysis）、切片分析（Slice Analysis）、公平性審計（Fairness Audit）。深入了解模型在哪些子群體或場景下表現不佳。

報告層：將評估結果轉化為可溝通的報告，包含信賴區間、統計顯著性檢定與視覺化圖表。

關鍵比較

離線評估 vs 線上評估

面向離線評估線上評估
速度快速，可自動化需時間累積流量
成本低，僅計算資源高，可能影響使用者體驗
真實性依賴歷史資料反映即時使用者行為
方法Hold-out、CVA/B 測試、MAB

單一指標 vs 多指標評估

單一指標便於比較與決策，但可能遺漏重要面向。多指標評估提供全面視角，但增加決策複雜度。實務上建議選定一個主要指標作為「北極星」，搭配 2-3 個護欄指標確保不犧牲其他面向。

點估計 vs 區間估計

僅報告指標的點估計（如 F1=0.85）資訊不足。應同時提供信賴區間（如 F1=0.85±0.03），透過 Bootstrap 或交叉驗證估計變異性，讓決策者了解結果的可靠程度。

應用場景

  1. 信用評分模型：重視 Recall（不遺漏高風險客戶），同時監控 Precision 避免過度拒絕
  2. 醫療診斷系統：使用 Sensitivity/Specificity 搭配 ROC 分析，設定業務可接受的閾值
  3. 推薦系統：結合離線指標（NDCG、Hit Rate）與線上指標（CTR、停留時間）
  4. 自然語言處理：依任務選擇 BLEU、ROUGE、BERTScore 等專用指標
  5. 異常偵測：在極度不平衡場景下使用 PR-AUC，避免 ROC-AUC 的樂觀偏差
  6. 模型可解釋性：使用 SHAP、LIME 等方法驗證模型決策邏輯是否合理
