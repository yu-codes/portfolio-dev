多分類與不平衡資料集

在現實世界的機器學習應用中，多分類問題與資料不平衡現象極為普遍。從醫療診斷到詐欺偵測，從影像辨識到自然語言處理，資料的類別分佈往往呈現高度偏斜。本文將系統性地探討多分類策略與不平衡資料的處理方法，提供實務可行的解決方案。

核心概念

多分類問題的本質

多分類問題將輸入樣本分配到三個或以上的類別中，主要策略包含：

  • One-vs-Rest (OvR)：為每個類別訓練一個二元分類器，將該類別視為正類，其餘為負類
  • One-vs-One (OvO)：為每對類別組合訓練一個分類器，透過投票機制決定最終類別
  • 原生多分類：直接使用支援多分類的演算法，如 Softmax 迴歸、決策樹、Random Forest

資料不平衡的定義與影響

資料不平衡指各類別的樣本數量存在顯著差異：

  • 輕度不平衡：多數類與少數類比例約 4:1 至 10:1
  • 中度不平衡：比例約 10:1 至 100:1
  • 嚴重不平衡：比例超過 100:1 甚至 1000:1

不平衡資料帶來的問題：

  • 模型傾向預測多數類，獲得虛假的高準確率
  • 少數類的決策邊界學習不充分
  • 評估指標可能產生誤導性結論

應用場景

場景一：醫療影像診斷（多分類 + 不平衡）

皮膚病變分類中，惡性病變的樣本遠少於良性：

  • 使用 Stratified K-Fold 確保每折都有各類別樣本
  • 採用 Focal Loss 讓模型聚焦於困難的少數類樣本
  • 使用 類別權重 反比於類別頻率：weight_i = N / (K * n_i)
  • 評估時使用 Macro-F1 與 Per-class Recall，確保少數類不被忽略

場景二：網路入侵偵測

正常流量遠多於攻擊流量，且攻擊類型多樣：

  • 使用 SMOTE-ENN 組合：先生成少數類樣本，再清理噪音
  • 採用 EasyEnsemble 集成方法，穩健處理極端不平衡
  • 使用 Micro-F1 評估整體偵測能力，Macro-Recall 確保各攻擊類型都能偵測
  • 設定 成本矩陣：漏報攻擊的成本遠高於誤報

場景三：客戶流失預測

流失客戶通常佔比 5-15%：

  • 使用 閾值調整：降低預測為流失的閾值，提高 Recall
  • 採用 成本敏感學習：流失客戶的誤分類成本設為非流失的 5-10 倍
  • 使用 AUC-PR 作為模型選擇的主要指標
  • 結合業務邏輯設定 Precision-Recall 的平衡點

場景四：多語言文本分類

不同語言的文本量差異懸殊：

  • 對低資源語言使用 資料增強：回譯、同義詞替換
  • 使用 Weighted Cross-Entropy 平衡各語言的損失貢獻
  • 採用 多任務學習 利用高資源語言的知識遷移到低資源語言
  • 使用 Weighted-F1 報告整體表現，同時監控各語言的個別 F1
