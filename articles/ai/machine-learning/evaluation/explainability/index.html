模型可解釋性

隨著機器學習模型在高風險領域（醫療、金融、司法）的廣泛應用，模型的可解釋性已從學術議題上升為合規要求。歐盟 GDPR 賦予公民「獲得解釋的權利」，這使得理解模型決策邏輯成為必要。本文將系統性地介紹主流的模型解釋方法，幫助讀者建立可解釋 AI 的實踐框架。

核心概念

可解釋性 vs 可詮釋性

  • 可詮釋性（Interpretability）：模型本身結構簡單，人類可以直接理解其決策邏輯（如線性迴歸、決策樹）
  • 可解釋性（Explainability）：透過事後分析方法，解釋複雜模型的決策原因（如對深度神經網路使用 SHAP）

解釋的範圍

  • 全域解釋（Global Explanation）：理解模型整體的行為模式與特徵重要性
  • 局部解釋（Local Explanation）：解釋模型對單一樣本的具體預測原因
  • 群組解釋（Cohort Explanation）：針對特定子群體分析模型行為

核心方法分類

  • 模型無關方法（Model-Agnostic）：適用於任何模型，如 SHAP、LIME、PDP
  • 模型特定方法（Model-Specific）：針對特定架構設計，如 Attention 權重、Grad-CAM
  • 內建可解釋模型（Inherently Interpretable）：模型本身即可解釋，如 EBM、規則學習

SHAP 方法架構

SHAP (SHapley Additive exPlanations)
├── 理論基礎
│   └── Shapley Value（合作博弈論）
├── 計算方法
│   ├── KernelSHAP（模型無關，近似計算）
│   ├── TreeSHAP（樹模型，精確且快速）
│   ├── DeepSHAP（深度學習，基於 DeepLIFT）
│   ├── LinearSHAP（線性模型，精確計算）
│   └── PartitionSHAP（文本/影像，層次結構）
└── 視覺化
    ├── Waterfall Plot（單一預測分解）
    ├── Force Plot（特徵貢獻方向）
    ├── Summary Plot（全域特徵重要性）
    ├── Dependence Plot（特徵與 SHAP 值關係）
    └── Interaction Plot（特徵互動效應）

應用場景

場景一：金融信貸審批

銀行需要向被拒貸的申請人解釋原因：

  • 使用 SHAP Waterfall Plot 展示每個特徵對決策的貢獻
  • 提供 Counterfactual Explanation：「如果您的年收入增加 10 萬，貸款將會被核准」
  • 使用 全域 SHAP Summary 確認模型沒有依賴敏感屬性（性別、種族）
  • 建立 模型審計報告，定期檢查特徵重要性的合理性

場景二：醫療診斷輔助

AI 輔助醫師判斷影像中的病灶：

  • 使用 Grad-CAM 高亮模型關注的影像區域，讓醫師確認注意力是否合理
  • 使用 Integrated Gradients 提供更精細的像素級重要性
  • 對結構化資料使用 SHAP 解釋風險因子的貢獻
  • 提供 不確定性估計：模型信心程度影響解釋的呈現方式

場景三：自然語言處理

情感分析或文本分類的解釋：

  • 使用 SHAP（PartitionSHAP） 解釋哪些詞語或片段影響分類
  • 使用 Attention 視覺化 展示模型的注意力分佈
  • 使用 LIME 透過遮蔽詞語觀察預測變化
  • 結合 Anchors 找出決定分類結果的充分條件

場景四：合規與公平性審計

確保模型符合法規要求：

  • 使用 全域 SHAP 分析受保護屬性的影響程度
  • 進行 群組公平性分析：比較不同群體的 SHAP 分佈差異
  • 建立 Model Card 文件化模型的用途、限制與偏差
  • 實施 持續監控：追蹤解釋結果是否隨時間漂移

延伸閱讀

推薦資源

  • 書籍：《Interpretable Machine Learning》— Christoph Molnar（免費線上版）
  • 論文：「A Unified Approach to Interpreting Model Predictions」— Lundberg & Lee (2017)
  • 套件：shap（Python）、lime（Python）、InterpretML（Microsoft）
  • 框架：Google What-If Tool、IBM AI Explainability 360

進階主題

  • 因果解釋（Causal Explanation）：超越相關性，提供因果層級的解釋
  • 概念級解釋（Concept-based Explanation）：使用人類可理解的概念而非原始特徵
  • 互動式解釋（Interactive Explanation）：允許用戶探索「如果…會怎樣」的場景
  • 解釋的評估：如何衡量解釋的忠實度、完整性與人類可理解性
