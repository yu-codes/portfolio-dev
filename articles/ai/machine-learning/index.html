機器學習（Machine Learning）是人工智慧的核心分支，讓電腦系統透過資料自動學習模式與規律，而非依賴明確的程式指令。從推薦系統、影像辨識到自然語言處理，機器學習技術已滲透到現代數位生活的每個角落。本文將系統性介紹機器學習的核心知識體系，涵蓋基礎理論、特徵工程、模型開發與評估等關鍵面向。

核心概念

學習範式分類

機器學習根據訓練資料的標註方式分為三大範式：

監督式學習（Supervised Learning）

模型從標註資料中學習輸入到輸出的映射關係。給定特徵 X 與標籤 Y，目標是學習函數 f: X → Y。

  • 分類（Classification）：預測離散類別（垃圾郵件偵測、影像分類）
  • 迴歸（Regression）：預測連續數值（房價預測、銷售預估）
  • 常見演算法：線性迴歸、決策樹、隨機森林、SVM、XGBoost

非監督式學習（Unsupervised Learning）

模型在無標註資料中發現隱藏的結構與模式。

  • 聚類（Clustering）：將相似資料分組（K-Means、DBSCAN）
  • 降維（Dimensionality Reduction）：壓縮特徵空間（PCA、t-SNE）
  • 異常偵測（Anomaly Detection）：識別異常資料點

強化學習（Reinforcement Learning）

智慧體透過與環境互動，依據獎勵信號學習最佳行為策略。

  • 核心元素：狀態、動作、獎勵、策略
  • 應用：遊戲 AI、機器人控制、推薦系統

偏差-變異權衡（Bias-Variance Tradeoff）

模型的泛化誤差由三部分組成：

$$ \text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error} $$

  • 高偏差（Underfitting）：模型過於簡單，無法捕捉資料模式
  • 高變異（Overfitting）：模型過於複雜，記住訓練資料的噪音
  • 最佳平衡：透過正規化、交叉驗證等技術找到甜蜜點

機器學習工作流程

完整的 ML 專案遵循系統化流程：

  1. 問題定義：明確業務目標與成功指標
  2. 資料收集：蒐集並整合相關資料來源
  3. 探索性分析（EDA）：理解資料分佈與特性
  4. 特徵工程：建構有意義的特徵表示
  5. 模型選擇與訓練：選擇適當演算法並訓練
  6. 評估與調參：交叉驗證與超參數最佳化
  7. 部署與監控：模型上線與效能追蹤

維度傳統機器學習深度學習

可解釋性較高較低

應用場景

場景一：推薦系統

電商與串流平台利用協同過濾與內容過濾結合的混合推薦，基於使用者行為與物品特徵預測偏好，提升使用者體驗與營收。

場景二：金融風控

信用卡詐欺偵測結合監督式分類與異常偵測，在極度不平衡的資料中識別可疑交易。時序特徵工程與即時推論是關鍵技術挑戰。

場景三：醫療診斷輔助

基於患者的臨床特徵、檢驗數據與醫學影像，輔助醫師進行疾病預測、風險分層與治療方案推薦。模型的可解釋性在醫療場景尤為重要。

場景四：自然語言處理

從文本分類、情感分析到機器翻譯，NLP 任務廣泛應用機器學習技術。隨著 Transformer 架構的突破，預訓練語言模型已成為 NLP 的主流範式。
