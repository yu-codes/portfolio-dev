資料清洗與轉換是資料工程與機器學習管線中最耗時卻最關鍵的環節。業界普遍認為資料科學家花費 60-80% 的時間在資料準備上。一套完善的清洗與轉換策略不僅能提升模型品質，更能大幅降低後續維護成本。

核心概念

資料清洗（Data Cleaning）

資料清洗的目標是識別並修正資料中的錯誤、不一致與缺失，使資料達到可用狀態。清洗不是一次性工作，而是持續性的過程。

資料轉換（Data Transformation）

資料轉換則是將清洗後的資料轉換為目標系統或分析需求所要求的格式與結構。這包括型別轉換、格式標準化、聚合計算與衍生欄位生成。

資料品質維度

維度定義衡量方式
完整性資料是否有缺失非空值比例
準確性資料是否反映真實世界與可信來源比對
一致性不同系統中的資料是否矛盾跨系統校驗規則
時效性資料是否足夠新鮮資料延遲指標
唯一性是否存在重複記錄重複率統計
有效性資料是否符合定義的規則驗證規則通過率

架構總覽

清洗轉換管線架構

典型的清洗轉換管線遵循分層設計：

Bronze 層（原始層）：保存來源系統的原始資料，不做任何修改。這是資料的「事實來源」，確保任何時候都能回溯。

Silver 層（清洗層）：執行資料清洗、去重、格式標準化。資料在這一層獲得結構化並通過品質驗證。

Gold 層（業務層）：根據業務需求進行聚合、衍生計算與最終轉換。這一層的資料直接服務於分析與模型。

關鍵比較

缺失值處理策略

策略適用場景優點缺點
刪除記錄缺失比例低（<5%）簡單直接損失資料量
均值/中位數填補數值型、缺失隨機不改變整體分佈降低變異性
眾數填補類別型資料保守穩定可能強化偏差
前值填補時序資料保持連續性不適用於跳躍式變化
模型預測填補缺失有規律準確度高計算成本高、可能引入偏差
標記為特殊值缺失本身有意義保留資訊增加特徵維度

異常值處理方法

方法原理適用資料分佈
Z-Score 法距離均值超過 N 個標準差常態分佈
IQR 法超出四分位距 1.5 倍任意分佈
百分位法超出指定百分位任意分佈
Isolation Forest隔離樹深度判斷高維資料
DBSCAN密度聚類邊界點空間分佈

轉換操作分類

結構性轉換：

  • 扁平化（Flattening）：將巢狀結構展開為平面表格
  • 樞紐（Pivot）：列轉行操作
  • 反樞紐（Unpivot）：行轉列操作
  • 合併（Join/Union）：多表關聯與合併

語意性轉換：

  • 標準化（Normalization）：將值映射到 [0,1] 區間
  • 正規化（Standardization）：轉換為標準常態分佈
  • 編碼（Encoding）：類別值轉數值表示
  • 分箱（Binning）：連續值離散化

應用場景

場景一：多來源客戶資料整合

企業常面臨客戶資料分散在 CRM、交易系統、客服系統的問題。清洗工作包括：姓名標準化（去除多餘空格、統一大小寫）、地址正規化（使用地址解析服務）、電話格式統一、以及跨系統的實體解析（Entity Resolution）。

場景二：NLP 文本前處理

自然語言處理中的資料清洗包括：HTML 標籤去除、特殊字元處理、Unicode 正規化、語言偵測與篩選、長度過濾、去重（使用 MinHash 或 SimHash）。對於 LLM 訓練資料，還需要移除有害內容與個人識別資訊。

場景三：時序感測器資料

IoT 感測器資料常見的問題包括：時間戳不對齊、感測器漂移、間歇性斷線造成的資料缺口。處理策略包括時間重新採樣（Resampling）、插值填補、以及基於物理規則的異常過濾。

場景四：特徵工程管線

將清洗後的資料轉換為機器學習可用的特徵。常見轉換包括：One-Hot Encoding、Target Encoding、時間特徵提取（年、月、週、小時）、滑動窗口統計量、以及文本向量化（TF-IDF、Word2Vec）。

最佳實踐

實踐說明
清洗規則版本化所有轉換邏輯應納入版本控制
資料剖析先行清洗前先進行資料探索性分析
保留原始資料永遠不修改原始資料層
漸進式清洗分階段處理，每階段有明確目標
自動化驗證轉換後自動執行品質檢查
文件化決策記錄每個清洗決策的理由
