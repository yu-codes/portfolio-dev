推論延遲優化是模型部署中的核心挑戰，直接影響使用者體驗與系統成本。從模型壓縮到硬體加速，從計算圖優化到服務架構調優，每一層都有顯著的優化空間。

核心概念

延遲組成分析

端到端推論延遲可分解為：

  • 網路延遲：請求從客戶端到伺服器的傳輸時間
  • 前處理延遲：資料序列化、特徵轉換、正規化
  • 排隊延遲：等待 GPU 資源或批次湊滿的時間
  • 計算延遲：模型前向傳播的實際計算時間
  • 後處理延遲：結果解碼、格式轉換
  • 回應延遲：結果回傳客戶端的時間

量化（Quantization）

降低模型數值精度以加速計算：

  • FP32 → FP16：半精度浮點，幾乎無精度損失
  • FP32 → INT8：8 位整數，需校準以維持精度
  • FP32 → INT4：4 位整數，適用於 LLM 推論
  • 混合精度：不同層使用不同精度

量化類型：

  • 訓練後量化（Post-Training Quantization, PTQ）：無需重訓練
  • 量化感知訓練（Quantization-Aware Training, QAT）：訓練時模擬量化

模型剪枝（Pruning）

移除模型中不重要的連接或結構：

  • 非結構化剪枝：移除個別權重（稀疏矩陣）
  • 結構化剪枝：移除整個通道或層（直接加速）
  • 動態剪枝：根據輸入動態跳過計算

知識蒸餾（Knowledge Distillation）

用大模型（教師）指導小模型（學生）的訓練：

  • 保留大模型的預測能力
  • 大幅降低計算量與模型大小
  • 特別適合部署到資源受限環境

計算圖優化

編譯器級別的優化手段：

  • 運算子融合：將連續的 Conv → BatchNorm → ReLU 合併為單一核心
  • 常數摺疊：預計算所有編譯時可確定的常數表達式
  • 記憶體規劃：優化張量的記憶體分配與複用
  • 排程優化：決定運算的最佳執行順序

快取策略

利用快取避免重複計算：

  • Embedding 快取：快取常見 ID 的 embedding 向量
  • KV-Cache：LLM 推論中快取注意力的 Key-Value
  • 結果快取：對相同或相似輸入直接回傳快取結果
  • 特徵快取：快取計算昂貴的特徵值

應用場景

大語言模型推論加速

LLM 推論面臨記憶體與計算的雙重瓶頸。透過 INT4 量化（GPTQ、AWQ）大幅降低記憶體需求，搭配 KV-Cache 優化（PagedAttention）與推測性解碼（Speculative Decoding）提升生成速度。

即時物體偵測

自動駕駛的物體偵測模型需要在 30ms 內完成推論。透過 TensorRT 的 FP16 + Layer Fusion 優化，配合模型架構設計（YOLO 系列）實現即時偵測。

搜尋排序模型

搜尋引擎的排序模型需要在 5ms 內為數百個候選文件評分。透過 INT8 量化、Embedding 快取與高效的批次推論實現嚴格的延遲要求。

語音助理即時回應

語音助理需要在 200ms 內完成語音辨識 + 語意理解 + 回應生成的完整管線。每個環節都需要極致的延遲優化，包括流式處理與管線平行化。
