角色與政策（Roles & Policies）是雲端身份與存取管理（IAM）的基礎構件。角色定義了「誰可以做什麼」，政策則精確描述了「在什麼條件下可以對哪些資源執行哪些操作」。正確的角色與政策設計是雲端安全的第一道防線，也是最關鍵的一環。

核心概念

IAM 基本模型

概念定義範例
主體（Principal）執行操作的實體使用者、服務帳號、角色
動作（Action）可執行的操作s3:GetObject、compute.instances.create
資源（Resource）操作的目標S3 Bucket、EC2 Instance
條件（Condition）額外的約束IP 限制、時間限制、MFA 要求
效果（Effect）允許或拒絕Allow / Deny

角色 vs 使用者

面向使用者（User）角色（Role）
身份類型長期身份臨時身份
憑證密碼 + 存取金鑰臨時安全憑證
有效期永久（直到撤銷）有限時間（通常 1-12 小時）
適用場景人類使用者服務、跨帳號、聯邦登入
安全風險憑證洩露風險較低（臨時性）
最佳實踐減少使用、啟用 MFA優先使用角色

RBAC vs ABAC

面向RBAC（基於角色）ABAC（基於屬性）
授權依據預定義的角色主體/資源/環境屬性
擴展性角色數量可能爆炸屬性組合靈活
管理複雜度低（預定義角色）高（屬性規則）
精確度角色級粒度屬性級粒度
適用場景組織結構明確動態、大規模環境
審計容易（查看角色）複雜（追蹤屬性）

政策設計最佳實踐

  1. 最小權限原則：僅授予完成任務所需的最小權限集
  2. 使用角色而非使用者：服務優先使用角色/服務帳號
  3. 定期審查：使用存取分析器識別未使用的權限
  4. 條件約束：添加 IP、時間、MFA 等條件限制
  5. 權限邊界：為委派管理設定權限上限
  6. 分離職責：關鍵操作需多角色配合

實踐說明

最小範圍精確到資源級別的權限
生命週期管理服務退役時同步清理帳號
