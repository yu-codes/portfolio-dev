最小權限原則（Principle of Least Privilege, PoLP）是資訊安全的基本原則之一：每個主體（使用者、服務、程式）僅應被授予完成其任務所必需的最小權限集合，且僅在需要的時間範圍內。在雲端環境中，權限的過度授予是最常見的安全風險來源之一。

核心概念

最小權限的維度

維度說明範例
操作最小化僅允許必要的 API 操作僅 s3:GetObject，不給 s3:*
資源最小化僅允許存取必要的資源特定 Bucket，不是所有 Bucket
時間最小化僅在需要時授予權限臨時提權，用完即收
條件約束添加額外的存取條件特定 IP、MFA、工作時間

權限過度的常見原因

原因說明後果
複製貼上複製既有角色而不審查繼承不必要的權限
快速修復為解決錯誤而添加寬泛權限遺留過度權限
萬用字元使用 * 省略精確定義權限範圍過大
缺乏審查從未回顧既有權限累積不需要的權限
預設角色使用雲端平台預設管理角色通常權限過寬

需要提權 → 申請 → 審批 → 臨時授予特權

  工作完成               自動撤銷（TTL）
    ↓                        ↓

服務間通訊的最小權限

# 不良實踐
Effect: Allow
Action: "s3:*"
Resource: "*"

# 最小權限實踐
Effect: Allow
Action:
  - "s3:GetObject"
  - "s3:PutObject"
Resource:
  - "arn:aws:s3:::my-bucket/uploads/*"
Condition:
  StringEquals:
    "aws:PrincipalOrgID": "o-xxxxx"
