Secret Manager 是雲端平台提供的託管式密鑰管理服務，專門用於安全地存儲、存取與管理敏感資訊（如 API 金鑰、資料庫密碼、憑證等）。相較於將密鑰硬編碼在程式碼或配置檔中，Secret Manager 提供了加密存儲、精細存取控制、自動輪換與完整審計能力。

核心概念

Secret Manager 的核心功能

功能說明價值
集中存儲所有密鑰存放在單一安全位置消除散落各處的密鑰
加密保護靜態加密（AES-256）+ 傳輸加密防止未授權存取
版本管理密鑰的歷史版本追蹤安全回滾、輪換支援
存取控制IAM 精細權限管理最小權限原則
審計日誌所有存取操作的記錄合規與安全調查
自動輪換定期自動更換密鑰降低洩露風險
跨區域複製密鑰在多區域可用高可用性

自動輪換設計

輪換排程器 → 觸發輪換 Lambda/Function
    → 產生新密鑰值
    → 存儲為新版本
    → 更新目標服務（DB 密碼等）
    → 驗證新密鑰可用
    → 標記新版本為 CURRENT
    → 通知相關服務更新快取

安全最佳實踐

  1. 永不記錄密鑰值：日誌中絕不輸出密鑰內容
  2. 細粒度存取：每個服務僅能存取其所需密鑰
  3. 強制輪換：所有密鑰設定最長有效期
  4. 監控異常存取：偵測非預期的密鑰讀取模式
  5. 災難恢復：確保密鑰的跨區域備份
  6. 版本清理：定期清理已不使用的舊版本

AWS 服務實作

AWS Secrets Manager

  • 管理密碼/金鑰/API Token 的儲存與自動輪替
  • 支援 RDS（MySQL/PostgreSQL/Aurora） 的原生密碼輪替
  • 其他服務可透過 Lambda 自訂輪替流程
  • 使用 KMS 加密秘密
  • 透過 CloudTrail 記錄存取實务

AWS Secrets Manager 主要功能

功能說明
秘密儲存安全儲存敏感資訊
自動輪替定期自動更換密碼
存取控制透過 IAM 政策控制存取
加密使用 KMS 加密秘密
審計透過 CloudTrail 記錄存取

支援的自動輪替

  • Amazon RDS（MySQL、PostgreSQL、Aurora）
  • Amazon Redshift
  • Amazon DocumentDB
  • 其他服務（透過 Lambda 自訂）

AWS Systems Manager Parameter Store

  • 集中管理密碼、環境變數（支援 KMS 加密）
  • 屬於 AWS Systems Manager（SSM）的一部分

使用 Secrets Manager

  • 資料庫密碼（特別是 RDS）
  • 需要自動輪替的敏感資訊
  • API 金鑰和存取令牌

使用 Parameter Store

  • 應用程式設定參數
  • 環境變數
  • 不需要自動輪替的設定値
  • 成本敏感的使用場景

AWS Systems Manager 其他功能

Run Command

  • 遠端對多台 EC2 批次下指令

Session Manager

  • 替代 Bastion Host，安全登入 Private Subnet 中的 EC2
  • 不需要開放 SSH 端口（Port 22）
  • 提供完整的操作審計記錄
