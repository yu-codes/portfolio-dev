網路安全

雲端網路安全防護

網路安全是雲端基礎架構的第一道防線。在傳統地端環境中，企業依賴硬體防火牆與實體網路隔離來保護資產；而在雲端世界，網路邊界變得模糊，攻擊面也隨之擴大。本文將深入探討雲端網路安全的核心概念、架構設計，以及如何透過多層防禦策略有效保護雲端工作負載。

核心概念

網路分層防禦（Defense in Depth）

雲端網路安全遵循「縱深防禦」原則，在多個層級設置安全控制：

  • 邊界層（Perimeter）：DDoS 防護、WAF（Web Application Firewall）、CDN 安全策略
  • 網路層（Network）：VPC/VNet 隔離、子網切分、路由表控制
  • 傳輸層（Transport）：TLS/mTLS 加密、VPN 通道、私有連線
  • 主機層（Host）：安全群組、網路 ACL、主機防火牆

軟體定義網路（SDN）

雲端網路安全高度依賴 SDN 技術，將網路控制平面與資料平面分離。這使得安全策略可以透過程式化方式管理，實現：

  1. 即時策略更新，無需重啟設備
  2. 微分段（Micro-segmentation），精確控制工作負載間的通訊
  3. 集中式日誌與監控，提升可視性

最小權限原則

網路安全設計應預設拒絕所有流量（Deny All），僅開放必要的通訊路徑。這意味著每條網路規則都必須經過明確授權，並定期審查是否仍然需要。

典型的雲端網路安全架構包含以下元件：

       │

│  公有子網 (DMZ)          │  ← 反向代理、Bastion Host

       │
┌──────▼──────────────────┐

│  來自 LB 的流量          │
└──────┬──────────────────┘
       │
┌──────▼──────────────────┐
│  私有子網 (Data)         │  ← 資料庫、快取

│  來自 App 層的流量       │
└─────────────────────────┘

此架構確保每一層僅能與相鄰層通訊，大幅降低橫向移動（Lateral Movement）的風險。

零信任 vs 邊界安全

傳統的邊界安全模型假設「內網可信」，一旦進入網路邊界就能自由通訊。零信任架構則徹底顛覆此假設：

  • 邊界安全：內網信任、一次性驗證、扁平網路
  • 零信任：永不信任始終驗證、持續身份確認、微分段隔離、加密所有通訊

應用場景

場景一：多租戶 SaaS 隔離

在多租戶環境中，每個租戶的資源必須完全隔離。透過 VPC Peering 或 Transit Gateway 搭配嚴格的路由策略，確保租戶間無法互相存取。同時使用私有端點連接共享服務（如 S3、RDS），避免流量經過公網。

場景二：混合雲安全連線

企業將部分工作負載遷移至雲端時，需要安全的混合連線方案。常見做法包括：

  • Site-to-Site VPN：透過 IPSec 通道加密地端與雲端之間的通訊
  • 專用線路（Direct Connect / ExpressRoute）：提供專屬的物理連線，不經過公網
  • SD-WAN 整合：智慧路由與加密，適合多站點部署

場景三：容器化微服務網路策略

在 Kubernetes 環境中，Network Policy 提供 Pod 層級的網路存取控制。結合 Service Mesh（如 Istio）的 mTLS 功能，可實現服務間的加密通訊與細粒度授權。

場景四：合規要求下的資料主權

某些法規（如 GDPR、資料在地化要求）限制資料傳輸的地理範圍。透過 VPC 端點策略、區域限制路由表，以及 DNS 防火牆，確保敏感資料不會離開指定區域。
