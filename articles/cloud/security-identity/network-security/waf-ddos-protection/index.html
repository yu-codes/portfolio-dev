WAF 與 DDoS 防護

Web 應用防火牆（WAF）與 DDoS 防護是保護雲端應用免受外部攻擊的關鍵安全服務。WAF 在應用層（L7）過濾惡意流量，防護 SQL Injection、XSS 等攻擊；DDoS 防護則在多層（L3/L4/L7）阻擋大規模的分散式阻斷服務攻擊。兩者結合形成完整的邊界防護體系。

WAF 工作原理

元素說明範例
規則（Rule）定義匹配條件與動作SQL Injection 偵測
規則群組（Rule Group）一組相關規則OWASP Top 10 規則集
Web ACL規則的容器，綁定到資源附加到 ALB/CloudFront

標籤（Label）標記匹配的請求後續規則參考

DDoS 攻擊類型

層級攻擊類型目標防護方式
L3/L4容量型攻擊網路頻寬清洗中心、黑洞路由
L3/L4協議攻擊連線表SYN Proxy、限流
L7應用層攻擊應用資源WAF、速率限制
L7慢速攻擊連線池超時設定、連線限制

    ↓

    ↓ 安全流量

    ↓

    ↓

    ↓

    ↓

WAF 調校流程

1. Count 模式部署 → 觀察匹配的請求
2. 分析誤報 → 識別合法請求被匹配
3. 建立例外規則 → 放行已知的合法模式
4. 切換 Block 模式 → 開始阻擋
5. 持續監控 → 調整規則、應對新攻擊
