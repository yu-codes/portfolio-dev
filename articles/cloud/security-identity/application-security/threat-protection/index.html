威脅防護

深入解析雲端應用的威脅防護策略，涵蓋偵測、回應與預防的完整安全體系

威脅防護（Threat Protection）是雲端安全的主動防禦層，涵蓋威脅偵測、自動回應與預防措施。現代雲端環境面臨的威脅日益複雜——從自動化的漏洞掃描到進階持續性威脅（APT），全面的威脅防護策略必須整合多層防禦與智慧化偵測能力。

核心概念

威脅防護三支柱

支柱目標實踐
預防（Prevent）阻止攻擊發生漏洞修補、安全配置、存取控制
偵測（Detect）發現攻擊行為異常偵測、日誌分析、威脅情報
回應（Respond）控制與修復自動隔離、事件回應、取證

雲端威脅分類

威脅類型說明典型攻擊防護措施
外部攻擊來自網際網路的攻擊DDoS、SQL InjectionWAF、DDoS 防護
內部威脅合法使用者的惡意行為資料外洩、權限濫用UEBA、DLP
供應鏈攻擊透過第三方滲透依賴投毒、CI/CD 攻擊SCA、SBOM
配置錯誤不安全的設定公開 S3、弱密碼CSPM、自動修復
帳號劫持憑證被盜釣魚、憑證填充MFA、異常偵測

偵測與回應流程

事件/日誌 → 收集 → 正規化 → 關聯分析 → 偵測規則/ML
    → 告警產生 → 優先級評估 → 自動/人工回應
    → 修復 → 事後檢討 → 改善偵測

應用場景

雲端威脅偵測場景

場景偵測信號自動回應
異常 API 呼叫非常規 API、非常規時間暫停 IAM 權限
加密貨幣挖礦高 CPU + 網路連線模式隔離實例
資料外洩大量 S3 下載、異常目標撤銷存取 + 告警
暴力攻擊大量失敗登入封鎖 IP + MFA 強制
橫向移動內部掃描、異常連線網路隔離
權限提升IAM 策略修改、新管理員回復變更 + 告警

事件回應自動化（SOAR）

    ↓
P1/P2：
├── 自動收集上下文（日誌、網路流、IAM 活動）
├── 自動隔離受影響資源
├── 通知安全團隊 + 建立事件頻道
└── 啟動事件回應流程

├── 自動收集上下文
├── 建立工單
└── 加入定期審查佇列
