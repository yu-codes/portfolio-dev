應用安全

雲端應用程式安全

應用安全（Application Security）是保護雲端應用程式免受攻擊、濫用與漏洞利用的綜合實踐。隨著應用程式從單體架構演進為分散式微服務，攻擊面大幅擴大 — 每個 API 端點、每個容器映像、每個第三方依賴都可能成為入侵向量。本文將深入介紹 WAF/DDoS 防護與執行期保護（Runtime Protection）兩大核心領域。

核心概念

Web 應用程式防火牆（WAF） 是部署在應用程式前方的安全控制層，檢查進入的 HTTP/HTTPS 流量並根據規則集阻擋惡意請求。WAF 可防禦 SQL Injection、Cross-Site Scripting（XSS）、Path Traversal 等 OWASP Top 10 攻擊。現代 WAF 結合簽章式規則與機器學習異常偵測，降低誤報率。

DDoS 防護 抵禦分散式阻斷服務攻擊，保護應用程式的可用性。防護分為三層：網路層（L3/L4）吸收大流量攻擊（如 SYN Flood、UDP Flood）；協定層（L4）處理連線耗盡攻擊；應用層（L7）偵測與阻擋精密的 HTTP Flood 與 Slowloris 攻擊。

執行期保護（Runtime Protection） 在應用程式執行時進行即時安全監控與防禦。包括 Runtime Application Self-Protection（RASP）— 嵌入應用程式內部，能夠觀察並攔截惡意行為；以及 Container Runtime Security — 監控容器內的系統呼叫、檔案存取與網路活動，偵測異常行為。

軟體供應鏈安全 確保應用程式依賴的所有組件（開源函式庫、容器基礎映像、建置工具）的安全性。Software Composition Analysis（SCA）掃描已知漏洞；SBOM（Software Bill of Materials）提供完整的依賴清單；簽章驗證確保映像未被篡改。

API 安全 保護 API 免受未授權存取、注入攻擊與資料洩漏。關鍵措施包括：OAuth 2.0/OIDC 身份驗證、速率限制、輸入驗證、回應過濾（避免回傳過多資料）與 API 閘道集中管控。

Shift-Left Security 將安全測試前移到開發階段，包括 SAST（靜態分析）在程式碼提交時掃描漏洞、DAST（動態分析）在部署後測試執行中的應用程式、IaC 掃描（如 Checkov）在基礎設施配置合併前偵測問題。

架構總覽

雲端應用安全的多層防禦架構：

邊緣防護層 — CDN（如 CloudFront、Cloudflare）提供第一道 DDoS 吸收層，利用全球分散的節點稀釋攻擊流量。Bot Management 識別並阻擋自動化攻擊工具。地理封鎖（Geo-blocking）阻止來自非業務區域的可疑流量。

WAF 層 — 部署在負載均衡器之後，檢查所有進入應用程式的請求。規則集包含：OWASP Core Rule Set（CRS）提供基礎防護；自訂規則針對業務邏輯攻擊；速率限制規則防止暴力破解；IP 信譽清單阻擋已知惡意來源。

API 閘道層 — 集中處理身份驗證、授權、速率限制與請求轉換。API Gateway（如 Kong、AWS API Gateway）強制執行 API Schema 驗證，阻擋不符合預期格式的請求。mTLS 確保服務間通訊的雙向身份驗證。

容器安全層 — 映像掃描（Trivy、Snyk Container）在 CI/CD 階段檢測基礎映像與依賴漏洞。Admission Controller（如 Kyverno、OPA Gatekeeper）阻止不合規的容器部署。Runtime Security（Falco、Sysdig）監控容器執行時行為。

應用內防護層 — RASP 嵌入應用程式 Runtime，能夠觀察實際的程式執行流程。相比 WAF 的黑盒規則，RASP 理解應用程式上下文，能更精準地判斷是否為攻擊（如區分正常的 SQL 查詢與注入攻擊）。

應用場景

多層 DDoS 防禦 — 電商平台在大促期間面臨混合型 DDoS 攻擊。邊緣 CDN 吸收 L3/L4 流量攻擊（峰值 500Gbps）；WAF 速率限制阻擋 L7 HTTP Flood；Bot Management 識別並挑戰可疑的自動化流量。合法使用者流量不受影響，攻擊在使用者無感知的情況下被化解。

零日漏洞快速回應 — 發現 Log4Shell（CVE-2021-44228）漏洞後，團隊立即在 WAF 新增規則阻擋含有 ${jndi: 的請求作為短期緩解。同時啟動 SCA 掃描識別所有受影響的服務，在 48 小時內完成修補。Runtime Protection 在此期間持續監控異常 JNDI Lookup 行為。

容器供應鏈安全 — 導入嚴格的映像管理流程：只允許來自受信任 Registry 的基礎映像；CI/CD 中自動執行漏洞掃描，Critical/High 漏洞阻止部署；所有映像使用 Cosign 簽章；Admission Controller 驗證簽章後才允許部署到生產環境。

API 安全加固 — 金融科技公司的 Open Banking API 實施多層防護：OAuth 2.0 + PKCE 身份驗證、每客戶速率限制、請求/回應 Schema 嚴格驗證、敏感欄位遮罩、完整的 API 存取稽核日誌。搭配 API 威脅偵測平台即時識別異常存取模式。
