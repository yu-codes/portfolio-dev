工作流編排（Workflow Orchestration）是管理多步驟分散式流程的技術，確保複雜業務流程的可靠執行、錯誤處理與狀態追蹤。在微服務架構中，跨服務的協調變得尤為重要。

核心概念

編排 vs 協調

面向編排（Orchestration）協調（Choreography）
控制方式中心化控制器各服務自行反應
流程可見性集中可視化分散、需追蹤
耦合度控制器與服務耦合服務間鬆耦合
錯誤處理集中處理各服務自行處理
適合場景複雜多步驟流程簡單事件反應
除錯難度低（集中日誌）高（分散追蹤）

工作流類型

類型說明範例持續時間
短期同步秒級完成API 聚合秒
長期非同步天到月審批流程天-月
定時批次排程執行ETL 管線分鐘-小時
人機互動等待人工操作審核流程不定
事件驅動事件觸發步驟訂單處理分鐘

工作流狀態

狀態說明處理方式
Running正在執行—
Waiting等待外部事件/計時器持久化等待
Completed成功完成記錄結果
Failed執行失敗觸發補償/重試
Cancelled被取消清理資源
Timed Out逾時觸發逾時處理

應用場景

電商訂單處理

  1. 驗證訂單 → 檢查庫存
  2. 支付處理 → 扣款（失敗→取消訂單）
  3. 庫存扣減 → 預留庫存（失敗→退款）
  4. 建立出貨 → 安排物流
  5. 通知客戶 → 發送確認

每步失敗都有補償動作（Saga Pattern）。

資料處理管線

  • Airflow DAG 排程每日執行
  • 資料擷取 → 清洗 → 轉換 → 載入
  • 任務依賴管理
  • 失敗重試與告警
  • SLA 監控

審批流程

  • 提交申請 → Wait 狀態
  • 發送通知給審批人
  • Callback 接收審批結果
  • 根據結果路由（批准/拒絕）
  • 支援多級審批與平行會簽
