控制平面（Control Plane）是雲端儲存架構中負責「管理」與「治理」的層級。如果說資料平面處理的是「資料存放在哪裡、如何存取」，控制平面則回答「資料如何被保護、何時該移動或刪除、誰有權限操作」等問題。一套完善的控制平面策略能確保資料的安全性、合規性與成本效率。

核心概念

資料治理（Data Governance）

資料治理是一套組織政策、流程與標準的集合，目的在確保：

  • 資料品質：資料正確、完整、一致且及時
  • 資料安全：適當的存取控制、加密與稽核
  • 資料合規：符合 GDPR、HIPAA、PCI-DSS 等法規要求
  • 資料可發現性：透過資料目錄（Data Catalog）讓使用者快速找到所需資料

備份策略的 3-2-1 原則

經典的備份策略遵循 3-2-1 規則：

  1. 保留至少 3 份資料副本
  2. 儲存在 2 種不同的儲存媒體上
  3. 其中 1 份存放在異地（Off-site）

在雲端環境中，這可演化為：主要區域的即時副本 + 跨區域複寫 + 歸檔層的長期保留。

不可變性（Immutability）

不可變備份確保資料在保留期間內無法被修改或刪除，即使是管理員也不行。這是防禦勒索軟體攻擊的關鍵策略：

  • Object Lock（WORM）：物件寫入後在指定期間內不可覆寫或刪除
  • Legal Hold：因法律訴訟需要無限期保留特定資料
  • Governance Mode vs Compliance Mode：前者允許特權使用者覆寫，後者連 root 都無法修改

應用場景

場景一：合規驅動的資料保留

金融業需保留交易紀錄 7 年、醫療業需保留病歷 30 年。透過生命週期策略，自動將過期的熱資料轉移至歸檔層（如 S3 Glacier Deep Archive），同時設定 Object Lock 確保在保留期間內不可刪除。

場景二：勒索軟體防禦

設定不可變備份策略，搭配跨帳號複寫（Cross-Account Replication），即使攻擊者取得主帳號的最高權限，也無法刪除或加密位於其他帳號的備份副本。同時啟用版本控制，確保可回溯至任何歷史版本。

場景三：多雲資料治理

組織使用多個雲端供應商時，透過統一的資料目錄（如 Apache Atlas、Collibra）串聯各平台的 metadata，提供一致的資料分類、標籤與血緣追蹤（Data Lineage），確保治理策略跨雲一致。

場景四：成本最佳化自動化

建立 CloudWatch/Azure Monitor 告警，當特定儲存桶的成本超過預算閾值時自動觸發生命週期加速轉換。搭配 S3 Storage Lens 或 Azure Storage Analytics 識別非活躍資料，推薦最佳分層策略。
