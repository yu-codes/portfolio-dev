元資料管理（Metadata Management）是對「描述資料的資料」進行系統化收集、存儲、維護與利用的實踐。在資料爆炸的時代，有效的元資料管理是實現資料可治理、可理解、可信賴的基礎。

應用場景

資料管線 Schema 管理

  • Kafka 訊息使用 Schema Registry
  • 相容性檢查防止破壞性變更
  • Schema 版本化追蹤演進歷史
  • 消費者自動適應新 Schema

資料品質監控

  • 元資料定義品質規則（非空、範圍、格式）
  • ETL 執行後自動檢查品質
  • 品質指標存入元資料系統
  • 異常時告警並追蹤血緣影響範圍

影響分析

  • 變更前評估影響範圍
