物件儲存（Object Storage）是雲端中最基礎且使用最廣泛的儲存服務，以扁平化的物件形式儲存非結構化資料。其無限擴展、高耐久性與低成本的特性，使其成為資料湖、備份、靜態網站等場景的首選。

AWS 服務實作

S3（Simple Storage Service）

核心概念

  • S3 是物件型儲存服務，每筆資料為一個 Object
  • Bucket 名稱需在全球唯一（類似網域）；3-63 字元，全部小寫，不可有底線，不可使用 IP 格式
  • 支援無限儲存數量、單一檔案最大 5TB；建議 100MB 以上使用 Multipart Upload；Single PUT 最大 5GB
  • 預設為私有存取，需明確開啟 Public Access
  • 每帳號 100 個 Bucket 軟限制，1000 個硬限制；單一 Bucket 可存無限物件

S3 Object 組成

元件說明
Value實際資料（圖像、影片、文件等）
Key完整路徑，如 /movies/comedy/abc.avi
Version ID物件版本（啟用版本控制時）
Metadata額外屬性資訊

存取控制

方式說明
ACL已淘汰（Legacy），不建議使用
S3 Bucket Policy推薦，JSON 格式 Resource Policy，可跨帳號或允許匿名主體
IAM Policy控制 IAM 使用者存取（Identity Policy）

使用情境建議：

場景建議方式
控制多種不同資源 / 同帳號存取Identity Policy
只控制 S3 / 匿名或跨帳號Bucket Policy
舊系統必須用 ACLACL（否則永遠不要用）

資料保護：

  • Versioning：一旦啟用，無法停用，只能暫停；未啟用時物件 ID 為 null；刪除不指定 ID 時新增 Delete Marker（可還原）；所有版本都佔用空間並計費
  • MFA Delete：變更版本控制狀態或刪除版本時，API 呼叫需傳入序號和驗證碼
  • Object Lock（WORM）：Retention Mode（保留期限）+ Legal Hold（法律保留）

Glacier Flexible 取用選項：

  • Expedited：1-5 分鐘

S3 Express One Zone（新）： 延遲低於 10 毫秒，使用 Directory Bucket（非一般 Bucket），適用於 ML 訓練、即時分析等高效能場景

S3 Intelligent-Tiering： 自動在不同層級間移動物件，有監控和自動化費用，適用於存取模式不明確的資料

加密機制（Server-side Encryption）

類型金鑰管理特點
SSE-S3AWS 管理AES-256，預設啟用
SSE-KMSKMS（可客戶管理）支援角色分離、金鑰旋轉；S3 管理員無法看未加密物件
SSE-C客戶自管上傳時提供金鑰，需 HTTPS
CSE（用戶端）客戶完全控制上傳前本地加密/解密

Bucket 不加密，物件才加密。Bucket 預設加密可透過 PUT header 指定類型（x-amz-server-side-encryption: "AES256" → SSE-S3；"aws:kms" → SSE-KMS）。PCI-DSS/HIPAA 建議 SSE-KMS 或 CSE。

上傳效能

方式限制特點
Single PUT最大 5GB單一串流，失敗即全失敗
Multipart Upload最小 100MB；每部分 5MB-5GB；最多 10,000 部分部分失敗可重傳，速度 = 所有部分總和
Transfer Acceleration需啟用透過 Edge Location，比公共網路更有效率，連線越差效果越明顯
Byte-Range FetchGET + Range Header只下載部分內容

Lifecycle Configuration（生命週期）

  • Transition Actions：自動轉換到其他儲存類別
  • Expiration Actions：在指定時間後刪除物件；可自動清除未完成 Multipart Upload

轉換順序（瀑布式，只能向下轉換）：

Standard → Standard-IA → Intelligent-Tiering → One Zone-IA → Glacier Instant → Glacier Flexible → Glacier Deep Archive

  • Standard → IA/One Zone-IA：至少儲存 30 天後才可轉換

複製（Replication）

類型說明
CRR（Cross-Region Replication）跨區域異步複製
SRR（Same-Region Replication）同區域複製
  • 來源與目的地都需開啟版本控制；不追溯：啟用前的物件不會自動複製
  • RTC（Replication Time Control）：確保 15 分鐘內同步
  • 支援 SSE-S3、SSE-KMS，不支援 SSE-C；單向複製（來源到目標，不反向）
  • 跨帳號複製需在目標 Bucket 加入 Bucket Policy 允許角色存取

進階功能

功能說明
靜態網站託管部署 HTML/CSS/JS；Route 53 設定自訂網域（Bucket 名稱需與網域相符）；Index 和 Error 文件
Pre-signed URL提供限時上傳/下載存取
S3 Select / Glacier Select對 CSV/JSON/Parquet 用 SQL 查詢部分內容，降低資料傳輸量
事件通知s3:ObjectCreated:* 可觸發 SNS/SQS/Lambda
VPC S3 Gateway Endpoint私網存取，節省資料傳輸費
CloudFront + OAI透過 CDN 快取公開檔案，限制非 CDN 來源存取
S3 Access Grants與企業目錄整合（Active Directory），按使用者/群組授權
S3 Object LambdaGET 請求時自動轉換資料（資料遮罩、格式轉換）
Server Access Logging紀錄物件存取紀錄（CloudTrail 不支援物件層級）

AWS Athena

  • Serverless SQL 查詢引擎，直接對 S3 物件查詢分析；無需管理基礎設施
  • 支援 CSV、JSON、Parquet、ORC 格式；結果輸出至 S3
  • 適用：一次性查詢、存取日誌分析、IoT 資料分析

S3 定價模型

  • 按 GB/月 收費；上傳（IN）免費；下載（OUT）收費
  • GET、PUT、POST 等操作按每 1000 次請求收費

S3 適用與不適用場景

適用場景不適用場景
大規模資料儲存、分發、上傳檔案系統用途
計算服務輸入/輸出需要掛載為磁碟（如 K:\ 或 /images）
靜態資源 Offload、資料湖、備份區塊儲存需求

S3 是物件儲存，無法掛載為磁碟，不適合頻繁小檔案隨機讀寫
