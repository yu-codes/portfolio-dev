檔案儲存（File Storage）提供基於檔案系統協定（如 NFS、SMB）的共享儲存服務，讓多個運算實例能同時存取相同的檔案資料。它是需要共享檔案存取場景的理想選擇。

核心概念

檔案儲存的定位

儲存類型存取模式共享能力典型用途
區塊儲存區塊級通常獨佔資料庫
檔案儲存檔案級多實例共享共享文件、CMS
物件儲存HTTP API無限共享靜態資料

AWS 服務實作

EFS（Elastic File System）

核心概念

  • POSIX 相容的受管共享檔案系統，支援共享與鎖定
  • 使用 NFS 協定，支援同時掛載至上千台 EC2（Linux 專用）
  • 自動擴展儲存空間，無需預配容量
  • 高可用性：Standard 類別跨多個 AZ 儲存資料

支援的服務：

  • EC2（Linux）
  • ECS
  • EKS
  • Lambda

EFS 效能模式

模式說明適用場景
General Purpose（預設）低延遲一般用途、Web 應用
Max I/O高併發吞吐但延遲略高大數據分析、媒體處理

吞吐量模式：

模式說明適用
Elastic（預設）隨工作負載自動擴展大多數場景
Provisioned自訂吞吐量高效能需求
Bursting基線 + 突發（舊模式）遺留使用

EFS 儲存類別

類別說明費用（參考）
Standard跨多個 AZ 儲存~$0.30/GB/月
One Zone單一 AZ 儲存，成本較低較低
Infrequent Access（IA）不常存取的資料，降低 92% 費用~$0.016/GB/月
One Zone-IA單一 AZ 不常存取最低
Archive極少存取最低

EFS Replication（複製）

  • 自動將資料複製到另一個區域
  • 支援災難復原和跨區域存取
  • RPO 約 15 分鐘

EFS 適用場景

  • 跨 AZ 共用檔案
  • 容器共享儲存（ECS/EKS + CSI Driver）；多 Pod 同時讀寫相同檔案
  • 軟體更新、CMS 共享媒體（如 WordPress）
  • 多用戶家目錄存取
  • 影像等大型共享資料

FSx 系列

FSx for Windows File Server

  • SMB 協定與 NTFS 檔案系統
  • 支援 Microsoft Active Directory 整合、ACL 與使用者配額
  • 適用：Windows 企業環境、DFS 命名空間、AD 整合
  • 可透過 DataSync 從地端 Windows File Server 遷移

FSx for Lustre

  • 高效能並行 Linux 檔案系統（POSIX 相容）
  • 延遲 < 1ms，具高吞吐、可達百萬 IOPS
  • 適用：ML 訓練、HPC、影片處理、EDA 等場景
  • 可與 S3 整合：Lustre 當熱儲存，S3 當冷儲存；訓練完成自動回寫 S3；多 GPU 實例平行讀取

FSx for Lustre 部署模式：

模式說明
Scratch短期使用，不持久，適合臨時分析
Persistent長期使用，持久儲存，適合長期工作負載

FSx for NetApp ONTAP

  • 企業 NAS 功能，支援 NFS/SMB/iSCSI
  • 適用：SAP 系統、企業應用遷移

FSx for OpenZFS

  • ZFS 檔案系統，POSIX 相容
  • 適用：Linux HPC 工作負載
