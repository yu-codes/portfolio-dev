運算基礎

雲端運算服務模型

運算（Compute）是雲端服務中最基礎也最核心的類別。所有的應用程式最終都需要運算資源來執行業務邏輯。雲端運算服務的演進從最初的虛擬機器，到容器化部署，再到無伺服器運算，反映了業界不斷追求更高抽象層次、更低管理負擔與更精細資源利用的趨勢。本文將深入解析這三種主要的運算模型，幫助你在不同場景中做出正確的技術選型。

核心概念

運算抽象層次的演進：

雲端運算的歷史可以理解為一個持續提升抽象層次的過程：

  1. 實體伺服器 → 完全控制，但購買與維護成本高。
  2. 虛擬機器 → 硬體層虛擬化，多租戶共享實體資源。
  3. 容器 → OS 層虛擬化，更輕量、更快啟動。
  4. 無伺服器 → 完全隱藏基礎設施，僅關注程式碼邏輯。

每一層的抽象都帶來更少的管理責任與更細粒度的資源計費，但同時也意味著更少的控制力與更多的供應商依賴。

運算資源的關鍵指標：

  • vCPU：虛擬 CPU 核心數，決定並行處理能力。
  • 記憶體：RAM 容量，影響能同時處理的資料量。
  • 網路頻寬：決定資料傳輸速度，常被忽略但影響顯著。
  • 儲存 IOPS：磁碟讀寫效能，對資料密集型應用至關重要。
  • GPU/加速器：機器學習訓練與推論、影像處理等特殊工作負載所需。

實例類型與家族：各供應商提供不同用途的實例家族——通用型（平衡 CPU 與記憶體）、運算優化型（高 CPU）、記憶體優化型（高 RAM）、儲存優化型（高 IOPS）與加速運算型（GPU/FPGA）。

架構總覽

虛擬機器實例（VM Instances）

虛擬機器是最傳統的雲端運算形態，提供完整的作業系統環境：

  • 使用者選擇 OS 映像檔（AMI / VM Image / OS Image）。
  • 選擇實例類型決定硬體規格。
  • 配置網路（VPC、Security Group）與儲存（區塊裝置）。
  • 擁有完整的 root/admin 權限。

適合場景：需要特定 OS 配置的傳統應用、需要持久連線的服務、需要安裝特殊軟體的工作負載。

容器服務（Container Services）

容器提供比 VM 更輕量的隔離，共享宿主機 OS 核心：

  • 以 Docker 映像檔封裝應用與所有依賴。
  • 透過容器編排平台（Kubernetes、ECS）管理生命週期。
  • 支援快速啟動（秒級）與高密度部署。
  • 聲明式管理（Deployment、Service、ConfigMap）。

適合場景：微服務架構、CI/CD 流水線、需要快速擴展的 Web 服務、開發與生產環境一致性要求高的系統。

無伺服器運算（Serverless Compute）

完全移除基礎設施管理，以函式或請求為計費單位：

  • 上傳程式碼，設定觸發條件（HTTP 請求、事件、排程）。
  • 平台自動處理佈建、擴展與回收。
  • 按實際調用次數與執行時間計費。
  • 內建自動擴展至零（Scale to Zero）。

適合場景：事件處理、API 後端、排程任務、資料轉換、Webhook 處理。

應用場景

  • 傳統企業應用上雲：將既有的 Windows Server/.NET 或 Linux/Java 應用直接遷移至 VM，作為雲端旅程的第一步（Lift and Shift）。
  • 微服務現代化：將單體應用拆分為容器化微服務，部署在 Kubernetes 上，獲得獨立部署與擴展的能力。
  • 事件驅動處理：使用 Lambda/Functions 處理 S3 上傳事件、SQS 訊息或 API Gateway 請求，完全免除伺服器管理。
  • 批次處理工作：大量資料處理可使用 Spot Instance（AWS）或 Preemptible VM（GCP）大幅降低成本。
  • 混合運算策略：核心 API 以容器提供穩定服務；周邊整合（通知、排程、Webhook）以 Serverless 處理；機器學習訓練以 GPU VM 執行。
  • 開發者體驗優化：本地開發用 Docker Compose 模擬微服務環境；CI/CD 使用容器確保環境一致；預覽環境使用 Serverless 節省成本。
