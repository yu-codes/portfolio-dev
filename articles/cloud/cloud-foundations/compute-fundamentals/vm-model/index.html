虛擬機（Virtual Machine，VM）是雲端運算中最基礎的服務型態，透過虛擬化技術將實體硬體資源抽象為多個獨立的運算環境。理解虛擬機的運作模型，是掌握雲端基礎設施的第一步。

世代技術特點代表
第一代全虛擬化完整模擬硬體VMware ESXi
第二代半虛擬化Guest OS 需修改Xen
第三代硬體輔助虛擬化CPU VT-x/AMD-V 支援KVM
第四代輕量級虛擬化microVM、最小攻擊面Firecracker、Cloud Hypervisor

應用場景

遷移上雲（Lift and Shift）

最直接的雲端遷移策略：

  1. 評估現有 VM 規格與相依性
  2. 選擇對應的雲端實例類型
  3. 使用遷移工具複製映像（AWS SMS、Azure Migrate）
  4. 驗證並切換流量

高效能運算（HPC）

  • 選用運算最佳化實例（C 系列）
  • 搭配 Placement Group 降低網路延遲
  • 使用 Spot 實例降低成本
  • 平行化分散任務到多個 VM

開發測試環境

  • 使用可突增實例（T/B 系列）
  • 排程啟停節省成本
  • 快照作為環境基準線
  • 搭配 Infrastructure as Code 快速重建

AWS 服務實作

EC2（Elastic Compute Cloud）概述

  • IaaS 模型的虛擬機器服務，可在雲端部署任意應用
  • 使用 Nitro 架構可達到最高 64,000 EBS IOPS，非 Nitro 僅 32,000
  • EC2 重新啟動時公網 IP 可能改變，若需固定 IP，使用 Elastic IP（每區預設上限 5 個）
  • 可透過 http://169.254.169.254/latest/meta-data 查詢 EC2 metadata
  • 建議同 AZ 部署多台 EC2 可節省區域間資料傳輸費用
  • 可用 VM Import/Export 匯入本地虛擬機映像檔轉為 EC2 AMI 使用

EC2 Hibernate

  • 可將 RAM 內容存至 EBS
  • 開機時載入，達成快速啟動
  • 適用於長時間運行的應用程式

EC2 實例類型分類

分類用途例子
T, M通用型Web 伺服器、小型資料庫、開發環境
C運算最佳化HPC、高效能批次處理、CPU ML
R, X, Z記憶體最佳化高效能資料庫、即時大數據分析
F, G, P加速運算GPU 密集應用、語音辨識、ML
D, H, I儲存最佳化高 I/O 作業如 Kafka、HDFS、ElasticSearch

EC2 啟動類型

類型說明適用場景
On-Demand按量計費，靈活但成本高短期、彈性需求
Reserved預付租用 1~3 年穩定、可預測工作負載
Scheduled Reserved指定週期性時段預約使用固定時段需求
Spot Instance最高折扣 90%，可容忍中斷批次處理、容錯應用
Dedicated Instance實體隔離，同租戶獨享硬體合規需求
Dedicated Host實體主機層級控管授權限制軟體

Reserved Instance 類型：

  • Standard：可轉售
  • Convertible：可轉換型態

Spot Instance 進階：

  • Spot Block：保留固定時段不中斷
  • Spot Fleet：組合 Spot + On-demand 依目標數量啟動

實例配額：

  • Reserved：20 台
  • On-demand：1152 vCPU
  • Spot：1440 vCPU
  • 可提出申請調升

EC2 網路強化功能

類型說明最高速度
ENI (Elastic Network Interface)虛擬網卡，可附掛至 EC2-
ENA (Elastic Network Adapter)支援 C4、D2、M4 等類型100 Gbps
EFA (Elastic Fabric Adapter)OS-bypass，HPC/ML 低延遲-
Intel 82599 VF早期虛擬網卡10 Gbps

ENI 特性：

  • 主要 ENI 不可移除
  • 次要 ENI 可 detach 並轉移至備援 EC2（Failover）

EC2 佈局策略（Placement Groups）

只能跨 AZ，不可跨 Region。

類型特性適用場景
Cluster同 AZ、同機架HPC、低延遲應用
Spread跨 AZ、跨機架，單 AZ 限 7 台高可用性需求
Partition同或跨 AZ，每 AZ 最多 7 分區Kafka、Cassandra

AMI（Amazon Machine Image）

  • 包含 OS、應用與設定的映像檔
  • 可用於快速建立新 EC2 實例
  • 可跨區複製，達成多區部署一致性
類型說明
Public AMIAWS 或社群提供
Private AMI自建私有 AMI
Marketplace AMI第三方廠商提供

AWS Nitro System

  • 所有現代 EC2 實例的底層虛擬化平台
  • 提供更高效能、更好的安全性和更低的延遲
  • 支援最高 64,000 EBS IOPS 和 100 Gbps 網路

Elastic Fabric Adapter (EFA)

  • 用於 HPC 和 ML 工作負載的低延遲網路介面
  • 支援 OS-bypass 技術，降低延遲
  • 支援 MPI（Message Passing Interface）

EC2 Capacity Blocks for ML

  • 預留 GPU 運算容量用於 ML 工作負載
  • 可預約特定時間段的 GPU 實例

Savings Plans（節省計畫）

類型折扣彈性
Compute Savings Plans最高 66%跨 EC2、Fargate、Lambda
EC2 Instance Savings Plans最高 72%特定實例系列和區域

Capacity Reservations（容量預約）

  • 確保在特定 AZ 中有可用的 EC2 容量
  • 無需長期承諾
  • 可與 Reserved Instances 或 Savings Plans 結合

虛擬化技術（AWS 視角）

類型說明效能
軟體模擬虛擬化軟體模擬硬體，OS 認為在真實硬體上運行慢
半虛擬化修改 OS 原始碼直接呼叫 Hypervisor較快
硬體輔助虛擬化硬體本身支援虛擬化快
SR-IOV網卡呈現多個邏輯裝置，減少主機 CPU 使用（即 Enhanced Networking）最快

AWS EC2 最佳實踐

  1. 選擇適當實例類型：根據工作負載特性選擇
  2. 使用 Reserved Instance：穩定負載可節省成本
  3. 利用 Spot Instance：容錯應用可大幅降低成本
  4. 配置 Placement Group：根據需求選擇適當策略
  5. 建立 AMI：標準化部署流程
  6. 使用 Graviton 實例：ARM 架構可提供更好的性價比
  7. 啟用詳細監控：使用 CloudWatch 監控效能
  8. 使用 Instance Metadata Service v2 (IMDSv2)：提高安全性
