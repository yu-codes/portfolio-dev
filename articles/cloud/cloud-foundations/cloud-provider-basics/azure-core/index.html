Microsoft Azure 是全球第二大公有雲平台，以其與企業 IT 生態系的深度整合著稱。Azure 在混合雲、企業身份管理與 Microsoft 產品整合方面具有獨特優勢，是許多大型企業數位轉型的首選平台。

核心概念

資源管理架構

Azure 採用 Azure Resource Manager（ARM） 作為統一的管理層：

層級說明用途
Management Group管理群組跨訂閱治理
Subscription訂閱計費邊界、存取控制
Resource Group資源群組生命週期管理單位
Resource個別資源VM、Storage Account

地理架構

概念說明數量
Geography地理區域（資料駐留）全球多個
Region區域（獨立基礎設施）60+ 區域
Availability Zone可用區域每區域 3 個
Region Pair配對區域（災難復原）自動配對

Azure Active Directory（Entra ID）

Azure AD（現稱 Microsoft Entra ID）是 Azure 的身份基礎：

  • Tenant：組織的專屬目錄實例
  • Managed Identity：Azure 資源的自動身份管理
  • RBAC：基於角色的存取控制
  • Conditional Access：條件式存取策略
  • PIM：特權身份管理（Just-in-Time）

應用場景

企業混合雲架構

  3. 統一管理：Azure Arc 跨環境管理
