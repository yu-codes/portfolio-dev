雲端基礎

雲端供應商、運算與架構原則

雲端基礎是所有雲端工程實踐的起點。在深入任何特定的架構模式或進階服務之前，工程師需要對雲端運算的基本概念、主流供應商的服務版圖以及核心架構原則有清晰的認識。本文將建立這些關鍵的基礎知識框架，幫助你在後續的學習旅程中擁有穩固的根基。

核心概念

雲端運算的定義：根據 NIST 的定義，雲端運算是一種模型，提供對共享可配置計算資源池（如網路、伺服器、儲存、應用程式與服務）的隨需、便利的網路存取，這些資源能以最少的管理工作或供應商互動快速佈建與釋放。

五大基本特徵：

  1. 隨需自助服務（On-demand Self-service）：使用者無需人工介入即可自行佈建資源。
  2. 廣泛的網路存取（Broad Network Access）：透過標準機制由各種平台存取。
  3. 資源池化（Resource Pooling）：多租戶模型動態分配資源。
  4. 快速彈性（Rapid Elasticity）：能力可彈性佈建與釋放，看似無限。
  5. 可量測服務（Measured Service）：自動控制並優化資源使用。

服務交付模型：

  • IaaS（基礎設施即服務）：提供虛擬機器、網路與儲存。使用者管理 OS 以上的一切。代表服務：EC2、Azure VMs、Compute Engine。
  • PaaS（平台即服務）：提供執行環境，使用者只管理應用程式與資料。代表服務：App Engine、Azure App Service、Elastic Beanstalk。
  • SaaS（軟體即服務）：完整的應用程式，使用者只需使用。代表服務：Gmail、Salesforce、Office 365。

部署模型：公有雲（Public）、私有雲（Private）、社群雲（Community）與混合雲（Hybrid），各有不同的安全性、控制度與成本取捨。

架構總覽

雲端基礎知識可組織為三大支柱：

1. 雲端供應商基礎

了解三大主流公有雲供應商的服務組織方式與核心差異：

  • AWS 以服務數量最多見長，擁有超過 200 項服務，適合需要全面解決方案的場景。
  • Azure 與 Microsoft 生態系深度整合，Active Directory、Office 365 與 .NET 堆疊無縫銜接。
  • GCP 在大數據（BigQuery）、機器學習（Vertex AI）與容器（GKE）領域有獨特優勢。

每個供應商都有自己的計費邏輯、區域佈局與認證體系，需要逐一理解。

2. 雲端架構原則

跨供應商通用的架構設計思維：

  • Well-Architected Framework 提供了系統化的架構評估方法。
  • 共同責任模型明確劃分了供應商與客戶的安全邊界。
  • 區域與可用區的概念是實現高可用部署的基礎。

3. 運算基礎

雲端最核心的服務類別——運算資源的不同形態：

  • 虛擬機器實例：最傳統的 IaaS 運算，完全控制 OS 環境。
  • 無伺服器運算：免除伺服器管理，專注業務邏輯。
  • 容器服務：輕量化封裝，支援高密度部署與快速啟動。

應用場景

  • 初學者入門：從單一供應商的免費方案開始，建立一台虛擬機器部署簡單 Web 應用。理解基本的網路（VPC、Security Group）與儲存（Block/Object）概念。
  • 遷移評估：現有地端系統上雲前，利用架構原則評估適合的服務模型（Rehost、Replatform 或 Refactor）。
  • 多雲策略起點：先深入理解各供應商的差異與優勢，再決定多雲的邊界——是為了避免供應商鎖定，還是利用各家最強項服務。
  • 認證考試準備：AWS Solutions Architect、Azure Administrator 或 GCP Cloud Engineer 認證都以這些基礎知識為核心考點。
  • 團隊技術選型：根據團隊既有技術堆疊、預算與業務需求，選擇最適合的供應商與運算模型組合。
