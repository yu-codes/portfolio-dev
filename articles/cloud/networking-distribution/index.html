網路與流量分配

雲端網路、CDN 與流量管理

雲端網路與流量分配是支撐所有雲端服務的基礎層。從虛擬私有網路（VPC）的設計到全球內容分發網路（CDN）的部署，網路架構的品質直接決定了應用的效能、安全性與可用性。本文將系統性地介紹雲端網路的核心元件與流量管理策略。

核心概念

雲端網路與流量分配涵蓋以下核心概念：

  • 虛擬私有雲（VPC/VNet）：在公有雲中劃分的邏輯隔離網路空間。支援自訂 IP 範圍、子網路劃分、路由表與安全群組。
  • 負載均衡（Load Balancing）：將流量分散至多個後端實例，提升可用性與效能。分為 L4（傳輸層）與 L7（應用層）兩種類型。
  • 內容分發網路（CDN）：將靜態內容快取至全球邊緣節點，降低終端使用者的存取延遲。
  • DNS 管理：網域名稱解析服務，支援健康檢查、地理路由與加權路由等進階功能。
  • 流量管理（Traffic Management）：透過 API Gateway、Ingress Controller 與服務網格等元件，實現精細的流量控制、路由策略與版本切換。
  • 網路安全：安全群組、網路 ACL、WAF 與 DDoS 防護構成的多層防禦體系。

架構總覽

雲端網路架構的典型設計遵循多層隔離原則：

邊緣層（Edge Layer）：CDN、WAF 與 DDoS 防護。流量首先到達邊緣節點進行快取命中判斷與攻擊過濾。

入口層（Ingress Layer）：全球/區域負載均衡器接收通過邊緣的流量，根據路由規則分發至不同的後端服務群。

應用層（Application Layer）：VPC 內部的私有子網路部署應用伺服器。透過 Internal Load Balancer 進行服務間通訊。

資料層（Data Layer）：資料庫與快取部署於最內層的私有子網路，僅允許應用層存取，不暴露公網。

互聯層（Connectivity Layer）：VPC Peering、Transit Gateway 或 VPN 連接多個 VPC 與地端網路。

各雲端的網路服務功能大致對齊，但定價模型與效能特性有差異。跨區域流量費用是架構設計時常被忽略的成本因素。

應用場景

全球化應用部署：電商平台在北美、歐洲與亞太部署應用副本，透過全球負載均衡器與 DNS 地理路由將使用者導向最近的區域，搭配 CDN 加速靜態資源。

零信任網路架構：放棄傳統的網路邊界防禦，每個服務間的通訊都經過身份驗證與加密（mTLS）。利用服務網格與 Micro-segmentation 實現最小權限存取。

混合雲網路互聯：透過 Direct Connect + VPN 建立地端與雲端的冗餘連接。Transit Gateway 集中管理數十個 VPC 的路由。

藍綠部署流量切換：利用加權路由 DNS 或 ALB 規則，將流量從舊版本漸進切換至新版本。支持 1%→10%→50%→100% 的金絲雀發布模式。

多租戶網路隔離：SaaS 平台為每個企業租戶建立獨立的 VPC 或子網路，搭配網路 ACL 與安全群組實現完全的網路隔離。
