API Gateway（API 閘道）是位於客戶端與後端服務之間的中介層，負責處理 API 呼叫、路由請求、驗證身份、限制流量與監控。在雲端架構中，API Gateway 是建構無伺服器架構與微服務的關鍵元件。

核心概念

API Gateway 的角色

功能說明
API 路由將請求導向對應的後端服務
身份驗證JWT、OAuth、API Key 驗證
限流保護Rate Limiting、Quota 管理
協定轉換REST → Lambda、HTTP → 後端服務
快取減少後端負載，提升效能
監控記錄請求、延遲、錯誤統計

API 架構類型

類型說明適用場景
REST API請求-回應模式，基於 HTTPWeb 應用、行動 App
HTTP API輕量 REST，低延遲簡單代理、Lambda
WebSocket API雙向即時通訊聊天、即時更新、遊戲
GraphQL API彈性查詢語言複雜資料需求
gRPC高效能 RPC微服務內部通訊

常見部署模式

Client → API Gateway → Lambda → DynamoDB
                     → HTTP 後端
                     → AWS Service（直接整合）
                     → VPC 內部服務（VPC Link）

AWS 服務實作

Amazon API Gateway 概述

  • Serverless API 管理服務，完全託管，自動擴展
  • 支援 HTTP、WebSocket、REST 類型
  • 提供限制頻率（Throttle Limit）與快取機制

API 類型比較

類型說明適用場景
REST API功能完整的 RESTful API需要進階功能（快取、WAF 等）
HTTP API輕量、低延遲，成本約便宜 70%簡單 HTTP 代理、Lambda
WebSocket API雙向即時通訊聊天、遊戲、即時更新

REST API vs HTTP API

項目REST APIHTTP API
成本較高較低（約 70% 便宜）
延遲較高較低
快取支援不支援
WAF 整合支援不支援
驗證IAM、Lambda Authorizer、CognitoIAM、Lambda Authorizer、JWT

驗證方式

方式說明
IAM Policy使用 IAM 角色和政策（AWS 服務間通訊）
Lambda Authorizer自訂驗證邏輯，整合現有身份系統
Cognito User Pool使用 Cognito 進行身份驗證，回傳 JWT Token

Lambda Authorizer

  • 自訂驗證邏輯，可整合現有身份系統
  • 支援 Token 類型（JWT、OAuth）和 Request 類型
  • 驗證結果可快取（減少 Lambda 呼叫）

Cognito User Pool

  • 託管用戶身份，支援社群登入（Facebook、Google、Apple）
  • 支援 MFA（SMS 或 TOTP）、SAML/OIDC 企業身份聯合
  • 回傳 JWT Token 供 API Gateway 驗證

整合類型

類型說明
Lambda呼叫 Lambda 函數（最常用）
HTTP代理到 HTTP 端點
AWS Service直接呼叫 AWS 服務（如 SQS、SNS）
VPC Link存取 VPC 內部的 NLB/ALB（私有整合）
Mock模擬回應（測試用，不呼叫後端）

部署與階段（Stages）

  • 代表 API 的不同版本（dev、staging、prod）
  • 每個階段有獨立的 URL：https://{api-id}.execute-api.{region}.amazonaws.com/{stage}
  • 階段變數（Stage Variables）：類似環境變數，可在不同階段指向不同後端

Canary 部署

  • 漸進式部署，將部分流量導向新版本
  • 可配置 Canary 比例（如 10% 流向新版）

限流與配額

帳號層級限制

限制預設值
請求限制10,000 請求/秒
並發限制5,000

使用計畫（Usage Plans）

  • 為 API 金鑰設定配額（每日/每月請求數）
  • 限流（Throttling）設定，保護後端服務

快取

  • 可快取端點回應，減少後端呼叫
  • TTL 範圍：0-3600 秒（預設 300 秒）
  • 可針對特定方法或整個階段啟用
  • 僅 REST API 支援快取

安全性

WAF 整合（僅 REST API）

  • 保護 API 免受 SQL Injection、XSS 等攻擊
  • 速率限制、IP 白名單/黑名單

CORS（跨來源資源共享）

  • 需在 API Gateway 設定，允許跨域存取

資源政策

  • 控制誰可以存取 API
  • 支援 IP 白名單/黑名單
  • 支援 VPC 端點存取

Private API

  • 僅能從 VPC 內存取，使用 VPC Interface Endpoint
  • 透過資源政策控制存取

Amazon Cognito 完整整合

Cognito User Pools

  • 提供使用者目錄進行註冊和登入，管理使用者設定檔
  • 發行 JWT Token
驗證方式說明
帳號密碼傳統驗證方式
社群登入Facebook、Google、Amazon、Apple
MFASMS 或 TOTP
SAML/OIDC企業身份聯合

Cognito Identity Pools（Federated Identities）

  • 授予使用者臨時 AWS 認證（STS AssumeRoleWithWebIdentity）
  • 可直接存取 AWS 服務（如 S3、DynamoDB）
  • 支援未經驗證的訪客存取

使用者遷移

  • 支援從現有使用者目錄遷移
  • Lambda 觸發器處理遷移邏輯

常見架構模式

Client → API Gateway → Lambda → DynamoDB
                     → SQS（非同步處理）
                     → Step Functions（工作流程）
                     → VPC Link → NLB → 後端服務（私有整合）

AWS API Gateway 最佳實踐

  1. 使用適當的 API 類型：簡單場景用 HTTP API（較便宜約 70%）
  2. 啟用快取（REST API）：減少後端負載，TTL 設 0-3600 秒
  3. 實施限流：帳號預設 10,000 RPS，為各 API 設定 Usage Plan
  4. 使用階段：分離開發（dev）和生產（prod）環境
  5. REST API 整合 WAF：保護 API 安全
  6. 監控延遲：追蹤 Latency 和 IntegrationLatency 差異
  7. 使用 Lambda Authorizer：自訂驗證需求
  8. 使用 Cognito：簡化使用者管理，支援社群登入和 MFA
  9. Private API + VPC Link：內部服務使用，確保安全

相關服務：Amazon CloudFront

CloudFront 常與 API Gateway 搭配使用，提供全球加速與快取。

概念說明

Regional Edge Cache區域邊緣快取（較大）
Origin內容來源（S3、ALB、API Gateway、自訂）

Origin 類型

S3 Origin

  • OAI/OAC：限制 S3 只能透過 CloudFront 存取
  • 建議使用 OAC（Origin Access Control），取代舊版 OAI

Custom Origin：ALB、EC2（需公開 IP）、API Gateway、自訂 HTTP 伺服器

TTL 與快取控制

  • Origin 可設定 Cache-Control Header，CloudFront 在 Min/Max TTL 範圍內遵守
  • 快取失效（Invalidation）：手動清除快取（有費用），考慮版本化 URL

SSL/TLS 與 ACM

來源區域

SNI（Server Name Indication）：允許多個憑證在同一 IP，預設免費，舊瀏覽器需要專用 IP（費用較高）。

選項說明
HTTP and HTTPS兩者皆可

  • 適用：內容授權限制、合規要求、成本控制

AWS Global Accelerator

  • 非 CDN，是網路最佳化服務，使用 AWS 骨幹網路
  • 提供兩個 Anycast 靜態 IP（防火牆白名單友好）
  • 支援 TCP/UDP，適合即時應用、遊戲
  • 提供健康檢查與容錯路由

Global Accelerator 使用案例

即時性需求遊戲、交易系統
跨 Region 故障轉移快速容錯路由

CloudFront 安全性

  • WAF 整合：保護 Distribution 免受 SQL Injection、XSS、速率限制攻擊
  • Field-Level Encryption：在邊緣加密敏感欄位（如信用卡），僅後端可解密
  • Origin Shield：額外的快取層，減少 Origin 負載，改善快取命中率

應用場景

無伺服器 API 架構

  • API Gateway + Lambda + DynamoDB
  • 按需付費，無需管理伺服器，自動擴展
  • CloudFront 前置，加速全球存取

微服務 API 路由

  • 單一入口，路由到不同微服務
  • 統一認證、限流、監控
  • 各服務獨立部署和擴展

混合雲 API

  • VPC Link 連接私有服務
  • Private API 限制僅 VPC 內存取
  • Direct Connect 整合地端服務
