負載平衡（Load Balancing）是將網路流量分散到多個伺服器的技術，確保單一伺服器不會過載，同時提高系統的可用性與回應速度。在雲端架構中，負載平衡器是連接使用者與後端服務的關鍵中介層。

AWS 服務實作

ELB（Elastic Load Balancing）概述

  • 提供負載平衡器，具有固定 DNS 名稱（如 myalb-123456789.us-east-1.elb.amazonaws.com）
  • 將請求導向 Target Group（可為 EC2、IP、Lambda）
  • 與 ASG、ECS、EKS 等整合，每個 AZ 有 LB 節點，DNS 輪詢返回所有節點 IP

負載平衡器類型

類型支援協定OSI 層特性
Application Load Balancer (ALB)HTTP, HTTPS, WebSocketLayer 7適合 Web 應用，支援進階路由
Network Load Balancer (NLB)TCP, UDP, TLSLayer 4超低延遲，處理大量流量
Gateway Load Balancer (GLB)GENEVE 第三方代理協定Layer 3整合第三方安全設備
Classic Load Balancer (CLB)HTTP, HTTPS, TCPL4/L7舊式 ELB，不建議新專案使用

Application Load Balancer (ALB) 詳解

主要特性

  • 支援基於路徑、主機名、Header 等進階路由
  • 加入 X-Amzn-Trace-Id 追蹤標頭，來源 IP/Port 透過 X-Forwarded-For 標頭取得
  • 可與 WAF 結合設限流、防 DDoS
  • 支援 SSL/TLS 終止，整合 ACM 憑證

路由規則（Listener → Rules → Target Group）

條件類型說明範例
Path基於 URL 路徑/api/*
Host基於主機名api.example.com
HTTP Header基於請求標頭X-Custom-Header
HTTP Method基於請求方法GET/POST
Query String基於查詢參數?user=admin
Source IP基於來源 IP10.0.0.0/8

動作類型

動作說明
forward轉發到 Target Group
redirectHTTP 重導向
fixed-response返回自訂回應
authenticate-oidcOIDC 認證
authenticate-cognitoCognito 認證

目標類型：EC2 實例、IP 地址、Lambda 函數、其他 ALB（ALB chaining）

Network Load Balancer (NLB) 詳解

主要特性

  • 處理波動大、極低延遲的流量，效能：數百萬 RPS，超低延遲（< 1ms）
  • 每個 AZ 可有 Elastic IP，支援固定靜態 IP
  • NLB 不修改封包，目標看到真實客戶端 IP
  • 支援 TLS 終止，支援 Proxy Protocol v2

使用場景：需要固定 IP、極低延遲、TCP/UDP 協定、高吞吐量應用

  • 僅支援 ALB、CLB，在 Target Group 層級設定

Cross-Zone Load Balancing

  • 啟用時：流量平均分配到所有 AZ 的目標
  • 停用時：流量僅在各自 AZ 內分配
類型預設狀態跨 AZ 費用
ALB啟用免費
NLB停用跨 AZ 流量需付費
CLB停用免費

Connection Draining（連線排空）

  • 也稱為 Deregistration Delay
  • 目標取消註冊時，允許現有連線完成，不接受新連線
  • 預設 300 秒，範圍 0-3600 秒
  • 長時間請求：增加排空時間；短時間請求：減少排空時間（加快擴縮）

SSL/TLS 終止

元件說明
SSL 憑證儲存在 ACM 或 IAM
SNI支援多域名憑證（Server Name Indication）
SSL Policy定義支援的協定和加密套件

SNI：允許多個 SSL 憑證在同一 ALB，客戶端在握手時指定主機名，ALB 選擇對應憑證。

Gateway Load Balancer (GLB) 詳解

  • 用於部署和管理第三方虛擬設備（防火牆、IDS/IPS）
  • 在 Layer 3（網路層）運作，使用 GENEVE 協定封裝流量
  • 支援透明網路閘道和負載平衡

流量流程

流量入口 → GLB 端點 → GLB → 安全設備 → GLB → 目的地

使用場景

場景說明
集中防火牆所有 VPC 流量經過防火牆
IDS/IPS入侵偵測和防禦
DLP資料外洩防護
流量鏡像複製流量進行分析

AWS ELB 最佳實踐

  1. Web 應用使用 ALB：利用進階路由功能（Path/Host/Header）
  2. 高效能場景使用 NLB：極低延遲（< 1ms），數百萬 RPS
  3. 啟用 Cross-Zone Load Balancing：確保流量均勻分配
  4. 設定適當的健康檢查：快速偵測不健康目標
  5. 使用 Target Group：靈活管理後端目標
  6. 安全設備使用 GLB：統一管理第三方網路設備
  7. ALB + WAF：Layer 7 安全防護
  8. NLB + EIP：需要靜態 IP 的應用（防火牆白名單）
