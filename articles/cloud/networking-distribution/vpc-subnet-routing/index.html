虛擬私有雲（Virtual Private Cloud，VPC）是雲端網路的基礎構建單元，提供邏輯隔離的虛擬網路環境。正確的 VPC 設計是所有雲端架構的根基，影響安全性、可擴展性與維運效率。

核心概念

VPC 基本元素

元素說明用途
VPC虛擬私有網路邏輯隔離的網路空間
Subnet子網VPC 內的 IP 位址範圍
Route Table路由表定義流量轉發規則
Internet Gateway網際網路閘道連接公網
NAT GatewayNAT 閘道私有子網存取外部
CIDR BlockIP 位址範圍定義網路大小

CIDR 規劃

CIDRIP 數量適用規模範例
/1665,536大型 VPC10.0.0.0/16
/204,096中型子網10.0.0.0/20
/24256小型子網10.0.1.0/24
/2816最小子網10.0.1.0/28

規劃原則：

  • VPC CIDR 不要與地端或其他 VPC 重疊
  • 預留足夠空間供未來擴展
  • 子網大小考量 AWS 保留 IP（每子網保留 5 個）

公有子網 vs 私有子網

特性公有子網私有子網
Internet Gateway有路由無路由
公有 IP可分配不分配
外部存取直接可達透過 NAT
適用資源LB、BastionApp、DB
安全等級中高

應用場景

企業級多帳號 VPC 設計

  1. 共享服務 VPC：DNS、目錄服務、監控
  2. 生產 VPC：應用與資料庫
  3. 開發 VPC：開發與測試環境
  4. DMZ VPC：面向外部的服務
  5. Transit Gateway：中心化連接所有 VPC

混合雲網路

  • VPN Gateway 連接地端與雲端
  • Direct Connect/Interconnect 專線
  • 路由傳播（BGP）自動更新路由
  • DNS 混合解析（條件轉發）

容器網路（EKS/GKE）

  • VPC CNI 模式：Pod 直接使用 VPC IP
  • 需規劃足夠 IP 空間（每 Pod 一個 IP）
  • 建議 /19 以上的子網給 K8s
  • 考量 Pod 最大數量對 IP 的需求
