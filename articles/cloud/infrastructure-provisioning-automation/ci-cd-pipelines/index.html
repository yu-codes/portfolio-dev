CI/CD（持續整合/持續交付）管線是現代軟體開發的核心基礎設施，將程式碼從開發者的提交到生產環境的部署自動化。良好設計的 CI/CD 管線能顯著提升軟體交付頻率、降低部署風險，並確保品質的一致性。

核心概念

CI vs CD vs CD

概念全稱定義自動化範圍
CI持續整合頻繁將程式碼整合到主幹建構、測試
CD (Delivery)持續交付確保程式碼隨時可部署建構、測試、預備
CD (Deployment)持續部署自動部署到生產環境建構、測試、部署

管線即程式碼最佳實踐

  1. 版本控制：管線定義與應用程式碼共存於 Git
  2. 可重用性：共享步驟、模板化常用管線
  3. 快速回饋：失敗盡早發現，快速通知
  4. 環境隔離：每次建構使用乾淨環境
  5. 密鑰管理：使用管線平台的密鑰功能，不硬編碼
  6. 可觀測性：管線執行指標、瓶頸分析

AWS 服務實作

AWS Step Functions（狀態機）

  • 無伺服器的流程編排服務
  • 使用 JSON 宣告式語法定義狀態機，可連接多個 Lambda 或其他服務流程
  • 適合用於編排複雜的部署流程
  • 可視化工作流程，支援錯誤處理與重試邏輯

AWS Simple Workflow Service（SWF）

  • 執行於 EC2 的流程編排服務（非 Serverless）
  • 支援流程中的外部訊號、子流程回傳值等進階需求
  • 新案建議改用 Step Functions

AWS 部署策略

CloudFormation 部署

  • 使用模板定義完整基礎設施
  • 支援 Stack 更新與自動回滾（預設啟用錯誤自動回滾）
  • StackSets 支援跨帳號、跨區域部署

Elastic Beanstalk 部署策略

策略說明停機風險
All at once一次全部更新有短暫停機
Rolling滾動更新部分實例更新時容量下降
Rolling with additional batch額外批次滾動維持完整容量
Immutable不可變部署（新 ASG）無停機
Blue/Green藍綠部署無停機，可快速回滾

與其他 AWS 服務整合

自動化觸發

  • EventBridge：可排程觸發部署任務（類似 cron），監聽 AWS 事件自動啟動管線
  • Lambda：可作為部署流程中的自動化腳本執行
  • SNS/SQS：部署通知與訊息佇列，整合警報與非同步處理

監控與回滾

  • CloudWatch：監控部署過程中的指標與告警
  • CloudTrail：稽核部署操作歷史，完整記錄 API 呼叫
  • Auto Scaling：部署後自動調整容量，配合部署策略彈性伸縮

AWS CodePipeline

完全託管的持續交付服務，自動化發布和部署流程，支援來源、建置、測試、部署多個階段。

階段類型

階段說明
Source從 CodeCommit、GitHub、S3 取得原始碼
Build使用 CodeBuild 編譯和測試
Test執行額外的整合測試或驗收測試
Deploy使用 CodeDeploy、Beanstalk、CloudFormation 部署

功能特點

  • 支援手動審批階段（Manual Approval）
  • 與 SNS 整合發送通知
  • 透過 CloudTrail 稽核所有操作
  • 支援平行執行多個動作

AWS CodeBuild

完全託管的持續整合服務，負責編譯、測試和打包程式碼，使用 buildspec.yml 定義建置步驟。

特性

  • 支援多種程式語言和框架
  • 支援自訂 Docker 映像作為建置環境
  • 建置快取加速後續建置（S3 快取或本地快取）
  • 支援 VPC 存取私有資源
  • 按使用時間計費，無需管理伺服器
# buildspec.yml 範例結構
version: 0.2
phases:
  install:
    commands:
      - npm install
  build:
    commands:
      - npm run build
  post_build:
    commands:
      - docker push $REPO_URI
artifacts:
  files:
    - '**/*'

AWS CodeDeploy

自動化應用程式部署服務，支援 EC2、Lambda、ECS 和本地伺服器。

部署策略

策略說明適用目標
In-place原地更新，停止應用、部署、啟動EC2、本地伺服器
Blue/Green藍綠部署，流量切換EC2、ECS
Linear線性漸進部署（每 N 分鐘增加 X%）Lambda
Canary金絲雀部署（先 10%，驗證後 100%）Lambda

功能特點

  • 自動回滾失敗的部署（CloudWatch Alarms 觸發）
  • 使用 AppSpec 檔案定義部署鉤子（lifecycle hooks）
  • 與 ELB 整合進行流量切換
  • 支援部署前/後執行腳本

AWS CodeCommit

完全託管的 Git 儲存庫服務，安全、高可用、可擴展。

功能

  • 支援所有標準 Git 功能（分支、標籤、合併）
  • Pull Request 和程式碼審查
  • 觸發器連接 Lambda 或 SNS（如程式碼推送時自動通知）
  • 跨區域複製以提高可用性
  • 與 IAM 整合，支援細粒度存取控制

AWS CI/CD 最佳實踐

  1. 基礎設施即程式碼：所有資源定義存放於版本控制
  2. 自動化測試：部署前在 CodeBuild 執行自動化測試
  3. 漸進式部署：使用 Rolling 或 Blue/Green 策略降低風險
  4. 監控與告警：部署後持續透過 CloudWatch 監控應用健康狀態
  5. 快速回滾機制：CodeDeploy 支援自動回滾，確保能快速還原至穩定版本
  6. 使用 CodePipeline：統一管理完整 CI/CD 流程，串接 CodeCommit → CodeBuild → CodeDeploy
  7. 啟用手動審批：生產環境部署前加入 Manual Approval 階段
