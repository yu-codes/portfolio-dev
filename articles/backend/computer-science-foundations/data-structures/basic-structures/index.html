資料結構是程式設計的基石，選擇正確的資料結構直接決定了演算法的效率與程式的品質。本文系統性地介紹基礎資料結構的原理、操作複雜度與適用場景，幫助開發者在實際工程中做出正確的選擇。

架構總覽

陣列 (Array) 與動態陣列

特性靜態陣列動態陣列
大小固定可變
擴容不可倍增策略
語言實作C array, Java arrayArrayList, Vec, slice
末端插入N/AO(1) 攤提
中間插入O(n)O(n)

動態陣列擴容策略：

策略方式空間浪費攤提複雜度
倍增容量 × 2最多 50%O(1)
1.5倍增容量 × 1.5最多 33%O(1)
固定增量容量 + k低O(n)

鏈結串列 (Linked List)

類型結構優點缺點
單向A → B → C → null最簡單只能前向遍歷
雙向null ← A ↔ B ↔ C → null雙向遍歷額外指標空間
環形A → B → C → A無需 null 檢查終止條件複雜
跳躍串列 (Skip List)多層鏈結O(log n) 搜尋額外空間

堆疊 (Stack) 與佇列 (Queue)

特性堆疊佇列雙端佇列 (Deque)
存取原則LIFO (後進先出)FIFO (先進先出)兩端皆可
核心操作push, pop, peekenqueue, dequeuepushFront, pushBack
實作方式陣列/鏈結串列環形陣列/鏈結串列環形陣列/雙向鏈結
典型應用函式呼叫、撤銷排隊、BFS滑動窗口
