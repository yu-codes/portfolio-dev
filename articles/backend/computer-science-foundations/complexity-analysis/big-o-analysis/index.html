Big-O 複雜度分析是評估演算法效能的基礎工具。它提供了一種獨立於硬體與常數因子的方式來比較演算法的可擴展性。本文系統性地介紹複雜度分析的方法、常見複雜度等級，以及在工程實踐中的應用。

關鍵比較

最佳/平均/最差情況

情況描述範例 (快速排序)
最佳 (Best)最理想的輸入O(n log n) — 每次均勻分割
平均 (Average)隨機輸入的期望O(n log n) — 隨機排列
最差 (Worst)最不利的輸入O(n²) — 已排序且選首元素為樞紐

常見陷阱

陷阱描述注意事項
忽略常數因子O(n) 可能比 O(n log n) 慢實際測量
忽略低階項小 n 時低階項可能主導考慮實際規模
混淆平均與最差雜湊表 O(1) 是平均注意最差情況
空間換時間降低時間複雜度的代價考慮記憶體限制
遞迴深度可能導致堆疊溢出考慮空間複雜度
