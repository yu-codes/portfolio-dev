搜尋與排序是最基本也最重要的演算法類別。理解各種演算法的效能特性與適用場景，是寫出高效能程式的基礎。本文系統性地比較主要的搜尋與排序演算法，分析它們的時間/空間複雜度與實際工程應用。

排序演算法總覽

演算法最佳平均最差空間穩定

選擇排序O(n²)O(n²)O(n²)O(1)❌
插入排序O(n)O(n²)O(n²)O(1)✅
合併排序O(n log n)O(n log n)O(n log n)O(n)✅
快速排序O(n log n)O(n log n)O(n²)O(log n)❌

搜尋演算法總覽

演算法時間複雜度前置條件適用場景

雜湊搜尋O(1)*雜湊表等值查詢

穩定性的意義

排序穩定性定義重要場景
穩定相等元素保持原始順序多鍵排序、UI 排序
不穩定相等元素順序可能改變單一鍵排序

架構總覽

比較型排序的下界

比較型排序的理論下界為 O(n log n)：

類別可突破 O(n log n)條件
比較型排序❌僅依靠比較操作
非比較型排序✅利用資料特性
計數排序✅ O(n+k)值域有限
基數排序✅ O(nk)固定位數
桶排序✅ O(n)*資料均勻分布

快速排序的變體

變體樞紐選擇最差情況實際表現
基本版第一個/最後一個O(n²) 已排序一般
隨機化隨機選取期望 O(n log n)良好
三數取中三個取中位數很少觸發最差優秀
雙樞紐兩個樞紐分三段O(n log n)最佳 (Java)
內省排序深度過大切換堆積排序O(n log n) 保證優秀 (C++ STL)

合併排序的變體

變體特點空間應用
自頂向下遞迴分割O(n)教學
自底向上迭代合併O(n)鏈結串列排序
Tim Sort利用已有順序O(n)Python, Java
原地合併減少空間O(1)理論研究
外部排序處理超大檔案有限記憶體資料庫

關鍵比較

排序演算法選擇指南

場景推薦演算法理由
小規模 (n < 50)插入排序常數因子小
通用排序Tim Sort / 快速排序平均最快
需要穩定性合併排序 / Tim Sort保證穩定
記憶體受限堆積排序O(1) 額外空間
整數排序 (小範圍)計數排序O(n+k) 線性
近乎有序插入排序 / Tim Sort接近 O(n)
外部排序 (超大檔案)外部合併排序分批處理
並行排序合併排序天然可分治
