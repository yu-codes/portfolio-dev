電腦科學基礎

資料結構、演算法與複雜度

計算機科學基礎是每位軟體工程師的必備素養。無論使用何種程式語言或框架，對資料結構、演算法與複雜度分析的深入理解，是寫出高效、可擴展程式碼的根本保證。這些知識不僅是技術面試的重點，更是日常開發中做出正確技術決策的基石——選擇合適的資料結構可能讓演算法從 O(n²) 降到 O(n log n)，在大規模資料處理中意味著秒級與小時級的差距。

核心概念

計算機科學基礎圍繞三個核心支柱：

資料結構（Data Structures） 是組織與儲存資料的方式，不同的資料結構適合不同的操作模式：

  • 陣列（Array）：連續記憶體、O(1) 隨機存取、但插入刪除需要 O(n)
  • 鏈結串列（Linked List）：非連續記憶體、O(1) 插入刪除、但搜尋需要 O(n)
  • 雜湊表（Hash Table）：平均 O(1) 查詢/插入/刪除，但最壞情況可能退化
  • 樹（Tree）：階層結構、BST 提供 O(log n) 操作、B-Tree 優化磁碟讀取
  • 圖（Graph）：表達複雜關係、支援路徑搜尋與網路分析
  • 堆（Heap）：優先佇列實現、O(log n) 插入與取出最值

演算法（Algorithms） 是解決特定問題的步驟化方法：

  • 排序：快速排序、歸併排序、堆排序各有適用場景
  • 搜尋：二分搜尋、BFS/DFS、A* 啟發式搜尋
  • 動態規劃：將複雜問題分解為重疊子問題，避免重複計算
  • 貪心演算法：每步選擇局部最優解，適合特定問題結構
  • 分治法：將問題分解、遞迴求解、合併結果

複雜度分析（Complexity Analysis） 提供評估演算法效能的數學框架：

  • 時間複雜度：演算法執行步驟數隨輸入規模的增長函數
  • 空間複雜度：演算法所需額外記憶體隨輸入規模的增長函數
  • Big-O 表示法：描述上界（最壞情況）的漸近行為

架構總覽

資料結構與演算法的知識體系可以用以下層次來理解：

問題建模層：將實際問題抽象為計算模型
    ↓
資料結構選擇層：根據操作模式選擇合適的資料結構
    ↓
演算法設計層：選擇或設計解決問題的演算法策略
    ↓
複雜度分析層：評估方案的時間與空間效率
    ↓
實作優化層：快取友善性、常數因子優化、並行化

在工程實踐中，這些抽象知識會映射到具體的系統設計決策：

  • 資料庫索引：B+ Tree 保證 O(log n) 查詢、Hash Index 提供 O(1) 精確匹配
  • 快取系統：LRU Cache 使用 Hash Map + 雙向鏈結串列實現 O(1) 操作
  • 路由系統：Trie 樹實現高效的 URL 前綴匹配
  • 任務排程：優先佇列（Min-Heap）保證高優先級任務先執行
  • 社群網路：圖演算法計算最短路徑、社群偵測、推薦系統

應用場景

搜尋引擎：倒排索引（Inverted Index）實現全文檢索、Trie 樹實現自動補全、PageRank 使用圖演算法計算網頁重要度、TF-IDF 演算法計算文件相關性。

推薦系統：協同過濾使用矩陣分解、KNN 演算法計算相似度、圖神經網路建模使用者-物品互動關係。

資料庫引擎：B+ Tree 索引支援範圍查詢、Hash Index 支援等值查詢、LSM-Tree（Log-Structured Merge Tree）優化寫入效能、Bloom Filter 快速判斷元素是否存在。

網路路由：Dijkstra 演算法計算最短路徑、動態規劃解決旅行推銷員問題的近似解、最小生成樹演算法設計網路拓撲。

壓縮演算法：Huffman Coding 基於頻率建構最優前綴碼、LZ77/LZ78 利用字典壓縮、動態規劃優化壓縮比。

子主題導覽

本計算機科學基礎系列涵蓋三大核心主題：

  • 資料結構（Data Structures）：從基礎的陣列、鏈結串列到進階的紅黑樹、Skip List、Bloom Filter，深入每種資料結構的內部實作與工程應用
  • 演算法（Algorithms）：排序搜尋、動態規劃、圖演算法、字串演算法等核心演算法的原理分析與程式碼實作
  • 複雜度分析（Complexity Analysis）：Big-O/Ω/Θ 表示法、攤銷分析、主定理（Master Theorem）與空間-時間權衡
