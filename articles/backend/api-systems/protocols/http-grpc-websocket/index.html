HTTP、gRPC 與 WebSocket 是現代後端系統中三種最重要的通訊協定。它們各自針對不同的通訊模式設計，理解它們的差異是選擇正確技術方案的基礎。本文深入比較這三種協定的設計理念、效能特性與適用場景。

核心概念

三種協定的定位

協定模型設計目標典型場景
HTTP/REST請求-回應通用 Web 通訊CRUD API
gRPC多種模式高效能服務間通訊微服務
WebSocket全雙工即時雙向通訊聊天、推送

HTTP 演進歷程

版本年份關鍵特性改進
HTTP/1.01996每次請求一個連接基礎
HTTP/1.11997持久連接、管線化減少連接建立
HTTP/22015多路複用、標頭壓縮、Server Push大幅提升效能
HTTP/32022QUIC (UDP)、0-RTT消除隊頭阻塞

gRPC 的核心特性

特性描述效果
Protocol Buffers二進位序列化格式體積小、解析快
HTTP/2底層傳輸多路複用
強型別合約.proto 文件定義編譯期檢查
串流支援四種通訊模式靈活的資料流
自動程式碼生成多語言支援跨語言一致性
