可觀測性

日誌、監控、追蹤與 SRE

Table of Contents

  • 延伸閱讀

可觀測性（Observability）是指從系統的外部輸出推斷其內部狀態的能力。在分散式微服務架構中，問題不再侷限於單一節點，而是可能跨越數十個服務的複雜調用鏈。可觀測性的三大支柱——日誌（Logging）、指標（Metrics）與分散式追蹤（Distributed Tracing）——搭配 SRE 原則，構成了現代系統維運的基礎。本篇將系統性地介紹可觀測性的核心框架與實踐方法。

核心概念

可觀測性包含四大領域：日誌（Logging）、監控（Monitoring）、分散式追蹤（Tracing）、以及 SRE 原則（Site Reliability Engineering）。

日誌是最基本的可觀測性信號，記錄系統中發生的離散事件。結構化日誌（Structured Logging）以 JSON 格式輸出，便於機器解析與查詢。集中式日誌管理（如 ELK Stack）將分散各節點的日誌彙整至統一平台，支援全文搜尋與關聯分析。

監控與指標關注系統的數值化度量。指標（Metrics）以時間序列形式記錄系統的量化狀態——CPU 使用率、請求延遲百分位數、錯誤率等。Prometheus 是雲原生生態中的標準指標收集與查詢引擎，搭配 Grafana 實現視覺化儀表板。

分散式追蹤解決的是「一個請求跨越多個服務時，瓶頸在哪裡」的問題。透過在請求中注入追蹤上下文（Trace Context），記錄每個服務的處理時間與調用關係，形成完整的調用鏈視圖。OpenTelemetry 正成為統一遙測資料收集的產業標準。

SRE 原則由 Google 提出，將軟體工程的方法論應用於維運工作。核心概念包括 SLI（Service Level Indicator）、SLO（Service Level Objective）與 Error Budget，以量化的方式平衡可靠性與開發速度。

架構總覽

一個完善的可觀測性平台通常包含以下元件：

  1. 資料收集層：

    • 應用程式 SDK（OpenTelemetry SDK）
    • 基礎設施代理（Node Exporter、Fluent Bit）
    • 自動儀器化（Auto-Instrumentation）

  2. 資料傳輸層：

    • OpenTelemetry Collector — 接收、處理、導出遙測資料
    • 支援多種協定（OTLP、Prometheus Remote Write、Zipkin）

  3. 資料儲存層：

    • 指標：Prometheus / Thanos / Cortex / Mimir
    • 日誌：Elasticsearch / Loki / ClickHouse
    • 追蹤：Jaeger / Tempo / Zipkin

  4. 視覺化與分析層：

    • Grafana — 統一的視覺化平台
    • 告警管理（Alertmanager）
    • SLO 儀表板與 Error Budget 追蹤

  5. 智慧層：

    • 異常偵測（Anomaly Detection）
    • 根因分析（Root Cause Analysis）
    • AIOps 自動化事件關聯

應用場景

微服務故障排查：使用者回報 API 回應緩慢。透過分散式追蹤找到調用鏈中的瓶頸 Span（例如某個下游服務的資料庫查詢耗時異常），結合該時間點的指標確認是 CPU 飽和還是記憶體壓力，最後查閱該服務的日誌取得詳細錯誤資訊。三大信號的關聯分析大幅縮短了 MTTR（Mean Time to Resolve）。

SLO 驅動的可靠性管理：定義核心 API 的 SLO 為「99.9% 的請求在 200ms 內完成」，透過 Prometheus 持續計算 SLI。Error Budget 消耗速率自動觸發告警：消耗超過 50% 時通知團隊，超過 80% 時凍結新功能上線，100% 耗盡時全力投入可靠性改善。

成本優化：日誌量隨業務增長爆炸式增加，透過 OpenTelemetry Collector 的 Processor 進行取樣（Sampling）、過濾與壓縮，將低價值日誌降級至冷儲存，高價值日誌保留於熱儲存，實現成本與可觀測能力的平衡。

自動化事件回應：Alertmanager 根據告警嚴重程度自動通知對應的值班人員（PagerDuty/Slack），搭配 Runbook Automation 對已知問題自動執行修復腳本（如重啟異常 Pod、擴容節點）。

效能基準與回歸偵測：每次部署後自動執行效能測試，將結果以指標形式寫入 Prometheus，Grafana 面板自動比對與前一版本的差異，顯著退化時阻止部署推進。
