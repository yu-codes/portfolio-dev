結構化日誌（Structured Logging）是以機器可解析的格式（如 JSON）記錄日誌的實踐，取代傳統的純文字日誌。結構化日誌讓日誌資料可以被高效地搜尋、過濾、分析和告警，是現代可觀測性基礎設施的基石。

核心概念

傳統日誌 vs 結構化日誌

維度傳統文字日誌結構化日誌
格式自由文字JSON/Key-Value
可解析性需要正規表達式原生解析
搜尋效率全文搜尋（慢）欄位索引（快）
上下文散落在文字中明確的欄位
聚合分析困難容易（按欄位聚合）
告警規則字串匹配條件運算式

日誌等級

等級用途生產環境範例
TRACE極度詳細的除錯資訊通常關閉函式進入/離開
DEBUG除錯用的詳細資訊通常關閉變數值、SQL 語句
INFO正常業務流程資訊開啟使用者登入、訂單建立
WARN預期外但可處理的情況開啟重試成功、降級觸發
ERROR錯誤但系統仍可運行開啟API 呼叫失敗、資料驗證失敗
FATAL系統無法繼續運行開啟資料庫連線失敗、OOM

結構化日誌的必要欄位

欄位說明範例
timestampISO 8601 格式時間2025-01-01T10:30:00.123Z
level日誌等級INFO
message人類可讀的訊息Order created successfully
service服務名稱order-service
trace_id分散式追蹤 IDabc123def456
span_id當前 Span IDspan789
environment運行環境production

應用場景

日誌規範設計

團隊統一的日誌規範：

規則說明理由
使用 JSON 格式統一格式機器可解析
時間用 UTC統一時區避免時區混亂
不記錄敏感資料PII 遮蔽合規要求
包含 trace_id串聯請求問題追蹤
錯誤包含 stack trace保留除錯資訊快速定位問題
使用一致的欄位名統一命名查詢便利

敏感資料處理

資料類型處理方式範例
密碼完全不記錄password: [REDACTED]
信用卡號遮蔽card: ****1234
Email部分遮蔽user: y***@example.com
Token截斷token: abc...xyz
IP 地址依合規需求可能需匿名化

日誌成本控制

策略方法節省比例
等級控制生產只記 INFO 以上50-70%
取樣高頻日誌取樣記錄30-50%
冷熱分層舊日誌移到冷儲存60-80%
壓縮傳輸和儲存時壓縮70-90%
精簡欄位移除不必要的欄位20-30%
生命週期設定自動過期刪除按保留期決定
