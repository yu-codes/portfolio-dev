自助式基礎設施

自助式基礎設施（Self-Service Infrastructure）讓開發者能夠自主申請和管理基礎設施資源，無需等待專門的運維團隊手動操作。這是平台工程的關鍵產出之一，它在維持安全和合規的前提下，大幅提升了開發者的自主性和交付速度。

核心概念

自助式的核心價值

傳統模式自助式模式
開發者提工單 → 等待審批 → 運維手動建立開發者自行操作 → 自動化驗證 → 即時建立
數天到數週數分鐘到數小時
運維成為瓶頸運維轉為平台建設
標準化低內建標準和護欄

自助式的設計原則

原則說明實踐
有護欄的自由在安全範圍內自由操作預定義的選項和限制
抽象複雜性隱藏底層實作細節高層次的 API
即時反饋操作結果立即可見即時狀態更新
可逆操作錯誤可以輕鬆恢復自助刪除和回滾
成本透明使用者了解資源成本成本估算和報告
合規內建安全和合規自動檢查Policy as Code

自助式能力的層級

完全自主所有操作僅高風險操作全環境管理

關鍵比較

自助式 vs 工單式 vs 完全自由

維度工單式自助式（有護欄）完全自由
速度慢（等待人工）快（自動化）最快
安全性高（人工審查）高（自動驗證）低
標準化依賴執行者內建強制無
可擴展差（線性增加人力）好-
成本控制中（人工判斷）好（自動配額）差
開發者體驗差好最好（短期）

安全的自助式生產環境存取

JIT（Just-In-Time）存取模式：

步驟動作有效期
1. 申請開發者說明原因和需要的權限-
2. 審批自動或主管審批-
3. 授權臨時授予最小權限1-4 小時
4. 使用操作被完整記錄有效期內
5. 撤銷到期自動撤銷自動
6. 審計操作日誌保存供審計長期
