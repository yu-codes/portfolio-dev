Golden Path（黃金路徑）是平台工程中的核心概念，指為開發者提供一條經過驗證、最佳化的標準開發路徑。它不是強制性的限制，而是一種「鋪好的道路」——遵循它可以獲得最好的體驗和支援，但開發者在有充分理由時仍可偏離。

核心概念

Golden Path 的哲學

原則說明類比
鋪好的道路提供最佳實踐路徑公園中的步道
非強制性可以偏離，但需承擔後果可以走草地，但沒有照明
減少決策疲勞預先做好技術選型菜單推薦而非自由搭配
內建品質路徑中內建安全和合規護欄而非路障
持續演進隨著經驗持續改善道路維護和升級

Golden Path vs 其他概念

概念強制性彈性目的
Golden Path低（建議）高降低認知負擔
標準規範高（強制）低確保一致性
最佳實踐文件低（建議）高知識分享
模板/腳手架中（起點）中快速啟動

Golden Path 涵蓋的面向

面向內容提供方式
技術選型語言、框架、資料庫決策文件
專案結構目錄結構、命名規範模板
CI/CD管線配置、測試策略管線模板
部署容器化、K8s 配置Helm chart
監控指標、日誌、追蹤預設整合
安全認證、授權、掃描內建護欄

關鍵比較

Golden Path 的成熟度模型

等級特徵開發者體驗
Level 0無標準，各自為政混亂、重複造輪子
Level 1有文件化的最佳實踐需自行閱讀和遵循
Level 2有模板和起始器快速啟動但維護困難
Level 3自動化的 Golden Path一鍵建立、持續維護
Level 4智能化推薦根據需求自動選擇路徑

應用場景

新服務建立的 Golden Path

一個完整的新服務建立流程：

步驟開發者操作平台自動化
1選擇服務類型（API/Worker/…）展示可選模板
2填寫服務名稱和描述驗證命名規範
3選擇依賴（DB/Cache/Queue）配置資源
4確認建立建立 Repo + CI/CD + 資源
5Clone 並開始開發服務已可部署到 Dev

Golden Path 的治理模型

活動頻率參與者產出
路徑審查季度平台團隊 + 架構師更新計畫
使用者回饋持續開發者改善清單
技術評估半年技術委員會技術選型更新

效果評估季度平台團隊成效報告
