資料庫

關聯式與非關聯式資料庫

資料庫是應用程式持久化儲存資料的核心組件，提供了資料的組織、查詢、更新與保護機制。從單機的 SQLite 到跨越多個資料中心的分散式叢集，資料庫技術在過去五十年間持續演進，衍生出關聯式、文件型、鍵值、列式、圖形等多種類型，各自針對不同的資料模型與存取模式進行最佳化。

核心概念

ACID 特性是傳統關聯式資料庫的基石：

  • 原子性（Atomicity）：事務中的所有操作要麼全部成功，要麼全部回滾，不會出現部分完成的狀態。
  • 一致性（Consistency）：事務執行前後，資料庫必須從一個合法狀態轉換到另一個合法狀態，所有約束條件得到滿足。
  • 隔離性（Isolation）：並行事務的執行結果等同於某種串行執行順序，避免髒讀、不可重複讀、幻讀等異常。
  • 持久性（Durability）：事務一旦提交，其結果永久保存，即使系統崩潰也不會遺失。

BASE 模型是 NoSQL 資料庫的設計哲學，相對於 ACID 的嚴格保證，BASE 接受：基本可用（Basically Available）、軟狀態（Soft State）、最終一致性（Eventually Consistent）。這種取捨換來了更高的可用性與擴展性。

儲存引擎是資料庫的底層核心，決定了資料如何在磁碟上組織與存取：

  • B-Tree 引擎（如 InnoDB）：以固定大小的頁面組織資料，支援高效的隨機讀寫，適合 OLTP 工作負載。
  • LSM-Tree 引擎（如 RocksDB、LevelDB）：將寫入緩衝在記憶體中，定期合併到磁碟，寫入效能極佳，適合寫入密集場景。
  • 列式儲存引擎（如 ClickHouse）：同一欄位的資料連續儲存，壓縮率高且適合聚合查詢，主要用於 OLAP。

架構總覽

SQL 資料庫（關聯式）：以結構化的表格儲存資料，表之間透過外鍵建立關係。SQL 提供強大的宣告式查詢語言，查詢優化器自動生成最佳執行計畫。PostgreSQL 是功能最完整的開源關聯式資料庫，支援 JSON、地理空間、全文搜尋等擴展功能。MySQL 以其簡潔和高效能成為 Web 應用的首選。

NoSQL 資料庫依據資料模型進一步細分：

  • 文件資料庫（Document）：MongoDB、CouchDB。以 JSON/BSON 文件為單位，Schema 彈性高，適合快速迭代的應用。
  • 鍵值資料庫（Key-Value）：Redis、DynamoDB。極簡的 Get/Put 介面，超高效能，適合快取與 Session 管理。
  • 寬欄資料庫（Wide-Column）：Cassandra、HBase。以列族（Column Family）組織資料，適合時序資料和大規模寫入。
  • 圖資料庫（Graph）：Neo4j、Amazon Neptune。以節點和邊描述實體關係，適合社交網路、知識圖譜。

分散式資料庫透過分片（Sharding）和複寫（Replication）實現水平擴展：

  • 分片將資料分散到多個節點，常用策略包括範圍分片（Range）、雜湊分片（Hash）和目錄分片（Directory）。
  • 複寫提供資料冗餘和讀取擴展，主要模式為主從複寫（Leader-Follower）、多主複寫（Multi-Leader）和無主複寫（Leaderless）。

選擇資料庫時需要考量：資料模型的匹配度、一致性需求、擴展性需求、查詢複雜度、運維能力、以及團隊的技術棧熟悉度。多數現代系統採用 Polyglot Persistence——針對不同子系統選用最適合的資料庫類型。

應用場景

金融系統：帳務核心使用 PostgreSQL 或 Oracle 保證嚴格的 ACID 事務；交易撮合引擎使用記憶體資料庫追求微秒級延遲；風控系統使用圖資料庫偵測異常關聯模式；歷史報表使用列式資料庫高效聚合。

內容管理平台：文章與媒體元資料使用 MongoDB 的彈性 Schema 適應頻繁的結構變更；標籤與分類使用關聯式資料庫維護一致性；搜尋功能使用 Elasticsearch 的倒排索引；使用者瀏覽記錄使用時序資料庫。

即時通訊系統：訊息存儲使用 Cassandra 處理大規模寫入；使用者狀態使用 Redis 提供即時查詢；對話歷史使用 MongoDB 支援靈活的查詢條件；使用者關係使用圖資料庫加速好友推薦。

遊戲後端：玩家資料使用 DynamoDB 提供穩定的低延遲；排行榜使用 Redis Sorted Set；遊戲世界狀態使用記憶體資料格線（In-Memory Data Grid）；日誌分析使用 ClickHouse 進行即時查詢。
