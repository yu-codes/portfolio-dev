資料系統

資料庫、快取與串流處理

資料系統是現代應用程式的核心支柱，負責可靠地儲存、高效地檢索、以及即時地處理資料。從簡單的 CRUD 應用到支撐數億用戶的全球化平台，資料系統的設計決定了系統的可用性、一致性與擴展性。理解資料系統的全貌，能幫助工程師在面對不同業務需求時做出正確的技術選型。

核心概念

資料系統的設計圍繞三個基本挑戰：

可靠性（Reliability）：系統在面對硬體故障、軟體錯誤或人為失誤時，仍能正確運作。這包括資料持久化保證（Durability）、錯誤偵測與恢復（Fault Tolerance）、以及資料一致性維護。

可擴展性（Scalability）：隨著資料量、流量和複雜度的增長，系統能合理地應對負載成長。擴展策略包括垂直擴展（Scale Up）增加單機資源、水平擴展（Scale Out）增加機器數量、以及功能分區（Functional Partitioning）將不同職責分離到不同服務。

可維護性（Maintainability）：系統在整個生命週期中保持可運維、可理解、可適應。這涉及系統的可操作性（Operability）、簡潔性（Simplicity）和可演化性（Evolvability）。

資料系統的另一個核心概念是 CAP 定理：分散式系統無法同時保證一致性（Consistency）、可用性（Availability）和分區容錯性（Partition Tolerance），必須在三者間取捨。實務上，由於網路分區不可避免，真正的選擇是在 CP（犧牲可用性）和 AP（犧牲強一致性）之間權衡。

架構總覽

資料系統可從功能角度劃分為以下子系統：

資料建模（Data Modeling）：定義資料的邏輯結構與關係。關聯式模型以表格和外鍵組織資料；文件模型以巢狀的 JSON/BSON 文件為單位；圖模型以節點和邊描述實體關係。模型的選擇影響查詢效率與應用程式的複雜度。

資料庫（Databases）：資料的持久化儲存引擎。SQL 資料庫（PostgreSQL、MySQL）提供 ACID 事務保證；NoSQL 資料庫（MongoDB、Cassandra）提供彈性的 Schema 與水平擴展能力；NewSQL 資料庫（CockroachDB、TiDB）嘗試兼顧兩者的優點。

快取與索引（Caching & Indexing）：加速資料存取的中間層。Redis/Memcached 提供記憶體級的讀取速度；CDN 快取靜態資源至邊緣節點；資料庫索引（B+Tree、Hash Index）減少磁碟 I/O。快取策略（Cache-Aside、Write-Through、Write-Behind）的選擇影響一致性與效能。

資料持久化（Data Persistence）：確保資料不會因系統故障而遺失。WAL（Write-Ahead Log）保證事務的原子性；複寫（Replication）提供資料冗餘；快照（Snapshot）支援時間點恢復。

資料串流（Data Streaming）：即時處理連續產生的資料流。訊息佇列（Kafka、RabbitMQ）解耦生產者與消費者；串流處理引擎（Flink、Spark Streaming）進行即時計算；事件溯源（Event Sourcing）將狀態變更記錄為不可變事件序列。

另一個重要維度是一致性模型的選擇：強一致性保證讀取到最新寫入但犧牲延遲；最終一致性提供更高可用性但可能讀到舊資料；因果一致性在兩者間取得平衡。

應用場景

電商平台：訂單系統使用 SQL 資料庫保證 ACID 事務；商品目錄使用文件資料庫支援彈性 Schema；購物車使用 Redis 快取提升讀取速度；使用者行為日誌使用 Kafka 進行即時串流分析以驅動推薦系統。

社群媒體：使用者關係圖使用圖資料庫；貼文動態時間線使用 Redis Sorted Set 快取；圖片與影片存放在物件儲存；即時通知使用 WebSocket + 訊息佇列分發。

金融交易系統：交易記錄使用 NewSQL 資料庫保證分散式 ACID；風控系統使用串流處理引擎進行即時欺詐偵測；報表系統使用 OLAP 引擎進行複雜聚合查詢；審計日誌使用事件溯源確保完整的變更歷史。

物聯網（IoT）平台：時序資料庫（TimescaleDB、InfluxDB）儲存感測器資料；邊緣運算節點使用嵌入式資料庫進行本地快取；中央平台使用串流處理進行即時異常偵測與告警。
