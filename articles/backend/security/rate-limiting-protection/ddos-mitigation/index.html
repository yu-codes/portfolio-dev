DDoS（Distributed Denial of Service）攻擊透過大量分散的流量來源癱瘓目標服務，使正常使用者無法存取。隨著 IoT 殭屍網路和攻擊即服務（DDoS-as-a-Service）的普及，DDoS 攻擊的規模和頻率持續增長。建立有效的 DDoS 緩解策略是維護服務可用性的必要措施。

DDoS 攻擊分類

類別攻擊層級目標典型攻擊規模
容量型L3-L4頻寬飽和UDP Flood、DNS 放大Tbps 級
協議型L3-L4連線資源耗盡SYN Flood、Ping of DeathMpps 級

常見攻擊手法

攻擊原理流量特徵防禦難點
UDP Flood發送大量 UDP 封包高頻寬、隨機端口量大難以完全過濾
SYN Flood大量半開放連線耗盡資源大量 SYN 無 ACK需區分正常連線
DNS 放大利用 DNS 反射放大流量來源偽造、放大 50-70x來源 IP 偽造
HTTP Flood大量合法 HTTP 請求看起來像正常流量難以區分正常請求
Slowloris保持連線但極慢傳送低流量佔用連線不觸發流量告警
NTP 放大利用 NTP 反射放大放大 500x+極大放大倍率

正常服務

Anycast 防禦原理

Anycast 將同一個 IP 位址宣告到全球多個節點：

  • 攻擊流量被分散到全球各節點
  • 每個節點只承受部分流量
  • 就近的節點處理最近的攻擊流量
  • 即使某節點被打滿，其他節點仍可服務

階段動作時間目標

應用層 DDoS 防禦

針對 HTTP Flood 等 L7 攻擊的防禦組合：

  1. 行為分析：建立正常流量基線，偵測異常模式
  2. JavaScript 挑戰：要求瀏覽器執行 JS 證明是真實瀏覽器
  3. CAPTCHA：可疑請求要求人機驗證
  4. 速率限制：限制單一 IP/Session 的請求頻率
  5. 地理分析：攻擊來源集中時封鎖特定地區
  6. 請求特徵過濾：過濾異常的 User-Agent、標頭模式
