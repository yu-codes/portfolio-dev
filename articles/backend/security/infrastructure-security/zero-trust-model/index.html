零信任模型

零信任安全架構的核心原則與實踐，涵蓋身份驗證、微分段、持續驗證與零信任網路存取

零信任（Zero Trust）是一種安全架構理念，核心原則為「永不信任，持續驗證」。不同於傳統的城堡護城河模型，零信任假設攻擊者可能已存在於網路內部，因此每一次存取請求都必須經過嚴格的身份驗證、授權和加密，無論請求來自網路內部或外部。

核心概念

零信任的基本原則

原則說明實踐
永不信任不因位置而給予信任內部流量也需認證
持續驗證不是一次性通過就永久可用即時評估存取風險
最小權限只授予完成任務的最小存取JIT（Just-In-Time）授權
假設已被入侵設計時考慮最壞情況限制爆炸半徑
明確驗證基於所有可用資料做決策身份+裝置+位置+行為

零信任的三大支柱

支柱範圍核心能力
身份安全使用者和服務身份MFA、SSO、身份治理
裝置安全端點裝置健康狀態裝置合規檢查、EDR
網路安全網路存取控制微分段、加密通訊

信任評估要素

零信任的存取決策基於多維度的信號：

信號類別具體指標風險高的情況
身份誰在請求未知帳號、異常登入
裝置什麼裝置未管理裝置、已越獄
位置從哪裡請求異常地理位置
時間什麼時候請求非工作時間
行為存取模式異常大量存取
資料敏感度存取什麼資料高機密資料

ZTNA vs 傳統 VPN

維度傳統 VPNZTNA（零信任網路存取）
存取範圍連入後可存取整個網路僅授權特定應用
身份驗證一次性認證持續認證
裝置檢查通常無必須通過合規檢查
攻擊面大（暴露整個網路）小（僅暴露代理）
使用體驗需手動連接透明接入
可擴展性集中式瓶頸分散式邊緣

傳統安全 vs 零信任

面向傳統模型零信任模型
信任邊界網路邊界（防火牆內=可信）無隱含信任
認證頻率入口認證一次每次存取都驗證
存取控制粗粒度（網路層）細粒度（應用+資料層）
監控焦點邊界流量所有東西向流量
橫向移動困難阻止微分段有效限制
遠端工作依賴 VPN原生支援

應用場景

零信任在微服務架構中的實踐

微服務的零信任實施重點：

  1. 服務身份：每個服務有唯一的身份憑證（mTLS 憑證）
  2. 服務間認證：所有服務通訊都經過雙向 TLS
  3. 細粒度授權：基於服務身份的 API 層級授權
  4. 可觀測性：記錄所有服務間的呼叫
  5. 動態策略：根據流量模式動態調整授權規則

漸進式零信任遷移路線

階段重點主要動作
第一階段身份基礎統一身份管理、MFA 全面啟用
第二階段裝置信任裝置健康檢查、合規評估
第三階段網路分段微分段、東西向加密
第四階段持續驗證行為分析、動態風險評估
第五階段資料保護資料分類、存取治理

零信任的常見挑戰

挑戰說明緩解方式
舊系統整合老舊應用不支援現代認證使用代理/閘道器封裝
效能開銷每次存取都驗證增加延遲Token 快取、就近決策
複雜度策略管理困難集中策略管理平台
文化阻力團隊不適應嚴格控制漸進式推行、透明化
