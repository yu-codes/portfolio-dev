網路安全是後端基礎設施防禦的第一道防線。透過合理的網路架構設計、存取控制策略與入侵偵測機制，可以有效降低攻擊面並限制潛在損害的範圍。在雲端原生時代，網路安全的概念已從傳統的邊界防禦擴展到更精細的微分段模型。

雲端安全群組設計原則

  • 最小開放：只開放必要的端口
  • 來源限制：限制允許連入的 IP 範圍
  • 分層控制：每一層有獨立的安全群組
  • 出站限制：限制對外連線（防止資料外洩）
  • 日誌記錄：記錄所有被拒絕的連線

傳統邊界防禦 vs 零信任

維度傳統邊界防禦零信任模型
信任模型內部網路可信永不信任，持續驗證
防禦重點邊界防火牆每個存取請求
橫向移動難以阻止微分段限制
遠端工作需要 VPN原生支援
可視性邊界流量所有流量

應用場景

Kubernetes 網路策略

K8s 中的 Network Policy 實現微分段：

  • 預設拒絕所有：建立預設拒絕策略
  • 白名單開放：明確允許必要的通訊
  • 命名空間隔離：不同團隊/環境的隔離
  • Egress 控制：限制 Pod 的對外連線

安全監控與告警

應持續監控的網路安全事件：

  • 未預期的對外連線（可能是資料外洩）
  • 暴力破解嘗試（大量認證失敗）
  • 異常流量模式（DDoS 前兆）
  • 內部橫向掃描（可能已被入侵）
  • 新開放的端口或服務
