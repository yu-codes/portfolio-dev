安全程式設計

安全程式設計的原則與實踐，涵蓋輸入驗證、錯誤處理、安全依賴管理與防禦性程式設計

安全程式設計（Secure Coding）不是在開發完成後再「加上」安全性，而是將安全思維融入每一行程式碼的撰寫過程中。遵循安全程式設計原則，能從根本上減少漏洞的產生，降低後期修補的成本。

安全的錯誤處理

錯誤處理的安全原則：

  • 對外：提供通用錯誤訊息，不洩露系統內部資訊
  • 對內：記錄詳細的錯誤資訊用於除錯
  • 分類：區分使用者錯誤、系統錯誤、安全事件
錯誤類型對使用者的回應內部處理
輸入錯誤明確指出哪個欄位有問題記錄來源 IP
認證失敗通用「帳號或密碼錯誤」記錄失敗次數
授權失敗「沒有權限」記錄並告警
系統錯誤「系統暫時無法服務」完整堆疊追蹤
安全事件與認證失敗相同回應立即告警

應用場景

安全的 API 設計原則

  1. 認證：所有 API 都需要認證（除了公開資源）
  2. 授權：檢查呼叫者是否有權執行該操作
  3. 輸入驗證：驗證所有請求參數、標頭、Body
  4. 速率限制：防止濫用和暴力攻擊
  5. 回應過濾：只回傳呼叫者有權看到的欄位
  6. 版本管理：安全修補時能快速更新

安全地使用第三方程式庫：

實踐說明工具

來源驗證驗證套件完整性npm audit signatures
最小引入只引入需要的功能避免全量導入

安全的組態管理

  • 敏感資訊：使用環境變數或密鑰管理服務（Vault、AWS Secrets Manager）
  • 組態分離：不同環境使用不同配置
  • 版本控制：.env 檔案不應進入版本控制
  • 加密：敏感配置在靜態儲存時加密
  • 輪換：定期更換密鑰和憑證
