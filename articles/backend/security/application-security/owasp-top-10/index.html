OWASP（Open Web Application Security Project）Top 10 是全球最具影響力的網頁應用安全風險清單，每隔數年更新一次。它代表了業界對當前最關鍵安全威脅的共識，是開發者、安全工程師與組織建立安全標準的重要參考。

風險評估公式

OWASP 使用以下因素評估風險嚴重度：

  • 威脅代理（Threat Agent）：誰可能利用此漏洞
  • 攻擊向量（Attack Vector）：利用漏洞的難易度
  • 安全弱點（Weakness）：漏洞的普遍程度
  • 技術影響（Technical Impact）：成功攻擊的後果
  • 商業影響（Business Impact）：對業務的損害程度

架構總覽

安全防禦深度模型

外部威脅
    ↓
[WAF / 防火牆]          ← 邊界防禦
    ↓
[負載均衡 / API Gateway] ← 流量過濾
    ↓
[應用安全層]            ← 輸入驗證、授權檢查
    ↓
[業務邏輯層]            ← 安全的程式設計實踐
    ↓
[資料存取層]            ← 參數化查詢、最小權限
    ↓
[資料儲存層]            ← 加密、存取控制

防禦不應依賴單一層，每層都應有獨立的安全機制。

依賴管理安全策略

針對 A06（易受攻擊的元件）的管理流程：

  1. 自動化掃描：使用 Dependabot、Snyk 等工具持續監控
  2. 版本鎖定：使用 lock file 確保一致性
  3. 漏洞評估：根據 CVSS 分數和可利用性評估風險
  4. 修補策略：
    • 嚴重（Critical）：24 小時內修補
    • 高（High）：7 天內修補
    • 中（Medium）：30 天內修補
    • 低（Low）：下次發版時修補

安全配置基線

針對 A05（安全配置錯誤）的檢查清單：

  • 移除不必要的功能、框架、元件
  • 停用除錯模式與詳細錯誤訊息
  • 配置安全標頭（CSP、HSTS、X-Frame-Options）
  • 更改所有預設密碼與帳號
  • 停用目錄列表
  • 實施最小權限原則
