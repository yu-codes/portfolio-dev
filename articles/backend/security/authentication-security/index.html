深入探討現代認證系統的安全設計，涵蓋密碼雜湊演算法選擇、Token 安全管理與多因子認證實踐，構建可靠的身份驗證機制。

核心概念

認證安全（Authentication Security）確保系統能正確識別使用者身份，防止未授權存取。認證是安全體系的基石——如果攻擊者能冒充合法使用者，其他所有安全控制都將失去意義。

密碼雜湊（Password Hashing） 是保護使用者憑證的最基本措施。絕不能以明文或可逆加密方式儲存密碼。現代密碼雜湊演算法如 bcrypt、scrypt 和 Argon2 專為密碼儲存設計，具備計算成本可調、內建鹽值（salt）和抗 GPU/ASIC 加速的特性。Argon2id 是目前推薦的首選演算法，它結合了抗時序攻擊（Argon2i）和抗 GPU 加速（Argon2d）的優勢。

Token 安全 涉及如何安全地發行、傳輸和驗證身份令牌。JWT（JSON Web Token）是最廣泛使用的無狀態 Token 格式，但其安全性取決於正確的實作：使用強密鑰、設定合理的過期時間、實施 Token 輪換機制，以及在登出時即時撤銷。Session-based 認證雖然需要伺服端狀態，但在某些場景中提供更好的安全控制（如即時撤銷）。

多因子認證（MFA） 要求使用者提供兩種以上的認證因子：知識因子（密碼）、持有因子（手機、安全金鑰）和固有因子（生物特徵）。即使密碼被洩漏，攻擊者仍無法僅憑密碼通過認證。TOTP（Time-based One-Time Password）和 WebAuthn/FIDO2 是目前最普及的 MFA 實作。

現代認證系統趨向分離認證關注點：應用程式不直接處理認證邏輯，而是委託給專門的身份提供者（IdP）如 Auth0、Keycloak 或雲端 IAM 服務。這降低了每個應用獨立實作認證的風險。

有狀態 vs 無狀態認證：無狀態 JWT 簡化了水平擴展，但犧牲了即時撤銷能力。短效期 Access Token（5-15 分鐘）配合可撤銷的 Refresh Token 是目前最佳的折衷方案。

應用場景

  1. 前後端分離 SPA：使用 OAuth 2.0 Authorization Code Flow with PKCE，Access Token 存放在記憶體中（非 localStorage），Refresh Token 使用 httpOnly、secure、SameSite=Strict Cookie 傳輸。

  2. 微服務內部認證：服務間使用 mTLS 或 JWT 進行認證。API Gateway 負責外部認證，內部服務信任 Gateway 轉發的身份資訊，減少重複驗證的開銷。

  3. 企業單一登入（SSO）：整合 SAML 2.0 或 OIDC 協定，讓員工使用企業帳號存取所有內部系統。配合條件式存取策略（設備合規、網路位置）提升安全性。

  4. 高安全性應用：銀行、政府系統要求 MFA 結合風險評估。異常登入行為（新設備、新地點、異常時間）觸發額外驗證步驟或暫時鎖定帳號。

  5. IoT 設備認證：設備使用 X.509 憑證或預共享金鑰進行認證，不適合密碼模式。設備身份需要與設備生命週期管理整合，支援憑證輪換和遠端撤銷。
