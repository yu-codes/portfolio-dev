副作用（Side Effects）是指函式除了回傳值之外對外部環境產生的任何影響。在前端組件中，副作用包括 API 呼叫、DOM 操作、訂閱、計時器等。正確管理副作用是建構穩定、可測試應用的關鍵。

核心概念

什麼是副作用

類型範例特性
網路請求fetch, WebSocket非同步、可能失敗
DOM 操作修改 DOM、測量尺寸同步、可能觸發重排
訂閱EventListener, Observable持續性、需要清理
計時器setTimeout, setInterval非同步、需要清理
瀏覽器 APIlocalStorage, geolocation可能失敗
日誌/追蹤console.log, analytics通常無需清理
全域狀態修改document.title, class需要還原

純粹渲染 vs 副作用

面向純粹渲染副作用
時機渲染階段渲染後（commit 階段）
可中斷可以（React Concurrent）不應該
決定性相同輸入必定相同輸出可能有外部影響
冪等性是不一定
測試快照測試需要 mock

副作用的分類

分類說明管理方式
同步副作用立即執行完畢useLayoutEffect / 直接執行
非同步副作用有延遲或等待useEffect + cleanup
訂閱型副作用持續監聽setup + teardown
一次性副作用只執行一次空依賴陣列
條件性副作用依賴變化時才執行依賴陣列控制

架構總覽

React Effect 模型

組件渲染
    ↓
React 更新 DOM
    ↓
useLayoutEffect 同步執行（阻塞繪製）
    ↓
瀏覽器繪製
    ↓
useEffect 非同步執行
    ├── 清理上一次的 effect（如果依賴變化）
    └── 執行新的 effect

Vue Watchers 與 Effects

API觸發時機用途
watchEffect立即執行 + 依賴變化時自動追蹤依賴的副作用
watch(source, cb)source 變化時明確監聽特定響應式來源
watch(source, cb, { immediate: true })立即 + 變化時初始化 + 持續監聽
onMounted + 手動邏輯掛載後DOM 相關操作

關鍵比較

useEffect vs Event Handler

面向useEffectEvent Handler
觸發渲染後、依賴變化使用者操作
用途同步外部系統回應使用者行為
是否必要系統需要的同步互動驅動的操作
範例訂閱即時資料按下按鈕送出表單
清理需要通常不需要
過度使用問題不必要的重新執行較少問題

副作用組合模式

模式說明實現
串行副作用A 完成後執行 Basync function 內 await
平行副作用A 和 B 同時執行Promise.all
條件副作用根據條件選擇性執行if 判斷在 effect 內

最佳實踐

原則說明
最小化 Effect能用事件處理的就不用 Effect
分離關注點每個 Effect 只處理一種副作用
穩定的依賴使用 useCallback/useMemo 穩定參照
正確清理所有訂閱和異步操作都要清理
避免效果連鎖Effect 觸發 state 更新再觸發另一個 Effect
提取為 Hook可重用的副作用邏輯提取為自訂 Hook
