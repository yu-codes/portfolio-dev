Islands Architecture（島嶼架構）將頁面視為靜態 HTML 的「海洋」中散佈的互動「島嶼」。每個島嶼獨立水合（Hydration），靜態區域保持純 HTML 不載入 JavaScript。這種模式特別適合內容為主的網站，能在保持豐富互動的同時達到近乎完美的效能表現。

應用場景

企業行銷網站

行銷頁面主要由靜態內容組成（品牌故事、產品特色、客戶見證），僅聯絡表單、動態定價計算器、互動示範需要 JavaScript。Islands 架構讓頁面在任何裝置上都能瞬間載入，同時保有必要的互動功能。

電商產品目錄

產品列表頁大量靜態呈現（圖片、價格、規格），僅加入購物車按鈕、篩選器、排序功能為互動島嶼。結合 View Transitions API 實現 MPA 中的流暢頁面切換，使用者感知近似 SPA 體驗。

多團隊協作網站

不同團隊可使用各自熟悉的框架開發島嶼元件（行銷團隊用 Vue、產品團隊用 React、資料團隊用 Svelte），所有島嶼在同一 Astro 頁面中共存，無需統一技術棧即可協作。
