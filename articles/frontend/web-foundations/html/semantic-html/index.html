語義化 HTML 是現代網頁開發的基石，透過使用具有明確意義的標籤來描述內容結構，讓瀏覽器、搜尋引擎與輔助技術能夠正確理解頁面資訊。本文將深入探討語義化標籤的核心原則、實務應用與最佳實踐。

核心概念

語義化 HTML（Semantic HTML）指的是選擇能夠準確傳達內容意義的 HTML 元素，而非僅依賴外觀樣式來組織頁面。與使用大量 <div> 和 <span> 的非語義化寫法不同，語義化標籤本身就攜帶了結構與意義資訊。

為什麼語義化很重要

  1. 可訪問性（Accessibility）：螢幕閱讀器依賴語義標籤來為視障使用者導覽頁面結構
  2. SEO 優化：搜尋引擎透過語義標籤判斷內容的重要性與層級關係
  3. 可維護性：開發者閱讀程式碼時能更快理解頁面結構
  4. 跨平台相容：不同裝置與瀏覽器能更好地呈現內容

常見語義化標籤

標籤用途取代對象
<header>頁首或區塊標頭<div class="header">
<nav>導覽連結區域<div class="nav">
<main>頁面主要內容<div class="content">
<article>獨立完整的內容單元<div class="post">
<section>主題性內容分組<div class="section">
<aside>側邊補充資訊<div class="sidebar">
<footer>頁尾或區塊結尾<div class="footer">
<figure>自含式內容（圖片、圖表）<div class="image-wrapper">
<time>日期與時間<span class="date">
<mark>標記/高亮文字<span class="highlight">

架構總覽

一個典型的語義化 HTML 頁面結構如下：

<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>語義化頁面範例</title>
</head>
<body>
  <header>
    <h1>網站名稱</h1>
    <nav>
      <ul>
        <li><a href="/">首頁</a></li>
        <li><a href="/about">關於</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>文章標題</h2>
        <time datetime="2025-01-01">2025 年 1 月 1 日</time>
      </header>
      <section>
        <h3>第一段落</h3>
        <p>內容文字...</p>
      </section>
      <footer>
        <p>作者：Yu</p>
      </footer>
    </article>

    <aside>
      <h3>相關文章</h3>
      <ul>
        <li><a href="#">推薦閱讀</a></li>
      </ul>
    </aside>
  </main>

  <footer>
    <p>&copy; 2025 版權所有</p>
  </footer>
</body>
</html>

標題層級原則

標題標籤 <h1> 到 <h6> 構成文件的大綱結構，必須遵循層級順序：

  • 每頁僅使用一個 <h1> 作為主標題
  • 不跳過層級（例如 <h2> 後直接使用 <h4>）
  • 標題層級反映內容的從屬關係

關鍵比較

語義化 vs 非語義化寫法

<!-- 非語義化 -->
<div class="header">
  <div class="nav">
    <div class="nav-item"><a href="/">Home</a></div>
  </div>
</div>
<div class="main">
  <div class="article">
    <div class="title">標題</div>
    <div class="content">內容</div>
  </div>
</div>

<!-- 語義化 -->
<header>
  <nav>
    <a href="/">Home</a>
  </nav>
</header>
<main>
  <article>
    <h2>標題</h2>
    <p>內容</p>
  </article>
</main>

<section> vs <article> vs <div>

  • <article>：可獨立存在、可被引用或轉載的完整內容（部落格文章、新聞報導、評論）
  • <section>：按主題分組的內容區塊，通常帶有標題
  • <div>：無語義意義的通用容器，僅用於樣式或腳本需求

<strong> vs <b> 與 <em> vs <i>

  • <strong>：語義上強調重要性（螢幕閱讀器會改變語調）
  • <b>：僅視覺上加粗，無語義意義
  • <em>：語義上的強調（改變句意理解）
  • <i>：僅視覺上斜體，無語義意義

應用場景

場景一：部落格網站

部落格最適合展示 <article> 標籤的價值。每篇文章作為獨立的 <article>，內部使用 <header> 包含標題與元資訊，<section> 分隔段落，<footer> 放置標籤與分享按鈕。

場景二：電商產品頁

產品資訊頁面可使用 <main> 包裹核心內容，<figure> 展示產品圖片，<details> 與 <summary> 實現可展開的規格說明，<aside> 放置推薦商品。

場景三：文件與技術文章

技術文件特別受益於正確的標題層級和 <code>、<pre>、<kbd> 等語義標籤，讓程式碼範例與操作指引更加清晰。

場景四：表單頁面

使用 <fieldset> 與 <legend> 分組表單欄位，<label> 明確關聯輸入控制項，大幅提升表單的可訪問性。
