HTML 表單是使用者與網站互動的核心介面，而表單驗證則確保使用者輸入的資料符合預期格式與規則。本文涵蓋原生 HTML5 表單控制項、內建驗證 API 與自訂驗證策略的完整實務。

核心概念

表單驗證分為兩個層級：

  1. 客戶端驗證（Client-side Validation）：在瀏覽器端即時回饋，提升使用者體驗
  2. 伺服器端驗證（Server-side Validation）：最終的安全防線，防止惡意資料進入系統

HTML5 引入了強大的內建驗證機制，讓開發者無需撰寫 JavaScript 就能實現基本驗證邏輯。

HTML5 表單控制項類型

類型用途內建驗證
email電子郵件自動驗證格式
url網址驗證 URL 格式
number數值min/max/step 限制
tel電話號碼配合 pattern 使用
date日期選擇日期格式驗證
range滑桿數值範圍限制
color顏色選擇器色碼格式
search搜尋清除按鈕支援
file檔案上傳accept 限制類型

驗證屬性

  • required：欄位必填
  • pattern：正規表達式驗證
  • minlength / maxlength：字串長度限制
  • min / max：數值範圍
  • step：數值遞增間隔
  • multiple：允許多值輸入

架構總覽

完整表單結構範例

<form action="/api/register" method="POST" novalidate id="registerForm">
  <fieldset>
    <legend>個人資料</legend>

    <div class="form-group">
      <label for="username">使用者名稱</label>
      <input type="text" id="username" name="username"
             required minlength="3" maxlength="20"
             pattern="[a-zA-Z0-9_]+"
             aria-describedby="username-hint">
      <small id="username-hint">3-20 字元，僅允許英數字與底線</small>
    </div>

    <div class="form-group">
      <label for="email">電子郵件</label>
      <input type="email" id="email" name="email" required
             placeholder="you@example.com">
    </div>

    <div class="form-group">
      <label for="password">密碼</label>
      <input type="password" id="password" name="password"
             required minlength="8"
             pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}">
    </div>

    <div class="form-group">
      <label for="birthdate">出生日期</label>
      <input type="date" id="birthdate" name="birthdate"
             max="2010-01-01">
    </div>
  </fieldset>

  <fieldset>
    <legend>偏好設定</legend>

    <div class="form-group">
      <label>通知方式</label>

Constraint Validation API

HTML5 提供了 JavaScript API 來進行進階驗證控制：

const form = document.getElementById('registerForm');
const email = document.getElementById('email');

// 自訂驗證訊息
email.addEventListener('invalid', (e) => {
  if (email.validity.typeMismatch) {
    email.setCustomValidity('請輸入有效的電子郵件格式');
  }
});

email.addEventListener('input', () => {
  email.setCustomValidity('');
});

// 表單提交驗證
form.addEventListener('submit', (e) => {
  if (!form.checkValidity()) {
    e.preventDefault();
    // 顯示所有錯誤
    form.reportValidity();
  }
});

關鍵比較

原生驗證 vs JavaScript 驗證

面向原生 HTML5 驗證JavaScript 自訂驗證
開發成本低，僅需屬性設定高，需撰寫邏輯
彈性有限，固定規則極高，自由定義
使用者體驗瀏覽器預設樣式完全可客製化
跨域驗證不支援可實現（如確認密碼）
非同步驗證不支援可呼叫 API 檢查

驗證時機比較

  • submit 事件：表單送出時統一驗證，適合簡單表單
  • blur 事件：離開欄位時驗證，即時回饋但不打擾填寫
  • input 事件：即時驗證，適合密碼強度指示器
  • 防抖（Debounce）：延遲驗證，適合需要呼叫 API 的場景（如帳號重複檢查）

novalidate 的使用策略

加上 novalidate 屬性後，原生驗證不會觸發瀏覽器預設 UI，但 Constraint Validation API 仍然可用。這讓開發者能用自訂 UI 呈現驗證結果，同時保留 API 的便利性。

應用場景

場景一：多步驟表單（Wizard Form）

大型表單拆分為多個步驟，每步僅驗證當前可見欄位：

function validateStep(stepElement) {
  const inputs = stepElement.querySelectorAll('input, select, textarea');
  let isValid = true;
  inputs.forEach(input => {
    if (!input.checkValidity()) {
      isValid = false;
      showError(input, input.validationMessage);
    }
  });
  return isValid;
}

場景二：即時搜尋與自動完成

搜尋欄位結合 <datalist> 提供建議選項，並使用防抖避免過多 API 呼叫。

場景三：檔案上傳驗證

透過 accept 限制檔案類型，再用 JavaScript 檢查檔案大小與數量：

fileInput.addEventListener('change', (e) => {
  const files = e.target.files;
  const maxSize = 5 * 1024 * 1024; // 5MB
  for (const file of files) {
    if (file.size > maxSize) {
      fileInput.setCustomValidity('檔案大小不得超過 5MB');
      return;
    }
  }
  fileInput.setCustomValidity('');
});

場景四：跨欄位驗證

密碼確認、日期範圍等需要比對多個欄位的場景，必須透過 JavaScript 實現。
