UX 工程

使用者體驗工程化

UX 工程（User Experience Engineering）是將設計理念轉化為可靠、可擴展技術實作的學科。它橋接了設計師的創意願景與工程師的技術實現，確保最終產品不僅「看起來對」，更「用起來對」。UX 工程涵蓋設計系統的技術架構、無障礙（Accessibility）的工程化保證、以及動態效果的效能最佳化。

核心概念

Design Tokens（設計權杖）：設計系統的原子單位，將視覺決策（顏色、間距、字型、圓角）抽象為命名化的變數。Tokens 在設計工具（Figma）與程式碼之間建立單一資料來源，確保設計更新能自動化同步到所有平台（Web、iOS、Android）。

無障礙工程（Accessibility Engineering）：無障礙不是事後添加的功能，而是從架構層面確保所有使用者（包括視障、聽障、動作障礙者）都能平等地使用產品。這包括語義化 HTML、ARIA 屬性管理、鍵盤導航支援、螢幕閱讀器相容性，以及自動化無障礙測試。

動畫與動態效果（Animation & Motion）：動畫不只是裝飾，它承載著 UX 語義——引導注意力、表達因果關係、提供操作回饋。UX 工程需要在視覺表現力與效能之間取得平衡，確保動畫在低階裝置上也能流暢運行（60fps）。

感知效能（Perceived Performance）：使用者感受到的速度往往比實際速度更重要。骨架屏（Skeleton）、漸進式載入、樂觀更新（Optimistic Update）等 UX 技巧讓應用「感覺」更快，即使底層資料還未載入完成。

應用場景

設計系統建置：大型組織需要跨團隊、跨平台的一致性 UI。Design Tokens 作為設計系統的基礎設施，從 Figma 變數匯出為 JSON，經 Style Dictionary 轉譯為 CSS Variables、iOS Swift、Android XML 等平台格式。Token 的變更觸發 CI/CD 自動更新所有消費端。

政府與金融無障礙合規：WCAG 2.1 AA 是許多國家的法律要求。UX 工程需建立自動化的無障礙保障流程——axe-core 整合到 CI/CD 測試、ESLint jsx-a11y 插件在開發時即時檢查、Storybook a11y addon 在元件開發時提供視覺化回饋。焦點管理（Focus Trap）在 Modal、Drawer 等覆蓋式元件中尤為關鍵。

品牌動效語言：如 Material Design 的 Motion 系統，定義了 Duration、Easing、Pattern 三個維度的設計規範。UX 工程將這些規範編碼為可配置的動畫 Token——--motion-duration-short: 150ms、--motion-easing-standard: cubic-bezier(0.4, 0, 0.2, 1)——確保所有組件的動畫行為一致且可調整。

漸進式載入體驗：在網路條件不佳的場景（新興市場、行動網路），感知效能比實際效能更重要。骨架屏需要與真實內容的佈局一致（避免 Layout Shift）、圖片使用 BlurHash 作為佔位、關鍵內容優先載入而次要元素漸進呈現。這些策略需要從組件架構層面統一規劃。

跨平台一致性：同一個產品需要在 Web、iOS、Android 上提供一致但不完全相同的體驗。UX 工程需要定義哪些面向跨平台一致（品牌色、間距系統），哪些面向應隨平台慣例（導航模式、手勢互動），並建立對應的 Token 階層（Global → Platform → Component）。
