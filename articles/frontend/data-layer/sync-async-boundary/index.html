同步非同步邊界（Sync-Async Boundary）是前端應用中同步的 UI 渲染邏輯與非同步的資料擷取/更新操作之間的介面。如何優雅地處理這個邊界，決定了應用的使用者體驗、資料一致性與程式碼可維護性。

核心概念

同步與非同步的本質差異

面向同步操作非同步操作
執行方式立即完成、阻塞式延遲完成、非阻塞
結果取得函式回傳值Promise/Callback/Observable
錯誤處理try/catch.catch / async try/catch
狀態確定的（有值或拋錯）不確定的（pending/success/error）
UI 影響立即反映需要載入/錯誤狀態

邊界產生的場景

場景同步側非同步側邊界處理
資料載入UI 渲染API 擷取Loading 狀態
使用者操作事件處理伺服器確認樂觀更新
初始化組件掛載設定載入Suspense/條件渲染
即時資料顯示WebSocket 推送狀態同步
檔案操作觸發上傳/下載進度追蹤
導覽路由切換預載資料路由守衛/Loading

三態模型（Tri-State）

任何跨越同步非同步邊界的操作都必須處理三種狀態：

狀態意義UI 表現
Pending操作進行中載入指示、Skeleton
Fulfilled操作成功顯示資料/成功訊息
Rejected操作失敗錯誤訊息/重試選項

進階模型可能還包含：

額外狀態意義UI 表現
Idle尚未觸發初始 UI
Stale有舊資料但正在更新顯示舊資料 + 更新指示
Refreshing背景重新擷取中可選的更新指示

關鍵比較

各種邊界處理方式

方式程式碼位置使用者體驗複雜度
條件渲染組件內 if/else基本低
Wrapper 組件包裝組件一致中
Suspense父層邊界最佳中
HOC高階組件一致中
Render Props函式子組件彈性中
Server Components伺服器最佳（無 Loading）框架依賴

同步渲染 vs 非同步渲染

面向同步渲染非同步渲染（Concurrent）
阻塞整個渲染不可中斷可中斷、可恢復
優先級全部相同可設定優先級
使用者體驗可能卡頓保持回應性
複雜度低高
代表React 17-, VueReact 18+ Concurrent
Transition不支援useTransition

即時 vs 最終一致性

面向即時一致（同步）最終一致（非同步）
UI 反映操作後立即正確可能暫時不一致
效能可能阻塞不阻塞
實現await 每個操作樂觀更新 + 背景同步
使用者感知較慢但可靠快速但偶爾回滾
適用金融、訂單社交互動、點讚

離線 → 線上同步佇列中的變更、重新驗證快取

Transition API

React 18 的 useTransition 處理邊界的過渡：

面向無 Transition有 Transition
UI 回應可能凍結保持舊 UI 可互動
載入指示立即顯示可延遲顯示
優先級全部一樣標記為低優先級
適用關鍵更新Tab 切換、篩選、導覽

設計原則

原則說明
明確標記邊界讓團隊知道哪裡是同步/非同步的分界
統一處理模式整個應用使用一致的 Loading/Error 模式
最小化邊界盡量減少需要跨邊界的操作
優雅降級非同步失敗時有合理的回退策略
漸進式載入先顯示結構再填入資料
避免瀑布平行擷取而非串行
