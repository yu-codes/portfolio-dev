Vue 生命週期

詳解 Vue 元件從建立到銷毀的完整生命週期，涵蓋各階段鉤子函式的觸發時機、使用場景與常見陷阱

Vue 元件的生命週期描述了元件從建立、掛載、更新到銷毀的完整歷程。每個階段都有對應的鉤子函式（Hook），讓開發者能在適當時機執行初始化、資料載入、DOM 操作與資源清理等工作。Vue 3 在保持生命週期概念的同時，提供了 Composition API 的新寫法。

核心概念

生命週期階段總覽

階段觸發時機可操作範圍
建立（Creation）元件實例初始化響應式資料、注入
掛載（Mounting）首次 DOM 渲染DOM 元素、第三方庫
更新（Updating）響應式資料變化新舊 DOM 狀態
卸載（Unmounting）元件從 DOM 移除清理、取消訂閱
錯誤處理子孫元件錯誤錯誤資訊、恢復

Options API 生命週期鉤子

鉤子階段DOM 可用典型用途
beforeCreate建立前❌插件初始化
created建立後❌API 呼叫、事件監聽
beforeMount掛載前❌最後修改機會
mounted掛載後✅DOM 操作、第三方庫
beforeUpdate更新前✅（舊）讀取更新前 DOM
updated更新後✅（新）存取更新後 DOM
beforeUnmount卸載前✅清理工作
unmounted卸載後❌最終清理
errorCaptured錯誤捕獲-錯誤處理
activatedkeep-alive 啟用✅恢復狀態
deactivatedkeep-alive 停用✅暫存狀態

父子元件生命週期順序

事件執行順序
建立父 beforeCreate → 父 created → 父 beforeMount → 子 beforeCreate → 子 created → 子 beforeMount → 子 mounted → 父 mounted
更新父 beforeUpdate → 子 beforeUpdate → 子 updated → 父 updated
銷毀父 beforeUnmount → 子 beforeUnmount → 子 unmounted → 父 unmounted

函式對應 Options Hook特點

onMounted()mounted可多次呼叫
onBeforeUpdate()beforeUpdate可多次呼叫
onUpdated()updated可多次呼叫
onBeforeUnmount()beforeUnmount可多次呼叫

onActivated()activatedkeep-alive

關鍵比較

各鉤子適用操作

操作類型建議鉤子原因
API 資料取得created / setup無需 DOM，越早越好
DOM 操作mounted確保 DOM 已渲染
第三方庫初始化mounted多數需 DOM 元素
事件監聽mounted + unmounted配對建立/清理
計時器mounted + unmounted配對建立/清理
路由守衛beforeRouteEnter/Leave路由層級
效能觀察mounted + updatedDOM 完成後測量

常見錯誤與修正

錯誤問題描述修正方式
created 操作 DOMDOM 尚未渲染移至 mounted
mounted 中同步修改 state觸發額外渲染使用 nextTick 或移至 created
未清理計時器記憶體洩漏unmounted 中 clearInterval
updated 中修改 state無限迴圈加條件判斷或使用 watch
SSR 中使用 window伺服器無 windowmounted 中或 onMounted

應用場景

第三方圖表庫整合

圖表庫（如 ECharts、D3）需在 DOM 掛載後初始化。標準模式為：mounted 中取得 DOM 元素並建立實例、watch 響應資料變化呼叫 update、beforeUnmount 中呼叫 dispose 釋放資源。搭配 ResizeObserver 處理容器尺寸變化。

keep-alive 快取管理

對於標籤頁或路由快取場景，使用 <KeepAlive> 保留元件狀態。activated 鉤子可在重新顯示時刷新資料，deactivated 鉤子可暫停計時器或動畫。搭配 include/exclude 控制快取範圍。

全域錯誤邊界

透過 onErrorCaptured 建立錯誤邊界元件，捕獲子孫元件的渲染錯誤與非同步錯誤。回傳 false 阻止錯誤向上傳播，搭配 app.config.errorHandler 實現全域錯誤上報機制。
