Vue 的響應式系統是其最核心的設計特徵，讓開發者能以直覺的方式操作資料，框架自動追蹤依賴並高效更新視圖。Vue 3 從 Object.defineProperty 遷移至 Proxy，帶來了更完整的攔截能力與更優異的效能表現。

架構總覽

依賴追蹤流程

  1. 建立副作用：元件渲染函式包裝為 ReactiveEffect
  2. 存取資料（GET）：觸發 track()，將當前 effect 加入依賴集合
  3. 修改資料（SET）：觸發 trigger()，通知所有相關 effect 重新執行
  4. 排程更新：Scheduler 批次處理，避免重複更新

應用場景

全域狀態管理

Vue 的響應式系統可獨立於元件使用。Pinia（官方狀態管理）即建構在 reactive/ref 之上，開發者可自行組合響應式 API 建立 store，享受自動追蹤與細粒度更新的優勢。

表單雙向綁定

v-model 背後是 Vue 響應式的典型應用。對於複雜表單，結合 reactive 建立表單模型、computed 處理衍生驗證狀態、watch 實現跨欄位連動，形成完整的表單響應式架構。

效能敏感場景

對於大型列表或高頻更新場景，可使用 shallowRef / shallowReactive 減少深層追蹤開銷，搭配 triggerRef 手動控制更新時機。虛擬列表（Virtual Scroll）中，僅可見範圍的資料需保持完整響應性。
