Composition API

全面解析 Vue 3 Composition API 的設計動機、核心模式與實戰技巧，探索邏輯組合與程式碼組織的新範式

Composition API 是 Vue 3 最重要的新特性，提供了一套以函式為基礎的 API，讓開發者能依邏輯關注點組織程式碼，而非被迫按選項類型拆分。它解決了 Options API 在大型元件中邏輯分散的問題，同時提供了更強大的 TypeScript 支援與邏輯複用能力。

型別推導this 型別推導困難純函式完整型別支援

應用場景

大型元件重構

當 Options API 元件超過 500 行時，相關邏輯可能分散在 data、computed、methods、watch 各處。透過 Composition API 重構，將搜尋邏輯、分頁邏輯、表單驗證邏輯各自提取為 composable，元件僅負責組合與渲染。

跨框架邏輯共享

@vue/reactivity 可獨立使用，composable 中的純邏輯部分可抽離為框架無關的函式。這使得核心業務邏輯可在 Vue、React（透過轉譯）甚至 Node.js 環境中複用。

漸進式遷移策略

Vue 3 支援在同一元件中混用 Options API 與 Composition API。團隊可在新功能中使用 Composition API，既有功能保持 Options API，逐步遷移而非一次性重寫。
