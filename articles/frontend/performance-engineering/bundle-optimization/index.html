前端打包優化是效能工程的起點——減少使用者需下載的程式碼量直接改善載入速度。從 Tree Shaking 移除無用程式碼，到 Gzip/Brotli 壓縮減少傳輸大小，再到智慧分割策略避免重複載入，每個環節都能顯著影響 Web Vitals 指標。

應用場景

CI/CD Bundle 大小監控

在 CI 流程中加入 Bundle 大小檢查：每次 PR 自動分析產出大小變化，超過閾值則阻止合併。工具如 bundlesize、size-limit 可設定各 chunk 的大小上限，防止無意間引入大型依賴。

漸進式最佳化

既有大型應用的優化路徑：先用分析工具找出最大的 chunk → 識別可 Tree Shake 的依賴 → 替換大型庫 → 實施路由級程式碼分割 → 優化共用 chunk 策略。每步都可量化效果。

多頁面共用最佳化

多入口應用（如管理後台 + 前台）共用框架程式碼。透過 splitChunks 配置提取共用的 React/Vue 核心為獨立 chunk，搭配 CDN 快取策略，使用者在不同頁面間切換時不需重複下載框架程式碼。
