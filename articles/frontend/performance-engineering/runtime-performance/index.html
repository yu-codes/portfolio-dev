執行期效能（Runtime Performance）關注的是應用載入後的流暢度——動畫是否卡頓、互動是否靈敏、捲動是否平滑。這涉及瀏覽器渲染管線的理解、JavaScript 執行效率的優化、記憶體使用的控制與主執行緒負載的管理。

應用場景

大型資料表格

顯示 10,000+ 行資料時，DOM 節點數直接影響記憶體與渲染效能。使用虛擬列表（如 @tanstack/virtual）僅渲染可見的 20-30 行，搭配 CSS transform 實現平滑捲動。即使數據量達百萬行，頁面仍保持 60fps 捲動。

即時圖表更新

股票行情或 IoT 感測器儀表板需高頻更新圖表。關鍵策略：使用 Canvas 而非 SVG 渲染（避免大量 DOM 操作）、requestAnimationFrame 控制更新頻率、Web Worker 處理資料計算，確保主執行緒專注渲染。

拖放排序介面

拖放操作期間需即時更新元素位置。使用 transform 而非 top/left（避免 Layout 觸發）、pointer events 而非 mouse events（觸控相容）、requestAnimationFrame 節流位置更新，確保拖放過程流暢無卡頓。
