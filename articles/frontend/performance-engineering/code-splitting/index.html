程式碼分割（Code Splitting）是將 JavaScript Bundle 拆分為多個較小區塊（chunks），在需要時才載入的技術。這直接減少了初始載入量，讓使用者更快看到首屏內容並開始互動。現代打包工具讓程式碼分割的實作變得簡單，但策略設計仍需開發者的架構思維。

應用場景

大型 SaaS 儀表板

儀表板包含十幾個功能模組（報表、設定、分析等），使用者單次只使用其中一兩個。路由級分割確保每個模組獨立載入，搭配 prefetch 預載入側邊欄中可見的模組連結。結果：初始載入從 2MB 降至 200KB。

條件功能載入

管理員面板、進階編輯器、除錯工具等僅特定使用者/條件需要的功能，使用動態 import 在條件滿足時才載入。避免普通使用者下載永遠不會使用的程式碼。

第三方庫延遲載入

重量級庫（圖表庫、PDF 渲染器、地圖 SDK）在使用者實際需要時才載入。例如：使用者點擊「產生報表」時才 import Chart.js，點擊「匯出 PDF」時才載入 PDF 生成庫。
