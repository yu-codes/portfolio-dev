前端工程是軟體開發中最貼近使用者的領域，負責將設計意圖轉化為可互動的數位體驗。從早期的靜態 HTML 頁面，到現今的單頁應用（SPA）、伺服器端渲染（SSR）與邊緣運算（Edge Computing），前端技術棧的深度與廣度已遠超想像。本文將系統性地梳理現代前端工程的知識體系，幫助開發者建立完整的技術視野。

核心概念

前端工程的三大支柱

現代前端工程建立在三個核心支柱之上：

  1. Web 標準（Web Standards）：HTML、CSS、JavaScript 以及不斷演進的 Web API（如 Web Components、Service Worker、WebAssembly）
  2. 框架生態（Framework Ecosystem）：React、Vue、Angular、Svelte 等框架提供了元件化、狀態管理與路由等抽象層
  3. 工程化實踐（Engineering Practices）：建置工具鏈、測試策略、CI/CD、效能監控與可觀測性

宣告式 vs 命令式

前端開發的範式轉變中，最重要的是從命令式走向宣告式：

  • 命令式（Imperative）：直接操作 DOM，告訴瀏覽器「如何做」。如 document.createElement()
  • 宣告式（Declarative）：描述 UI 應該長什麼樣子，框架負責更新。如 JSX、Vue Template

宣告式的優勢在於：程式碼更易理解、更好維護、框架可以進行更多最佳化。

元件化思維

現代前端的核心思維是將 UI 拆解為獨立、可重用的元件：

  • 關注點分離：每個元件封裝自己的結構（HTML）、樣式（CSS）與行為（JS）
  • 單向資料流：父元件透過 props 傳遞資料給子元件，子元件透過事件向上溝通
  • 組合優於繼承：透過元件組合建構複雜介面，而非使用繼承階層

應用場景

場景一：大型企業應用

需要嚴謹的架構規範、完善的測試策略、微前端（Micro Frontends）拆分、Design System 標準化。技術選型偏向 React + TypeScript + Next.js，搭配 Module Federation 實現團隊獨立部署。

場景二：內容導向網站

部落格、文件站、行銷頁面等 SEO 導向的內容網站。優先選擇 SSG（如 Astro、Next.js Static Export）或 ISR 策略，搭配 CMS（如 Contentful、Sanity）管理內容，確保最佳的首屏載入效能。

場景三：即時互動應用

聊天應用、協作工具、遊戲等需要 WebSocket/WebRTC 的即時通訊場景。關注狀態同步、衝突解決（CRDT）、離線支援與漸進式載入策略。

場景四：跨平台開發

一套程式碼同時覆蓋 Web、iOS、Android。可選方案包括 React Native（行動端）、Electron/Tauri（桌面端）、PWA（漸進式 Web 應用），各有其效能與體驗的取捨。
