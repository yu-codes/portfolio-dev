Web 平台 API 是瀏覽器提供給開發者的一系列程式介面，讓網頁應用能夠存取裝置功能、實現離線體驗、處理背景任務，甚至建構接近原生應用程式的使用者體驗。隨著 Web 標準的持續演進，現代瀏覽器已經支援數百個 API，從基礎的 DOM 操作到進階的檔案系統存取、推送通知、藍牙連接等，Web 平台的能力邊界正不斷擴展。

核心概念

Service Workers

Service Worker 是在瀏覽器背景執行的 JavaScript 腳本，獨立於網頁主執行緒，充當網頁應用與網路之間的代理伺服器。它是實現離線功能、推送通知、背景同步等 PWA 核心能力的基礎：

// 註冊 Service Worker
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js')
    .then(registration => {
      console.log('SW registered:', registration.scope);
    });
}

// sw.js - 快取策略
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(cached => cached || fetch(event.request))
  );
});

Service Worker 的生命週期包含：安裝（install）、等待（waiting）、啟動（activate）三個階段。理解這個生命週期對於實現可靠的快取更新策略至關重要。

Web Components

Web Components 是一組瀏覽器原生技術，讓開發者能夠建立可重用的自定義 HTML 元素。它由三個核心規範組成：

  • Custom Elements：定義全新的 HTML 標籤及其行為
  • Shadow DOM：提供樣式與 DOM 的封裝隔離
  • HTML Templates：宣告可重用的 DOM 片段
class MyCard extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        :host { display: block; border: 1px solid #ddd; padding: 1rem; }
        ::slotted(h2) { color: #333; }
      </style>
      <slot name="title"></slot>
      <slot></slot>
    `;
  }

  connectedCallback() {
    // 元素被加入 DOM 時觸發
  }
}

customElements.define('my-card', MyCard);

Web Components 的最大優勢在於框架無關性——無論使用 React、Vue 或純 HTML，都能直接使用自定義元素。

瀏覽器 API 生態系

現代瀏覽器提供了豐富的 API 供開發者使用：

儲存類 API：

  • localStorage / sessionStorage：同步鍵值對儲存
  • IndexedDB：大量結構化資料的非同步儲存
  • Cache API：HTTP 回應的程式化快取管理
  • File System Access API：直接讀寫使用者的本地檔案

通訊類 API：

  • Fetch API：現代化的 HTTP 請求介面
  • WebSocket：全雙工即時通訊
  • WebRTC：點對點音視訊通訊
  • Broadcast Channel：同源分頁間通訊

裝置類 API：

  • Geolocation：地理位置資訊
  • Web Bluetooth：藍牙裝置連接
  • Web USB：USB 裝置存取
  • Notification API：系統通知推送

Progressive Web Apps（PWA）

PWA 是一種利用現代 Web 技術建構的應用程式模式，結合了網頁的可及性與原生應用的使用體驗。PWA 的三大支柱：

  1. 可靠性：透過 Service Worker 實現離線可用
  2. 效能：預快取關鍵資源，即時回應使用者操作
  3. 可安裝性：透過 Web App Manifest 支援「新增至主畫面」
{
  "name": "My PWA App",
  "short_name": "MyApp",
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#4A90D9",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}

快取策略比較

策略說明適用場景離線支援
Cache First優先從快取讀取靜態資源（CSS、JS、圖片）完整
Network First優先發送網路請求API 回應、即時資料有（使用舊快取）
Stale While Revalidate回傳快取同時更新非關鍵性即時資料有
Network Only僅使用網路敏感操作（支付、認證）無
Cache Only僅使用快取預先載入的離線頁面完整

應用場景

離線優先新聞應用：利用 Service Worker 實現 Cache First 策略，預快取文章列表與內容，使用者在無網路環境下依然可以閱讀已載入的新聞。當網路恢復時，透過 Background Sync 同步使用者的閱讀記錄與書籤。

即時協作編輯器：結合 WebSocket 實現多人即時協作，使用 SharedWorker 在多個瀏覽器分頁間共享連線狀態，並以 IndexedDB 本地儲存編輯歷史以支援離線編輯。

可重用元件庫：以 Web Components 建構跨框架的 UI 元件庫，透過 Shadow DOM 確保樣式隔離，使用 Slots 提供靈活的內容插入點，讓組件可在 React、Vue、Angular 或純 HTML 專案中通用。

地理位置追蹤應用：使用 Geolocation API 持續追蹤使用者位置，搭配 Notification API 在使用者接近特定地標時發送提醒，並透過 Service Worker 在背景持續運作。

高效能資料視覺化：將繁重的資料計算任務委託給 Web Worker，避免阻塞主執行緒的 UI 渲染，再透過 postMessage 將計算結果傳回主執行緒進行 Canvas 或 SVG 繪製。
