Web Components 是一組瀏覽器原生的 API 標準，讓開發者能夠建立可重用、封裝良好的自訂 HTML 元素。它由 Custom Elements、Shadow DOM 和 HTML Templates 三大核心技術組成，提供了框架無關的組件化解決方案。

核心概念

三大核心技術

技術功能解決的問題
Custom Elements定義自訂 HTML 標籤語意化、行為封裝
Shadow DOM隔離的 DOM 與樣式樣式衝突、DOM 污染
HTML Templates可複用的 HTML 片段模板定義、延遲渲染

Custom Elements

概念說明
自主元素（Autonomous）繼承自 HTMLElement，完全自訂
內建元素擴展繼承自特定元素如 HTMLButtonElement
命名規則必須包含連字號（如 my-component）
註冊方式customElements.define('my-comp', MyComp)

生命週期回呼

回呼觸發時機用途
constructor()元素建立時初始化狀態、附加 Shadow DOM
connectedCallback()插入 DOM 時設定事件監聽、開始觀察
disconnectedCallback()從 DOM 移除時清理資源、移除監聽
attributeChangedCallback()觀察的屬性變更時回應屬性變化
adoptedCallback()移至新文件時罕見，處理跨文件

Shadow DOM 模式

模式說明外部存取
open可透過 element.shadowRoot 存取是
closed無法從外部存取否

架構總覽

Shadow DOM 結構

<my-component>
  #shadow-root (open)
  ├── <style>...</style>     ← 隔離的樣式
  ├── <slot name="header">   ← 具名插槽
  │   └── (分配的節點)
  ├── <div class="content">
  │   └── <slot>             ← 預設插槽
  └── <slot name="footer">
  
  <!-- Light DOM（使用者提供的內容） -->
  <h1 slot="header">標題</h1>
  <p>正文內容</p>
  <footer slot="footer">底部</footer>
</my-component>

樣式封裝機制

機制說明用途
Shadow DOM 邊界內外樣式互不影響避免全域污染
:host選取組件自身設定組件根樣式
:host()條件性選取 host依據 host 屬性/類別
::slotted()選取分配到 slot 的內容有限的投影內容樣式
:host-context()根據祖先環境主題切換
CSS Custom Properties穿透 Shadow 邊界可設定的主題值
::part()外部樣式化內部元素暴露可自訂的部件

事件傳播

事件行為說明
組合（Composed）composed: true 的事件可穿越 Shadow 邊界
重新定向（Retargeting）事件 target 在外部看到的是 host 元素
冒泡Shadow 內部正常冒泡，穿越邊界後重新定向
自訂事件需要明確設定 composed: true, bubbles: true

關鍵比較

Web Components vs 框架組件

面向Web ComponentsReact/Vue/Angular 組件
標準瀏覽器原生標準框架特定
依賴無需框架需要框架執行時期
樣式隔離原生 Shadow DOMCSS Modules/Scoped/CSS-in-JS
資料流屬性 + 事件Props + State + Context
伺服器渲染Declarative Shadow DOM框架 SSR
生態系統較小豐富
學習曲線原生 API 較繁瑣框架抽象較簡潔
效能無框架開銷虛擬 DOM/編譯器開銷

Shadow DOM vs iframe vs CSS 命名空間

面向Shadow DOMiframeBEM/CSS Modules
隔離程度樣式 + DOM 結構完全隔離僅命名避免衝突
效能輕量重（獨立文件環境）輕量
通訊自訂事件、屬性postMessage直接存取
SEO支援（Light DOM）不利正常
可及性正常支援較困難正常

應用場景

設計系統

Web Components 適合建構跨框架的設計系統：

優勢說明
框架無關React、Vue、Angular 都能使用
版本獨立不受框架升級影響
團隊自治不同團隊可用不同框架
長期穩定基於瀏覽器標準

微前端整合

作為微前端架構的封裝邊界：

用途說明
樣式隔離各微應用不互相污染
生命週期管理connectedCallback/disconnectedCallback
通訊界面屬性 + 自訂事件
漸進增強未註冊時優雅降級

現有框架整合

框架使用 Web Components建構 Web Components
React需要包裝器處理事件可用 React 渲染至 Shadow DOM
Vue原生良好支援defineCustomElement()
Angular原生良好支援Angular Elements
Svelte原生支援customElement: true 選項
Lit—專為 WC 設計的框架

效能注意事項

問題原因解決方案
大量元素效能每個都有獨立 Shadow Root虛擬化列表
首次渲染JS 載入後才渲染Declarative Shadow DOM
記憶體使用每個 Shadow DOM 有獨立樣式使用 adoptedStyleSheets 共享
可及性Shadow DOM 與 ARIA正確設定 role、aria-*
