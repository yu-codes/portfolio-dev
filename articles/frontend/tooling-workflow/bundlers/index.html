前端打包工具（Bundlers）是現代前端工程的核心基礎設施。它們將分散的模組、資源檔案整合為瀏覽器可高效載入的最佳化產物。從 Webpack 開創的模組打包時代，到 Vite 引領的原生 ESM 開發體驗，再到 esbuild 展示的極速建構能力，打包工具的演進深刻影響著前端開發的每一個環節。

核心概念

打包工具的核心職責包含以下幾個面向：

模組解析（Module Resolution）：從入口檔案出發，遞迴解析所有 import/require 語句，建構完整的依賴圖（Dependency Graph）。需處理各種模組格式（ESM、CJS、UMD）與路徑別名。

程式碼轉譯（Transpilation）：將 TypeScript、JSX、現代 JavaScript 語法轉譯為目標瀏覽器可執行的程式碼。通常委託 Babel、SWC 或 esbuild 的轉譯器完成。

Tree Shaking：靜態分析 ESM 的 import/export，移除未被引用的程式碼（Dead Code Elimination）。這依賴 ESM 的靜態結構特性，CommonJS 模組因動態特性難以完美 Tree Shake。

Code Splitting：將程式碼按路由或功能邊界分割為多個 Chunk，實現按需載入。動態 import() 是分割點的主要標記方式。

資源處理：CSS、圖片、字型等非 JavaScript 資源也需經過處理——CSS Modules 編譯、圖片壓縮與 Base64 內聯、字型子集化等。

最佳化：程式碼壓縮（Minification）、作用域提升（Scope Hoisting）、持久化快取（Content Hash）等生產環境最佳化。

架構總覽

三大主流打包工具的架構差異：

Webpack 採用全量打包策略——無論開發或生產，都將所有模組打包為 Bundle：

所有模組 → Loader 轉譯 → 依賴圖 → Chunk 分割 → Plugin 處理 → Bundle

Webpack 的強大在於 Loader 與 Plugin 系統的無限擴展性，但代價是設定複雜度高且冷啟動慢。

Vite 在開發階段使用原生 ESM + 按需編譯，生產環境使用 Rollup 打包：

開發：瀏覽器請求模組 → Vite 攔截 → 即時轉譯 → 回傳 ESM
生產：Rollup 全量打包 → Tree Shaking → Code Split → 最佳化

Vite 的核心創新是利用瀏覽器原生 ESM 支持，省去開發時的打包步驟，實現毫秒級 HMR。依賴預打包（Pre-bundling）使用 esbuild 處理 node_modules。

esbuild 使用 Go 語言撰寫，多執行緒平行處理，追求極致速度：

所有模組 → Go 平行解析/轉譯/打包 → 產物

esbuild 速度比 Webpack 快 10-100 倍，但功能較精簡，常作為其他工具的底層引擎（如 Vite 的依賴預打包、tsup 的函式庫建構）。

新興打包工具：

  • Turbopack：Webpack 作者以 Rust 重寫，Next.js 14+ 預設 Dev Server
  • Rolldown：Vite 團隊以 Rust 重寫 Rollup，未來將統一 Vite 的開發/生產建構
  • Rspack：字節跳動以 Rust 重寫 Webpack，API 相容 Webpack

應用場景

場景一：新建 React/Vue 專案。使用 Vite 作為建構工具，享受即時啟動的 Dev Server 與快速 HMR。vite.config.ts 簡潔明瞭，大多數場景零設定即可運作。搭配 vite-plugin-compression 啟用 Brotli 壓縮。

場景二：大型企業 Legacy 專案。繼續使用 Webpack 5，利用 Module Federation 實現微前端架構。透過 webpack-bundle-analyzer 分析 Chunk 體積，設定 splitChunks 最佳化快取策略。

場景三：npm 函式庫發布。使用 tsup（基於 esbuild）快速建構，同時產出 ESM 與 CJS 格式。配置 package.json 的 exports 欄位實現條件導出，支持 Tree Shaking。

場景四：Monorepo 開發體驗最佳化。使用 Vite 搭配 vite-plugin-dts 為共用套件產生型別宣告。透過 pnpm workspace 的 link: 實現套件間即時連結，修改共用元件後主應用 HMR 即時更新。
