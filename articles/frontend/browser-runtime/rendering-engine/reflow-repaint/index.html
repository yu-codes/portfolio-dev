重排與重繪

重排（Reflow）與重繪（Repaint）是瀏覽器渲染過程中最關鍵的效能瓶頸。當 DOM 或樣式發生變化時，瀏覽器需要重新計算元素的幾何資訊（重排）或重新繪製元素的外觀（重繪）。過度的重排與重繪是導致頁面卡頓、動畫不流暢的首要原因。

核心概念

重排（Reflow）

重排是指瀏覽器重新計算元素的位置和幾何尺寸的過程。由於 DOM 元素之間存在相互影響的佈局關係，一個元素的變更可能導致其父元素、兄弟元素甚至整個文件的重新佈局。

重排的特性：

  • 計算量大，涉及整棵渲染樹或其子樹
  • 必然伴隨重繪（重新計算位置後需重新繪製）
  • 是渲染管線中最昂貴的操作之一
  • 可能被 JavaScript 同步觸發（強制同步佈局）

觸發重排的常見操作：

  • 新增、刪除或修改可見 DOM 元素
  • 元素的位置或尺寸改變（margin、padding、border、width、height）
  • 頁面初始渲染
  • 瀏覽器視窗大小變更（resize 事件）
  • 字型大小變更
  • CSS 偽類觸發（如 :hover）
  • 讀取特定屬性（offsetTop、scrollHeight、getComputedStyle()）

重繪（Repaint）

重繪是指元素的外觀發生變化但幾何資訊不變時，瀏覽器重新繪製該元素的過程。重繪的成本低於重排，因為不需要重新計算佈局。

僅觸發重繪的操作：

  • 修改顏色（color、background-color）
  • 修改可見性（visibility: hidden，注意 display: none 觸發重排）
  • 修改文字裝飾（text-decoration）
  • 修改背景圖片
  • 修改邊框顏色
  • 修改陰影（box-shadow）
  • 修改輪廓（outline）

強制同步佈局（Forced Synchronous Layout）

當 JavaScript 修改了 DOM 的幾何樣式後，立即讀取佈局相關的屬性，瀏覽器被迫中斷正常的批次處理流程，立即執行佈局計算以返回最新值。這種模式稱為強制同步佈局，是效能殺手。

// 強制同步佈局範例
element.style.width = '100px';     // 標記佈局為 dirty
const height = element.offsetHeight; // 強制立即重排以取得最新值

佈局抖動（Layout Thrashing）

在迴圈中反覆讀寫佈局屬性，導致每次迭代都觸發強制同步佈局，造成嚴重的效能問題：

// 佈局抖動：N 次迭代 = N 次強制重排
for (let i = 0; i < items.length; i++) {
  items[i].style.height = container.offsetHeight + 'px'; // 每次都觸發重排
}

瀏覽器的批次優化

現代瀏覽器會將多個 DOM 修改批次處理，在下一個畫格（frame）統一執行佈局和繪製。但讀取佈局屬性會打破此機制：

會強制刷新佈局佇列的屬性：

  • offsetTop / offsetLeft / offsetWidth / offsetHeight
  • scrollTop / scrollLeft / scrollWidth / scrollHeight
  • clientTop / clientLeft / clientWidth / clientHeight
  • getComputedStyle()
  • getBoundingClientRect()

應用場景

場景一：批量 DOM 操作

// 差：多次重排
const list = document.getElementById('list');
for (let i = 0; i < 1000; i++) {
  list.innerHTML += `<li>Item ${i}</li>`; // 每次觸發重排
}

// 好：使用 DocumentFragment，僅一次重排
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}
list.appendChild(fragment);

場景二：動畫效能優化

// 差：使用 left/top 觸發重排
function animateBad(element) {
  let pos = 0;
  setInterval(() => {
    pos += 1;
    element.style.left = pos + 'px'; // 觸發重排
  }, 16);
}

// 好：使用 transform 僅觸發合成
function animateGood(element) {
  let pos = 0;
  function frame() {
    pos += 1;
    element.style.transform = `translateX(${pos}px)`; // 僅合成
    requestAnimationFrame(frame);
  }
  requestAnimationFrame(frame);
}

場景三：樣式批量修改

// 差：逐一修改樣式（可能多次重排）
element.style.width = '100px';
element.style.height = '200px';
element.style.margin = '10px';

// 好：使用 cssText 或 className 批量修改
element.style.cssText = 'width:100px; height:200px; margin:10px;';
// 或
element.classList.add('new-style');

場景四：使用 fastdom 防止佈局抖動

import fastdom from 'fastdom';

// 自動排程讀寫操作，避免佈局抖動
items.forEach(item => {
  fastdom.measure(() => {
    const height = item.offsetHeight;
    fastdom.mutate(() => {
      item.style.height = height * 2 + 'px';
    });
  });
});
