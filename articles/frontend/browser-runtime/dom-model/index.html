DOM 模型

DOM 操作與事件系統

DOM（Document Object Model）是瀏覽器將 HTML 文件轉換為程式可操作的樹狀結構。它是 JavaScript 與網頁內容互動的橋樑，也是所有前端框架底層運作的基礎。理解 DOM 的結構、操作方式與事件系統，是掌握前端開發的必備知識。即使在框架盛行的今天，了解原生 DOM API 仍有助於除錯、效能最佳化與理解框架的設計決策。

核心概念

DOM 樹的節點類型

DOM 將 HTML 文件表達為一棵節點樹，主要節點類型包括：

  • Document：整棵樹的根節點，代表整個文件
  • Element：HTML 標籤對應的節點，如 <div>、<p>、<span>
  • Text：元素內的文字內容
  • Attribute：元素的屬性（在 DOM 中作為元素的特性存在）
  • Comment：HTML 註解節點
  • DocumentFragment：輕量級容器，用於批次 DOM 操作

每個節點都實作了 Node 介面，擁有 parentNode、childNodes、nextSibling 等屬性，允許程式碼遍歷整棵樹。

DOM 操作的效能代價

DOM 操作是前端效能的主要瓶頸之一，原因在於：

  1. 跨界通訊成本：JavaScript 引擎與渲染引擎是分離的，每次 DOM 操作都需要跨越兩者之間的邊界
  2. 重排（Reflow）觸發：修改元素的幾何屬性會導致瀏覽器重新計算佈局
  3. 重繪（Repaint）觸發：修改外觀屬性會導致重新繪製
  4. DOM 查詢的開銷：querySelector 等方法需要遍歷 DOM 樹

最佳化策略：批次修改、使用 DocumentFragment、避免讀寫交錯、善用 CSS class 切換。

事件傳播模型

DOM 事件的傳播分為三個階段：

  1. 捕獲階段（Capture Phase）：事件從 window 向下傳播到目標元素的父節點
  2. 目標階段（Target Phase）：事件到達觸發事件的元素本身
  3. 冒泡階段（Bubble Phase）：事件從目標元素向上冒泡回 window

大多數事件預設在冒泡階段被監聽，但可透過 addEventListener 的第三個參數 { capture: true } 切換到捕獲階段。

應用場景

場景一：高效能列表渲染

當需要渲染數千個列表項目時，逐一建立 DOM 節點效率極低。最佳做法是使用 DocumentFragment 批次建立所有節點，最後一次性插入 DOM。或採用虛擬捲動（Virtual Scrolling），僅渲染可視範圍內的項目。

const fragment = document.createDocumentFragment();
items.forEach(item => {
  const li = document.createElement('li');
  li.textContent = item.name;
  fragment.appendChild(li);
});
list.appendChild(fragment); // 僅觸發一次 Reflow

場景二：事件委派實現動態列表

在 Todo List 等動態增刪項目的場景中，使用事件委派可避免為每個新增項目綁定事件：

document.querySelector('.todo-list').addEventListener('click', (e) => {
  if (e.target.matches('.delete-btn')) {
    e.target.closest('.todo-item').remove();
  }
  if (e.target.matches('.toggle-btn')) {
    e.target.closest('.todo-item').classList.toggle('completed');
  }
});

場景三：MutationObserver 監控 DOM 變化

當第三方腳本或框架修改 DOM 時，可透過 MutationObserver 非同步監控特定節點的變化，用於：分析工具追蹤 DOM 變更、懶載入元素的偵測、無障礙（Accessibility）輔助功能。

場景四：自訂元素與 Shadow DOM

Web Components 透過 Custom Elements 與 Shadow DOM 實現真正的元件封裝。Shadow DOM 建立獨立的 DOM 子樹，外部的 CSS 和 JavaScript 無法穿透，實現了樣式與行為的完全隔離。
