IndexedDB 是瀏覽器內建的大型結構化資料儲存系統，提供了一個完整的客戶端資料庫解決方案。與 LocalStorage 的簡單鍵值對不同，IndexedDB 支援索引、交易、游標等資料庫特性，能夠儲存大量結構化資料，是建構離線優先（Offline-First）應用的關鍵技術。

核心概念

資料庫結構

IndexedDB 採用階層式結構組織資料：

  • Database：最頂層容器，一個來源可擁有多個資料庫
  • Object Store：類似關聯式資料庫的「表」，儲存記錄的集合
  • Index：為 Object Store 中的特定屬性建立索引，加速查詢
  • Record：實際的資料記錄，以鍵值對形式存在

交易模型（Transaction）

IndexedDB 所有操作都必須在交易中進行：

  • readonly：只讀交易，可以並行執行多個
  • readwrite：讀寫交易，同一時間只能有一個作用於相同的 Object Store
  • versionchange：升級交易，用於修改資料庫結構

交易具有原子性：要麼全部成功，要麼全部回滾。

非同步 API

IndexedDB 完全基於事件驅動的非同步 API：

const request = indexedDB.open('myDB', 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  const store = db.createObjectStore('users', { keyPath: 'id' });
  store.createIndex('email', 'email', { unique: true });
};

request.onsuccess = (event) => {
  const db = event.target.result;
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  store.add({ id: 1, name: 'Alice', email: 'alice@example.com' });
};

版本管理

IndexedDB 使用整數版本號管理結構變更：

  • 開啟資料庫時指定版本號
  • 若版本高於現有版本，觸發 onupgradeneeded 事件
  • 在升級事件中執行結構遷移（建立/刪除 Object Store、Index）
  • 不支援向下降級

離線優先架構

IndexedDB 在離線優先應用中的典型位置：

  • Service Worker：攔截網路請求，決定使用快取或網路
  • IndexedDB：儲存應用資料和待同步的變更
  • Sync Manager：當網路恢復時，將本地變更同步到伺服器
  • Conflict Resolver：處理同步時的衝突

資料同步模式

線上模式：App → IndexedDB → Server（即時同步）
離線模式：App → IndexedDB（本地暫存）
恢復連線：IndexedDB → Sync Queue → Server（批次同步）

應用場景

離線優先 Web 應用

將完整應用資料儲存在 IndexedDB：

  • 郵件客戶端的離線讀取
  • 筆記應用的本地儲存
  • 地圖應用的離線地圖資料

大型資料集快取

避免反覆下載大量資料：

  • 電商產品目錄的本地快取
  • 搜尋建議的離線索引
  • 圖片和媒體的 Blob 儲存

漸進式 Web 應用（PWA）

PWA 的核心儲存層：

  • 應用資料的持久化儲存
  • 背景同步的待處理佇列
  • 推播通知的本地歷史記錄

資料密集型前端應用

需要複雜查詢能力的場景：

  • 資料視覺化工具的本地資料處理
  • 表格/試算表應用的大量資料管理
  • 日誌分析工具的本地查詢引擎
