JavaScript 採用基於原型（Prototype-based）的物件導向模型，與傳統基於類別（Class-based）的語言有著根本性的差異。理解原型鏈是掌握 JavaScript 物件系統的關鍵，即使 ES6 的 class 語法也只是原型機制的語法糖。

核心概念

原型（Prototype）

每個 JavaScript 物件都有一個內部屬性 [[Prototype]]，指向另一個物件或 null。這個指向形成了原型鏈。

概念說明存取方式
[[Prototype]]物件的內部原型參照Object.getPrototypeOf(obj)
__proto__非標準存取器（已棄用）obj.__proto__
prototype建構函式的原型物件Constructor.prototype
原型鏈終點Object.prototype.__proto__null

原型鏈（Prototype Chain）

屬性查找沿著原型鏈進行：

myObj → MyClass.prototype → Object.prototype → null
操作行為是否遍歷原型鏈
屬性讀取沿鏈向上查找是
屬性寫入只在自身物件上操作否（有例外）
in 運算子檢查自身與原型鏈是
hasOwnProperty只檢查自身否
delete只刪除自身屬性否

建構函式與 new 運算子

new 運算子執行以下步驟：

  1. 建立空物件
  2. 將物件的 [[Prototype]] 設為建構函式的 prototype
  3. 以該物件為 this 執行建構函式
  4. 若建構函式未回傳物件，則回傳步驟 1 的物件

Class 語法結構

class Animal
├── constructor()
├── 實例方法 → Animal.prototype
├── static 方法 → Animal 本身
├── getter/setter → Animal.prototype（描述符）
├── #private 欄位 → 實例內部槽位
└── extends → 設定原型鏈
    └── super → 呼叫父類別

應用場景

Mixin 模式

在 JavaScript 中實現功能混入：

const Serializable = (Base) => class extends Base {
  serialize() { return JSON.stringify(this); }
};

const Validatable = (Base) => class extends Base {
  validate() { /* ... */ }
};

class Model extends Serializable(Validatable(BaseModel)) {}

設計模式應用

  • 工廠模式：隱藏建構細節
  • 單例模式：使用 Class + static
  • 觀察者模式：事件發射器繼承
  • 策略模式：利用多型替換演算法

效能考量

  • 過深的原型鏈影響屬性查找效能
  • 避免在熱路徑中修改原型（導致隱藏類別失效）
  • Object.create(null) 建立無原型字典物件，適合用作 Map
