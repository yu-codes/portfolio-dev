JavaScript 的執行模型是理解非同步程式設計的關鍵。單執行緒的設計搭配事件迴圈機制，讓 JavaScript 能在不阻塞使用者介面的情況下處理大量 I/O 操作。理解這個模型是撰寫正確非同步程式碼、診斷效能瓶頸的基礎。

核心概念

事件迴圈（Event Loop） 是 JavaScript 並行模型的核心機制。它不斷地從任務佇列中取出任務並執行，形成一個無限迴圈。其運作可以簡化為：

  1. 執行呼叫堆疊（Call Stack）中的同步程式碼直到清空
  2. 清空所有微任務佇列（Microtask Queue）中的任務
  3. 若有必要，執行渲染更新（Rendering）
  4. 從巨集任務佇列（Macrotask Queue）中取出一個任務執行
  5. 回到步驟 2

微任務（Microtask） 與 巨集任務（Macrotask） 的區分至關重要：

  • 微任務：Promise.then/catch/finally、queueMicrotask、MutationObserver
  • 巨集任務：setTimeout、setInterval、setImmediate、I/O、UI Rendering

微任務在當前任務結束後立即執行，且會一直清空直到佇列為空。這意味著遞迴的微任務可能阻塞渲染。

Promise 是對非同步操作結果的抽象表示。一個 Promise 處於三種狀態之一：pending（等待中）、fulfilled（已完成）、rejected（已拒絕）。Promise 的狀態一旦確定便不可逆轉。

async/await 是 Promise 的語法糖，讓非同步程式碼看起來像同步程式碼。async 函式總是回傳 Promise，await 暫停函式執行直到 Promise 解決。在底層，async/await 透過生成器（Generator）實現狀態機式的暫停與恢復。

Web Workers 提供了真正的多執行緒計算能力。Worker 在獨立的執行緒中運行，擁有自己的事件迴圈，透過 postMessage 與主執行緒通訊。

Promise 組合模式：

  • Promise.all()：全部成功才成功，一個失敗即失敗
  • Promise.allSettled()：等待全部完成，不論成功或失敗
  • Promise.race()：取第一個完成的結果
  • Promise.any()：取第一個成功的結果

應用場景

高效能資料處理：當需要在前端處理大量資料（如 CSV 解析、影像處理）時，將計算密集型任務移至 Web Worker 可避免阻塞主執行緒。搭配 Transferable Objects 可實現零拷貝的大型資料傳遞。

並行 API 請求：使用 Promise.all() 並行發送多個獨立的 API 請求，而非逐一 await。搭配 Promise.allSettled() 可以優雅地處理部分失敗的情境。

競態條件防護：在搜尋建議（autocomplete）場景中，使用 AbortController 取消過時的請求，確保只有最新的查詢結果被顯示。這是前端最常見的競態條件問題。

漸進式載入：利用 async/await 配合動態 import()，實現「用到才載入」的元件懶載入模式。搭配 Suspense（React）或 defineAsyncComponent（Vue）提供優雅的載入狀態。

離線優先應用：Service Worker 攔截網路請求，優先從快取提供資源。搭配 Background Sync API，在網路恢復時自動同步離線期間的操作。
