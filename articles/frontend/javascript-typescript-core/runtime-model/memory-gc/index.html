記憶體管理與 GC

JavaScript 使用自動記憶體管理，開發者無需手動分配與釋放記憶體。然而，理解垃圾回收（Garbage Collection）機制對於撰寫高效能程式碼、避免記憶體洩漏至關重要。

核心概念

記憶體生命週期

階段動作JavaScript 中的表現
分配（Allocation）為值配置記憶體空間宣告變數、建立物件、呼叫函式
使用（Usage）讀寫已分配的記憶體存取變數、修改屬性
釋放（Deallocation）不再需要時回收由 GC 自動執行

記憶體空間分類

空間儲存內容特性管理方式
堆疊（Stack）原始值、參照地址、執行上下文快速分配/釋放、大小固定隨執行上下文自動回收
堆積（Heap）物件、陣列、函式動態大小、較慢存取由 GC 管理
程式碼空間編譯後的程式碼JIT 編譯結果引擎管理

V8 堆積結構

區域功能大小GC 策略
新生代（Young Generation）新建立的物件1-8 MBScavenge（Minor GC）
老生代（Old Generation）存活較久的物件數百 MB~數 GBMark-Sweep-Compact（Major GC）
大物件空間（Large Object）超過閾值的物件動態不移動，直接標記清除
程式碼空間JIT 編譯碼動態特殊管理

架構總覽

垃圾回收演算法

引用計數（Reference Counting）

特性說明
原理追蹤每個物件被引用的次數
回收時機引用數歸零時立即回收
優點即時回收、無暫停
缺點無法處理循環引用
現代使用已不作為主要策略（部分用於 COM 物件）

標記清除（Mark-and-Sweep）

1. 標記階段：從根集合（roots）出發，遍歷所有可達物件並標記
2. 清除階段：遍歷堆積，回收未標記的物件
3. 壓縮階段（可選）：整理記憶體碎片
根集合（GC Roots）說明
全域變數window/global 上的屬性
呼叫堆疊中的變數目前執行函式的區域變數
閉包引用被閉包捕獲的外部變數
DOM 樹中的 JS 參照事件處理器等

V8 的世代式 GC

GC 類型目標空間演算法頻率暫停時間
Minor GC (Scavenge)新生代複製演算法（Cheney）高短（1-2ms）
Major GC老生代標記-清除-壓縮低較長（可達數十ms）
增量式標記老生代分步標記—分散暫停
並行 GC老生代多執行緒—減少主執行緒暫停
併發 GC老生代背景執行—幾乎無暫停

各種 GC 策略比較

策略吞吐量延遲記憶體使用適用場景
Stop-the-world高高延遲低批次處理
增量式中低延遲中互動式應用
並行高中中多核環境
併發中最低較高UI 敏感應用

WeakRef vs 強引用

特性強引用（一般變數）WeakRefWeakMap/WeakSet
阻止 GC是否鍵不阻止 GC
存取方式直接存取.deref() 可能回傳 undefined.get(key)
可預測性高低（何時回收不確定）鍵被回收則項目消失
用途一般使用快取、觀察者元資料關聯
可迭代——WeakMap 不可迭代

應用場景

常見記憶體洩漏模式

洩漏類型原因解決方案
全域變數意外建立全域變數使用嚴格模式、let/const
被遺忘的計時器setInterval 未清除元件卸載時 clearInterval
DOM 參照JS 保留已移除的 DOM 節點及時解除參照
閉包捕獲閉包持有大物件手動設為 null
事件監聽器未移除事件監聽removeEventListener
集合累積Map/Set/Array 持續成長設定上限或使用 WeakMap
遊離的 DOM 樹從 DOM 移除但 JS 仍參照使用 WeakRef

工具功能使用時機

Allocation Timeline即時分配追蹤找出分配熱點

Node.js –inspect遠端除錯伺服器端分析

最佳化建議

  • 避免在熱路徑中建立臨時物件（使用物件池）
  • 大型陣列用完後設為 null 協助 GC
  • 使用 WeakMap 儲存 DOM 元素的附加資料
  • 在 SPA 中注意路由切換時的清理
  • 使用 FinalizationRegistry 註冊清理回呼
