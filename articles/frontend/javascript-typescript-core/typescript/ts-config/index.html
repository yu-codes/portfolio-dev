tsconfig.json 是 TypeScript 專案的核心配置檔案，它控制著編譯器的行為、型別檢查的嚴格程度與模組解析策略。正確配置 tsconfig 對於專案的型別安全性、編譯效能和開發體驗至關重要。

核心概念

tsconfig.json 結構

頂層屬性功能說明
compilerOptions編譯器選項最核心的配置區塊
include包含的檔案Glob 模式指定要編譯的檔案
exclude排除的檔案排除 include 中的某些檔案
files明確指定檔案精確列舉要編譯的檔案
extends繼承配置從基底配置檔繼承
references專案參照多專案構建設定

嚴格模式選項

選項功能建議
strict啟用所有嚴格檢查新專案必開
strictNullChecksnull/undefined 檢查避免最常見錯誤
strictFunctionTypes函式參數逆變檢查函式型別安全
strictBindCallApplybind/call/apply 型別檢查正確推斷 this
strictPropertyInitialization類別屬性初始化避免未初始化存取
noImplicitAny禁止隱式 any確保型別標註
noImplicitThis禁止隱式 this明確 this 型別
useUnknownInCatchVariablescatch 變數為 unknown安全的錯誤處理

模組系統選項

選項值說明
moduleESNext, CommonJS, NodeNext輸出的模組格式
moduleResolutionNode, NodeNext, Bundler模組解析策略
targetES2020, ESNext 等編譯目標版本
esModuleInteroptrue/falseES Module 與 CJS 互操作
allowSyntheticDefaultImportstrue/false允許合成預設匯出
resolveJsonModuletrue/false允許匯入 JSON

應用場景

Monorepo 配置

使用專案參照（Project References）管理多套件：

tsconfig.base.json（共用設定）
├── packages/core/tsconfig.json（references: []）
├── packages/ui/tsconfig.json（references: [core]）
└── apps/web/tsconfig.json（references: [core, ui]）
選項功能必要性
composite啟用專案參照被引用的套件必須
references宣告依賴引用方必須
declarationMap跳轉到原始碼開發體驗
incremental增量編譯效能提升

效能最佳化

策略設定效果
增量編譯incremental: true重複編譯加速 50%+
專案參照composite + references只重建變更的套件
排除不必要的檔案exclude減少掃描範圍
skipLibCheckskipLibCheck: true跳過 .d.ts 檢查
使用 tsc –watch—檔案監聽模式

遷移策略

從 JavaScript 逐步遷移至 TypeScript：

步驟設定目的
1allowJs: true, checkJs: false讓 TS 和 JS 共存
2checkJs: trueJS 檔案也做型別檢查
3逐步重新命名 .js → .ts正式遷移
4啟用 strict提高型別安全
5移除 allowJs完成遷移
