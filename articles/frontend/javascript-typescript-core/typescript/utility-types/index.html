TypeScript 的工具型別（Utility Types）是一組預定義的泛型型別，用於常見的型別轉換操作。理解這些工具型別的實作原理，不僅能更好地使用它們，還能幫助我們設計自己的進階型別工具。

核心概念

工具型別的基礎構件

構件語法說明
映射型別（Mapped Types）{ [K in keyof T]: ... }遍歷鍵並轉換
條件型別（Conditional Types）T extends U ? X : Y型別層級的 if-else
索引存取型別T[K]取得屬性的型別
keyof 運算子keyof T取得所有鍵的聯合
infer 關鍵字infer R在條件型別中提取型別
模板字面量型別`${T}`字串型別操作

內建工具型別分類

類別工具型別功能
屬性修飾Partial<T>, Required<T>, Readonly<T>改變屬性的可選/必要/唯讀性
屬性選取Pick<T, K>, Omit<T, K>選取或排除特定屬性
聯合操作Exclude<T, U>, Extract<T, U>過濾聯合型別成員
函式相關ReturnType<T>, Parameters<T>提取函式的型別資訊
字串操作Uppercase<T>, Lowercase<T>字串字面量轉換
PromiseAwaited<T>解包 Promise 型別

架構總覽

屬性修飾型別的實作

工具型別實作原理效果
Partial<T>{ [K in keyof T]?: T[K] }所有屬性變為可選
Required<T>{ [K in keyof T]-?: T[K] }所有屬性變為必要
Readonly<T>{ readonly [K in keyof T]: T[K] }所有屬性變為唯讀
Mutable<T> (自訂){ -readonly [K in keyof T]: T[K] }移除唯讀

選取與排除型別的實作

工具型別實作原理
Pick<T, K>{ [P in K]: T[P] }
Omit<T, K>Pick<T, Exclude<keyof T, K>>
Exclude<T, U>T extends U ? never : T
Extract<T, U>T extends U ? T : never
NonNullable<T>T extends null | undefined ? never : T

函式型別提取

工具型別功能實作方式
ReturnType<T>取得回傳型別T extends (...args: any) => infer R ? R : any
Parameters<T>取得參數型別元組T extends (...args: infer P) => any ? P : never
ConstructorParameters<T>建構函式參數類似 Parameters
InstanceType<T>實例型別T extends new (...) => infer R ? R : any
ThisParameterType<T>this 參數型別T extends (this: infer U, ...) => any ? U : unknown

關鍵比較

Pick vs Omit 的選擇

情境推薦原因
需要少數屬性Pick明確列出所需
排除少數屬性Omit列出不需要的
屬性可能增加Omit新屬性自動包含
嚴格 API 契約Pick新屬性不會意外暴露
安全性考量Pick白名單比黑名單安全

Record vs Map vs Object

面向Record<K, V>Map<K, V>普通物件
型別層級型別工具執行期資料結構型別定義
鍵的型別string/number/symbol任意值string/number/symbol
存取點語法/方括號.get()/.set()點語法/方括號
迭代順序不保證插入順序部分保證
效能一般物件效能最佳化的 Hash Map一般物件效能
序列化JSON.stringify 直接支援需要自訂JSON.stringify 直接支援

常見工具型別替代方案

需求內建方案自訂替代方案
深層 Partial無DeepPartial<T> 遞歸實作
深層 Readonly無DeepReadonly<T> 遞歸實作
選擇性必要無RequiredPick<T, K> 組合
可為空值無Nullable<T> = T | null
路徑型別無Path<T> 遞歸字串生成

應用場景

表單處理型別

// 表單欄位的各種狀態
type FormField<T> = {
  value: T;
  error?: string;
  touched: boolean;
  dirty: boolean;
};

type FormState<T> = {
  [K in keyof T]: FormField<T[K]>;
};

type FormValues<T> = {
  [K in keyof T]: T[K] extends FormField<infer V> ? V : never;
};

API 回應型別

工具型別用途
Partial<User>PATCH 請求的 Body
Pick<User, 'id' | 'name'>列表回應（精簡欄位）
Omit<User, 'password'>公開回應（排除敏感欄位）
Required<CreateUserDTO>確保所有建立欄位必填

自訂進階工具型別

自訂型別功能使用場景
DeepPartial<T>遞歸可選深層物件的部分更新
Paths<T>提取所有路徑表單欄位路徑
FlattenUnion<T>展平巢狀聯合簡化型別結構
Merge<A, B>深層合併兩型別設定檔合併
StrictOmit<T, K>嚴格排除（K 必須是 T 的鍵）避免拼寫錯誤

模板字面量型別應用

型別效果用途
`on${Capitalize<string>}`匹配 onXxx 格式事件處理器命名
`${string}.${string}`匹配含點的字串路徑格式
`#${string}`以 # 開頭CSS 顏色、ID 選擇器

設計原則

  • 優先使用內建工具型別
  • 組合簡單型別而非建立複雜的單一型別
  • 注意遞歸深度限制（TypeScript 預設限制）
  • 加上 JSDoc 說明自訂工具型別的用途
  • 使用型別測試確保工具型別行為正確
