模組化設計

深入探討前端模組化的演進歷程、模組系統比較與現代模組化實踐，涵蓋 ESM、動態載入與依賴管理策略

模組化是管理程式碼複雜度的核心策略，將大型應用拆分為獨立、可組合的小單元。JavaScript 模組系統從早期的 IIFE、AMD 演進至今日的 ES Modules，經歷了根本性的標準化。理解模組化的原理與實踐，是建構可維護前端應用的基礎。

應用場景

設計系統套件

設計系統需精心設計模組邊界：每個元件為獨立子模組（支援 tree-shaking），theme 和 utils 為共用模組，index.ts 提供便利的全量匯入。搭配 package.json exports 確保消費者只能匯入公開 API。

外掛/擴充系統

模組化是外掛架構的基礎。定義核心 API 介面，外掛透過動態 import() 載入，使用依賴注入模式取得核心服務。支援第三方擴充同時保持核心穩定。

漸進式遷移

大型 CJS 專案遷移至 ESM 可逐步進行：新模組直接使用 ESM；共用模組以 dual package（同時提供 CJS + ESM）過渡；工具鏈逐步升級至支援 ESM 的版本。
