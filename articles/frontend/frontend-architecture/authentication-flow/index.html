認證流程

深入解析前端認證與授權的架構設計，涵蓋 Token 管理、OAuth 整合、Session 策略與安全最佳實踐

前端認證流程是應用安全的第一道防線，涉及使用者身份驗證、Token 管理、權限控制與安全儲存等多個面向。從傳統的 Cookie-Session 到現代的 JWT + OAuth 2.0，每種方案都有其適用場景與安全考量。正確實作認證流程，需兼顧安全性、使用者體驗與架構彈性。

應用場景

BFF（Backend For Frontend）模式

最安全的 SPA 認證架構：前端不直接持有 Token，而是透過同域 BFF 伺服器管理 Session。BFF 以 HttpOnly Cookie 維持與前端的 Session，同時以 Token 與下游 API 通訊。完全消除前端 Token 儲存的 XSS 風險。

多租戶 SSO

企業應用常需支援多種身份提供者（Azure AD、Google Workspace、Okta）。前端實作通用 OAuth PKCE 流程，根據租戶配置動態選擇身份提供者。登入後統一為內部 Token 格式，簡化下游服務整合。

無感刷新體驗

結合記憶體 Token + HttpOnly Refresh Cookie：Access Token 存於記憶體（安全且快速），過期時透過 refresh endpoint（使用 HttpOnly Cookie）取得新 Token。搭配請求攔截器和請求佇列，確保並行請求不會觸發多次刷新。
